Configuring and Verifying the FireWire or HyperWire™ Card

Use these procedures to configure a FireWire or HyperWire card to operate with the A3200 and to verify the configuration.

To configure the FireWire card
  1. Open INtime Device Manager (Start → All Programs → INtime → INtime Configuration. Then double-click INtime Device Manager).
  2. In the INtime devices tree, find the FireWire card that you want to use. Right-click the FireWire card and select Pass to INtime using polling. Now the FireWire card is under INtime control.
    3. INtime Device Manager — Pass to INtime

  3. Save the configuration by clicking File → Save the configuration.
  4. If you get a message that you must restart your computer, restart and continue to the next step.
  5. If you do not get a message to restart, close all applications and continue to the next step.
  6. Open A3200 Configuration Manager (Start → All Programs → Aerotech → Automation 3200 → A3200 Configuration Manager).
  7. Open the Connection Settings dialog by right-clicking Controller and selecting Connections Settings.
  8. Click the Drive Communication tab.
  9. If only one card is listed in the list of devices, you can either select the card or select Automatically select a card. If more than one card is listed, you must select the card that you want to use.
    10. Connection Settings — Select the FireWire Card

  10. Click OK to close Connection Settings
  11. Close Configuration Manager.
To configure the HyperWire card
  1. Open INtime Device Manager (Start → All Programs → INtime → INtime Configuration. Then double-click INtime Device Manager).
  2. In the INtime devices tree, find the HyperWire card that you want to use. Right-click the HyperWire card and select Pass to INtime using MSI. Now the HyperWire card is under INtime control.
    3. INtime Device Manager — Pass to INtime

  3. Save the configuration by clicking File → Save the configuration.
  4. If you get a message that you must restart your computer, restart and continue to the next step.
  5. If you do not get a message to restart, close all applications and continue to the next step.
  6. Open A3200 Configuration Manager (Start → All Programs → Aerotech → Automation 3200 → A3200 Configuration Manager).
  7. Open the Connection Settings dialog by right-clicking Controller and selecting Connections Settings.
  8. Click the Drive Communication tab.
  9. If only one card is listed in the list of devices, you can either select the card or select Automatically select a card. If more than one card is listed, you must select the card that you want to use.
    10. Connection Settings — Select the HyperWire Card

  10. Click OK to close Connection Settings
  11. Close Configuration Manager.
To verify FireWire or HyperWire card operation
  1. Run the Status Utility from the Automation 3200 folder.
    • If the Status Utility does not open and you get a CommunicationsCardConfiguration or CommunicationsConfiguration error, refer to the To Configure the FireWire Card or the To Configure the HyperWire Card section of this topic.
    • If you get a different error message, refer to the Troubleshooting section of the Help file for more information.
  2. In the Controller tab of the Status Utility, find the row labeled Real Time Error Total. Then, in the Drive Interface tab, find the Total Errors row. These rows are highlighted in the figures that follow. The numbers in these rows should be zero. If not, there might be a problem with the cable or the drive. Or, the FireWire card might not be compatible with the A3200. Refer to Compatible FireWire Cards for more information. FireWire, HyperWire, and Real Time errors can also be caused by an SMC task failure.

    0

    Data was transmitted with no problems.

    Figure: No Problems

    0

    m

    The axis and FireWire are operating correctly. But, the data generation did not complete within 1 msec. "m" is the number of times the data generation did not complete within 1 msec.

    This state is usually caused by a defective PC ship set, CPU throttling, power conservation, or because another low level PC program is running. Refer to the PC recommendations in Optimizing A3200 Performance for more information.

    In the figure that follows, m = 3. There were three times that the data generation did not complete within 1 msec.

    Figure: Data Generation Failure

    n

    m

    The axis is operating, but it did not correctly transmit the data through the FireWire. "n" is the number of times the data was not correctly transmitted.

    This state is usually caused by problems with a drive or a loose or disconnected FireWire card. Also, a very bad "0 / m" state can cause this state.

    In the figure that follows, n = 1 and m = 3. There was one time that the data was not correctly transmitted. There were three times that the data generation did not complete within 1 msec.

    Figure: Data Transmission Failure

NOTE: If the motion is erratic or if other problems occur, look at the Status Utility for FireWire errors. Start the Status Utility and leave it running while the problems occur. FireWire Error should always be 0.

NOTE: FireWire errors can cause unpredictable problems. A3200 will not operate correctly until the cause of the error has been corrected.
