Modbus Protocol

The A3200 supports additional I/O through Ethernet devices that support the Modbus/TCP protocol. These devices include the following.

  • WAGO Fieldbus couplers - model 750-341, 750-342, 750-841, 750-842
  • Automation Direct T1H-EBC100
  • DVT smart image sensor
  • Cognex In-Sight camera
  • Festo CPX-FEC Ethernet I/O/pneumatic valve controller

Table:     Modbus Configurations Supported by the A3200

Type Description

Drive Modbus

Modbus I/O modules are attached to the onboard Ethernet controller of the drive either directly with a crossover cable (Aerotech Part Number: NCONNECT-IO-XXX) or with a regular Ethernet cable through a network switch.

PC Modbus

A dedicated network adapter card is configured to run under a real-time subsystem. The Modbus I/O module plugs into the network adapter either directly with a crossover cable (Aerotech Part Number: NCONNECT-IO-XXX) or with a regular Ethernet cable through a network switch.

NOTE: When you do a Modbus configuration, make sure that you also refer to the documentation from the manufacturer. This documentation has important information about your device that is not included in the A3200 Help file.
Comparison of Drive Modbus and PC Modbus

Table:     Drive Modbus and PC Modbus Comparison

Action Drive Modbus PC Modbus

Implementation

Firmware running within the drive.

A process running at the same time as the Software Motion Controller (SMC) under INtime.

Connections

Supports one Modbus master connection per drive.

Supports one Modbus master connection and one Modbus slave connection, or up to 16 Modbus master connections with the Professional Option.

Performance

About 3 msec delay at the firmware level, around 12 to 22 msec delay for setting each of the outputs of one 8-output bit output module (tested with WAGO Output module 750-530) and waiting for its status to change from an AeroBasic program.

About 1 msec delay at the RTSS process level, around 4 to 10 msec delay for setting each of the outputs of one 8-output bit output module (tested with WAGO Output module 750-530), and waiting for the status to change from an AeroBasic program.

Ease of configuration

Very simple.

Moderately difficult (configure only once).
Modbus Register Addresses and Function Codes

When you specify a Modbus register address, do not include the highest-order digit of the address.

For Example

Say that you want to use FC 2 (0x02) to read an input bit at register address 0x10012. To do this, you specify 0x12 for the register address. Then the function code automatically selects the correct register address based on the applicable range of addresses, which is 0x10000 - 0x1FFFF.

NOTE: Your Modbus device might require the highest-order digit in a register address. Refer to the documentation from the manufacturer to find the correct way to specify a Modbus register address for your device.
