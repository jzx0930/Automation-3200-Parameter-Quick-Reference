Building Programs in Motion Composer

Build Actions

Build actions always use the active file.

  • Build. Tries to build the active file. Compiler output is sent to the Output window, and errors are shown in the Error List.
    • Note: Only .pgm files may be built.
    Note: Axis names from the active parameter file can be used. When there is no active parameter file, default axis names will be used.
  • Load. Builds the current AeroBasic program, loads it onto the controller, and sets the task to the Program Ready state. You can then execute the program.

Output

Output from the compiler is sent directly to the output window. This output contains detailed information about the build process. For a list of build errors only, refer to the Error List.

Error List

When building AeroBasic programs, any errors found during compilation will be displayed here. Double-clicking on any error line will try to open the file containing the error and put the cursor on the line containing the error.

  • Line. This displays the line number of the error.
  • File. This displays the full path of the file containing the error.
  • Type. This displays the type of error.
  • Message. This displays a message about the error and may contain instructions for correcting it.
