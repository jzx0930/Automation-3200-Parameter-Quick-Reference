Converting Plot Files to Matlab Files

Digital Scope can convert your plot files to MATLAB files. You can access this ability by using the command line to call the Digital Scope executable file with a series of arguments, which are explained in the following sections. You can find the Digital Scope executable file in the [Installation Directory]\bin folder.

Syntax:

 
digitalscope -matlab <InputFile> [-dominantunit <DominantUnit>] 
[-dependentunit <DependentUnit>] [-timeunit <TimeUnit>] 
[-o <OutputFile>] [-silent] [-responsetype <ResponseType>]

Example: 

digitalscope -matlab "c:\plot files\A3200Data.plta" -dominantunit 3 
-dependentunit 2 -timeunit 2
; Converts a plot file to a matlab data file 
; ("c:\plot files\A3200Data.mat") using dominant primary units,  
; dependent primary units / 1000 and milliseconds. If 
; "c:\plot files\A3200Data.mat" exists, the user will be prompted to 
; choose whether or not it is overwritten.
digitalscope -matlab "c:\plot files\A3200Data.plta" 
-o  "c:\plot files\A3200Output.mat"
; Converts a plot file to a matlab data file 
; ("c:\plot files\A3200Output.mat"). No unit conversion will be done so 
; the matlab data units will be counts and seconds. If 
; "c:\plot files\A3200Data.mat" exists, the user will be prompted to 
; choose whether or not it is overwritten.

Arguments

<InputFile>

This required argument specifies name of the scope or loop transmission file that Digital Scope will convert to a MATLAB file. This argument is always required.

-dominantunit <DominantUnit>

This optional argument specifies the units to use in the MATLAB file. The <DominantUnit> value specifies the units for dominant axes.

<DominantUnit> Value Description

0

counts (default)

1

secondary units

2

secondary units / 1000

3

primary units

4

primary units / 1000
-dependentunit <DependentUnit>

This optional argument specifies the units to use in the MATLAB file. The <DependentUnit> value specifies the units for dependent axes.

<DependentUnit> Value Description

0

counts (default)

1

primary units

2

primary units / 1000
-timeunit <TimeUnit>

This optional argument specifies the units for the time data in the MATLAB file.

<TimeUnit> Value Description

0

minutes

1

seconds (default)

2

milliseconds

-o <OutputFile>

This optional argument allows you to specify the name of the MATLAB output file. If you do not specify a name, the resulting output file will have the same name as the input file, but with a .mat extension.

-silent

By default, Digital Scope asks you to confirm that an output file should be overwritten if it already exists. This optional argument will prevent the application from displaying the prompt and will always cause an existing file to be overwritten.

-responsetype <ResponseType>

This optional argument specifies the response type to use when exporting a Loop Transmission (*.ltda) file.

<ResponseType> Value Description

0

Open-loop (default)

1

Plant

2

Sensitivity

3

Complementary Sensitivity

4

Process Sensitivity

5

Closed-loop

6

Closed-loop error

7

PID Controller

8

PID Controller and Servo Filters

9

Feedforward

10

Inverse Feedforward
