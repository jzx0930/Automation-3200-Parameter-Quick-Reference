Camming Example

Before you run an AeroBasic example program, copy the example to your User Files directory. Copying the example makes sure that you have access to compile and run the program and prevents changes that you make from being overwritten if you install updated software.

View the example program in A3200 Motion Composer. Motion Composer adds color in the code which makes it easier to read. For best results, do one of the following.

  • Open a new file in Motion Composer. Copy and paste the code into the file.
  • OR
  • To examine the source code, open the example program from your User Files directory. You might have to copy the example program from the \A3200\Samples\AeroBasic folder first.

This example is one of many example programs. For a complete list of examples, refer to Example Programs.

To see a list of available hardware manuals, refer to Other Documentation. 

'
'				Camming.pgm 
'    This sample shows you how to do use the LOADCAMTABLE command to do 
'    simple camming motion. You can also use the LOADCAMVAR command to do camming.
'    The program also shows in-feeding motion on the slave axis while camming.
'    X is the master axis. Y is the slave axis.
'
' 
DVAR $TableNumber
DVAR $Interpolation
DVAR $MasterStartPosition
DVAR $SlaveStartPosition
DVAR $TrackingMode
DVAR $SyncMode
DVAR $MasterMotionRange
'
' Set motion values
$MasterMotionRange = 100
$MasterStartPosition = 50
$SlaveStartPosition = 20
'
' Specify Interpolation type:
'   1 for third order splines (recommended unless you are trying to reduce spline overshoots)
'   0 for linear interpolation
$Interpolation = 1 
'
' Specify Tracking mode. The slave position command is driven by:
'   0 = master axis feedback position            
'		(WARNING: servo loop jitter in the master axis will jitter the slave axis)
'   1 = master axis command position
'   2 = master axis auxiliary position feedback
$TrackingMode = 1 
'
' Specify table number. Up to 99 tables allowed.
$TableNumber = 3 
'
' Specify synchronization mode:
'   1 = on  (relative synchronization)
'   2 = on  (absolute synchronization)
'   3 = on  (VELOCITY synchronization)
' Suppose at the time of synchronization the master axis is at position "m". And suppose that 
' in the table, when the master axis is at position=m, then the slave axis is told to be at
' position "s". Also, suppose that the slave axis is currently at position "t". If s=t, then
' relative and absolute modes operate the same. If s!=t, then in absolute mode, the SYNC command
' does a RAPID move of the slave axis to position t, before synchronizing to the table. The
' SYNC command does not return until this motion is complete. But, in relative mode, (t-s) is
' specified as an offset, and all slave positions in table are shifted by (t-s) for this
' synchronization cycle.
$SyncMode = 1           
'
' Turn off camming (if it was on).
CAMSYNC Y $TableNumber 0  
FREECAMTABLE $TableNumber
'
' Some motion setup.
PRIMARY 
MINUTES
VELOCITY OFF
ENABLE X Y
HOME X Y
LINEAR X $MasterStartPosition Y $SlaveStartPosition F 1000
'
' Start data collection. You can collect:
'    6,000 points at one millisecond period
'    X (master axis), Position command, Velocity command, Acceleration command
'    Y (slave axis), Position command, Velocity command, Acceleration command
' or you can import the Camming.dcca file into Digital Scope.
' (see "Saving and Loading Data Collection Configurations" in Help).
scopetrig
'
' Load cam table.
LOADCAMTABLE X, $TableNumber, Y, $Interpolation, $TrackingMode, "SimpleCam.cam" NOWRAP
'
' Sync to the table. 
' NOTE: Only one active table can reference a specified slave axis.
' NOTE: Slave axes can also be master axes to other axes.
' NOTE: If you are in absolute synchronization mode and slave motion starts, then this
' command waits until the slave axis stops moving.
CAMSYNC Y $TableNumber $SyncMode 
'
' Move the master axis (the slave axis also moves because the table is active).
MoveInc X $MasterMotionRange 1000
DWELL 2.5
'
' At this time, move the slave axis on top of its master motion (this is an in-feed)
' NOTE: As an alternative to the LINEAR command, you can use the RAPID command.
' NOTE: Do not use the ABSOLUTE command during in-feeding because it causes the
' end position to be incorrect.  
'  
INCREMENTAL
LINEAR Y 500 F 10000
'
' Turn off camming.
CAMSYNC Y $TableNumber 0  
FREECAMTABLE $TableNumber

END PROGRAM
