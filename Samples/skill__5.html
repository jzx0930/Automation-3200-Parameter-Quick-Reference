PSO Programming Examples

Hardware: PSO Programming Examples do not apply to the Nmark SSaM and the Nstep.

The PSO firing examples describe PSO capabilities and command syntax.

The pulses drawn in some examples assume a rising edge pulse (active high) instead of normal falling edge pulse (active low) configuration. To verify correct operation of the PSO, the DATAACQ (data acquisition) commands are used for data collection based on a PSO event.

When the PSO output is masked (turned on and off) because of some condition (such as WINDOW mode), data acquisition prevails because the PSO firing events are still present. More position data can be collected than the actual number of positions where PSO pulses are fired. However, the set of positions where PSO pulses are fired must be a subset of all collected positions.

The PSO hardware operates in encoder counts, even if you command the axes to move in user units.

The PSO hardware cannot do non-integer scaling. For true vector firing, all axes being tracked must have the same distance unit per encoder count, even when trying to simultaneously fire based on the position of both a linear and rotational axis. If you are programming a rotational axis in linear units based on the circumference of the part, each encoder count will still represent an unequal linear distance that can not be normalized (scaled to like units) by the integer pre-scalar available in the PSO hardware.

For the following examples, a typical configuration of 1,000 encoder counts per primary distance unit is used. The CountsPerUnit parameter sets the number of encoder counts per primary distance unit.

Note: The example files are located on your hard drive. By default, these examples are installed in the PSO folder (C:\Program Files\Aerotech, Inc.\A3200\Samples\AeroBasic\PSO). For the links in the table below to work correctly, the .pgm files need to be associated with a text editor program, like Notepad, on your computer. Refer to the documentation that was supplied with your computer for information on how to associate a file extension with a file type.

Before you run an AeroBasic example program, copy the example to your User Files directory. Copying the example makes sure that you have access to compile and run the program and prevents changes that you make from being overwritten if you install updated software.

View the example program in A3200 Motion Composer. Motion Composer adds color in the code which makes it easier to read. For best results, do one of the following.

  • Open a new file in Motion Composer. Copy and paste the code into the file.
  • OR
  • To examine the source code, open the example program from your User Files directory. You might have to copy the example program from the \A3200\Samples\AeroBasic folder first.
