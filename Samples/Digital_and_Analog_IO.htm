Digital and Analog IO Example

Before you run an AeroBasic example program, copy the example to your User Files directory. Copying the example makes sure that you have access to compile and run the program and prevents changes that you make from being overwritten if you install updated software.

View the example program in A3200 Motion Composer. Motion Composer adds color in the code which makes it easier to read. For best results, do one of the following.

  • Open a new file in Motion Composer. Copy and paste the code into the file.
  • OR
  • To examine the source code, open the example program from your User Files directory. You might have to copy the example program from the \A3200\Samples\AeroBasic folder first.

This example is one of many example programs. For a complete list of examples, refer to Example Programs.

To see a list of available hardware manuals, refer to Other Documentation. 

'				IO.pgm
'
'
' This example shows how to use the digital, analog,
' virtual binary, and virtual register IO commands.
' Further details can be found in the help file.

' Inputs and outputs are always two letter codes, where the last letter 
' is "O" for outputs, and "I" for inputs.


'
' Analog inputs and outputs are double-precision floating point values.
' Digital inputs and outputs are binary values.
' You can set and get analog and digital outputs.
' You can get analog and digital inputs.
'
' Set analog output #1
$AO[1].X = 2.0   
' Clear digital output #2
$DO[2].X = 0                      
' Clear digital output indicated by global[0]
$DO[$global[0]].X = 0             
' Set digital output #2 if digital input #1 is on
IF ($DI[1].X == 1)                  
	$DO[2].X = 1                     
ENDIF
' You can get as well as set outputs 
$DO[3].X = $DO[2].X                    


'
' Virtual binary inputs and outputs are binary values.
' Virtual binary inputs and virtual binary outputs are bidirectional, 
' which means that they can either be read from or written to.
'
' Set binary output #1
$BO[1] = 1                      
' Clear binary output #2
$BO[2] = 0       
' Clear binary output indicated by global[0]
$BO[$global[0]] = 0  
' Set binary output #2 if binary input #1 is on
IF ($BI[1] == 1)                  
	$BO[2] = 1                     
ENDIF
' You can get as well as set binary outputs and inputs
$BI[3] = $BO[2]                     


'
' Virtual register inputs and outputs are double-precision floating point values.
' Virtual register inputs and virtual register outputs are bidirectional, 
' which means that they can either be read from or written to.
'
' Set register output #1
$RO[1] = 2.0                        
' Clear register output #2
$RO[2] = 0.0                      
' Clear register output indicated by global[0]
$RO[$global[0]] = 0.0 
' Set register output #2 if register input #1 is on              
IF ($RI[1] == 1.0) 
   $RO[2] = 9.5  
ENDIF
' You can get as well as set register outputs and inputs
$RI[3] = $RO[2]   


END PROGRAM
