Fixed Distance, Bit Map Mode Example

Before you run an AeroBasic example program, copy the example to your User Files directory. Copying the example makes sure that you have access to compile and run the program and prevents changes that you make from being overwritten if you install updated software.

View the example program in A3200 Motion Composer. Motion Composer adds color in the code which makes it easier to read. For best results, do one of the following.

  • Open a new file in Motion Composer. Copy and paste the code into the file.
  • OR
  • To examine the source code, open the example program from your User Files directory. You might have to copy the example program from the \A3200\Samples\AeroBasic folder first.

This example is one of many example programs. For a complete list of examples, refer to Example Programs.

To see a list of available hardware manuals, refer to Other Documentation.

Program Summary

Hardware: This example program does not apply to the Nmark SSaM and the Nstep.

This program demonstrates the use of the bit map functionality of the PSO. The primary feedback for the X axis is tracked and a PSO firing event is generated each time the feedback increments by 1000 counts. An array, which specifies the output states of the PSO, is created and downloaded to the PSO hardware. At each firing event the state of the PSO output is set to the next bit in the array.

'                     FixedDistanceBitMap.pgm
'
' This sample demonstrates the use of the bit map functionality of the PSO.  An array
' containing the ON/OFF bit values is downloaded to the hardware and the PSO output
' is simply set to the corresponding state at each fixed-distance firing event.

' NOTE: All PSO samples are written assuming 1000 counts per unit.
' NOTE: PSO pulse outputs are intentionally long so that the PSO output state can
'       be easily viewed using the "PSO Status" item in the Digital Scope.


' Define the number of bits we want to put in our array.  Using a #define statement
' makes it easier to change the number of bits later without having to remember all
' of the places in the program that would need to be changed.
#define NUM_BITS 10

' Only the most-significant bit of each 32-bit array value is used for the bit map
' output, so here we create defines for the ON and OFF states so we don't have to
' remember this fact later when creating the bit map array.
#define PSO_BIT_OFF 0x00000000
#define PSO_BIT_ON  0x80000000

DVAR $BitmapArray[NUM_BITS]

' Fill the local array with the bit map values.
$BitmapArray[0] = PSO_BIT_ON
$BitmapArray[1] = PSO_BIT_OFF
$BitmapArray[2] = PSO_BIT_OFF
$BitmapArray[3] = PSO_BIT_ON
$BitmapArray[4] = PSO_BIT_ON
$BitmapArray[5] = PSO_BIT_OFF
$BitmapArray[6] = PSO_BIT_ON
$BitmapArray[7] = PSO_BIT_ON
$BitmapArray[8] = PSO_BIT_ON
$BitmapArray[9] = PSO_BIT_OFF

' Write the bit map values from the local variables into the drive array.
ARRAY X WRITE $BitmapArray[0] 0 NUM_BITS

' Enable and home the axis.
ENABLE X
HOME X

' Reset the PSO.  This resets the internal state of the PSO hardware, including the distance
' tracking counters and the window counters.
PSOCONTROL X RESET

' Uncomment the following line to configure the PSO hardware to send the PSO signal out the
' auxiliary marker.  If using the dedicated PSO output, this command should not be used.
' Refer to your drive hardware manual for more information.
'PSOOUTPUT X CONTROL 0 1

' Configure the PSO for single axis tracking from the primary encoder input.
PSOTRACK X INPUT 0

' Configure the PSO to generate a firing event at a fixed distance.
PSODISTANCE X FIXED 1000

' Specify the output of the PSO to come from the bit map logic.
PSOOUTPUT X BIT MAP 0 NUM_BITS

' Arm the PSO to start the distance tracking.
PSOCONTROL X ARM

' Move the axis so as to cause the PSO to fire.
LINEAR X 10 F 5

' Disable the PSO pulse generator.
PSOCONTROL X OFF

END PROGRAM
