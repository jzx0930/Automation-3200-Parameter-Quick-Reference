Acceleration and Deceleration Ramping Overview

The controller uses acceleration and deceleration ramping to gradually change speeds. Refer to the Ramp Type and Peak Acceleration section. Ramping applies to all motion except handwheel, gearing, and camming motion. You can control the ramping operation by using the parameters and commands in the Ramping Commands and Parameters section of this topic. If you change the value of one or more of these parameters during motion, the changes do not have an effect until the next move starts.

Velocity Profiling

When an axis is commanded to move, the controller generates a velocity profile. At the rate that you specify for the MotionUpdateRate parameter, the controller calculates position and velocity values and sends these values to the position loop of the servo loop. The velocity profile includes the following ramp segments.

  • Acceleration
  • Constant Velocity
  • Deceleration

This topic gives you information about the different properties of the acceleration and deceleration ramps.

Figure: Acceleration and Deceleration

Types of Motion

Acceleration and deceleration ramping operates differently for different types of motion.

Coordinated Motion

Coordinated motion includes LINEAR, CW, CCW, and BEZIER commands. Because coordinated motion synchronizes motion across multiple axes, you must configure ramping per task instead of per axis. Refer to the Ramping Commands and Parameters section.

Independent Motion

Independent motion includes RAPID, HOME, and the asynchronous motion commands. Because the controller ramps RAPID, HOME, and asynchronous motion independently on each axis, you must configure ramping per axis instead of per task. Refer to the Ramping Commands and Parameters section.

Abort Motion

If you issue an ABORT command before the controller generates its target position, the controller might use a different deceleration rate. Refer to the ABORT command and the AbortDecelerationRate parameter for more information. Because the controller ramps Abort motion independently on each axis, you must configure ramping per axis instead of per task.

HOME Motion

HOME motion ramping, by default, is the same as independent motion ramping. But, you can use the HomeRampRate parameter to specify a different type of ramping for HOME. Because the controller ramps HOME motion independently on each axis, you must configure ramping per axis instead of per task.

Ramp Types and Peak Acceleration

You can configure the ramp as one of the following acceleration and deceleration ramp types. If you specify an acceleration or deceleration rate for the ramp type that you are using, the specified rate is the average rate across the time interval over which the acceleration or deceleration occurs. That time interval is the same for all of the ramp types. Use the RAMP TYPE command to specify the type of ramp that you want to use.

  • Linear
  • Sinusoidal Half-Sine This is the default. To set the ramp type to sinusoidal (full-sine), clear the Half Sine Ramp bit of the SystemCompatibility global parameter in Configuration Manager.
  • S-curve

Changes in Velocity Acceleration

The following ramps shows three moves that use a speed of 1,000 mm/sec and a specified ramp rate of 1,000 mm/sec2.

Figure: Linear, Half-Sine, and S-curve Ramping

The first ramp uses Linear ramping.

  • Shows the lowest peak acceleration, which is 1,000 mm/sec2, of all three ramps.
  • Shows sudden changes in acceleration at the segment boundaries.

The second ramp uses Sinusoidal Half-Sine ramping.

  • Shows a higher peak acceleration, which is 1,570 mm/sec2 , than linear ramping.
  • Sudden changes in acceleration do not occur at the segment boundaries.

The third ramp uses S-curve ramping at 50 percent.

  • Shows a higher peak acceleration, which is 1,333 mm/sec2 , than linear ramping.
  • Sudden changes in acceleration do not occur at the segment boundaries.
Ramp Modes

You can specify rate-based and time-based acceleration and deceleration. Acceleration and deceleration occur over the same period with either a sinusoidal or linear ramp type. Aerotech recommends that you use rate-based ramping as an alternative to time-based ramping because the maximum acceleration does not change when the commanded velocity changes. Refer to RAMP MODE RATE command.

NOTE: When you use time-based ramping, if a move does not have sufficient distance to get to full speed, the ramp time might be smaller than the value that you specified for the RAMP TIME command. If this occurs the A3200 continues to use the ramp rate that was initially set with the full speed and the specified ramp time, which causes a smaller than expected ramp time.

This has an effect on the LINEAR, CW, CCW, BEZIER, RAPID, MOVEINC, and MOVEABS motion commands.
Peak Acceleration

In the A3200, you specify acceleration and deceleration ramp rates as an average acceleration value across the ramp. You can use the RAMP RATE command to specify this value by rate or you can use the RAMP TIME command to specify this value by time. The value that you specify by rate or time is always an average value. Refer to the Ramp Modes section of this topic for more information.

During acceleration and deceleration ramping, the average acceleration value and the ramp type that you specify have an effect on the value of the peak acceleration. On the A3200, you can use the RAMP TYPE command to specify the ramp type as LINEAR, SINE, or SCURVE.

This figure shows the velocity and acceleration for an S-curve ramp type that has a value of 50.

Linear Ramping

Linear ramping is equivalent to S-curve ramping that has a value of 0.

  • The peak acceleration value is equal to the average acceleration value throughout the ramp.
  • This type of ramping causes the lowest possible peak acceleration.
  • The ramp instantaneously gets to peak acceleration.
  • High jerk values can occur at the ramp boundaries.

Sinusoidal Ramping

  • The peak acceleration value is 1.57 times the average acceleration value.
  • This type of ramping causes higher peak accelerations than the linear ramp type.
  • The ramp gradually gets to peak acceleration.
  • High jerk values are decreased.

Refer to the Ramp Types and Peak Acceleration section of this topic for more information.

S-curve Ramping

S-curve ramping that has a value of 0 is equivalent to Linear ramping. For S-curve ramping that has values between 0 and 100, the peak acceleration values are between 1.0 and 2.0 times the average acceleration. For example, an S-curve ramp that has a value of 50 causes a peak acceleration that is 1.33 times the average acceleration.

S-curve Ramping that has a value of 100

  • The peak acceleration value is 2.0 times the average acceleration value.
  • This type of ramping causes higher peak accelerations than the linear ramp type.
  • The ramp gradually gets to peak acceleration.
  • High jerk values are decreased.

Refer to the SCURVE command and the Ramp Types and Peak Acceleration section of this topic for more information.

ACCEL and DECEL Keywords

If you do not specify the ACCEL and DECEL keywords, the value for the acceleration and deceleration is the same. The controller applies this value equally to the acceleration and deceleration ramps.

If you want different behavior for the acceleration and deceleration ramps, use the ACCEL and DECEL keywords to specify different values for acceleration and deceleration in the RAMP TIME and RAMP RATE commands.

Additional Ramping Considerations
  • In velocity profiling mode (VELOCITY ON), ramping can occur across multiple moves.
  • Master/Slave motion does not do ramping of the slave axis. The motion of the master axis controls the acceleration and deceleration of the slave axis.
  • PVT motion does not use ramping. The values that you specify in the PVT command for <Position>, <Velocity>, and <Time> control the acceleration and deceleration.
  • If an axis fault causes deceleration to occur, none of the values that you specify for the ramp commands and parameters in this topic have an effect on the ramping. As a result, the ramp type is always linear and the ramp rate is always the value that you specify for the AbortDecelRate parameter.
