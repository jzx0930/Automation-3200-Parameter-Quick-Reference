Callback Commands

Callback commands are special commands that execute in Windows. For example, the controller running under a real-time operating system cannot access files on the PC, so when a file access command is encountered on the controller, it pauses the AeroBasic program and triggers execution of the command in the Windows environment. There is a full list of callback commands at the end of this topic.

Some callback commands include commands that display text. Because the controller cannot directly write to the display, these commands are passed to the host PC. The process is as follows.

  1. The controller pauses execution of the AeroBasic program.
  2. A callback interrupt is generated by the controller. The CallbackHoldActive bit of the TaskStatus0 status value is set True.
  3. The Communication Service, running under Windows, detects the callback interrupt and signals the controller that it received the callback interrupt. The CallbackResponding bit of the TaskStatus0 status value is set to True.

  4. Note: If this step does not occur quickly enough the controller will generate the Timeout Responding to Callback error. This generally indicates that no application is registered to handle the callback.
  5. A host application retrieves the callback data and carries out the appropriate function.
  6. Upon completion of the callback command, the host application tells the controller to continue, clearing the CallbackResponding bit of the TaskStatus0 status value.
  7. The controller continues execution of the AeroBasic program.

  8. Note: Callback commands set a task error for most error conditions. But, for some file-related errors the command sets the Info Variables and does not set a task error. The Help topic for the callback command tells you if information is returned in the info variables. If the topic does not give information about info variables, then the command always sets a task error for error conditions.

Table:     Related TaskStatus0 Status Value Bits

CallbackHoldActive

True if the controller is waiting on the AeroBasic program line for the host application to handle the callback. The host application may or may not have received the callback data.

CallbackResponding

True if the host application running on the PC received the callback data but did not finish handling the callback.
