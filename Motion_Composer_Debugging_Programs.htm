Debugging Programs in Motion Composer

In Motion Composer, you can run more than one program at the same time, but you can only run one program at a time on each task.

Note: You can test or debug programs without moving the axes. Refer to Simulation of Motion for more information.

Debug Actions

Debug actions usually apply to the AeroBasic program that is associated with the active task. The following actions are on the Debug menu in Motion Composer.

Run Runs, or continues to run, the selected AeroBasic program.

Pause Causes the active program to pause before it executes the next command. Because the command that is currently running does not stop, motion might not stop immediately.

Stop Stops executing the program that is associated with the active task.

Stop All Stops executing all programs in all tasks.

Step Over Executes one line of the program that is associated with the active task. If the program line is a subroutine call (CALL or FARCALL), it executes all of the subroutine and moves to the next program line.

NOTE: Step Over does not stop on lines in a subroutine unless there are breakpoints in the subroutine.

Step Into Executes one line of the program that is associated with the active task. If the program line is a subroutine call (CALL or FARCALL), it steps one level into the subroutine.

Toggle Breakpoint To add or remove a breakpoint from a program line, put your cursor on the line, open the Debug menu, and select Toggle Breakpoint. If the program is running, the breakpoint is set immediately. If the program is not running, the breakpoint is set when the program is loaded.

NOTE: Breakpoints on non-executable lines do not cause the program to pause.

Clear All Breakpoints Clears all of the breakpoints in the program that is associated with the active task. If the program is running, the breakpoints are cleared immediately.

Toggle Line Highlighting Select Toggle Line Highlighting to tell Motion Composer to either highlight each line that executes or to not highlight each line that executes while a program runs.

NOTE: Lines are highlighted when Motion Composer gets status information from the controller, but are not highlighted in real time.

NOTE: Motion Composer always highlights the lines where execution is paused or where an error occurs. Toggle Line Highlighting does not change this behavior.

Variable Watch

To open the Variable Watch window, click the Variable Watch tab.

Use the Variable Watch window to view and edit the value of Global Variables, Task Variables, Program Variables, Virtual Binary I/O Bits, and Virtual I/O Registers.

To add a variable to the Watch window, in the Name column, click the cell and type the name of the variable. The name is case-insensitive. If the variable can be resolved, the Value, Type, Address, and Context cells are populated automatically.

Each variable value in the Watch Window updates when you change the current task selection, when you change the current task state, or when you click the Update Now button . To have variable values continuously update, click the Turn on Continuous Updating button . To stop continuous updates, click the Turn off Continuous Updating button .

Note: Task variables are resolved using the current task.

Note: Program variables are resolved using the program that is associated on the current task.

Note: You can add only the individual elements of Program Array variables to the watch window.

To edit a variable in the Watch window, in the Value column, click the cell and type the new value.

To remove a variable from the Watch window, in the Name column, click the cell and delete the text.

Note: To put a Name or Value cell in Edit mode, select the cell and press F2 on your keyboard.

Setting the Next Program Line

Danger: Setting the program line can cause unpredictable and possibly dangerous machine operation. Make sure that all of the code necessary for the correct and safe operation of your system will be executed after your program starts on the line that you select.

Warning: If you are not sure how to correctly use this setting, contact Aerotech Global Technical Support.

You can set the next program line only when the task is in one of the following states.

  • Program Ready
  • Program Paused
  • Program Complete

To set the next program line, right-click on the line and then click Set As Next Program Line.

The current-line indicator moves to the line that you select. When the program runs, the program execution begins on that line.

After the Set As Next Program Line function is called, the task is in the Program Paused state.

Note: If you select a program line that is non-executable, such as a comment, the next executable program line is set as the next line that the program executes when the program runs.
