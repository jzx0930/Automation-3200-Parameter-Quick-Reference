SMC Arguments

Do not change the SMC command line arguments unless you are instructed to do so by an Aerotech service representative.

Depending on the arguments and the system configuration, changes that you make to the command line arguments might not have an effect until you shut down the A3200 and then reset the controller.

Table:     SMC Arguments

Argument

Description

-Dn

INtime Console mode for debug

WARNING: Nonzero values for n can cause real-time performance problems. This argument is only for debugging and must be removed after debugging.

n

Description

0

No INtime Console output (default - this is mode when no -D specified)

1

Errors only

2

Normal debug and errors

3

Extended debug and errors

4

Extended debug, but only during reset/restart process

-G

FireWire or HyperWire card is not used.

-Pn

Use Poll time to detect FireWire activities in microseconds (μsec). The default value for n is 100 μsec. n must be less than 1000 μsec and is rounded to the nearest Kernel Clock Rate. To access the Kernel Clock Rate, click Start>Programs>INtime>Time Configuration, then double click Node Management, select AerotechNode0, select the Kernel tab, and then select the Kernel Clock Rate.

-Mn

Number of context programs running on the PC. The only valid values for n are 0, 1, 2, or 4 (the default value is 0). If n is 0, it will automatically choose the lowest possible number needed (1, 2, or 4) to service the drives connected. Lower numbers of context programs reduce FireWire traffic on the PCI bus when FireWire errors occur due to PCI bus contention. To see the number of context programs that are configured, refer to the Drive Interface tab of the Status Utility.

-ENABLEVME

Enables support for a controller-level VME card. By default, the controller does not scan the PCI bus for VME cards because this can interfere with the APIs supplied by the card vendor. Enable this argument to control the VME card directly with the VME READ function and the VME WRITE command.

The following arguments are available for use but are not typically used.

-K

Use to communicate to 1394 compatible drives; alters certain reset functionality (internal use only). Ndrive CP, ML, MP, Npaq, Npaq MR, Nservo, and Nstep are not 1394 compatible.

Use with -F.

-S

Indicates FireWire bandwidth used (0x0 for 100 MHz, 1 for 200 MHz, 2 for 400 MHz).

2 is the default.

Warning: Do not change this setting unless you are instructed to do so by Aerotech personnel.

-F

Skips the "ack complete" pending check that occurs during asynchronous communications. Use for communicating to drives that are 1394 compatible (internal use only).

(Ndrive CP, ML, MP, Npaq, Npaq MR, Nservo, and Nstep are not 1394 compatible). Use with -K.

-Tn

Adjusts all sleep timings (in msec) used in the reset process (internal use only).

n

Name

Default

Description

a

ShutdownWaitMsec

500

Sleep after successful bus mastering completed on a reset prior to startup. A time too low or unsuccessful bus mastering can lead to errors during reset.

b

BusResetWaitMsec

2000

Sleep after all bus resets at startup before checking for correct bus mastering.

c

RestartWaitMsec

2000

Sleep after a drive release command before starting asynchronous communications.

e

StopCycleWaitTimeMsec

10

Sleep after stopping the context programs before verifying that they are stopped.

f

SoftResetTimeMsec

50

Sleep after a FireWire soft reset before verifying if bus is reset. A time too low can lead to errors during reset.

g

HCControlSet

100

Sleep after setting the HCControlSet register at startup (no verification).

-Z

Doubles the -Tb time above.
