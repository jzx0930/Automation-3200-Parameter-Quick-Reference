Camming Motion

For each relationship between a master axis and slave axis, you must supply a cam table of coordinates that specifies a slave axis position for each master axis position. You can use the LOADCAMTABLE command or the LOADCAMVAR command to make and load your cam table.

Figure: Master/Slave Profile

If the master axis position on the controller is equal to the master axis position of a table entry, the controller uses the value of that table entry as the slave axis position. If the master axis position on the controller is between two of the master axis positions in the table, the controller uses linear or cubic spline interpolation to calculate a slave axis position that is between the two corresponding slave axis positions in the table. Refer to LOADCAMTABLE or LOADCAMVAR command for more information about linear or cubic spline interpolation.

The following are optional changes that you can make when you configure your cam table.

  • For master axis positions in the cam table, you can specify corresponding slave axis velocities as an alternative to slave axis positions. Refer to Mode 3 in Table: Synchronization Modes of CAMSYNC command for more information.
  • You can add an infeed to the slave axis during camming motion. Refer to Infeeding Overview for more information.
  • A master axis can have two or more slave axes. You can also make the master axis into a slave axis for a different master axis. Refer to Synchronizing Multiple Axes for more information.

Use the following procedure to direct master and slave motion.

To direct master and slave motion

  1. Use LOADCAMTABLE or LOADCAMVAR command to load the cam table.
  2. Synchronize the axes.
  3. Immediately after you synchronize the axes, the controller uses the cam table to change master axis motion into slave axis motion.
  4. Unsynchronize the axes to stop the controller from changing master axis motion into slave axis motion. Refer to Mode 0 in Table: Synchronization Modes of CAMSYNC command for more information.
  5. Issue the FREECAMTABLE command to release the cam table.

Note: If you enable cam table motion and the cam table is configured incorrectly, sudden changes in slave axis velocity can occur. To prevent this, make sure that the master axis is stationary when you enable or disable cam table motion.

The MaxSpeedClamp axis parameter does not limit the velocity of slave axes during camming motion.

Note: You can use the SLAVEADVANCE and SLAVEOFFSET commands to offset the master axis position in the cam table. Thus the master axis position that the controller uses to do the table lookup is a sum of the master axes position commands and an offset value of these commands.
