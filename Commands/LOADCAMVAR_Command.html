LOADCAMVAR Command

Uses global variables as the cam table.

Syntax:

 
LOADCAMVAR <MasterAxis>, <TableNumber>, <SlaveAxis>, <InterpolationType>, <TrackingMode> RANGE <MasterStart>, <MasterDelta>, $global[<VariableStart>], <VariableNumber> UNITS WRAP
 
LOADCAMVAR <MasterAxis>, <TableNumber>, <SlaveAxis>, <InterpolationType>, <TrackingMode> RANGE <MasterStart>, <MasterDelta>, $global[<VariableStart>], <VariableNumber> COUNTS WRAP
 
LOADCAMVAR <MasterAxis>, <TableNumber>, <SlaveAxis>, <InterpolationType>, <TrackingMode> RANGE <MasterStart>, <MasterDelta>, $global[<VariableStart>], <VariableNumber> UNITS NOWRAP
 
LOADCAMVAR <MasterAxis>, <TableNumber>, <SlaveAxis>, <InterpolationType>, <TrackingMode> RANGE <MasterStart>, <MasterDelta>, $global[<VariableStart>], <VariableNumber> COUNTS NOWRAP

Example:

 
LOADCAMVAR X, 0, Y, 0, 1 RANGE 0.0, 1.0, $global[0], 100 UNITS NOWRAP

Table:     LOADCAMVAR Command Arguments

Argument

Type

Description

<MasterAxis>

Axis

The axis that is set as the master axis.

<TableNumber>

32-bit signed integer

The table number, from 0 to 99.

<SlaveAxis>

Axis

The axis that is set as the slave axis.

<InterpolationType>

Other

The interpolation mode to use for camming. Set to 0 for linear splining or 1 for cubic splining.

<TrackingMode>

Other

Set to 0 to have the slave axis track the position feedback of the master axis. Set to 1 to have the slave axis track the master axis position. Set to 2 to have the slave axis track the auxiliary encoder channel of the master axis.

<MasterStart>

64-bit IEEE floating point

The master axis starting position.

<MasterDelta>

64-bit IEEE floating point

The master axis position delta.

<VariableStart>

32-bit signed integer

The variable where the table starts.

<VariableNumber>

32-bit signed integer

The number of points in the variables to read.

<Wrapping>

32-bit signed integer

(Optional)

If the master axis position is outside of the range of master axis values that you specified in the cam table, this argument specifies which wrapping condition the controller uses. Specify NOWRAP for the NOWRAP condition or WRAP for the WRAP condition. Refer to the <Wrapping> section for more information.

NOTE: This command applies only when you use camming motion.

Use the LOADCAMVAR command to load a cam table from variables that contain master axis position and slave axis position or slave axis velocity information and to configure the master and slave axes for the cam table.

The LOADCAMVAR command does not enable camming. Use the CAMSYNC Command to enable camming.

NOTE: Motion results can be different depending on if you specify a value of 0 for master position feedback or a value of 1 for master position command to the <TrackingMode> argument.

The LOADCAMVAR command is almost the same as the LOADCAMTABLE Command. These are the differences between each command:

  • For the LOADCAMVAR command, the cam table gets its points from the command arguments and variable values on the controller.
  • For the LOADCAMTABLE command, the cam table gets its points from a file on the PC. Thus the controller does not execute the LOADCAMTABLE command in real time.
  • For the LOADCAMVAR command, the format of this command keeps a constant distance between the master axis positions in the table.
  • For the LOADCAMTABLE command, the format does not keep a constant distance between the master axis positions in the table.

The controller uses the <VariableNumber>, <MasterStart>, and <MasterDelta> arguments to calculate the span of master axis positions in the cam table. These master axis positions are evenly spaced <MasterDelta> apart, starting at <MasterStart> and ending at (<MasterStart> + <MasterDelta> * (<VariableNumber> -1)).

You can find the first slave axis position, for master axis position = <MasterStart>, in the variable that you specify for the <VariableStart> command argument. You can find subsequent slave axis positions in the variables that immediately follow <VariableStart>. You must make sure that these variables are in your cam table.

If the master axis position on the controller is equal to the master axis position in a variable, the controller uses the value of that variable as the slave axis position. If the master axis position on the controller is between two of the master axis positions in the variables, the controller uses linear or cubic spline interpolation to calculate a slave axis position that is between the two corresponding slave axis positions in the variables.

If you use the UNITS keyword, the master positions are in distance units. If you use the COUNTS keyword, the master positions are in encoder counts.

NOTE: The SMC automatically disables feedforward (GainAff and GainVff) on the slave axis when you are camming or gearing from an external position or position feedback.
Interpolation Types: Linear or Cubic Splining

The <InterpolationType> argument specifies linear or cubic spline interpolation. Use a cubic spline for maximum smoothness. Use a linear spline for minimum rippling. Aerotech recommends that you use cubic spline interpolation because it makes smoother motion.

<Wrapping>

If the master axis position on the controller is outside of the range of master axis positions that you specified in the cam table, then one of the following conditions is true. The value that you specify for the <MasterAxis> argument in the LOADCAMTABLE or LOADCAMVAR command has an effect on which condition applies.

  • NOWRAP The master axis position on the controller is equal to the master axis position in the cam table that is nearest to the side of the table that the master axis position on the controller overrides. This condition is the default and is correct for most applications.
  • WRAP The master axis position on the controller is wrapped and is in the range of the master axis positions that you specified in the cam table. You can use this condition for some rotary applications.

If WRAP is true and the first and last slave axis positions that you specified in the cam table are not the same, the controller does a rollover of the slave axis position. This can be helpful in a rotary application if the first slave axis position is 0° and the last slave axis position is 360°.
