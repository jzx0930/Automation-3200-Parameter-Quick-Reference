DFS Command

Defines a subroutine in a program.

Syntax:

 
DFS <Subroutine>
<Code>
ENDDFS

Example:

 
DFS SquareGlobal
    $global[0] = $global[0] * $global[0]
ENDDFS

Table:     DFS Command Arguments

Argument

Type

Description

<Subroutine>

String

The name that you want to give the subroutine.

<Code>

Other

The block of commands in the subroutine.

The DFS and ENDDFS commands define the start and end of a subroutine block. Within this block is any number of program commands. Typically, this group of commands performs a particular task required multiple times during the execution of a program.

Subroutine names must be less than 32 characters long. If you enter a subroutine name that is 32 characters or longer, you get an error message.

Subroutines are called using the CALL command. The subroutine executes and returns to the next program line following the CALL command. A RETURN command is also available to return from the subroutine before the ENDDFS. If the RETURN command is omitted, the ENDDFS acts as a return.

You cannot return values from a subroutine. However, subroutines can freely access program, task, and global variables.

Subroutines can be passed parameters (refer to Call Arguments), however they cannot return values through these parameters (parameter passing is "call by value").

Subroutines can call themselves (recursion) or other subroutines. The CallStackSize task parameter limits the number of nested calls. There is no limit to the CallStackSize parameter except the limit that is imposed by the quantity of available memory on the controller.

When a program returns from a subroutine, the modal settings of the program are not restored. To save the modal status of the program, use the PUSHMODES command before you call the subroutine and use the POPMODES command after the subroutine.

For example

Say that in your program you issue an ABSOLUTE command and then you call a subroutine that executes the INCREMENTAL command. When the program returns from the subroutine, INCREMENTAL mode is active.

If you use the PUSHMODES and POPMODES commands, when the program returns from the subroutine, ABSOLUTE mode is active.

Note: If execution falls into the subroutine without a call, the compiler detects this condition and delivers a warning.
