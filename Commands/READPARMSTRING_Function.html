READPARMSTRING Function

Reads a string parameter from a parameter file.

Syntax:

 
<return> = READPARMSTRING <ParameterFileName>, <ParameterId>
 
<return> = READPARMSTRING <ParameterFileName>, <AxisOrTaskIndex>, <ParameterId>

Example:

 
$strglob[0] = READPARMSTRING "MyParameters.prma", PARAMETERID_UserString0

Table:     READPARMSTRING Function Arguments

Argument

Type

Description

<ParameterFileName>

String

The name of the parameter file to read from. The name must include the .prma extension.

<AxisOrTaskIndex>

32-bit signed integer

(Optional) The axis index or task on which to issue the command. Axis indices are zero-based integers. To specify the current task, use the $taskindex variable.

<ParameterId>

Other

Specifies the parameter to read from. Syntax is PARAMETERID_<Name of the Parameter>

RETURNS

String

The value of the parameter.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the READPARMSTRING function to read the value of a string parameter in the specified parameter file.

When you read the value of an axis or task parameter, you can specify which axis or task to read the parameter from by using the <AxisOrTaskIndex> argument. If you do not specify a value for this argument, it defaults to a value of zero. When you read the value of a system parameter, the <AxisOrTaskIndex> argument does not have an effect.

If you do not specify an absolute file path in the file name, the file path is relative to the program that is currently executing.

This function is a callback which causes the controller to send a message to the operating system. This function sets the Info Variables if Windows finds an error while it is opening, reading, or closing the specified parameter file. All other error conditions set the Task Error.
