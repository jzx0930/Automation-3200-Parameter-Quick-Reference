PULSE SET OUTPUT Command

Configures the pulse output mode.

Syntax:

 
PULSE SET <InputAxis> [ <InputAxis> ... ] OUTPUT <OutputAxis>
 
PULSE SET <InputAxis> [ <InputAxis> ... ] OUTPUT <OutputAxis>, <OutputMode>
 
PULSE SET <InputAxis> [ <InputAxis> ... ] OUTPUT <OutputAxis>, <OutputMode> SCALE <InputScaleFactor> [ , <InputScaleFactor> ... ]

Example:

 
PULSE SET Z OUTPUT X
PULSE SET X Y OUTPUT Y

Table:     PULSE SET OUTPUT Command Arguments

Argument

Type

Description

<InputAxis>

Axis

One or more input axes to track.

<OutputAxis>

Axis

The axis that will output the pulse signal.

<OutputMode>

32-bit signed integer

(Optional) Specifies the pulse output mode.

<InputScaleFactor>

64-bit IEEE floating point

(Optional) Specifies a scale factor to apply to each input axis when calculating the vector velocity.

Use the PULSE SET OUTPUT command to configure the pulse output mode.

The controller tracks the vector velocity by calculating the square root of the sum of squares of the components of each axis that you give with an <InputAxis> argument.

The controller tracks the distance moved by each input axis in encoder counts. If you want to track the input axes in units that are not encoder counts, use the optional <InputScaleFactor> argument to give a scale factor for each input axis. The scale factors should be in the same order as your input axes. The distance moved by each input axis is multiplied by the scale factor before the controller calculates the vector velocity that is sent to the output axis.

When you configure the pulse output to use one input axis, the vector sum will be the speed and direction of the input axis. When you configure the pulse output to use multiple input axes, the vector sum will be the vector velocity with no direction signal. You can track real or virtual input axes.

Issue the PULSE ON command to turn on the pulse output mode. Issue the PULSE OFF command to turn off the pulse output mode.

PULSE Output Timing

When pulse output mode is enabled, the controller tracks the input axes and generates a pulse output to the output axis. The controller sends encoder counts to the specified output axis at an 8 kHz rate. The table that follows shows the maximum rate for the pulse output for each drive type.

Drive Type Maximum Output Rate

HEX RC

Ndrive FCL, HLe, and HPe

Nmark GCL

Npaq

24,000,000 encoder counts per second

All HyperWire-based drives

25,000,000 encoder counts per second
