Display Command Category

Use the Display commands and functions to automate the Messages tab in the operator interface, to show messages to the user with simple button-selected responses, and to get data from the user. You can show warnings or information as illuminated text lamps in the Messages tab by using the MSGLAMP command. You can ask the user to select from a list of options by using the MSGMENU command.

For debugging, you can also use these commands and functions in Motion Composer. Motion Composer shows a message for each command that is used to automate the operator interface, so that the program executes correctly.

Example

DVAR $Width, $Height
MSGDISPLAY 1, "Performing program startup"; Type 1 message
MSGDISPLAY 2, "You can create long" " strings by concatenating them."

' Clear all type 1 messages from the display.
MSGCLEAR 1
$Width = 1
$Height = 2
MSGDISPLAY 3, "The square is: " $Width " millimeters, by " $Height
 " millimeters."

' Clear all messages from the display.
MSGCLEAR -1

Note: Motion Composer, CNC Operator Interface, Motion Simulator, or custom application must be open for this command to work. By default, Motion Composer, CNC Operator Interface, and Motion Simulator all try to register for these callbacks. The first application that registers will get the callbacks. The next application that tries to register will get an error. You can change the default behavior in Preferences of each application.

Commands

  • MSGBOX
  • MSGCLEAR
  • MSGDISPLAY
  • MSGFILESELECT
  • MSGHIDE
  • MSGINPUT
  • MSGLAMP
  • MSGMENU
  • MSGSHOW
