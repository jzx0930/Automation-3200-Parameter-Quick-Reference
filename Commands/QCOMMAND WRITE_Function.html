QCOMMAND WRITE Function

Writes motion profile data to the command queue of a task.

Syntax:

 
<return> = QCOMMAND WRITE <Task>, <Axis> [ <Axis> ... ], <PosArray>, <VelArray>, 
 
<return> = QCOMMAND WRITE <Task>, <Axis> [ <Axis> ... ], <PosArray>, <VelArray>, <AccArray>

Example:

 
$global[0] = QCOMMAND WRITE 1, X Y, $PosArray, $VelArray

Table:     QCOMMAND WRITE Function Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Task>

64-bit IEEE floating point

The task on which to execute the command.

<PosArray>

String

The position data, in counts, of the motion profile of the specified axes.

<VelArray>

String

The velocity data, in counts per millisecond, of the motion profile of the specified axes.

<AccArray>

64-bit IEEE floating point

(Optional) The acceleration data, in counts per millisecond2.

RETURNS

other

The variable that stores the return value. The value is either a string or a 64-bit IEEE floating point value.

Warning: If you use this command incorrectly, the axis can jerk and might cause damage to the equipment. Before you use this command on physical hardware, test it with simulated motion. Refer to Simulation of Motion.

Aerotech recommends that only advanced users use this command. Before you read this topic, read the following.

  • Kinematic Transformation Overview
  • QCOMMAND Features and Limitations
  • Transformed Motion example program

Use the QCOMMAND WRITE function with QCOMMAND READ to change or transform the motion profile in the command queue entry of a task. You can also use this function to create new motion profile data that adds new entries to the queue.

Note: This function applies only to motions that use the queue of a task, which include RAPID, LINEAR, CW, CCW, and PVT motion.

The function returns a value that identifies a one entry that is a specific motion point in the queue. The value is returned in the variable that you specify in the <return> argument. Each task has a separate command queue. Thus, if axes X and Y are the input axes and are being commanded on task 1, the value of the <Task> argument must be set to 1.

If you are using QCOMMAND WRITE with QCOMMAND READ to transform the motion profile, then the value returned by both functions must be the same. If the value is not the same, then the transformation is not writing over the record that it wrote, which causes unknown results. This problem can occur if you do not use CRITICAL START correctly.

If you are writing new profile data and using only QCOMMAND WRITE, then the value returned must be -1. If the value is not -1, then another command (normally LINEAR or CW) is also writing to the queue, which will cause unknown results.

The <Axis> arguments specify each axis that is used as the output of the transformation. If no valid queue entry is available for any of the specified axes, then the function returns a value of -1, the transforms do not occur and the <PosArray>, <VelArray>, <AccArray> variables are not assigned. If one or more of the axes that you specify are not enabled, then an error will occur.

The data in the <PosArray>, <VelArray>, and <AccArray> array variables is not packed. For example, if the axis mask is X Z and you have not changed the default axis names, then <PosArray>[0] is the X position and <PosArray>[2] is the Z position.

The velocities are instantaneous velocities, not average velocities. You cannot change the profile time (T1-T0) with this function.
