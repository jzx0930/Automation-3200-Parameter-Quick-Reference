$DO Function

Accesses the value of a specified digital output bit.

Syntax:

 
<return> = $DO[<Bit>].<Axis>
 
$DO[<Bit>].<Axis> = <Value>

Example:

 
$DO[4].X = 1
$global[0] = $DO[0].X

Table:     $DO Function Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Bit>

32-bit signed integer

The digital output bit to access the value of.

<Value>

32-bit signed integer

(Optional) The value to set the specified digital output bit to.

RETURNS

32-bit signed integer

(Optional) The value of the specified digital output bit.

Use the $DO function to set a digital output bit on a specified axis.

For information about how I/O is mapped to different drive types, refer to Specifying I/O Numbers.

Input and output bits are numbered starting with 0. For more information about digital I/O connections, refer to the hardware manual for your drive.

You can view input and output states in the Status Utility or plot them in Digital Scope.

Drives with opto-isolated inputs will show a value of 1 when the opto-coupler is energized.

To access more than one digital input or output at the same time, use the $WI or $WO functions.

Note: When you set a digital or analog output on a virtual axis, the corresponding input is set to the same value. For example, if you set $WO[0].X = 45, then $WI[0].X becomes 45.

When you use the $WO function to get the value of the digital output bits as a 32-bit value, Bit 0 is the least significant bit.

Latency Notes

Note: Digital outputs have a two to three millisecond latency before the outputs are actually set. They are however precisely synchronized with the commanded motion.

Note: When setting output, you must wait a minimum of six milliseconds before the corresponding read will report the same value. Refer to the example that follows.

$DO[6].X = 1
' Without this line, the next line does not assign $global[0] equal to 1.
DWELL .006           
							
$global[0] = $DO6.X
