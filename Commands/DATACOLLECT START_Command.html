DATACOLLECT START Command

Starts real-time data collection.

Syntax:

 
DATACOLLECT START <FileHandle>, <NumSamples>, <SampleTime>
 
DATACOLLECT START <FileHandle>, <NumSamples>, <SampleTime>, <Mode>
 
DATACOLLECT START <FileHandle>, <NumSamples>, <SampleTime>, <WaitTime>
 
DATACOLLECT START <FileHandle>, <NumSamples>, <SampleTime>, <Mode>, <WaitTime>

Example:

 
DATACOLLECT START $fileHandle, 1000, 1

Table:     DATACOLLECT START Command Arguments

Argument

Type

Description

<FileHandle>

32-bit signed integer

The file handle (reference number) of a file for which you want the controller to write data samples.

<NumSamples>

32-bit signed integer

Specifies the number of data samples to be collected.

<SampleTime>

32-bit signed integer

Specifies the sample time in milliseconds. To specify the sample time in sub-millisecond intervals, refer to Table: <SampleTime> Argument Values for Sub-millisecond Sample Times.

<Mode>

32-bit signed integer

(Optional) Specifies the data collection mode. To see the possible values, refer to Table: <Mode> Argument Values.

<WaitTime>

32-bit signed integer

(Optional) Specifies the time (in milliseconds) that the PC data collection thread sleeps while waiting for the next block of data to be uploaded. The default is 100 milliseconds.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the DATACOLLECT START command to trigger the real-time data collection that has been previously configured by DATACOLLECT ITEM commands. The data samples are written to a file on the PC.

The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value Description

0

(default)

Finite size data collection. Data collection stops after the <NumSamples> are collected. Then, you must issue the DATACOLLECT STOP command.

1

Infinite size data collection. Data is collected continuously until the controller executes a DATACOLLECT STOP command.

In this mode, <NumSamples> specifies the size of the queue that is used for continuous data collection. Data points are lost if the PC data collection thread cannot upload samples from the queue before the controller overwrites them. To find if this has occurred, look for a nonzero value of $info0 after you issue DATACOLLECT STOP.

For a solution to this problem, specify a value of <NumSamples> that is large enough to make sure that the PC has time to upload all data points before the controller overwrites them.

The following table shows the values you can use to specify sub-millisecond sample times.

Table:     <SampleTime> Argument Values for Sub-millisecond Sample Times

Value Sample Time

-2

Specifies a 1/2 millisecond sample time.

-4

Specifies a 1/4 millisecond sample time.

-8

Specifies a 1/8 millisecond sample time.

-24

Specifies a 1/24 millisecond sample time.

-48

Specifies a 1/48 millisecond sample time.

-192

Specifies a 1/192 millisecond sample time.
