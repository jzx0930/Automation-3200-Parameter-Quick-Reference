ANALOG SET Command

Configures the analog vector-tracking mode.

Syntax:

 
ANALOG SET <Axis> [ <Axis> ... ] <Mode> <VelocityPosition> OUTPUT <DACAxis>, <DACNumber>, <MinVoltage>, <MaxVoltage>
 
ANALOG SET <Axis> [ <Axis> ... ] <Mode> <VelocityPosition> UNITS OUTPUT <DACAxis>, <DACNumber>, <MinVoltage>, <MaxVoltage>

Example:

 
ANALOG SET X Y ANALOG_TRACK_CMD_VEL 50 OUTPUT X 0, 0, 10

Table:     ANALOG SET Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Mode>

Other

The vectorial quantity that the analog voltage tracks. To see the possible values, refer to Table: <Mode> Argument Values.

<VelocityPosition>

32-bit signed integer

The vector position or velocity value at which the output voltage equals the <MaxVoltage> argument.

<DACAxis>

Axis

The axis that contains the analog output. On the GL4 and the Nmark GCL, you can use only the first axis. On all other drives, you can use any axis.(1)

<DACNumber>

32-bit signed integer

Specifies which analog output to use. You must specify a value of 0 or 1.(1)

<MinVoltage>

64-bit IEEE floating point

Specifies the minimum voltage for the analog output.

<MaxVoltage>

64-bit IEEE floating point

Specifies the maximum voltage for the analog output.

(1) For example, if you want to use analog output 3 on an Npaq, you must set the <DACAxis> argument to Y (the second axis of the Npaq) and set the <DACNum> argument to 1.

Use the ANALOG SET command to configure the analog vector-tracking mode. The tracked value is calculated as the square root of the sum of squares of the components of each axis that is specified in an <Axis> argument. The tracked value is always positive or zero. In position tracking modes, the distance is relative to the axis position value at which the ANALOG ON command is issued. Velocity is always relative to zero.

The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value

Description

ANALOG_TRACK_CMD_POS

The analog voltage tracks the command position of the axes.

ANALOG_TRACK_FDBK_POS

The analog voltage tracks the feedback position of the axes.

ANALOG_TRACK_PROG_POS

The analog voltage tracks the programmed position of the axes.

ANALOG_TRACK_CMD_VEL

The analog voltage tracks the command velocity of the axes.

ANALOG_TRACK_FDBK_VEL

The analog voltage tracks the feedback velocity of the axes.

ANALOG_TRACK_PROG_VEL

The analog voltage tracks the programmed velocity of the axes.

The encoder value is converted to a voltage using the <MinVoltage>, <MaxVoltage>, and <VelocityPosition> arguments.

<VelocityPosition> can be a velocity or position. The output voltage equals the <MinVoltage> argument value when the tracked velocity value is zero or when the tracked position value is equal to the axis position where the ANALOG ON command was issued.

For Example

If you use ANALOG_TRACK_FDBK_POS on Axis X only, and X=3 at the time that you issue the ANALOG SET command, then the <MinVoltage> analog output occurs at X=3.

The output voltage is clamped by the <MinVoltage> and <MaxVoltage> argument settings. The <VelocityPosition>, <MinVoltage>, and <MaxVoltage> arguments can be changed dynamically while this mode is active. All of the other arguments change only when the ANALOG ON command is issued.

You can use the optional UNITS keyword to specify that the positions or velocities are in distance units instead of encoder counts. The velocities are interpreted in the current time base (distance units per time base) instead of encoder counts per millisecond.
