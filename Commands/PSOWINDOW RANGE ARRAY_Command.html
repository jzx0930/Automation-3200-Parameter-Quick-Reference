PSOWINDOW RANGE ARRAY Command

Specifies the window range based on data in the PSO array.

Syntax:

 
PSOWINDOW <Axis> <WindowNumber> RANGE ARRAY <StartIndex>, <NumElements>
 
PSOWINDOW <Axis> <WindowNumber> RANGE ARRAY <StartIndex>, <NumElements> EDGE <EdgeCode>

Example:

 
PSOWINDOW X 1 RANGE ARRAY 5 10 EDGE 1

Table:     PSOWINDOW RANGE ARRAY Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<WindowNumber>

32-bit signed integer

The window to configure. Set to 1 or 2.

<StartIndex>

64-bit IEEE floating point

The index of the first array value to load into the window.

<NumElements>

64-bit IEEE floating point

The number of values to sequentially load into the window.

<EdgeCode>

64-bit IEEE floating point

(Optional) An optional argument that lets the controller update the PSO window range based on how the axis exits the PSO window.

Hardware: This command applies only when used with the GL4, the Ndrive FCL, FLS, HLe, HPe, ML, or QLe, the Nmark CLS or GCL, the Npaq, the Nservo, the XC2, the XC4, the XC4e, the XL4s, or the XR3. It does not apply to the Npaq MR.

NOTE: If you are doing single-axis tracking, configure only window 1. If you are doing multi-axis tracking, you must configure both window 1 and window 2.

Use the PSOWINDOW RANGE ARRAY command to specify window ranges that are based on data in the PSO array. The window-range values in the array are specified in encoder counts and must be in the range of ±2,147,483,647 encoder counts for the Ndrive HLe, HPe, ML, and QLe, the Nmark CLS and GCL, the Npaq, the Nservo, the XC2, the XC4, the XC4e, and the XR3. For all other drives, these values must be in the range of ±8,388,607 encoder counts. Each pair of array elements corresponds to the low and high limits of the PSO window range.

To load window-range values that you specify for the PSOWINDOW RANGE Command into the PSO array, use the ARRAY WRITE Command. First load the value of the <LowLimit> argument into the PSO array. Then load the value of the <HighLimit> argument into the PSO array.

If you want your AeroBasic application to specify window ranges in user units, you must manually convert the values of the <LowValue> and <HighValue> arguments into counts. When you write these values to the PSO array by using the ARRAY WRITE Command, this command rounds the values to the nearest number of counts.

Because the ARRAY WRITE Command rounds these values, the PSO window might not occur when you expect it to occur. There might be a small difference between when you specify the PSO window to occur and when the PSO window actually occurs. If this command rounds up the value, the window range becomes longer by a maximum of a 1/2 count. If this command rounds down the value, the window range becomes shorter by a maximum of a 1/2 count.

Each time the axis exits the PSO window that is currently active, the controller loads more window-range values into the PSO hardware. After the controller downloads all the window-range values, it stops updating the window ranges. The last window range stays active.

Optional EDGE Keyword and <EdgeCode> Argument

The optional EDGE keyword and <EdgeCode> argument let the controller update the window range only when the axis exits, in a specified direction, the PSO window that is currently active. This table shows the possible values of the <EdgeCode> argument and how they have an effect on how the controller updates the PSO window range.

Table:     <EdgeCode> Argument Values

Setting

Description

0

The axis exits from the PSO window in a positive or negative direction.

1

The axis exits from the PSO window in a positive direction.

2

The axis exits from the PSO window in a negative direction.
