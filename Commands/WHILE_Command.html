WHILE Command

Executes a set of code while a specified condition is true.

Syntax:

 
WHILE <Condition>
<Code>
ENDWHILE
 
WHILE <Condition> DO
<Code>
ENDWHILE

Example:

 
WHILE $OUTER < 10 DO
    $global[0] = $global[0] + 1
ENDWHILE

Table:     WHILE Command Arguments

Argument

Type

Description

<Condition>

Other

The condition to examine.

<Code>

Other

The code to execute while the specified condition is true.

The WHILE command executes a group of program lines while a specific condition is true (!= 0). When the WHILE command is executed, the controller examines the specified conditional expression. If the conditional expression is true, then the controller executes the program lines that are between the WHILE and ENDWHILE commands. Then, the conditional expression is examined again. This process occurs again and again until the condition is false (zero). When the condition is false, the program flow goes to the command that immediately follows the ENDWHILE command. If the condition is false the first time it is examined, the program lines in the WHILE loop are not executed.

You can nest WHILE loops in each other, limited only by the available memory.

You can use the DO keyword with the WHILE command. It does not change how the command is executed.

Example

' Define the variables OUTER and INNER.
DVAR $OUTER $INNER

' Initialize the outer loop counter to zero.
$OUTER = 0

' Execute this loop while the variable OUTER is less than 10.
WHILE $OUTER < 10 DO

    ' Increment the loop counter.
    $OUTER = $OUTER + 1

' End of the WHILE construct.
ENDWHILE

' Execute this loop while the variable OUTER is greater than or equal to 1.
WHILE $OUTER >= 1 DO
				
    ' Initialize the inner loop counter.
    $INNER = 0

    ' You can nest WHILE loops.
    WHILE $INNER <= 5 DO

        INNER = $INNER + 1

        ' Conditional commands can be in WHILE loops.
        IF INNER == 5 THEN
            $OUTER = $OUTER -1
        ENDIF
    ENDWHILE
ENDWHILE
