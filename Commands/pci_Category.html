PCI Bus Command Category

Use the PCI Bus commands and functions to read and write to cards on the PCI bus from within a program. Before you can read or write to a card, you must open the card for communication. You must close the card when communication is complete.

Note: Do not give other Windows applications access to the cards that you open with the PCI OPEN function or the PCI Bus commands will not work correctly.

Table:     PCI Bus Command Descriptions

Command Description

PCI OPEN

Open communications to a card on the PCI bus.

PCI CLOSE

Close communications to a card on the PCI bus.

PCI READ BYTE

Read a byte of data from a card on the PCI bus.

PCI WRITE BYTE

Write a byte of data to a card on the PCI bus.

PCI READ WORD

Read a word of data from a card on the PCI bus.

PCI WRITE WORD

Write a word of data to a card on the PCI bus.

Example

' Declare the variables.
DVAR $ByteValue, $CardID
' Open communication with the PCI card with a vendor ID of 0x135E and device ID of 0x8008.
$CardID = PCI OPEN 0H135E 0H8008 0
' Read from the PCI card.
$ByteValue = PCI READ BYTE $CardID 0 1
' Write to the PCI card.
PCI WRITE BYTE $CardID 0 3 0HFF 0H80
' Close communication with the card.
PCI CLOSE $CardID

Commands

  • PCI CLOSE
  • PCI OPEN
  • PCI READ BYTE
  • PCI READ WORD
  • PCI WRITE BYTE
  • PCI WRITE WORD
