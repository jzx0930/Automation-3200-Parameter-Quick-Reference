FARCALL Command

Calls a subroutine in another program or executes a called program as a subroutine.

Syntax:

 
FARCALL <ProgramName>
 
FARCALL <ProgramName>, <Label>
 
FARCALL <ProgramName>, <Subroutine>
 
FARCALL <ProgramName>, <ArgumentList>
 
FARCALL <ProgramName>, <Label>, <ArgumentList>
 
FARCALL <ProgramName>, <Subroutine>, <ArgumentList>

Example:

 
FARCALL "prog2.pgm" "MYSUB"
FARCALL "" "MYSUB"

Table:     FARCALL Command Arguments

Argument

Type

Description

<ProgramName>

String

The name of the program to which the controller transfers program execution.

<Label>

Label

(Optional) The label to which the controller transfers program execution.

<Subroutine>

Label

(Optional) The name of the subroutine to which the controller transfers program execution.

<ArgumentList>

Label

(Optional) A mixture of comma-separated integer, double, and string arguments. The controller concatenates these arguments into one text string.

This command calls a subroutine in another program or executes the called program as a subroutine. After the subroutine executes, or a RETURN command is executed, program flow returns to the next line in the calling program. An END PROGRAM within a called program terminates execution and control does not return to the calling program.

The <ProgramName>, <Subroutine>, and <Label> arguments can be Global String variables and define the program execution entry point. If you specify "" for the <ProgramName> argument, the controller uses the label in the calling program. You can also use the CALL Command to do this operation.

Note: Program names must be less than 187 characters long. Refer to Program Names for more information.

Note: Subroutine names must be less than 32 characters long. Refer to Subroutines Overview for more information.

For information about using labels for the <Label> argument, refer to Labels. For information about using a subroutine name for the <Subroutine> argument, refer to DFS Command.

If you do not specify the <Label> or <Subroutine> argument or if you specify "" for one of these arguments, execution starts at the first line in the called program. Programs can call themselves or other programs. There is no limit to the nesting of calls, except for the limits imposed by the available memory and by the CallStackSize task parameter.

Before the controller executes a program called by the FARCALL command, you must download that program onto the controller. When the program is on the controller, it prevents an invalid file name or path error from occurring. Download the program at startup by using the Program Automation feature. Make sure that you set the Mode for the execution type to Download.

After a FARCALL is made to a program, the program variables of the original program are no longer available. Each program has its own set of program variables, even if a program calls another version of itself. Call arguments or global variables can be used for passing data to a called program. Call arguments can be passed to a program called as a subroutine. For more information, refer to Subroutine Arguments.

When a program returns from a subroutine, the modal settings of the program are not restored. To save the modal status of the program, use the PUSHMODES command before you call the subroutine and use the POPMODES command after the subroutine.

For example

Say that in your program you issue an ABSOLUTE command and then you call a subroutine that executes the INCREMENTAL command. When the program returns from the subroutine, INCREMENTAL mode is active.

If you use the PUSHMODES and POPMODES commands, when the program returns from the subroutine, ABSOLUTE mode is active.

The program variables declared within each program are initialized to zero on the first FARCALL into any subroutine within that program. On subsequent calls into the subroutines within that file, the program variable values are retained.

Example

' Main.Pgm
FARCALL "c:\prog2.pgm" "entry3"

' c:\Prog2.pgm
' The controller does not execute this line.
$global[0] = 1
:entry3
' The controller executes this line. 
global[0] = 2
