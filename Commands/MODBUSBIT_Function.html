MODBUSBIT Function

Provides access to a Modbus bit set.

Syntax:

 
MODBUSBIT <BitName>, <BitNumber>, <Value>
 
MODBUSBIT <ConnectionNumber>, <BitName>, <BitNumber>, <Value>
 
<return> = MODBUSBIT(<BitName>, <BitNumber>)
 
<return> = MODBUSBIT(<ConnectionNumber>, <BitName>, <BitNumber>)
 
MODBUSBIT <Axis>, <BitName>, <BitNumber>, <Value>
 
<return> = MODBUSBIT(<Axis>, <BitName>, <BitNumber>)

Example:

 
MODBUSBIT X, MasterOutputBits, 4, 1
$Var = MODBUSBIT(0, MasterOutputBits, 7)

Table:     MODBUSBIT Function Arguments

Argument

Type

Description

<BitName>

Other

The bit set to access. Either MasterInputBits, SlaveInputBits, MasterOutputBits, or SlaveOutputBits. The input bits have read-only access, and the output bits have read and write access.

<BitNumber>

32-bit signed integer

The bit number in the set. This argument can be an expression and must be a positive integer.

<Value>

32-bit signed integer

(Optional) The value to write to the bit. Either 0 or 1.

<ConnectionNumber>

32-bit signed integer

(Optional) Identifies the Modbus connection to use. Specify the axis name on which to use the Modbus register set. Specify a number from 0 - 15 to use the corresponding PC Modbus register set (1 - 15 only available with the Professional option).

<Axis>

Axis

(Optional) The name of the axis for drive Modbus.

RETURNS

32-bit signed integer

(Optional) The stored value.

Obsolete. This command will be removed in a future release. Refer to Fieldbus Mapping Dialog for more information.

Use the MODBUSBIT command to read or write 1-bit integer values to the Modbus register on the drive or the PC.

Note: All drive Modbus I/O on the Npaq is referenced by the first axis within the Npaq. This does not apply to the Npaq MR.
