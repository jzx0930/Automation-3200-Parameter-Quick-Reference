INSTANTACCEL Command

Immediately change from one program velocity to another.

Syntax:

 
INSTANTACCEL <MotionCommand>

Example:

 
INSTANTACCEL LINEAR X5 F500

Table:     INSTANTACCEL Command Arguments

Argument

Type

Description

<MotionCommand>

Other

The motion command after which the stop occurs.

Warning: This command is obsolete and will only be supported for a limited time. This Help topic is supplied only for compatibility with previous software versions. To use instantaneous ramping, issue the RAMP RATE command with a value of 0.0.

Use the INSTANTACCEL command to immediately change from one program velocity and a new program velocity. It disables automatic acceleration and causes the motion to instantaneously accelerate or decelerate to the new programmed feedrate at the beginning of a coordinated move (INSTANTACCEL applies only to LINEAR, CW, and CCW moves). Use this command with the TrajectoryIIRFilter axis parameter.

Example

' Set the velocity to F50.
LINEAR X5 F50
' Immediately change the velocity from F50 to F500.
INSTANTACCEL LINEAR X5 F500

The figure (Velocity Profile with INSTANTACCEL) displays a velocity profile with the INSTANTACCEL command. Without this command, acceleration is performed based upon the settings of the acceleration mode. INSTANTACCEL applies only to coordinated motion, not RAPID or asynchronous motion. An INSTANTACCEL must be on every line if instantaneous acceleration or deceleration is required on each line.

Note: INSTANTACCEL overrides VELOCITY ON/VELOCITY OFF for the current program line.

Example

' Enable incremental programming mode.
INCREMENTAL
' Set the time base to seconds.
SECONDS
' Move the X axis 1 unit at 100 units/s.  
INSTANTACCEL LINEAR X1 F100
' Move the X axis 1 unit at 200 units/s.
INSTANTACCEL LINEAR X1 F200
' Move the X axis 1 unit at 100 units/s.
INSTANTACCEL LINEAR X1 F100
Figure: Example of a Velocity Profile with INSTANTACCEL

Note: If motion is executing in the INSTANTACCEL mode and is placed into feedhold, the deceleration to zero is instantaneous. This also applies to MFO adjustments and the Abort button during a INSTANTACCEL mode move.

Note: The acceleration cannot occur in a period that is less than the value specified by the MotionUpdateRate parameter.
