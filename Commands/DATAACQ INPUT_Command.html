DATAACQ INPUT Command

Specifies the data element collected when a trigger occurs.

Syntax:

 
DATAACQ <Axis> 1 INPUT <SourceSignal>

Example:

 
DATAACQ X 1 INPUT 1

Table:     DATAACQ INPUT Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<SourceSignal>

32-bit signed integer

The source signal to be collected.

Hardware: This command does not apply when used with the Ndrive FCL, FLS, QL, or QLe, the Nmark SSaM, or the Nstep.

Hardware: When you use a multi-axis drive such as the Npaq or Nservo, all axes in the drive share the same array space. You can specify any axis in the drive for the <Axis> argument.

Use the DATAACQ INPUT command to specify the data element that is collected when a trigger occurs. The <SourceSignal> argument selects the data element and has different meanings for each drive type.

These DATAACQ commands must be issued in the following order.

  1. DATAACQ TRIGGER
  2. DATAACQ INPUT
  3. DATAACQ ARRAY
  4. DATAACQ OFF (optional)

Note: If the VelocityFeedbackType parameter is set to Analog Input, you cannot use the DATAACQ INPUT command to capture analog inputs.

The sections that follow show the possible values of the <SourceSignal> argument for each drive type.

<SourceSignal> Argument Values for Npaq

At each sample trigger, you can collect data from multiple <SourceSignal> arguments (refer to the following table). To collect multiple quantities, issue the DATAACQ INPUT command once for each sample type you want to collect. All values will be put into the same drive array, which is shared across all axes of the Npaq.

When you specify multiple inputs on the Npaq by issuing the DATAACQ INPUT command multiple times, the value of the <SourceSignal> argument of each DATAACQ INPUT command determines the order in which the interleaved data is returned. The order in which you issue the DATAACQ INPUT commands does not affect the order of the data that is returned. Data is always returned from the lowest <SourceSignal> argument to the highest <SourceSignal> argument.

Data collection rate is 16,000 samples per second (except as noted).

<SourceSignal> Argument Values for Npaq MR, Ndrive ML, and Ndrive MP

The Npaq MR contains multiple ML or MP drives. To get the data collection rate, refer to the ML or MP values. The data collection rates are as follows:

  • For an Ndrive ML, the data collection rate is 32,000 samples per second.
  • For an Ndrive MP, the data collection rate is 8,000 samples per second.
