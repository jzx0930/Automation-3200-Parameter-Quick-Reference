CALLDLL Function

Calls a function in a custom dynamic link library (DLL).

Syntax:

 
<return> = CALLDLL <FileName>, <Function>
 
<return> = CALLDLL <FileHandle>, <Function>
 
<return> = CALLDLL <FileName>, <Function>, <ArgumentList>
 
<return> = CALLDLL <FileHandle>, <Function>, <ArgumentList>
 
CALLDLL <FileName>, <Function>
 
CALLDLL <FileHandle>, <Function>
 
CALLDLL <FileName>, <Function>, <ArgumentList>
 
CALLDLL <FileHandle>, <Function>, <ArgumentList>

Example:

 
CALLDLL "ExampleDLL.dll", "ExampleFunctionName", $ExampleArg1, $ExampleArg2
$global[0] = CALLDLL "ExampleDLL.dll", "ExampleFunctionName", $ExampleArg1, $ExampleArg2

Table:     CALLDLL Function Arguments

Argument

Type

Description

<FileName>

String

(Optional) The name and path of the .dll file. Refer to File Names for more information.

<FileHandle>

32-bit signed integer

(Optional) The file handle (reference number) of the .dll file that you want to use. This file handle is returned by the OPENDLL function.

<Function>

String

The name of the function that you want to call.

<ArgumentList>

Other

(Optional) A mixture of comma-separated integer, double, and string arguments. The controller concatenates these arguments into one text string.

RETURNS

other

(Optional) The double or string value returned from the function that you call.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the CALLDLL function to call a function in a custom dynamic link library (DLL). There are different ways to use CALLDLL function.

To call one function in a .dll file

  1. Use the CALLDLL function to load the .dll file, call the function, and close the .dll file.
  2. Use the <FileName> argument and specify the name of the .dll file that you want to use.

To call more than one function in the same .dll file

This way is more efficient because the .dll file is not opened and closed each time you call a function in the same file.

  1. Use the OPENDLL function to load the .dll file and get the file handle.
  2. Call each function by using a CALLDLL function. Use the <FileHandle> argument and specify the value that was returned by the OPENDLL function.
  3. Use the CLOSEDLL command to close the file.

You must specify the <Function> argument. You can get the optional arguments to use in <ArgumentList> argument by using A3200CallbackArgs C library functions. Refer to A3200 Programming Help for more information.

Note: The CALLDLL command can only call a function that is in a native Win32 DLL. You can call other DLLs or COM objection from the Win32 DLL.

When the CALLDLL function executes, the AeroBasic program stops on the CALLDLL line and stays there until the DLL tells the program to continue. Make sure that the function that you call tells the program when to continue. You can also return double or string data from the called function into program variables. Refer to A3200CallbackReturn C library functions in A3200 Programming Help for more information.

The function called by the CALLDLL function must have a special prototype. Refer to Writing Custom CALLDLL Handling Functions with the C Library for more information.

Example

DVAR $ExampleQuantity1
DVAR $ExampleQuantity2

$ExampleQuantity1 = CALLDLL "ExampleDLL.dll", "GetExampleQuantity", 1
$ExampleQuantity2 = CALLDLL "ExampleDLL.dll", "GetExampleQuantity", 2

MSGDISPLAY 1, $ExampleQuantity1
MSGDISPLAY 1, $ExampleQuantity2

NOTE: Aerotech does not support C++/CLR DLLs. In the Project Settings of your Visual Studio project, make sure that Common Language Runtime Support is set to No Common Language Runtime Support.
