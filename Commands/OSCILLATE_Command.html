OSCILLATE Command

Generates sinusoidal oscillation on an axis.

Syntax:

 
OSCILLATE <Axis>, <Distance>, <Frequency>, <Cycles>
 
OSCILLATE <Axis>, <Distance>, <Frequency>, <Cycles>, <NumIterations>

Example:

 
OSCILLATE X, 25, 1, 5

Table:     OSCILLATE Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Distance>

64-bit IEEE floating point

Defines the peak-to-peak displacement of the motion. The amplitude of the oscillate sequence is 1/2 of the peak-to-peak displacement. The arithmetic sign specifies the initial direction of the motion.

<Frequency>

64-bit IEEE floating point

The frequency of oscillation, in hertz. You must specify a frequency that is greater than 0 Hz and less than or equal to 500 Hz. Because the controller generates sinusoidal positions and velocities at a 1 kHz rate, less accurate motion might occur if you specify higher oscillation frequencies to this command.

<Cycles>

32-bit signed integer

The number of sinusoidal cycles to generate during the oscillate sequence. Each cycle is one full sinusoidal cycle.

<NumIterations>

32-bit signed integer

(Optional) The number of oscillate sequences to execute. After the controller executes the first oscillate sequence, the following changes occur for the subsequent execution of each of the other sequences. The amplitude of the motion decreases to 1/4 of its previous value, the frequency is doubled, and the number of cycles is doubled. If you do not specify a value for this argument, it defaults to a value of one.

Use the OSCILLATE command to generate sinusoidal motion on a single axis. This command is asynchronous and program execution continues immediately to the next program line. The controller does not wait for the motion to complete before continuing on to the next command.

Use the ABORT command to stop the axis motion before the specified number of cycles completes.

This command is an asynchronous motion command, which means that program execution immediately continues to the next program line before motion is complete. Use the WAIT MOVEDONE or WAIT INPOS command to wait for motion to complete.
