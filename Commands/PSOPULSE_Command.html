PSOPULSE Command

Configures the pulse sequence that is used for PSO.

Syntax:

 
PSOPULSE <Axis> TIME <TotalTime>, <OnTime>
 
PSOPULSE <Axis> TIME <TotalTime>, <OnTime> CYCLES <NumCycles>
 
PSOPULSE <Axis> TIME <TotalTime>, <OnTime> DELAY <DelayTime>
 
PSOPULSE <Axis> TIME <TotalTime>, <OnTime> CYCLES <NumCycles> DELAY <DelayTime>

Example:

 
PSOPULSE X TIME 10, 10
PSOPULSE Y TIME 1000, 500 CYCLES 10

Table:     PSOPULSE Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<TotalTime>

64-bit IEEE floating point

The total time of generated pulse in microseconds.

<OnTime>

64-bit IEEE floating point

The time of the cycle when the PSO output is in the ON state.

<NumCycles>

64-bit IEEE floating point

(Optional) The number of pulses to generate in a single pulse event.

<DelayTime>

64-bit IEEE floating point

(Optional) The quantity of time to delay between a fire event and the laser output.

Hardware: This command does not apply when used with the Nmark SSaM or the Nstep.

Use the PSOPULSE command to configure the PSO pulse generator. The output of the pulse generator will be used as the PSO output when using the PSOOUTPUT PULSE, PSOOUTPUT PULSE WINDOW MASK, or PSOOUTPUT PULSE BIT MASK commands.

The pulse generator can be configured to output a single pulse or multiple pulses when each PSO firing event occurs. Firing events can be generated based on distance tracking when using a PSOTRACK command or under manual control using the PSOCONTROL FIRE command. Firing events that occur during a currently active pulse are ignored, which can occur if firing events occur faster than the <TotalTime> argument.

The frequency of the pulse generator output is defined by the <OnTime> and <TotalTime> arguments. The resolution of these values is 0.02 microseconds, with a minimum value of 0.04 microseconds. The maximum differs depending on the type of drive being used, as shown in the following table.

If the optional CYCLES keyword is omitted, a single pulse is generated for each firing event. When using the CYCLES keyword, the <NumCycles> argument specifies the number of output pulses generated for each firing event and must be less than 16,777,215.

If the optional DELAY keyword is omitted, the output of the pulse generator is not delayed. When using the DELAY keyword, the <DelayTime> argument specifies the time, in μsec, that the output of the pulse generator is delayed when a firing event occurs.

Hardware: The DELAY keyword applies only to the GL4, the Ndrive FCL, FLS, HLe, HPe, ML, and QLe, the Nmark CLS and GCL, the XC2, the XC4, the XC4e, the XL4s, and the XR3.
