PROGRAMCONTROL Command

Restricts (or allows) the program operations that can be done on a task.

Syntax:

 
PROGRAMCONTROL RESTRICT
 
PROGRAMCONTROL <Task> RESTRICT
 
PROGRAMCONTROL ALLOW
 
PROGRAMCONTROL <Task> ALLOW

Example:

 
PROGRAMCONTROL RESTRICT

Table:     PROGRAMCONTROL Command Arguments

Argument

Type

Description

<Task>

TaskID

(Optional) The task on which to restrict or allow program operations. If you do not specify a task, the command uses the current task.

PROGRAMCONTROL RESTRICT Command

Use the PROGRAMCONTROL RESTRICT command to restrict the program operations that you can do on a task. This command prevents the execution of a critical task from being interfered with by the buttons of the application that you are using or by other tasks. Aerotech recommends that you use this command with background tasks that are always running to monitor status and to handle errors, faults, and safety conditions.

After the PROGRAMCONTROL RESTRICT command is executed on a task, the following commands and features are disabled on that task. When these commands and features are given the task argument of the restricted task, and are executed from a different task or from the library, they fail to execute and cause an error.

Restricted Commands

EXECUTEMODE Command

FEEDHOLD Command

INTERRUPTMOTION ON/OFF Command

MFO Command

MSO Command (CNC Option only)

PROGRAM PAUSE Command

PROGRAM RESET Command

PROGRAM STOP Command

RETRACE Command (CNC Option only)

Features

Breakpoints

Immediate Commands

NOTE: When you issue the PROGRAMCONTROL RESTRICT command on a task, all of the commands that are disabled on that task can still be executed by the AeroBasic program that is running on the restricted task.

Task Restrictions

The task restrictions are removed when one or more of the following occurs.

  • You issue the PROGRAMCONTROL ALLOW command from a different task or from the AeroBasic program that is running on the restricted task.
  • The AeroBasic program that is running on the restricted task completes its execution.
  • The AeroBasic program that is running on the restricted task executes an END PROGRAM Command or a PROGRAM STOP Command.
  • A task error occurs on the restricted task.
  • The AeroBasic program that is running on the restricted task causes an axis fault.
  • You reset the controller.

To make a task that cannot be stopped, you must do all of the following.

  • Issue the PROGRAMCONTROL RESTRICT command.
  • Set the TaskTerminationAxes Parameter to 0 to prevent an axis fault from causing a task error.
  • In the restricted program, use the ONGOSUB Command to configure a handling routine to recover from task errors.

Refer to the Restricted Program Example for more information.

NOTE: To keep the program that is on the restricted task running and to keep the task restrictions, issue the ONGOSUB Command. The Restricted Program Example for an example program that shows you how to do this.
