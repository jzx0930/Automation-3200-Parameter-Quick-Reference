QCOMMAND RESET Command

Makes the pre-transformation position commands the same as the actual post-transformation position commands that were generated from a kinematic transformation task

Syntax:

 
QCOMMAND <Axis> [ <Axis> ... ] RESET

Example:

 
QCOMMAND X Y RESET

Table:     QCOMMAND RESET Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

Warning: If you use this command incorrectly, the axis can jerk and might cause damage to the equipment. Before you use this command on physical hardware, test it with simulated motion. Refer to Simulation of Motion.

Aerotech recommends that only advanced users use this command. Before you read this topic, read the following.

  • Kinematic Transformation Overview
  • QCOMMAND Features and Limitations
  • Transformed Motion example program

Use the QCOMMAND RESET command to make the pre-transformation position commands the same as the actual post-transformation position commands that were generated from a kinematic transformation task. Use it only with kinematic transform. It is not related to other transformations. Refer to the QCOMMAND WRITE and QCOMMAND READ functions.

A kinematic transformation task transforms the position commands that are supplied in the user program into possibly different position commands that are delivered to the drive. After the kinematic transformation task stops, the pre-kinematic transform position commands are different from the post-kinematic transform position commands. Because relative motion is measured from the pre-kinematic transformation coordinates, relative motion after the kinematic transformation task stops can cause motion to incorrect targets.

The QCOMMAND RESET command makes the pre-transformation and the post transformation position commands the same. It does not execute any motion. You can get the pre-kinematic transformation positions with the ProgramPositionCommand status item. You can get the post-kinematic transform positions with the PositionCommand status item. Refer to AXISSTATUS for information about these status items.

Warning: You must use this command after you stop the kinematic transformation program or unexpected jerks in the motion occur.
