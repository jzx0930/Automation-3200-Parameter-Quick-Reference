ARRAY READ Command

Transfers drive array values into the specified controller array variables.

Syntax:

 
ARRAY <Axis> READ <ControllerVariable> <StartIndex> <NumberOfElements>
 
ARRAY <Axis> READ <ControllerVariable> <StartIndex> <NumberOfElements> FAST

Example:

 
ARRAY X READ $global[21] 20 34

Table:     ARRAY READ Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<ControllerVariable>

64-bit IEEE floating point variable

The controller variable on which to write data that is read from the drive array.

<StartIndex>

32-bit signed integer

The starting index of the drive array from which the data is read.

<NumberOfElements>

32-bit signed integer

The number of elements to be read.

Hardware: When you use a multi-axis drive such as the Npaq or Nservo, all axes in the drive share the same array space. You can specify any axis in the drive for the <Axis> argument.

Use the ARRAY READ command to transfer drive array values into specified controller array variables. The drive array values are 32-bit signed integers. They will be converted to 64-bit IEEE floating point variables when they are transferred into the controller array variables. You can use ARRAY READ to get data that is collected by the DATAACQ command. For more information about the drive array, refer to Drive Array Overview.

When you specify multiple inputs on the Npaq or Nservo by issuing the DATAACQ INPUT command multiple times, the value of the <SourceSignal> argument of each DATAACQ INPUT command determines the order in which the interleaved data is returned. The order in which you issue the DATAACQ INPUT commands does not affect the order of the data that is returned. Data is always returned from the lowest <SourceSignal> argument to the highest <SourceSignal> argument.

The ARRAY READ command will not complete until it retrieves all of the specified values from the drive. When you are not using the FAST qualifier, the controller retrieves 500 values per second. To increase the number of values the controller can retrieve, use the FAST qualifier as shown in the following table.

FAST uses the full bandwidth of communications. Do not use FAST mode while you do other operations on the drive (such as homing). You cannot use the FAST qualifier while the controller is collecting data through Digital Scope or the DATACOLLECT commands.
