VELOCITY Command

Turns on and turns off the velocity profiling mode.

Syntax:

 
VELOCITY ON
 
VELOCITY OFF

Example:

 
VELOCITY ON
VELOCITY OFF

Note: This feature causes the controller to activate lookahead. For more information, refer to Program Block Lookahead Overview.

Use the VELOCITY command to blend multiple coordinated motion commands into one continuous motion path. In this velocity profiling mode, the controller does not decelerate to zero between consecutive coordinated moves. During the move sequence you can change the feedrate (velocity) by using the F command (or the E command for dependent axes) if necessary. The axes will increase or decrease in speed in a coordinated way such that the programming path is maintained.

Activate velocity profiling mode with a VELOCITY ON command. VELOCITY on is modal and remains in effect until VELOCITY OFF is issued. The state of the modal command is indicated by the Motion Continuous bit of the Task Mode status item. Use the FORCEDDECEL command on a coordinated motion line to override the VELOCITY ON setting only for that line.

If you are in velocity profiling mode and you issue a VELOCITY OFF command, the controller will decelerate to zero velocity after the last coordinated move before the VELOCITY OFF command. If two coordinated moves are separated by a command that pauses program execution (for example, DWELL), the controller forces a deceleration to zero velocity. Refer to Program Block Lookahead Overview for more information.

Without velocity profiling mode, the controller will always decelerate to zero velocity between consecutive coordinated moves. The following examples illustrate the difference between these two modes.

Example with Velocity Profiling 
' Enable velocity blending mode.
VELOCITY ON
' Enable incremental programming mode.
INCREMENTAL
' Set the vector feedrate to 60.
F60. 
' Move the X axis 1.0.
LINEAR X1.
' Set the vector feedrate to 120.
F120.
' Move the X axis 2.0.
LINEAR X2.
' Set the vector feedrate to 60.
F60.      
' Move the X axis 1.0.
LINEAR X1.   
' Dwell for one second.
DWELL 1      
' This move accelerates from 0 velocity because a DWELL command preceded it.
LINEAR X1.

Figure: Example of a Velocity Profile with VELOCITY ON

Example without Velocity Profiling 
' Enable forced deceleration mode.
VELOCITY OFF
' Enable incremental programming mode.
INCREMENTAL
' Set the vector feedrate to 60.
F60.          
' Move the X axis 1.0.
LINEAR X1.  
' Move at a velocity of 120.
F120.        
' Move the X axis 2.0.
LINEAR X2.    
' Set the vector feedrate to 60.
F60.           
' Move the X axis 1.0.
LINEAR X1.     
' Dwell for one second.
DWELL 1   
' Move the X axis 1.0.
LINEAR X1.

If the move distance is too short, the specified user feedrate might not be achieved. However, in VELOCITY ON mode, the acceleration continues into subsequent moves until the feedrate is achieved or until a new feedrate is supplied. Refer to F and E Commands for more information.

Note: When you are in VELOCITY ON mode and using corner rounding, use the CoordinatedCircularAccelLimit parameter to limit axis accelerations caused by traveling around arcs that are inserted by corner rounding. The radius of the arcs inserted by corner rounding can be very short, causing high accelerations. High axis accelerations can cause position errors, servo loop oscillation, and other motion problems.

Note: When you are in VELOCITY ON mode, use the CoordinatedAccelLimit parameter to limit axis accelerations caused by changes in direction (non-tangential moves) in multi axis motion and reversals of direction in single axis motion. High axis accelerations can cause position errors, servo loop oscillation, and other motion problems. Refer to CoordinatedAccelLimit and DependentCoordinatedAccelLimit parameters for more information.

Note: When you are in VELOCITY ON mode and using BEZIER, or if you are using CW, CCW, G2, or G3 arcs with small radii, use the CoordinatedCircularAccelLimit parameter to limit the axis accelerations caused by traveling the arc. High axis accelerations can cause position errors, servo loop oscillation, and other motion problems.

Note: If you use the SLICE command while you are doing velocity profiling with a VELOCITY ON - VELOCITY OFF block, then an error occurs.

Note: VELOCITY ON mode does not blend PVT commands with LINEAR, CW, CCW, or BEZIER commands. After the final PVT move in a sequence, the controller always immediately returns to zero velocity, even if VELOCITY ON is active. To prevent axis jerking, set the final PVT move to return to zero velocity. LINEAR, CW, CCW, and BEZIER commands decelerate to zero before proceeding to a PVT command, even if VELOCITY ON is active.
