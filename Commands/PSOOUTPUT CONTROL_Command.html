PSOOUTPUT CONTROL Command

Sets the PSO output mode.

Syntax:

 
PSOOUTPUT <Axis> CONTROL <Pin> <Mode>

Example:

 
PSOOUTPUT X CONTROL 1 1
PSOOUTPUT X CONTROL 0 1

Table:     PSOOUTPUT CONTROL Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Pin>

32-bit signed integer

Applies only to Npaq, Nservo, and XR3. Does not apply to Npaq MR. Specifies the pin number of PSO output.

<Mode>

32-bit signed integer

Does not apply to the XR3. Used to select the mode of PSO output.

Hardware: This command does not apply when used with the Nmark SSaM, the Nstep, or the XL4s.

Use the PSOOUTPUT CONTROL command to change the default PSO output locations. Refer to your drive hardware manual for more information. To see a list of available hardware manuals, refer to Other Documentation.

The <Pin> argument is used by the Npaq (does not include the Npaq MR), Nservo, and XR3 because they have multiple PSO outputs. All other drives use only the <Mode> argument, so on these drives, you should specify 0 for the <Pin> argument.

Auxiliary Marker Output

The auxiliary marker is a dual-function, bidirectional pin pair. By default, the controller uses the auxiliary marker together with the auxiliary encoder. Auxiliary Marker Output

The auxiliary marker is a dual-function, bidirectional pin pair. By default, the controller uses the auxiliary marker together with the auxiliary encoder. The bidirectional pins of the auxiliary marker are inputs. Refer to the drive hardware manual for more information. When you configure the PSO output to use the auxiliary marker, the bidirectional pins become outputs. In differential mode, the auxiliary marker output signal uses an RS-422 differential buffer. You can invert the active polarity by reversing the electrical connections to the drive.

In single-ended mode, the two pins are driven independently as TTL outputs where the positive signal supplies an active-high output and the negative signal supplies an active-low output.

Auxiliary Marker Output

The auxiliary marker is a dual-function, bidirectional pin pair. By default, the controller uses the auxiliary marker together with the auxiliary encoder. The bidirectional pins of the auxiliary marker are inputs. Refer to the drive hardware manual for more information. When you configure the PSO output to use the auxiliary marker, the bidirectional pins become outputs. In differential mode, the auxiliary marker output signal uses an RS-422 differential buffer. You can invert the active polarity by reversing the electrical connections to the drive.

In single-ended mode, the negative signal supplies an active-high TTL output, and the positive signal is not driven.

PSO Output 2 (Isolated and TTL)

The XR3 has three dedicated PSO output pins that can be used by any axis in the chassis. You cannot configure more than one axis to use the same PSO output pin. To free one of the PSO output pins so that it can be used by another axis, issue the PSOOUTPUT CONTROL command and specify a <Pin> value of 0, or issue the PSOCONTROL Command with a RESET argument.
