PSOWINDOW RESET Command

Resets the window counter to 0 based on the encoder marker signal.

Syntax:

 
PSOWINDOW <Axis> <WindowNumber> RESET <BitMask>

Example:

 
PSOWINDOW X 1 RESET 2

Table:     PSOWINDOW RESET Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<WindowNumber>

32-bit signed integer

The window to configure. The Ndrive CP, MP, QL, and Npaq MR with MP drives must be set to 1. All other drives can be set to 1 or 2.

<BitMask>

32-bit signed integer

The mask of possible conditions that hold the tracking counter in reset.

Hardware: This command does not apply when used with the Nmark SSaM or the Nstep.

NOTE: If you are doing single-axis tracking, configure only window 1. If you are doing multi-axis tracking, you must configure both window 1 and window 2.

Use the PSOWINDOW RESET command to specify conditions under which the PSO window counter will be reset automatically. The available options are listed in the following table. You can select multiple options at the same time by specifying the sum of their values for the <BitMask> argument.

Table:     <BitMask> Argument Values

Bit Number

Value

Description

0

0

(default)

The PSO window counter does not reset on each marker pulse.

1

The PSO window counter resets on each marker pulse.

1

0 (default)

The PSO window counter is not affected by the marker pulse.

2

The PSO window counter is held in reset until the first marker is seen and begins tracking. Subsequent marker pulses do not reset the counter.
