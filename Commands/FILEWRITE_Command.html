FILEWRITE Command

Writes data to a file or serial port.

Syntax:

 
FILEWRITE <FileHandle>, <ArgumentList>
 
FILEWRITE <FileHandle>, <FormatSpecifier>, <ArgumentList>
 
FILEWRITE <FileName>, <ArgumentList>
 
FILEWRITE <FileName>, <FormatSpecifier>, <ArgumentList>

Example:

 
FILEWRITE "C:\MyFolder\MyFile.dat", 127, " ", 4096.000, " ", $global[0], " "

Table:     FILEWRITE Command Arguments

Argument

Type

Description

<FileHandle>

32-bit signed integer

(Optional) The file handle (reference number) of a file or serial port for which you want to write data. This file handle is returned by the FILEOPEN function.

<FileName>

String

(Optional) The name (and path) of the ASCII file that you want to write to.

<FormatSpecifier>

String

(Optional) The characters that determine the format for each numerical value that is shown. For more information, refer to Argument Lists and Format Specifiers.

<ArgumentList>

Other

A mixture of comma-separated integer, double, and string arguments. The controller concatenates these arguments into one text string.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the FILEWRITE command to write one line of data to a file or serial port. This command automatically appends the carriage return and line feed terminators, <CR> and <LF>, to the end of each data line. The data to be written is specified last in the command and can be a combination of character strings and floating-point expressions.

The <FileHandle> argument specifies the file handle (reference number) of the file or serial port for which you want to write data. If you use <FileHandle> as the first argument, the file or serial port stays open. After all writing completes, you must use the FILECLOSE command to close the file or serial port.

The <FileName> argument specifies the name of the file to which the data is written. If you use the file name as the first argument, the file automatically opens, the data is appended to the end of the file, and the file is automatically closed after the file write is completed. If the file does not exist, it is created. The directory must already exist. If you do not specify an absolute file path in the file name, the file path is relative to the program that is currently executing.

You can also include an optional format specifier to control how variables and expressions are converted to text. If you do not include a format specifier, the command formats variables and expressions with six digits after the decimal point.

This command sets the Info Variables if one or more of the following error conditions occur.

  • Windows finds an error while it is writing to the file or serial port.
  • Windows finds an error while it is opening the file that is specified in the <FileName> argument.

All other error conditions set the Task Error.
