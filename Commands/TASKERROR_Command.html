TASKERROR Command

Causes a specified task error on a task.

Syntax:

 
TASKERROR <ErrorCode>, <ErrorLocation>
 
TASKERROR <TaskNumber>, <ErrorCode>, <ErrorLocation>

Example:

 
TASKERROR $info0, $info1

Table:     TASKERROR Command Arguments

Argument

Type

Description

<Task>

64-bit IEEE floating point

(Optional) The task on which to cause the error. If you do not specify a task, the command causes the error on the current task.

<ErrorCode>

64-bit IEEE floating point

The error that you want to cause.

<ErrorLocation>

64-bit IEEE floating point

The type of the error that you specified in the <ErrorCode> argument.

Use the TASKERROR command to cause a specified task error on a task. If you do not specify a task, the command causes the error on the current task.

Each task error has a number of location codes that give specific information about the error. In order to generate an error with the TASKERROR command, you must supply the code for the task error in the <ErrorCode> argument and the specific location in the <ErrorLocation> argument.

If you cause an error on a task, that task enters the Error task state. The Tasks tab of Status Utility and the Task List in Motion Composer give the state of the task. When a task is in the Error task state, you cannot execute more program lines in that task until that task is removed from the Error task state. To remove a task from the Error task state, do one of the following operations.

  • Issue the TASKERROR command with an <ErrorCode> and <ErrorLocation> of zero to clear a task error.
  • Click the Acknowledge All button or issue an ACKNOWLEDGEALL command to clear task errors on all tasks.
  • Click the Stop button or issue a PROGRAM STOP command to stop the current task and clear a task error on the current task.
  • Click the Abort button to abort all motion, stop the current task, and clear a task error on the current task.

When a task error occurs the controller will abort the axes specified in the TaskErrorAbortAxes parameter for that task.

For more information about task errors, refer to Fault and Error Overview.

Example

DVAR $fileHandle

' The FILEOPEN command does not stop the task if it encounters an error.
' Instead, it returns the error code and error location to $info0 and $info1.
$fileHandle = FILEOPEN "C:\MyFile.txt", 0 

' If you want the task to stop, use the TASKERROR command.
IF ($info0 != 0)
	TASKERROR $info0, $info1   	    
ENDIF
