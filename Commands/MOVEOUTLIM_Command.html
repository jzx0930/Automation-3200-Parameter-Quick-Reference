MOVEOUTLIM Command

Moves an axis out of a limit condition.

Syntax:

 
MOVEOUTLIM <Axis>

Example:

 
MOVEOUTLIM X

Table:     MOVEOUTLIM Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

Use the MOVEOUTLIM command to move an axis out of a limit condition. This is an asynchronous command which means that program execution continues to the next program line without waiting for the move to complete. Use the WAIT MOVEDONE Command or WAIT INPOS Command to wait for motion to complete.

This command has an effect only if the axis is in a limit condition. Refer to the applicable drive type that follows.

For FireWire Drives

Use the MOVEOUTLIM command to move an axis out of an active hardware end-of-travel limit or an active software limit. The controller moves the axis out of one limit condition at a time:

  • If a hardware end-of-travel limit and a software limit are active, the controller moves an axis out of the hardware end-of-travel limit first. If the software limit stays active, then you can issue the MOVEOUTLIM command again to move the axis out of the software limit.
  • If a hardware end-of-travel limit is active, the controller moves an axis only out of that limit.
  • If a software limit is active, the controller moves an axis only out of that limit.

The limit conditions must be active. But the faults that correspond to them can be inactive. To configure the active state of a hardware end-of-travel limit, use the EndOfTravelLimitSetup Parameter. Software limits are active after you home the axis.

Faults Related to Hardware End-of-Travel Limits and Software Limits
  • CW/Positive End-of-Travel Limit Fault
  • CCW/Negative End-of-Travel Limit Fault
  • CW/Positive Software Limit Fault
  • CCW/Negative Software Limit Fault

During the move, the motion starts to decelerate after the limit condition is declared inactive. For a hardware end-of-travel limit, the LimitDebounceDistance Parameter and the LimitDebounceTime Parameter have an effect on how the limit condition is determined.

Specify the speed of the move and the acceleration and deceleration rates by configuring the parameters that follow:

HomeSpeed Parameter specifies the speed of the move.

HomeRampRate Parameter specifies the acceleration and deceleration rates of the move. The acceleration and deceleration ramp types are always linear.

For HyperWire Drives

Use the MOVEOUTLIM command to move an axis out of all the hardware end-of-travel limits and the software limits that are currently active:

  • If a hardware end-of-travel limit and a software limit are active, the controller moves an axis out of the two limits at the same time.
  • If a hardware end-of-travel limit is active, the controller moves an axis only out of that limit.
  • If a software limit is active, the controller moves an axis only out of that limit.

The limit conditions must be active. But the faults that correspond to them can be inactive. To configure the active state of a hardware end-of-travel limit, use the EndOfTravelLimitSetup Parameter. Software limits are active after you home the axis.

  • CCW/Negative End-of-Travel Limit Fault
  • CW/Positive Software Limit Fault
  • CCW/Negative Software Limit Fault
