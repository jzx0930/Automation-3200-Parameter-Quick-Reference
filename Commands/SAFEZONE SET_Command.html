SAFEZONE SET Command

Sets the safe zone boundaries.

Syntax:

 
SAFEZONE <Zone> SET <Axis> <Low>, <High>

Example:

 
SAFEZONE 0 SET X -5, 15

Table:     SAFEZONE SET Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Zone>

32-bit signed integer

The zero-based index of the safe zone.

<Low>

64-bit IEEE floating point

The safe zone low boundary, specified in user units.

<High>

64-bit IEEE floating point

The safe zone high boundary, specified in user units.

Use the SAFEZONE SET command to set a dimension for a safe zone. A dimension has an axis (<Axis>), a low value (<Low>), and a high value (<High>). Safe zones can have 1 to 32 dimensions.

The values of the safe zone boundaries are always relative to the home position, which is normally at 0.0. When you use the HomePositionSet parameter, the application shows a value that is different from 0.0 at the home position. But the configuration of the safe zone boundaries stays relative to a zero home position.

To Calculate the values of the safe zone boundaries

  1. Move the axis to the lower boundary. Examine the value of the Position Command or Position Feedback that the application shows.
  2. Subtract the value of the HomePositionSet parameter from the value that you examined in the previous step. The new value that you get is the value to use for the <Low> argument.
  3. Move the axis to the upper boundary. Examine the value of the Position Command or Position Feedback that the application shows.
  4. Subtract the value of the HomePositionSet parameter from the value that you examined in the previous step. The new value that you get is the value to use for the <High> argument.

The low boundary must always have a lower numbered value than the high boundary. If the value of the low boundary is greater than the value of the high boundary, then one of the following occurs.

  • A NoExit zone is always violated.
  • A NoEnter zone is never violated.

The arithmetic sign of the safe zone boundaries is determined by the encoder direction. The following example shows you how to set the safe zone boundaries.

For Example

If ReverseMotionDirection is set to No on this axis, set the boundaries to the following values.
