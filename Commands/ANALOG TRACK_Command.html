ANALOG TRACK Command

Configures the system to continuously set an analog output to be a real-time internal servo loop value.

Syntax:

 
ANALOG TRACK <Axis>, <AnalogOutput>, <ServoValue> SCALE <ScaleFactor>
 
ANALOG TRACK <Axis>, <AnalogOutput>, <ServoValue> SCALE <ScaleFactor> OFFSET <OffsetValue>
 
ANALOG TRACK <Axis>, <AnalogOutput>, <ServoValue> SCALE <ScaleFactor> OFFSET <OffsetValue> LIMIT <MinVoltage>, <MaxVoltage>

Example:

 
ANALOG TRACK X, 0, 5 SCALE 0.33

Table:     ANALOG TRACK Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis from which the controller tracks the servo loop value.

<AnalogOutput>

32-bit signed integer

The analog output to which the servo loop value is sent. For information about how to use this argument with different drive types, refer to the Using the <AnalogOutput> Argument section in this topic.

<ServoValue>

32-bit signed integer

Specifies the servo loop value that the controller tracks. To see the possible values, refer to Table: <ServoValue> Argument Values.

<ScaleFactor>

64-bit IEEE floating point

The value that you use to change the servo loop units to the analog output voltage.

For Example

If you track the Position Command for the <ServoValue> argument and you want 1 volt to equal 1000 counts for the Position Command, set the value of the <ScaleFactor> argument to .001.

<OffsetValue>

64-bit IEEE floating point

(Optional) This value is applied with the tracking value to the analog output. Use this argument if you want to track position on a stage where the position can never be negative. The units are volts.

<MinVoltage>

64-bit IEEE floating point

(Optional) Specifies the minimum voltage for the analog output.

<MaxVoltage>

64-bit IEEE floating point

(Optional) Specifies the maximum voltage for the analog output.

Use the ANALOG TRACK command to configure the system to continuously set an analog output to be a real-time internal servo loop value. You can have one ANALOG TRACK command set to ON for each analog output in the system.

Using the <AnalogOutput> Argument

On the XR3, the Npaq, and all single-axis drives, you must use the analog output that is related to the axis from which the controller tracks the specified <Axis><ServoValue>.

On all other multi-axis drives, you can use any analog output of the drive to track the specified <Axis><ServoValue>. You are not limited to using only the analog output that is related to the specified <Axis>.

For information about how I/O is mapped to different drive types, refer to Specifying I/O Numbers.

Examples of Tracking the <ServoValue> from the GL4

Say that you have a GL4 where Axis X is the first axis and Axis Y is the second axis. Analog outputs 0 and 1 are related only to Axis X. The <ServoValue> is set to 5, which is the value of the Current Command. The <ScaleFactor> is set to 0.33. You want to track the <ServoValue> of the first axis and can do so as follows.

To track the Current Command from the first axis (X) of a GL4 on analog output 0:

ANALOG TRACK X, 0, 5 SCALE 0.33

To track the Current Command from the first axis (X) of a GL4 on analog output 1:

ANALOG TRACK X, 1, 5 SCALE 0.33

Now you want to track the <ServoValue> of the second axis. You keep this value set to 5 and keep the <ScaleFactor> set to 0.33. Even though the analog outputs are related to Axis X, you use the ANALOG TRACK command to track a <ServoValue> from Axis Y.

To track the Current Command from the second axis (Y) of a GL4 on analog output 0:

ANALOG TRACK Y, 0, 5 SCALE 0.33

To track the Current Command from the second axis (Y) of a GL4 on analog output 1:

ANALOG TRACK Y, 1, 5 SCALE 0.33
