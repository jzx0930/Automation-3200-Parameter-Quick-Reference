JOYSTICK Command

Configures and turns on joystick mode.

Syntax:

 
JOYSTICK <Dimensions> RESET
 
JOYSTICK <Dimensions> ON
 
JOYSTICK <Dimensions> AXISGROUP <GroupNumber>, <Input0Axis>, <Input1Axis>
 
JOYSTICK <Dimensions> AXISGROUP <GroupNumber>, <Input0Axis>, <Input1Axis>, <Input2Axis>

Example:

 
JOYSTICK 2D RESET
JOYSTICK 2D ON
JOYSTICK 2D AXISGROUP 0, X, Y

Table:     JOYSTICK Command Arguments

Argument

Type

Description

<Dimensions>

32-bit signed integer

The dimension representation of the number of axes you are using. Specify 2D (two-dimensional) to use a two-axis joystick. Specify 3D to use a three-axis joystick.

<GroupNumber>

32-bit signed integer

(Optional) The number of the axis group that you want to specify. If you specify one axis group, you must specify a value of 0 for this argument. If you specify two or more axis groups, start with a value of 0 for the first axis group. Then for each of the other axis groups, increase the value of this argument in a sequence. You can specify a maximum of eight axis groups (0 - 7) for this argument.

<Input0Axis>

32-bit signed integer

(Optional) The first axis that is used in the joystick axis group. To configure an axis group that has one axis, specify NONE for either the <Input0Axis> argument or the <Input1Axis> argument.

<Input1Axis>

32-bit signed integer

(Optional) The second axis that is used in the joystick axis group. To configure an axis group that has one axis, specify NONE for either the <Input0Axis> argument or the <Input1Axis> argument.

<Input2Axis>

32-bit signed integer

(Optional) The third axis that is used in the joystick axis group. Use this argument only if you are configuring a 3D joystick axis group.

Use the JOYSTICK ON command to activate the joystick. This program will execute the JOYSTICK ON command and stay on that program line until you deactivate the joystick. When the joystick is active, the axes being controlled are shown in yellow in the Axis Manager of Motion Composer.

Before you activate the joystick, you must use the JOYSTICK AXISGROUP command to specify a minimum of one axis group for the <GroupNumber> argument. Refer to the Axis Group Settings section of Joystick Configuration for more information. You can also specify a maximum of seven other axis groups for this argument. To change between these axis groups, use Joystick Button A.

If you issue the JOYSTICK ON command and the joystick voltages are outside of the deadband voltage range (refer to JoystickInput0Deadband parameter), the joystick stick and the joystick buttons will not operate until after you first enter the deadband voltage range. When the joystick voltages are outside of the deadband, the JOYSTICK command continues to execute but does not control any axes until the voltages are in the deadband range.

To deactivate the joystick, click Joystick Button C or the Abort button in Motion Composer. You can also use the JoystickInterlock digital input (refer to JoystickInterlock parameter) to deactivate the joystick, or you can issue a Program Stop or an ABORT command. If a task error or an axis fault error occurs on any of the active joystick axes, this also deactivates the joystick.

If you specify NONE for <Input0Axis> or <Input1Axis>, the axis does not respond to movement on the corresponding joystick input, which lets you configure a one-dimensional joystick.

Use the JOYSTICK RESET command to remove all the configured axis groups.

Joystick Status

When the JOYSTICK command is executed, the Joystick Control bit of Axis Status changes to ON for each axis status that the joystick stick is controlling. When the JOYSTICK command is running and the joystick stick has entered the deadband voltage range a minimum of one time, the Joystick Active bit of Task Status 0 changes to ON for that task. If you are using multiple axes, all of the axes must be in the deadband range. The joystick does not control the axes and Joystick Button A does not operate until the Joystick Active bit of Task Status 0 changes to ON. You can use Joystick Button B and Joystick Button C when the joystick is inside or outside of the deadband range.

If the interlock drops (refer to JoystickInterlock parameter), then a task warning occurs. When the joystick is in low speed mode, the Joystick Low Speed Active bit of Task Status 0 changes to ON. Refer to JoystickLowSpeed parameter.

Note: When the speeds change in a JOYSTICK command, the relevant acceleration and deceleration ramping uses the following specifications:

  • The user-specified acceleration is used, regardless of if the speed is being increased or decreased, and regardless of what deceleration information the user specifies.
  • The controller uses a linear ramp type for the JOYSTICK command. The ramp type that you specify to the RAMP TYPE command does not have an effect.
  • A rate-based mode is used. If the user specifies acceleration ramping in time-based mode, the rate is defined by the maximum joystick speed variation (maximum joystick deflection) over the specified time period.
