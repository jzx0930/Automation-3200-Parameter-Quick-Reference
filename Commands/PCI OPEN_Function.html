PCI OPEN Function

Opens communication to a card on the PCI bus and returns a handle (reference number) for the card.

Syntax:

 
<return> = PCI OPEN <VendorID>, <DeviceID>, <Location>

Example:

 
$CardID = PCI OPEN 0H135E, 0H8008, 0

Table:     PCI OPEN Function Arguments

Argument

Type

Description

<VendorID>

32-bit signed integer

The Vendor identification number for the card. You can get this information from the manufacturer of the card, or from the .inf file that you used to install the device driver of the card.

<DeviceID>

32-bit signed integer

The Device identification number for the card. You can get this information from the manufacturer of the card, or from the .inf file that you used to install the device driver of the card.

<Location>

32-bit signed integer

The location of the card that you opened. This number is zero-based and relates to the location of the card on the PCI bus.

RETURNS

32-bit signed integer

The card handle (reference number) of the card that you opened.

Use the PCI OPEN function to find a card on the PCI bus and get a card handle (reference number) for the card. If the specified card is not found on the PCI bus, an error occurs. You can open a maximum of 10 PCI cards with this function. Each card handle is valid until you reset the system or you close communication by issuing the PCI CLOSE command for the specified card.

Note: Do not give other Windows applications access to the cards that you open with the PCI OPEN function or the PCI Bus commands will not work correctly.

The <Location> argument is a zero-based number that relates to the location of the card on the PCI bus. If you have one PCI card, or if the Vendor ID and the Device ID are different for each card, then specify each <Location> argument as a value of 0. If you have more than one PCI card with the same Vendor ID and Device ID, use the <Location> argument to specify which card to open. The controller searches for devices on the PCI bus by Bus Number, then Device Number, and then Function Number. To find the PCI Bus Number, Device Number, and Function Number, use the Windows Device Manager to see the properties of the location information of the PCI card. To specify the lowest-numbered card on the PCI bus, specify the <Location> argument as a value of 0. For the next card, increase the value by one.

For Example

You have two PCI cards, Card A and Card B. Both cards have the same Vendor ID and Device ID. Card A is identified on the PCI bus as Bus 2, Device 7, Function 0. Card B is identified on the PCI bus as Bus 2, Device 8, Function 0. To get the card handle for each card, you must issue the PCI OPEN function one time for each card. When you set the <Location> argument to 0, the card handle for Card A is returned. When you set the <Location> argument to 1, the card handle for Card B is returned.

Example

' Declare the variables.
DVAR $ByteValue, $CardID
' Open communication with the PCI card with a vendor ID of 0x135E and device ID of 0x8008.
$CardID = PCI OPEN 0H135E 0H8008 0
' Read from the PCI card.
$ByteValue = PCI READ BYTE $CardID 0 1
' Write to the PCI card.
PCI WRITE BYTE $CardID 0 3 0HFF 0H80
' Close communication with the card.
PCI CLOSE $CardID
