CW Command

Executes a clockwise circular move on the axes.

Syntax:

 
CW <Axis1> <Axis1Endpoint> <Axis2> <Axis2Endpoint> R <Radius>
 
CW <Axis1> <Axis1Endpoint> <Axis2> <Axis2Endpoint> I <Axis1Center> J <Axis2Center>
 
CW <Axis1> <Axis1Endpoint> <Axis2> <Axis2Endpoint> R <Radius> F <CoordinatedSpeed>
 
CW <Axis1> <Axis1Endpoint> <Axis2> <Axis2Endpoint> I <Axis1Center> J <Axis2Center> F <CoordinatedSpeed>

Example:

 
CW X2.5 Y1 I.5 J-1.5

Figure: Circular Arc that the CW Syntax Example Specifies

Note: This command applies only to dominant axes. Refer to Axis Type Overview for more information about dominant axes.

The CW command generates a clockwise circular arc. To create the arc, you must use this command to specify the center of the arc and to explicitly specify the end point of the arc. If you use the INCREMENTAL distance mode, the end point that you specify is related to the current axis positions. If you use the ABSOLUTE distance mode, the end point that you specify is related to the home or software home position.

To specify the center of the arc, use the IJ Method or the R Method.

For information about how to prevent positional errors that can occur in CW and CCW motion, refer to Imprecision of Generated Circular Motion.

Note: When you are in VELOCITY ON mode and using BEZIER, or if you are using CW, CCW, G2, or G3 arcs with small radii, use the CoordinatedCircularAccelLimit parameter to limit the axis accelerations caused by traveling the arc. High axis accelerations can cause position errors, servo loop oscillation, and other motion problems.

Note: VELOCITY ON mode does not blend PVT commands with LINEAR, CW, CCW, or BEZIER commands. After the final PVT move in a sequence, the controller always immediately returns to zero velocity, even if VELOCITY ON is active. To prevent axis jerking, set the final PVT move to return to zero velocity. LINEAR, CW, CCW, and BEZIER commands decelerate to zero before proceeding to a PVT command, even if VELOCITY ON is active.

Methods of Determining the Circular Center Point

Use one of these methods to specify the center of an arc. To specify the center of an arc as an offset from the start of a circle, use the IJ Method. To specify the center of an arc as a radius, use the R Method.

IJ Method

Use this method to specify the center of the arc as an incremental offset from the starting position of a circle. The controller always specifies the offsets as INCREMENTAL so that they are related to the axis position. The ABSOLUTE or INCREMENTAL distance mode that you specify does not have an effect on how the controller specifies the offsets.

The radius of the generated arc is the distance between the starting position and the specified offset. The I offset corresponds to the first axis that you specify in the end point. The J offset corresponds to the second axis that you specify in the end point.

Creating a Full Circle with the IJ Method

To create a full 360° circle with the IJ Method, specify an end point that is the same location as the starting point of the CW motion.

Example

' The controller moves the axes to (4,5) and then generates a circle of
' radius 1 that has a center point at (5,5).
' To create a full circle, the specified endpoint must have 
' the current axis locations.
' The center point (I,J) is relative to the current location
' of the axes.
' This example circle has a center point that is one unit in
' the positive X direction from the current axis locations.
ABSOLUTE
' Move to the start point.
LINEAR X4 Y5
' Generate the circle.
CW X4 Y5 I1 J0
R Method

Note: You cannot use the R Method to create a full 360° circle. To create a full circle, use the IJ Method.

Aerotech recommends that you do not use the R Method to program arcs that are almost full circles or almost half circles. In these arcs, a small error in the location of the end point causes a much larger error in the center point of the arc. As an alternative, Aerotech recommends that you either divide the arc into two or more smaller arcs or use the IJ Method to construct the arc.

If you use the R Method, make sure that you specify the correct arc that you want to use.

  • If the radius is positive, the controller generates a shorter arc that is always 180° or less.
  • If the radius is negative, the controller generates a longer arc that is always between 180° and 360°.
