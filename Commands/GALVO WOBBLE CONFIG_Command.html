GALVO WOBBLE CONFIG Command

Configures the wobble feature, which generates a wobble pattern that is added to the motion command of the GL4 or the Nmark CLS, GCL, or SSaM.

Syntax:

 
GALVO WOBBLE <GalvoAxis> CONFIG <AmplitudeParallel>, <AmplitudePerpendicular>, <Frequency>, <WobbleMode>, <WobbleType>

Example:

 
GALVO WOBBLE x CONFIG 0.1, 0.2, 15, GALVOWOBBLEMODE_TimeBased, GALVOWOBBLETYPE_Figure8Parallel

Table:     GALVO WOBBLE CONFIG Command Arguments

Argument

Type

Description

<GalvoAxis>

Axis

The first axis of the GL4 or the Nmark CLS, GCL, or SSaM on which to generate the wobble motion.

<AmplitudeParallel>

64-bit IEEE floating point

The amplitude of the wobble shape parallel to the vector path.

<AmplitudePerpendicular>

64-bit IEEE floating point

The amplitude of the wobble shape perpendicular to the vector path.

<Frequency>

64-bit IEEE floating point

The frequency of the wobble oscillation. Specified in hertz for time-based mode or user units for distance-based mode.

<WobbleMode>

32-bit signed integer

Specifies whether the wobble is repeated based on a fixed time or a fixed vector distance.

<WobbleType>

32-bit signed integer

The type of figure that is generated by the wobble.

Use the GALVO WOBBLE CONFIG command to configure the galvo wobble feature. When the GL4 or the Nmark CLS, GCL, or SSaM does marking motion, the galvo wobble feature generates an oscillatory pattern and the controller combines that pattern with the position command of the drive. The oscillatory pattern increases the width of the marking line.

Set the <AmplitudePerpendicular> argument to control the width of the marking line. Set the <AmplitudeParallel> argument with the <Frequency> argument to control the overlap that occurs between each wobble oscillation.

<WobbleMode>

Set the <WobbleMode> argument to control if the <Frequency> of the repeated wobble pattern is time-based or distance-based. When you use time-based mode, the controller repeats the wobble pattern at a fixed time interval. When you use distance-based mode, the controller repeats the wobble pattern at a fixed distance. In distance-based mode, the speed of the marking vector controls the time at which the controller repeats the wobble pattern. To create a uniform overlap of the wobble pattern during acceleration and deceleration, Aerotech recommends that you use the distance-based mode.

Table:     Galvo Wobble Mode Values

Value AeroBasic Constant Name Description

0

GALVOWOBBLEMODE_TimeBased

TimeBased

Wobble is repeated at a fixed time interval.

1

GALVOWOBBLEMODE_DistanceBased

DistanceBased

Wobble is repeated at a fixed vector distance.
<WobbleType>

Set the <WobbleType> argument to control the type of wobble pattern that the controller generates.

Table:     Galvo Wobble Type Values

Value AeroBasic Constant Name Description

0

GALVOWOBBLETYPE_Ellipse

Ellipse

An ellipse shape.

1

GALVOWOBBLETYPE_Figure8Parallel

Figure8Parallel

A figure 8 shape parallel to the vector path.

2

GALVOWOBBLETYPE_Figure8Perpendicular

Figure8Perpendicular

A figure 8 shape perpendicular to the vector path.
