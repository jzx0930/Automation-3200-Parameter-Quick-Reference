SLAVEOFFSET Command

Offsets the position of the master axis that is specified in the cam table.

Syntax:

 
SLAVEOFFSET <SlaveAxis> <OffsetUnits>

Example:

 
SLAVEOFFSET Y 10

Table:     SLAVEOFFSET Command Arguments

Argument

Type

Description

<SlaveAxis>

Axis

The axis that is set as the camming slave axis.

<OffsetUnits>

64-bit IEEE floating point

The distance of offset in user units. To shift the cam table in the negative direction, specify a positive value. To shift the cam table in the positive direction, specify a negative value.

Use the SLAVEOFFSET command to shift, or offset, the start of the cam table without creating a new cam table. If the value of the <OffsetUnits> argument is negative, then the cam table shifts to the right by the value that you specify. If the value of the <OffsetUnits> argument is positive, then the cam table shifts to the left by the value that you specify. Specify a value of zero to remove the offset. Before the slave motion is generated, the offset is added to the proportional value that is applied by the SLAVEADVANCE Command.

Warning: Before you use this command, use the ABORT Command on the master axis to make sure that the master axis is not moving. Then, make sure that the velocity is zero for the master axis and for the slave axis. If you use the SLAVEOFFSET command while the master axis is moving, then the slave axis will suddenly change its position.

For Example

A cam table specifies that at a master position of 3 inches, the slave must be at a specific position. If you want a master position of 4 inches to correspond to the same slave position, set the <OffsetUnits> value a to -1 inch.
