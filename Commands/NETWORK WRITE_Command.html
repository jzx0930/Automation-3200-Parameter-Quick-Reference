NETWORK WRITE Command

Writes the specified data to the connection stream.

Syntax:

 
NETWORK WRITE <Id>, <DataType> <Value> [ , <DataType> <Value> ... ]

Example:

 
NETWORK WRITE $serverHandle, INT 5, DOUBLE 3.1415

Table:     NETWORK WRITE Command Arguments

Argument

Type

Description

<Id>

32-bit signed integer

The ID of the connection

<DataType>

Other

(Optional) The data type of the data. Valid data types include all typecasts (such as BIT, BOOL, and UDINT) and STRING.

<Value>

Other

(Optional) The value of the data.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the NETWORK WRITE command to send data to the remote machine of the connection.

If NETWORK AUTOFLUSH is ON, then the specified data is sent immediately to the remote machine. If NETWORK AUTOFLUSH is OFF, then the data waits in queue until you issue a NETWORK FLUSH command.

You can specify more than one pair of type and value data. For example, you can execute NETWORK WRITE 1, BYTE 1, BIT 0. When performance is an issue, this is useful because there is overhead each time you issue a NETWORK command.

Values of type STRING are not null-terminated. If the remote machine expects STRING values to be null-terminated, then you must send the null byte manually, for example, NETWORK WRITE 1, STRING $strglob0, BYTE 0.
