LINEAR Command

Executes a linear move on axes.

Syntax:

 
LINEAR <Axis> <Distance> [ <Axis> <Distance> ... ]
 
LINEAR <Axis> <Distance> [ <Axis> <Distance> ... ] F <CoordinatedSpeed>
 
LINEAR <Axis> <Distance> [ <Axis> <Distance> ... ] E <DependentSpeed>

Example:

 
LINEAR X10 Y5 F2

Table:     LINEAR Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Distance>

64-bit IEEE floating point

The distance to move the axis.

<CoordinatedSpeed>

64-bit IEEE floating point

(Optional) The vectorial speed at which to move the axes.

<DependentSpeed>

64-bit IEEE floating point

(Optional) The vectorial speed at which dependent axes move.

Use the LINEAR command to generate synchronized motion along a straight line for one or more axes. All specified axes start and stop motion at the same time. If the incremental distance mode is in effect, the motion occurs relative to the current axis positions. If the absolute distance mode is in effect, the motion occurs relative to the home or software home position.

When you use the LINEAR command in velocity profiling mode (VELOCITY ON), the controller blends the motion of multiple commands without stopping between each move. Use the VELOCITY OFF command as the last command in a velocity profiled sequence of moves to force the axis motion to stop at the completion of the target position of the previous program line.

Use the RAMP commands to configure the coordinated acceleration and deceleration. Refer to Acceleration and Deceleration Ramping Overview for more information.

If the LINEAR command contains a mixture of dominant and dependent axes, the controller uses the speed that is specified by the F command and only includes the dominant axes in the speed calculation. The dependent axes move at a rate that maintains synchronism with the dominant axes. The LINEAR command uses the speed that is specified by the E command if only the dependent axes are contained in the move. For more information about dominant and dependent axes, refer to Axis Type Overview. The E command is modal and you do not need to specify it on each program line. If you do not specify the E command, the controller uses the previously specified value of the E command for the motion.

The LINEAR command is modal and you do not need to specify it on each program line.

Advanced Topics
  • You can use a LINEAR command to infeed an axis while camming, gearing, or handwheel motion is active.
  • You can specify a default feedrate for each task by using the DefaultCoordinatedSpeed task parameter.

Note: VELOCITY ON mode does not blend PVT commands with LINEAR, CW, CCW, or BEZIER commands. After the final PVT move in a sequence, the controller always immediately returns to zero velocity, even if VELOCITY ON is active. To prevent axis jerking, set the final PVT move to return to zero velocity. LINEAR, CW, CCW, and BEZIER commands decelerate to zero before proceeding to a PVT command, even if VELOCITY ON is active.

Example

' Enable the axes.
ENABLE X Y
' Home the axes.
HOME X Y
' Enable incremental programming mode.			
INCREMENTAL
' Set the time base to seconds.
SECONDS
' Move the X and Y axes 10 user units at 100 user units/s.
LINEAR X10 Y10 F100
' Move the X axis -10 user units at 100 user units/s.
LINEAR X-10
' Move the Y axis -10 user units at 100 user units/s.
LINEAR Y-10
' Enable absolute programming mode.
ABSOLUTE
' Move to absolute position x=50, y=20.
LINEAR X50 Y20
' Move to absolute position X=0, Y=0.
LINEAR X0 Y0
' Enable incremental programming mode.
INCREMENTAL
' Enable velocity profiling mode.
VELOCITY ON
' Start a profile sequence. The speed is 50 user units/s.
LINEAR X10 Y10 F50
LINEAR X5 Y5
' Stop at the end of this move.				
LINEAR X10 Y10
' Force a stop at the end of the previous line.
VELOCITY OFF
' This move starts and stops at 0 speed.
LINEAR X-20 Y-20
