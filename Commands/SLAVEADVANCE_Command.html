SLAVEADVANCE Command

Advances the motion of a slave axis in proportion to the velocity of the master axis.

Syntax:

 
SLAVEADVANCE <SlaveAxis> <Time>

Example:

 
SLAVEADVANCE Y 10

Table:     SLAVEADVANCE Command Arguments

Argument

Type

Description

<SlaveAxis>

Axis

The axis that is set as the slave axis.

<Time>

64-bit IEEE floating point

The duration of the advance, in milliseconds. To advance the motion of the slave axis, specify a positive value. To delay the motion of the slave axis, specify a negative value.

Use the SLAVEADVANCE command to advance the motion of the slave axis in proportion to the velocity of the master axis. This compensates for the position following an error of the slave axis and communication lag caused by transmission delay.

Use the equation that follows to calculate the proportional value by which the position of the master axis is shifted before slave motion is generated.

If the value of the <Time> argument is positive, then the motion of the slave axis is advanced in time. If the value of the <Time> argument is negative, then the motion of the slave axis is delayed in time. Specify a value of zero to remove the time advance or delay. Before the slave motion is generated, the advance or delay is added to the offset that is applied by the SLAVEOFFSET Command.

Note: When you increase the value of the <Time> argument, the quantization noise on the velocity command also increases.

Warning: Before you use this command, use the ABORT Command on the master axis to make sure that the master axis is not moving. Then, make sure that the velocity is zero for the master axis and for the slave axis. If you use the SLAVEADVANCE command while the master axis is moving, then the slave axis will suddenly change its position.

Camming and the SLAVEADVANCE Command

If the slave axis has a camming relationship with the master axis, then this value is used to shift, or offset, the relationship between the master positions and the slave positions that are specified in the cam table. Thus, you can shift the cam table to the left or right without creating a new cam table. The figure that follows shows this relationship.
