RAPID Command

Executes single or multi-axis point-to-point motion.

Syntax:

 
RAPID <Axis> <Distance> [ <Axis> <Distance> ... ]
 
RAPID <Axis> <Distance> [ <Axis> <Distance> ... ] <Axis>F<Speed> [ <Axis>F<Speed> ... ]

Example:

 
RAPID X4.5 Y2 XF100 YF120

Table:     RAPID Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Distance>

64-bit IEEE floating point

The distance to move the axis.

<Speed>

64-bit IEEE floating point

(Optional) The speed at which to move an axis.

Use the RAPID command to generate single or multi-axis point-to-point motion. For a multi-axis move, all axes start motion at the same time but do not end at the same time. Use the RAPID command for operation where you must move axes from one position to another as fast as possible without regard for coordination during the move or the path of the movement.

If INCREMENTAL distance mode is in effect, the motion will occur relative to the current axis positions. If ABSOLUTE distance mode is in effect, the motion will be executed relative to the home or software home position.

Specify each axis speed by using the <Axis>F<Speed> syntax with no spaces. If you are using MINUTES mode, then the axis speed is in user units per minute. If you are using SECONDS mode, then the axis speed is in user units per second. Because axis speeds are modal, it is not necessary to specify them on each program line. If you reset the controller, the axis speeds go back to the value that you specified for the DefaultSpeed parameter.

The RAPID command can be configured for rate-based, time-based, or distance-based ramping using either linear or sinusoidal ramping. These are configured using RAMP <Type> <Axis> commands. In addition, the acceleration and deceleration characteristics can be set independently using task parameters.

Advanced Topics

  • You can use a RAPID command to infeed an axis while camming, gearing, or HANDWHEEL motion is active.
  • The RAPID command internally clamps the commanded axis speeds to 100 percent MFO. Axis acceleration and deceleration rates are lower when the MFO is less than 100 percent. For more information, refer to MFO Overview.

Examples

' Specify independent axis speeds and distances.
RAPID X4.5 Y2 XF100 YF120
ABSOLUTE
SECONDS
' Move to X=10 at 20 user units per second. Move to Y=5 at 50 user units per second.
RAPID X10 Y5 XF20 YF50
