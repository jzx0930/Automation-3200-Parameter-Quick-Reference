AXISSTATUSFAST Command

Gets the specified status item from the specified axis at a rate that is faster than 1 kHz.

Syntax:

 
AXISSTATUSFAST <Axis>, <DataItem>, <AdditionalData>, <ControllerVariable>, <SampleRate>

Example:

 
AXISSTATUSFAST X, DATAITEM_PositionCommandRaw, 0, $global10, 8

Table:     AXISSTATUSFAST Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<DataItem>

Other

Specifies the axis status to retrieve. Table: AXIS STATUS Data Items shows the values that you can use.

<AdditionalData>

32-bit signed integer

Additional data for the specified data item. Some data items require additional data. If additional data is not required, specify a value of 0 for this argument.

<ControllerVariable>

64-bit IEEE floating point variable

The controller variable on which to retrieve the data. The value of the controller variable must be sufficient to store the number of samples that are specified by the <SampleRate> argument.

<SampleRate>

32-bit signed integer

The number of samples to collect. This must be 1, 2, 4, 8, 16, 24, or 48.

Use the AXISSTATUSFAST command to read axis status values from the controller at a rate that is faster than the standard 1 kHz rate. When you issue this command, the controller collects the number of values that you specify for the <SampleRate> argument and returns them from the same 1 millisecond period. The controller puts these values into a sequence of AeroBasic variables that starts with the variable that you specify for the <ControllerVariable> argument. You must make sure that the value of the <ControllerVariable> argument is sufficient to store all the samples.

The controller does not update all axis status values at a rate that is faster than 1 kHz. Use the AXISSTATUSFAST command only if you must do computations on values available at 2 kHz, 4 kHz, 8 kHz, 16 kHz, 24 kHz, or 48 kHz rates. If you use the AXISSTATUSFAST command to read values at a rate that is faster than the update rate of the controller, the controller collects the same values two or more times. If you do not require data at a faster rate, Aerotech recommends that you use the AXISSTATUS Function to retrieve status at the standard 1 kHz rate.

Some values are not available at the faster rate. The table that follows shows the data items that are available at a rate that is faster than 1 kHz.

Example

' This sample shows a simple transformation of motion from
' one axis to a different axis, at an 8 kHz rate. 
DVAR $sample
DVAR $positions[8]
DVAR $velocities[8]

ABSOLUTE

CRITICAL START
WHILE TRUE	

        ' The line that follows makes sure the controller executes
        ' one iteration of the loop each millisecond.
        DWELL 0.001
	
        ' Get the samples.
        AXISSTATUSFAST X, DATAITEM_PositionCommandRaw, 0, $positions, 8
        AXISSTATUSFAST X, DATAITEM_VelocityCommandRaw, 0, $velocities, 8

        FOR $sample = 0 to 7
		   
           ' Copy the position and velocity of the X axis to the Y axis.
           PVT Y $positions[$sample], $velocities[$sample] TIME (1/8)
	    
        next $sample

ENDWHILE
CRITICAL END

END PROGRAM
