PSODISTANCE Command

Specifies the distance that an axis travels between firing events.

Syntax:

 
PSODISTANCE <Axis> FIXED <FireDistance>
 
PSODISTANCE <Axis> ARRAY <StartIndex> <NumberOfFiringEvents>
 
PSODISTANCE <Axis> OFF

Example:

 
PSODISTANCE X FIXED 1000
PSODISTANCE Y ARRAY 0 100

Table:     PSODISTANCE Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<FireDistance>

64-bit IEEE floating point

(Optional) The distance, in encoder counts, that an axis must travel before a firing event triggers the pulse generator.

<StartIndex>

32-bit signed integer

(Optional) The element of the array that contains the first incremental firing distance.

<NumberOfFiringEvents>

32-bit signed integer

(Optional) The number of firing events.

Hardware: This command does not apply when used with the Nmark SSaM or the Nstep.

Use the PSODISTANCE command to specify the incremental distance between PSO firing events. To specify constant firing distances, use the PSODISTANCE FIXED command. To specify firing distances in which the data in the PSO array has an effect, use the PSODISTANCE ARRAY command. If you do multi-axis tracking, the distance that you specify is a vector distance. The controller compares the vector distance with the square root of the sum of the squares of the individual axis components.

Because the PSODISTANCE command uses information that you specify in the PSOTRACK INPUT command, you must issue the PSOTRACK INPUT command before you issue the PSODISTANCE command.

You can see the PSO distance and window counters from the Drive Information Tab of the Status Utility. You can plot the PSO Counter 1, PSO Counter 2, PSO Counter 3, PSO Window 1, and PSO Window 2 items in Digital Scope.

For information about how to specify firing distances, refer to the sections that follow.

PSODISTANCE FIXED - Fixed-Distance Firing

Example

PSODISTANCE Y FIXED 2000

Use the PSODISTANCE FIXED command to specify a fixed incremental distance at which a PSO firing event occurs. The controller generates a firing event each time the axis moves the distance specified by the <FireDistance> argument. The controller measures this distance, which is the firing distance, from the location where the previous firing event occurred.

The value that you specify for the <FireDistance> argument is in encoder counts and must be a positive integer value that is less than 8,388,607 encoder counts. The PSO algorithm requires firing distances to be in encoder counts because it operates at the hardware level.

If you want your AeroBasic application to specify firing distances in user units, you must manually convert the value of the <FireDistance> argument into counts. Then you must supply this value to the PSODISTANCE FIXED command, which rounds it to the nearest number of counts.

For Example

Assume that your system uses an ABRT-150-AS rotary stage and you want to specify a fixed-distance firing event that occurs at each interval of 30.0 degrees. This rotary motor has 11,840 counts per revolution, which is 32.8888889 counts per degree.

Thus, PSODISTANCE X FIXED ABS(UNITSTOCOUNTS(X, 30.0)) fires at each interval of 987 counts because it rounds up 986.666667 to 987. An interval of 987 counts is equivalent to an interval of 30.01 degrees, not 30.0 degrees, which causes an error of 0.01 degree.

After 100 firing events occur, the counts per degree of the rotary motor is incorrect by 1 degree. After 1,000 firing events occur, the counts per degree of the rotary motor is incorrect by 10 degrees.

Because the PSODISTANCE FIXED command rounds this value, PSO might not fire when you expect it to fire. There might be a small difference between the firing distances that you specify and the firing distances that occur. If this command rounds up the value, the firing distance becomes longer by a maximum of a 1/2 count. If this command rounds down the value, the firing distance becomes shorter by a maximum of a 1/2 count.

When many firing events occur, the firing distances continue to become longer or shorter because the controller measures each distance from the location where the previous firing event occurred. After many firing events, there can be a large difference between the firing distances that you specify and the firing distances that occur.

For Example

The controller generates 100 firing events and the PSODISTANCE FIXED command rounds up each value specified for the <FireDistance> argument. As a result, the position where the last firing event actually occurs is a maximum of 50 counts farther from where you think the firing event will occur.

PSODISTANCE ARRAY - Array-Based Distance Firing

Hardware: This command applies only when used with the GL4, the Ndrive FCL, FLS, HLe, HPe, ML, or QLe, the Nmark CLS or GCL, the Npaq, the Nservo, the XC2, the XC4, the XC4e, the XL4s, or the XR3. It does not apply to the Npaq MR.

Example

PSODISTANCE Y ARRAY 3 77

Use the PSODISTANCE ARRAY command to specify the distances between firing events. These distances are based on data that is in the PSO array. Each array element contains an incremental distance and supplies the distance between firing events. For multi-axis tracking, the array elements supply the vector distances between firing events.

After the axis moves through the current distance, a firing event triggers the pulse generator and loads the next value from the array. The PSO hardware uses the new value as the next incremental firing distance.

To write the firing distances into the array, use the ARRAY WRITE Command. The firing distances in the array are specified in encoder counts and must be positive integer values that are less than 8,388,607 encoder counts.

To prevent unexpected PSO firing events, make sure that your AeroBasic program does not overwrite firing distances in the array until the PSO hardware consumes them. You can do this by splitting the PSO array in half. Thus the PSO hardware consumes distances from one half of the PSO array while your AeroBasic program writes new distances to the other half. You can use the PSO Distance Array Index value on the Drive Info tab of the Status Utility to monitor the array index from which the PSO hardware consumes points. You can also get this information by using the DRIVEINFO Function.

Each drive has a maximum number of elements that you can download from the PSO array. For information about the array size and the PSO FIFO Load Rate, refer to Table: Array Size by Drive Type in Drive Array Overview.

Finite Number of Firing Distances

If you want a finite number of firing distances, you can use the ARRAY WRITE Command to download and use some or all of the elements from the PSO array. Make sure that you specify the correct values for these arguments.

<NumberOfFiringEvents> The value of this argument must be less than or equal to the number of elements that you download from the PSO array. Thus, the <NumberOfFiringEvents> argument of the PSODISTANCE ARRAY Command must be less than or equal to the <NumberOfElements> argument of the ARRAY WRITE Command.

<StartIndex> You can specify a value of 0. But if you want the first incremental firing distance to start where you put the data, specify the <StartIndex> argument of the PSODISTANCE ARRAY Command with the same value that you use for the <StartIndex> argument of the ARRAY WRITE Command.

The PSO hardware uses the new value as the next incremental firing distance. When the PSO hardware consumes all distances in the PSO array, the controller stops generating the PSO firing events.

Infinite Number of Firing Distances

If you want an infinite number of firing distances, you can use the PSO array as a circular buffer. When you do this, you must use the ARRAY WRITE Command to download and use all of the elements from the PSO array. You cannot download and use only some of the elements. Make sure that you specify the correct values for these arguments.

<NumberOfFiringEvents> Specify a value of -1.

<StartIndex> Specify a value of 0.

To get an infinite number of firing distances that is valid, make sure that the PSO firing rate is lower than the rate at which you can download new distances into the PSO array. When the PSO hardware consumes the last firing distance at the end of the array, it wraps around from the end of the array space to the start of the array for the next firing distance. This is continuous until you stop the PSO firing.

To disable distance-based firing events, use the PSODISTANCE OFF command. After you issue this command, the controller completes all pulse sequences that were triggered by previous firing events. If you configured the distance counter, it continues to track the feedback sources that you specified for the PSOTRACK INPUT Command.

Note: The PSO hardware operates on encoder counts and cannot do non-integer scaling. For true vector firing, all axes being tracked must have the same distance unit per encoder count.
