FREERUN Command

Freeruns the axes.

Syntax:

 
FREERUN <Axis> <Velocity>
 
FREERUN <Axis> STOP

Example:

 
FREERUN X 25

Table:     FREERUN Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Velocity>

64-bit IEEE floating point

(Optional) The velocity at which to run the axes.

Issue the FREERUN command to start continuous axis motion at the specified speed. The arithmetic sign of <Velocity> determines the direction.

This is an asynchronous command which means that program execution continues to the next program line without waiting for the move to complete.

While a FREERUN command is executing, you can issue a new FREERUN, MOVEABS, or MOVEINC command. This changes the velocity of the axis without stopping. Based on the current ramp values, the axis ramps up or down to the requested speed at the axis-based acceleration rate.

Issue the FREERUN STOP command to stop motion on an axis that is executing a FREERUN command.

Velocity

You can specify a velocity of zero, in which case no motion occurs, but the axis is still considered to be moving. A negative value for the <Velocity> argument reverses the direction of motion.

The velocity is limited by the MaxSpeedClamp axis parameter.

Ramping

The axis does not immediately get to the requested speed. Based on the current ramp values, the axis ramps up or down to the requested speed at the axis-based acceleration rate. (Refer to Acceleration and Deceleration Ramping Overview for more information.)

NOTE: If the axis is already moving when this command is issued, the acceleration axis ramp is used to change speeds.

Status

When the axis generates motion, it sets the Jogging bit and the Wait Done bit of the AxisStatus status item. When the axis is done generating motion, it clears the Jogging bit and the Wait Done bit of the AxisStatus status item.

In addition to the "no longer generating motion" condition, there can be other conditions that must be set in order for the Wait Done bit to be set. For more information, refer to Motion Done Overview.

This command is an asynchronous motion command, which means that program execution immediately continues to the next program line before motion is complete. Use the WAIT MOVEDONE or WAIT INPOS command to wait for motion to complete.
