PORT Command

Configures the direction of bidirectional I/O.

Syntax:

 
PORT <Axis> DIRECTION <Port> INPUT
 
PORT <Axis> DIRECTION <Port> OUTPUT

Example:

 
PORT X DIRECTION 0 OUTPUT

Table:     PORT Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Port>

32-bit signed integer

The port on which to set the direction of the I/O.

Hardware: This command applies only when used with the Npaq, Nservo or Nstep.

Use the PORT command to configure the direction of the bidirectional signals on the Npaq, Nservo, and Nstep.

Npaq

The Npaq has three bidirectional pins on the J8 high-speed I/O connector. By default, these pins are configured as inputs. However, you can change the direction of the pins by using the PORT command with a <Port> argument of 0. When these pins are configured as inputs, you can use the $DI function to read their values. When they are configured as outputs, you can use the $DO command to set their values.

Nservo

The Nservo has six 8-bit bidirectional ports on the J111 expansion I/O connector. By default, all ports are configured as inputs. You can change the direction of each port by using the PORT command. When you configure a port as an input, you can read the value of the port by using the $PORT function. Use the $DI function to read the value of each individual input. When you configure a port as an output, you can use the $PORT function to set its value. Use the $DO command to set the value of each individual input. Because of limitations in the $DI function and $DO command, not all bits can be accessed on the same axis. Individual I/O bits are mapped to a given port. Refer to the following table.

Table:     Nservo I/O Bit Port Map

Port Number

Bit Numbers

0

16 - 23 on the first axis

1

24 - 31 on the first axis

2

0 - 7 on the second axis

3

8 - 15 on the second axis

4

16 - 23 on the second axis

5

24 - 31 on the second axis

Nstep

The Nstep has two 8-bit bidirectional ports. By default, all ports are configured as inputs, but you can use the PORT command to change the direction of each port. When you configure a port as an input, you can use the $DI function to read the value of each individual input. When you configure a port as an output, you can use the $DO command to set the value of each individual input. Individual I/O bits are mapped to a given port. Refer to the following table.

Table:     Nstep I/O Bit Port Map

Port Number Bit Numbers

0

0 - 7

1

8 - 15
