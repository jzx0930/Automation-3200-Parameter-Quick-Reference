MOVEABS Command

Executes an absolute move on an axis.

Syntax:

 
MOVEABS <Axis> <Position> <Speed>

Example:

 
MOVEABS X 5 10

Table:     MOVEABS Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis to move.

<Position>

64-bit IEEE floating point

The absolute position to move to.

<Speed>

64-bit IEEE floating point

The speed, in user units/time base, at which to move an axis. The arithmetical sign is ignored and a speed of zero causes a task error.

Issue the MOVEABS command to generate a point-to-point move of the specified absolute distance for a single axis.

This is an asynchronous command which means that program execution continues to the next program line without waiting for the move to complete.

While a MOVEABS command is executing, you can issue a new MOVEABS, MOVEINC, or FREERUN command. The axis ramps up or down to the requested speed at the axis-based acceleration rate and changes the target distance to the distance specified by the new command. If the new command specifies a distance in the same direction as the current motion, the controller updates the target position of the current move to the new target. If the new command specifies a distance in the opposite direction of the current motion, the axis reverses direction and moves to the new target position without stopping.

Issue the ABORT command to stop motion before the target is reached on an axis that is executing a MOVEABS command.

Speed

The speed is limited by the MaxSpeedClamp parameter.

Ramping

The axis does not immediately get to the requested speed. Based on the current ramp values, the axis ramps up or down to the requested speed at the axis-based acceleration rate. When the axis gets to the specified distance, it ramps down based on the current deceleration ramp values. For more information, refer to Acceleration and Deceleration Ramping Overview.

NOTE: If the axis is already moving when this command is issued, the acceleration axis ramp is used to change speeds.

Status

When the axis generates motion, it sets the Jogging bit and the Wait Done bit of the AxisStatus status item. When the axis is done generating motion, it clears the Jogging bit and the Wait Done bit of the AxisStatus status item.

In addition to the "no longer generating motion" condition, there can be other conditions that must be set in order for the Wait Done bit to be set. For more information, refer to Motion Done Overview.

This command is an asynchronous motion command, which means that program execution immediately continues to the next program line before motion is complete. Use the WAIT MOVEDONE or WAIT INPOS command to wait for motion to complete.
