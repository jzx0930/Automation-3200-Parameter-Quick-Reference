PCI READ BYTE Function

Reads one byte of data from a specified card on the PCI bus.

Syntax:

 
<return> = PCI READ BYTE <CardHandle>, <ResourceID>, <AddressOffset>

Example:

 
$ByteValue = PCI READ BYTE $CardID, 0, 1

Table:     PCI READ BYTE Function Arguments

Argument

Type

Description

<CardHandle>

32-bit signed integer

The card handle (reference number) of the card on the PCI bus that you want to read. This card handle is returned by the PCI OPEN function.

<ResourceID>

32-bit signed integer

The PCI card resource to use to get the base address of the data that you want to read.

<AddressOffset>

64-bit IEEE floating point

The offset from the base address that is specified by the <ResourceID> argument.

RETURNS

32-bit signed integer

The byte of data from the PCI card.

Use the PCI READ BYTE function to read one byte of data from a specified card on the PCI bus.

The <ResourceID> argument specifies which resource of the card to use. A PCI card can claim memory or I/O resources. The value of the <ResourceID> argument starts at 0. For the next resource, increase the value by one.

Table:     Example of a PCI Card Resource Claim

Resource <ResourceID> Value

I/O Range: DC70 - DC77

0

I/O Range: DC80 - DCFF

1

Memory Range: FF6FF400 - FF6FF47F

2

The actual address from which the data is read is given by the following equation.

The following program uses the PCI resource claims from the previous example.

Example

' Declare the variables.
DVAR $ByteValue, $CardID
' Open communication with the PCI card with a vendor ID of 0x135E and device ID of 0x8008.
$CardID = PCI OPEN 0H135E 0H8008 0
' Read a byte from address 0xDC71.
$ByteValue = PCI READ BYTE $CardID 0 1
' Close communication with the card.
PCI CLOSE $CardID

Note: Do not give other Windows applications access to the cards that you open with the PCI OPEN function or the PCI Bus commands will not work correctly.
