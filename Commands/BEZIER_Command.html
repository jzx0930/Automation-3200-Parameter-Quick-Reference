BEZIER Command

Executes a Bezier move on the specified axes.

Syntax:

 
BEZIER QUAD <AxisH>, <P0_H>, <P1_H>, <P2_H>, <AxisV>, <P0_V>, <P1_V>, <P2_V>
 
BEZIER QUAD <AxisH>, <P0_H>, <P1_H>, <P2_H>, <AxisV>, <P0_V>, <P1_V>, <P2_V> TOLERANCE <Tolerance>
 
BEZIER CUBIC <AxisH>, <P0_H>, <P1_H>, <P2_H>, <P3_H>, <AxisV>, <P0_V>, <P1_V>, <P2_V>, <P3_V>
 
BEZIER CUBIC <AxisH>, <P0_H>, <P1_H>, <P2_H>, <P3_H>, <AxisV>, <P0_V>, <P1_V>, <P2_V>, <P3_V> TOLERANCE <Tolerance>

Example:

 
BEZIER CUBIC X, 0.000, 0.126, 0.367, 0.584, Y, 0.000, 0.278, -0.148, 0.112

Use the BEZIER command to generate synchronized motion of two axes along a quadratic or cubic Bezier curve.

Quadratic Bezier Curves

To create a quadratic Bezier curve, use the BEZIER QUAD syntax. You must specify all three control points (P0, P1, and P2) that define the quadratic Bezier curve. Specify the horizontal axis components of the control points with the <P0_H>, <P1_H>, and <P2_H> arguments. Specify the vertical axis components of the control points with the <P0_V>, <P1_V>, and <P2_V> arguments.

Figure: Quadratic Bezier Curve

Cubic Bezier Curves

To create a cubic Bezier curve, use the BEZIER CUBIC syntax. You must specify all four control points (P0, P1, P2, and P3) that define the cubic Bezier curve. Specify the horizontal axis components of the control points with the <P0_H>, <P1_H>, <P2_H>, and <P3_H> arguments. Specify the vertical axis components of the control points with the <P0_V>, <P1_V>, <P2_V>, and <P3_V> arguments.

Figure: Cubic Bezier Curve

For all types of Bezier curves, you must specify all Bezier control points in absolute coordinates and you must use the ABSOLUTE programming mode. Also, you must specify all control points as numerical values because the BEZIER command does not support program variables. The controller generates a task error if the value of the first control point (<P0_H> and <P0_V>) is not the same as the program position of the horizontal and vertical axes.

The controller approximates the Bezier motion to within a positional tolerance. To specify the tolerance, use one of the methods that follow.

  • Specify the TOLERANCE keyword and the <Tolerance> argument for each BEZIER command.
  • Issue the #bezier tolerance preprocessor directive near the start of your program to specify the tolerance in user units before you issue a motion command.

Set the tolerance to a positioning error that is sufficient for your application. Tolerance values that are too large can cause the fit to not be accurate. Tolerance values that are too small can cause computational burden on the controller.

Note: Programs that run in queue mode cannot use the #bezier tolerance preprocessor directive. If you are running programs in queue mode, use the BEZIER command with the TOLERANCE keyword to specify the tolerance. Refer to PROGRAM BUFFEREDRUN Command.

Use the F command to define the combined axis speed, also called coordinated speed, that is used for the motion.

Use the RAMP commands to configure the coordinated acceleration and deceleration. Refer to Acceleration and Deceleration Ramping Overview for more information.

The controller will automatically generate a continuous motion path for a single BEZIER command move, regardless of the VELOCITY ON or VELOCITY OFF setting. Use the VELOCITY ON command in your AeroBasic program to make sequential BEZIER command moves part of a continuous motion path. Because Bezier curves usually have many curvature changes, use the CoordinatedAccelLimit parameter and the CoordinatedCircularAccelLimit parameters to prevent over-acceleration of the axes.

Example

' Set the Bezier positional tolerance to 0.001 units.
#bezier tolerance 0.001

' To generate a cubic Bezier curve, use the control points that follow:
' P0={0.000, 0.000}
' P1={0.126, 0.278}
' P2={0.367,-0.148}
' P3={0.584, 0.112} 
BEZIER CUBIC X, 0.000, 0.126, 0.367, 0.584, Y, 0.000, 0.278, -0.148, 0.112
