FILEOPEN Function

Opens a data file or serial port.

Syntax:

 
<return> = FILEOPEN <FileName>, <Mode>

Example:

 
$hFile = FILEOPEN "COM2", 2
$hFile = FILEOPEN "C:\MyFolder\MyFile.dat", 0

Table:     FILEOPEN Function Arguments

Argument

Type

Description

<FileName>

String

The name (and path) of the user file or serial port to open. Refer to File Names for more information.

<Mode>

32-bit signed integer

Specifies the mode of access to the file when the file is opened. To see the possible values, refer to Table: <Mode> Argument Values.

RETURNS

32-bit signed integer

The file handle (reference number) of a file or serial port that you opened. This file handle is required by the FILEREAD function and the FILEWRITE command.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the FILEOPEN function to open a data file or serial port. The <FileName> is the full file path and file name of the file or serial port that you opened.

To open a file, specify a file name and file path for <FileName>. The file name and file path can be any valid Windows file name and file path (maximum 31 characters). If you do not specify an absolute file path in the file name, the file path is relative to the program that is currently executing.

To open a serial port, specify a communications port as the <FileName> ("COM1", "COM2", "COM3", and so on). To use a COM port that is greater than COM9, you must use a special syntax to specify the COM port.

For example

If you want to use COM10, specify the COM port as follows.

$hFile = FILEOPEN "\\.\COM10", 2

Use the COMMINIT Command to configure the serial port. To specify the timeouts for the reads from a serial port, use the COMMSETTIMEOUT Command.

The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value

Description

0

Opens a new file and overwrites the existing file.

1

Opens an existing file only for reading.

2

Opens an existing file only for appending records to the end of the file. You must open serial ports in mode 2.

The FILEOPEN function returns the file handle (reference number) that is assigned to the file when the file is opened. You must use the file handle as the value of the <FileHandle> argument for subsequent FILEREAD, FILEWRITE, and FILECLOSE commands. When you use the <FileHandle> argument, make sure that the applicable file is open.

This function sets the Info Variables if Windows finds an error while it is opening the file. All other error conditions set the Task Error.
