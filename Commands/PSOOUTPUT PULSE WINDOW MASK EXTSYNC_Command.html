PSOOUTPUT PULSE WINDOW MASK EXTSYNC Command

Sets the PSO output mode.

Syntax:

 
PSOOUTPUT <Axis> PULSE WINDOW MASK EXTSYNC
 
PSOOUTPUT <Axis> PULSE WINDOW MASK HARD EXTSYNC
 
PSOOUTPUT <Axis> PULSE WINDOW MASK EDGE <EdgeMode> EXTSYNC
 
PSOOUTPUT <Axis> PULSE WINDOW MASK HARD EDGE <EdgeMode> EXTSYNC

Example:

 
PSOOUTPUT X PULSE WINDOW MASK EXTSYNC

Table:     PSOOUTPUT PULSE WINDOW MASK EXTSYNC Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<EdgeMode>

32-bit signed integer

(Optional) Specify pulse output behavior. This argument is required if you use the EDGE keyword.

Hardware: This command does not apply when used with the Ndrive CP, MP, or QL, the Nmark SSaM, the Npaq, the Npaq MR with MP drives, the Nservo, the Nstep, or the XC2.

Use the PSOOUTPUT PULSE WINDOW MASK EXTSYNC command to configure the PSO output to be generated from the PSO pulse generator, masked by the state of the PSO window output. You must first configure the pulse generator by using the PSOPULSE command. No pulses will be output unless firing events are being generated via the PSODISTANCE command or firing is triggered manually using the PSOCONTROL FIRE command.

If the window output is inactive, the PSO firing event is masked or suppressed, which prevents the pulse generator from being triggered. If the window output is active, the firing event occurs and triggers the pulse generator. Once a pulse sequence begins, it will run to completion, even if the window output becomes inactive. If the optional HARD keyword is specified, the pulse sequence will be immediately truncated as soon as the window output becomes inactive.

The optional EDGE keyword and <EdgeMode> argument allow the pulse generator to be triggered when entering or exiting the window, in addition to the triggering defined by the PSODISTANCE command. The table that follows shows the possible values of the <EdgeMode> argument and how these values have an effect on pulse generation.

Table: <EdgeMode> Argument Values

Mode

Description

0

No pulse output.

1

The pulse output is triggered on exit from window.

2

The pulse output is triggered on enter into window.

3

The pulse output is triggered on both enter into and exit from window.

When a firing event triggers the pulse generator, the pulse generator waits to generate the pulse output until after the external synchronization signal transitions high. This applies to distance-based firing events that you configure with the PSODISTANCE Command. It also applies when you use the EDGE keyword and <EdgeMode> argument to trigger the pulse generator when the window counter enters or exits a window range.
