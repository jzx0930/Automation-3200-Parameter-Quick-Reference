SCOPETRIG Command

Starts or stops data collection in a front-end application such as Digital Scope.

Syntax:

 
SCOPETRIG
 
SCOPETRIG CONTINUOUS
 
SCOPETRIG ID <ApplicationID>
 
SCOPETRIG CONTINUOUS ID <ApplicationID>
 
SCOPETRIG STOP

Example:

 
SCOPETRIG
SCOPETRIG STOP

Table:     SCOPETRIG Command Arguments

Argument

Type

Description

<ApplicationID>

32-bit signed integer

(Optional) Specifies which application responds to the SCOPETRIG command. If you do not specify a value then Digital Scope responds to the SCOPETRIG command. Values 0-99 are reserved for Aerotech use and should not be used.

Use the SCOPETRIG command to trigger data collection in the desired application. The application will collect one sample based on its current settings. This is useful for collecting a sample at a specific location of interest in a motion program.

Use the SCOPETRIG CONTINUOUS command to have the application collect multiple, consecutive samples. The application will continue to collect data until you use the SCOPETRIG STOP command to stop data collection.

If the target application is not open when SCOPETRIG is executed, the command is ignored. Use the SCOPEOPEN command to open Digital Scope.

With the optional ID argument you can control which program will respond to data collection on the controller. Aerotech programs (such as the Digital Scope) will not respond to data collection on the controller if a different program is specified in the SCOPETRIG command. For your convenience, you can use the following constants to specify the desired application.

  • SCOPETRIG_ID_Default (triggers the Digital Scope)
  • SCOPETRIG_ID_DigitalScope
  • SCOPETRIG_ID_MotionSimulator

You can use the optional ID argument to allow user created .NET, C, LabVIEW, or MATLAB programs to respond to data collection on the controller in place of the Digital Scope. Because the Digital Scope will not respond to data collection when you issue, for example, a SCOPETRIG ID 150 command, you can write your .NET, C, LabVIEW, or MATLAB program to respond when a SCOPETRIG ID 150 is issued, ensuring that no conflicts will occur.

Note: When you design user programs to respond to SCOPETRIG ID, do not use 0-99 for the <ApplicationID>. Values 0-99 are reserved for use by Aerotech.
