FILEREADARRAYBINARY Function

Reads data from a file or serial port into variables on the controller.

Syntax:

 
<return> = FILEREADARRAYBINARY <FileHandle> <DataType> <ControllerVariable>, <NumVariables>

Example:

 
$nItems = FILEREADARRAYBINARY $fil BYTE $task[10] 50

Table:     FILEREADARRAYBINARY Function Arguments

Argument

Type

Description

<FileHandle>

32-bit signed integer

The file handle (reference number) of a file or serial port for which you want to read data. This file handle is returned by the FILEOPEN function.

<DataType>

Other

The data type of the data to read.

<ControllerVariable>

64-bit IEEE floating point variable

The name of the first controller variable to write to.

<NumVariables>

32-bit signed integer

The number of variables to read from the file or serial port.

RETURNS

32-bit signed integer

The number of items read.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the FILEREADARRAYBINARY function to read bytes of data from a file or serial port into variables on the controller. To specify the file or serial port to read, use the file handle (reference number) of that file or serial port as the value of the <FileHandle> argument.

The settings of the <DataType> argument control how the data is read from the file or serial port. To see the possible values of the <DataType> argument, refer to Data Type Keywords.

Each element in the file or serial port data is read into a separate variable on the controller, starting with the <ControllerVariable> argument.

If the file or serial port data contains less elements than the number that you specified in the <NumVariables> argument, then all of the elements in the file or serial port data are copied into variables, and the value of the <return> argument is less than the value of the <NumVariables> argument. You must allocate a sufficient number of variables for the number of global variables that you want to read.

This function sets the Info Variables if Windows finds an error while it is reading from the file or serial port. All other error conditions set the Task Error.
