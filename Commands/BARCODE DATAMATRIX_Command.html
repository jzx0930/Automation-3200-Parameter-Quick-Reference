BARCODE DATAMATRIX Command

Encodes a string into the binary representation of a Data Matrix ECC 200 barcode.

Syntax:

 
BARCODE DATAMATRIX <StringToEncode>, <EncodingType>, <EncodingFormat>, <BarcodeOptions>, <OutputArray>, <NumberOfRows>, <ItemsPerRow>

Example:

 
BARCODE DATAMATRIX "Aerotech", DATAMATRIXENCODING_Ascii, DATAMATRIXFORMAT_Auto, BARCODEOPTIONS_None, $OutputArray, $NumberOfRows, $ItemsPerRow

Table:     BARCODE DATAMATRIX Command Arguments

Argument

Type

Description

<StringToEncode>

String

The string to encode.

<EncodingType>

32-bit signed integer

The type of encoding to use. Select the encoding based on the input data.

<EncodingFormat>

32-bit signed integer

The format of the encoding to use. The format determines the size of the output data matrix.

<BarcodeOptions>

32-bit signed integer

A bitwise set of options for encoding the data.

<OutputArray>

64-bit IEEE floating point variable

The double array variable where the output data is kept.

<NumberOfRows>

32-bit signed integer variable

The variable that is used to keep the number of rows of encoded data.

<ItemsPerRow>

32-bit signed integer variable

The variable that is used to keep the number of items (variables) per row of encoded data.

Use the BARCODE DATAMATRIX command to encode a string into the binary representation of a Data Matrix ECC 200 barcode. The encoded data is kept in the <OutputArray> variable and is in the correct format that is necessary for the PSOOUTPUT BIT MAP command, where each bit in the output array corresponds to one of the cells in the 2D (two-dimensional) barcode. The encoding algorithm adds one row of quiet space (padding) to the top and bottom of the barcode and one column of quiet space to the left and right of the barcode.

<EncodingType>

Specify the <EncodingType> based on the type of data that is in the input string. When you use the DATAMATRIXENCODING_Auto option, the BARCODE DATAMATRIX command selects the optimal encoding type.

Example Program

The output array is formatted for use with the PSO. Use the ARRAY WRITE command to download the encoded data to the drive array. The example program that follows shows you how to configure the PSO for the GL4, the Nmark CLS, and the Nmark GCL. It also shows you how to download the barcode information to the PSO array and sequence the marking motion.

Example

DVAR $OuterLoopIndex
DVAR $InnerLoopIndex
DVAR $OutputArray[100]
DVAR $NumberOfRows
DVAR $ItemsPerRow
DVAR $PassesPerRow
DVAR $RowWidth

' Size of a single barcode cell in user units.
#define CellSize 0.6

' Spot size of the laser used to mark the barcode.
#define LaserSpotSize 0.02

' Configure the PSO for fixed-distance firing.
PSOCONTROL X RESET
PSOTRACK X INPUT 0
PSOOUTPUT X CONTROL 0 1
PSODISTANCE X FIXED (ABS(UNITSTOCOUNTS(X, CellSize)))

' Encode the barcode.
$strglob[0] = "Aerotech"
BARCODE DATAMATRIX $strglob[0], DATAMATRIXENCODING_Auto, DATAMATRIXFORMAT_Auto, BARCODEOPTIONS_InvertPolarity, $OutputArray, $NumberOfRows, $ItemsPerRow

' Calculate the distance that you must travel to traverse a single row.
$RowWidth = 32 * $ItemsPerRow * CellSize

' Calculate the number of passes that are necessary to mark a single row.
$PassesPerRow = (CellSize / LaserSpotSize)

' Do the marking motion.
INCREMENTAL
CRITICAL START

FOR $OuterLoopIndex = 0 TO $NumberOfRows - 1

        ' Write the marking sequence for this row to the PSO array.
        ARRAY X WRITE $OutputArray[$OuterLoopIndex * $ItemsPerRow] 0 $ItemsPerRow
		
        FOR $InnerLoopIndex = 0 TO $PassesPerRow - 1
	    
                ' Configure the PSO.
                PSOOUTPUT X BIT MAP 0 $ItemsPerRow 1
                PSOCONTROL X ARM 
		
                ' Mark the pattern.
                LINEAR X $RowWidth
                WAIT INPOS X
		
                ' Index move for the next row.
                RAPID X -$RowWidth Y -LaserSpotSize
                WAIT INPOS X Y
		
        NEXT $InnerLoopIndex
NEXT $OuterLoopIndex

CRITICAL END

END PROGRAM
