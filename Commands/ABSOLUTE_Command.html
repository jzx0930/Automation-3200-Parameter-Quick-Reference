ABSOLUTE Command

Sets motion commands to be in absolute mode.

Syntax:

 
ABSOLUTE

Example:

 
ABSOLUTE

Use the ABSOLUTE command to have the controller interpret move target positions as absolute. Absolute mode means that move target positions are relative to a zero location established by homing or a POSOFFSET command. Absolute mode is set by default but can be changed at any time during program execution. This command is modal and stays in effect until incremental mode is specified.

You can set the default mode used by the task by using the DefaultProgrammingMode parameter.

The absolute programming mode state is indicated by Bit 1 of the TaskMode data item of the TASKSTATUS Function.

If you move an axis with a nonzero value for the RolloverCounts axis parameter in ABSOLUTE mode, the target position of the axis is adjusted to within the RolloverCounts value. The axis travels the shortest distance to the target position.

The example program that follows executes as if the axes start at position [0, 0].

Example

' Enable absolute programming mode.
ABSOLUTE
' Set the feedrate for subsequent moves.
F100.
' Move the X and Y axes to absolute coordinate 10,10.
LINEAR X10.0 Y10.0
' Move the X and Y axes to absolute coordinate 15,25.
LINEAR X15.0 Y25.0
' Move the X and Y axes to absolute coordinate 15,10.
LINEAR X15.0 Y10.0
