VME WRITE Command

Writes a value to a specified VME register address.

Syntax:

 
VME WRITE <Address>, <Value>, <Mode>

Example:

 
VME WRITE 0h110000, 0h400, VME_32_Bit_Access

Table:     VME WRITE Command Arguments

Argument

Type

Description

<Address>

64-bit IEEE floating point

The address of the VME register that you want to write to.

<Value>

64-bit IEEE floating point

The value that you want to write to the register address.

<Mode>

Other

The mode that you want to use the write to the register address. To see the possible values, refer to Table: <Mode> Argument Values.

Use the VME WRITE command to write a value to a specified VME register address.

You can use the <Mode> argument to specify if you want to write in 8-bits, 16-bits, or 32-bits. The values of the <Mode> argument are constants which are defined in aerparam.pgm. The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value Description

VME_32_Bit_Access

Writes to the VME register address in 32-bits.

VME_16_Bit_Access

Writes to the VME register address in 16-bits.

VME_8_Bit_Access

Writes to the VME register address in 8-bits.

NOTE: By default, support for a controller-level VME card is disabled. To enable support for a VME card, use the ENABLEVME argument. Refer to SMC Arguments. Or, you can communicate with a VME card by using the APIs that are supplied by the card vendor.
