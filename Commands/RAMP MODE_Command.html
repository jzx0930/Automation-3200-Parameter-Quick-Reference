RAMP MODE Command

Specifies the ramp mode calculation type to use.

Syntax:

 
RAMP MODE <Mode>
 
RAMP MODE <Mode> <Axis> [ <Axis> ... ]

Example:

 
RAMP MODE RATE X
RAMP MODE RATE

Table:     RAMP MODE Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Mode>

Other

The ramp mode to use. To see the possible values, refer to Table: <Mode> Argument Values.

Use the RAMP MODE command to specify rate-based or time-based acceleration and deceleration for coordinated motion and axis-based motion.

If you specify an axis, the specified <Mode> will be applied to axis-based (non-coordinated) motion on that axis. If you do not specify an axis, the specified <Mode> will be applied to coordinated motion. The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value Description

RATE

The currently active acceleration and deceleration parameters specify the vector acceleration and deceleration.

TIME

The currently active acceleration time and deceleration time parameters, along with the programmed feedrate, specify the vector acceleration and deceleration.

The active mode state for coordinated motion is indicated by Bit 3 of the TaskMode status item.

The DefaultRampMode parameter sets the default ramp mode.

Example

' Specify rate-based mode for the X axis.
RAMP MODE RATE X
' Specify rate-based mode for coordinated motion.
RAMP MODE RATE
' Specify time-based mode for the X and Y axes.				
RAMP MODE TIME X Y
' Specify time-based mode for coordinated motion.
RAMP MODE TIME
