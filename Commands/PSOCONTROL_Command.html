PSOCONTROL Command

Enables and disables the PSO hardware.

Syntax:

 
PSOCONTROL <Axis> <Mode>

Example:

 
PSOCONTROL X ON

Table:     PSOCONTROL Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Mode>

Other

The mode of operation of the PSO hardware.

Hardware: This command does not apply when used with the Nmark SSaM or the Nstep.

Use the PSOCONTROL command to cause immediate actions on the PSO, including readying for pulse generation (ARM), turning off pulses (OFF), software triggered pulse generation (FIRE), and resetting the hardware to the default state (RESET). Typically, the first command issued when configuring the PSO should be PSOCONTROL RESET and the last command should be PSOCONTROL ARM. Many of the PSO commands do not have an effect until the PSOCONTROL ARM command is issued.

Hardware: If you are using an Npaq (not including the Npaq MR), make sure that the PSO interlock is driven or disabled by setting the IOSetup axis parameter. For more information, refer to the hardware manuals for your drives.

The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value Description Description

ARM

Enables PSO tracking.

Use the ARM argument as the last command in a sequence of PSO commands. It causes all previously specified PSO commands to activate and allows the PSO hardware to begin tracking the axis positions.

This argument also causes the distance counters to be loaded with the user specified values. Because this creates the reference point for PSO counters, make sure that the servo is "in position" before you issue this command. This can be done by putting a delay ( DWELL ) command in the program or by checking axis in position status.

This argument can be used after a PSOCONTROL OFF command to allow the PSO hardware to resume pulse generation. The PSOTRACK RESET command controls what occurs on the counters during the OFF state.

ARM GALVO

Enables manual firing.

Hardware: This command applies only when used with the GL4 or the Nmark CLS or GCL.

Use the ARM GALVO argument as the last command in a sequence of PSO commands. The ARM GALVO argument activates all of the previously-specified PSO commands. Then you can manually fire the PSO pulse generator.

To configure the PSO pulse generator to manually fire, issue the PSOOUTPUT PULSE LASER MASK Command, which configures the pulse generator to be triggered by the laser status.

  • If you want the laser to fire while the axis is not moving, then manually fire the pulse generator by issuing the GALVO LASEROVERRIDE ON Command, then a MOVEDELAY Command of at least 1, and then a GALVO LASEROVERRIDE OFF Command.
  • If you want the laser to fire while the axis is moving, issue the GALVO LASEROVERRIDE ON Command and GALVO LASEROVERRIDE OFF Command to control the firing during your profile.

When you use this argument, the distance counters are held in reset so that if you previously configured a PSO tracking mode, tracking is no longer enabled.

ARM PWM

Enables PSO tracking.

Hardware: This command applies only when used with the Ndrive HLe, HPe, or ML.

Use the ARM PWM argument to do the same functions as the PSOCONTROL ARM command, such as activating all previous PSO commands and allowing position tracking to begin, but with the following additional functionality.

When you use this argument in conjunction with the PSOOUTPUT PULSE BIT MASK command, the drive generates fixed frequency output pulses (not dependent on axis position) with a duty cycle that can be changed at each firing event by values stored in the PSO array. The pulse output is also masked with the state of the bit in each array location, just as in the normal BIT MASK case. Note that the masking of the output and changes to the duty cycle will not take effect until the completion of the current pulse and, therefore, might not occur at the exact positions that a normal firing event would occur.

When the PSOCONTROL ARM PWM command is issued, the PSO output immediately begins pulsing using the specifications given in the PSOPULSE TIME command. (The optional CYCLES argument is ignored, however). It continues to pulse until you turn it off with the PSOCONTROL OFF command. To prevent the PSO output from pulsing until the first firing event, specify the first bit of the BIT MASK values as zero. You can initiate changes to the duty cycle and the mask bit at either fixed distances (using PSODISTANCE FIXED) or variable distances (using PSODISTANCE ARRAY).

FIRE

Fires one or a sequence of pulses.

 Use the FIRE argument to cause the pulse generator to produce one pulse or pulse sequence under software control.

FIRE CONTINUOUS

Fires one or a sequence of pulses.

 Use the FIRE CONTINUOUS argument to cause the pulse generator to repeatedly generate pulses or pulse sequences until the PSOCONTROL OFF command is given.

ON

Turns on PSO output.

Use the ON argument to immediately set the PSO output signal to its active state until a PSOCONTROL OFF command is given. No pulse generation occurs in this mode and the output is continuously on.

OFF

Turns off PSO output.

Use the OFF argument to immediately set the PSO output signal to its inactive state. The PSOTRACK RESET command controls what occurs on the internal counters when the OFF command is given. The inactive state is a hardware fail safe level defined in the individual drive manuals; typically this is the high impedance state.

To resume operation, issue a PSOCONTROL ARM command.

PULSE DISABLE

Prevents all applicable sources from triggering the pulse generator.

Use the PULSE DISABLE value to prevent all applicable sources from triggering the pulse generator. When you use this value, the pulse generator completes all pulses that it is currently generating. But it does not generate new pulses. Examples of sources that can trigger the pulse generator include firing events and when the window counter enters or exits a window range.

To start generating new pulses again, issue the PSOCONTROL ARM value.

RESET

Reverts all PSO states to the default condition

Use the RESET argument to set all PSO states to their default conditions and remove all PSO data from the internal hardware FIFO buffers. This argument should be used to restart PSO operations after faults or other process interruptions occur.
