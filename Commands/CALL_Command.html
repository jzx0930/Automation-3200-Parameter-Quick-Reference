CALL Command

Calls a subroutine that is defined in the program.

Syntax:

 
CALL <Subroutine>
 
CALL <Subroutine> <ArgumentName> <ArgumentValue> [  <ArgumentName> <ArgumentValue> ... ]

Example:

 
CALL MYSUB X 78 Y $global[8]

Table:     CALL Command Arguments

Argument

Type

Description

<Subroutine>

String

The name of the subroutine that you want to call.

<ArgumentName>

Other

(Optional) The name of the argument that you want to pass to the subroutine.

<ArgumentValue>

64-bit IEEE floating point

(Optional) The value of the argument that you want to pass to the subroutine.

The CALL command calls a specified subroutine that is defined in the program file. When the program returns from the subroutine, execution resumes at the program line that follows the CALL command. You can pass values to subroutines by using arguments.

The number of subroutines that you can nest is limited by the CallStackSize parameter. You cannot use string variables as the subroutine name in the CALL command. But, you can use string variables as the subroutine name in the FARCALL command.

When a program returns from a subroutine, the modal settings of the program are not restored. To save the modal status of the program, use the PUSHMODES command before you call the subroutine and use the POPMODES command after the subroutine.

For example

Say that in your program you issue an ABSOLUTE command and then you call a subroutine that executes the INCREMENTAL command. When the program returns from the subroutine, INCREMENTAL mode is active.

If you use the PUSHMODES and POPMODES commands, when the program returns from the subroutine, ABSOLUTE mode is active.
