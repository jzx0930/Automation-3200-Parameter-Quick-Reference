CAMSYNC Command

Starts synchronizing the master axis to the slave axis.

Syntax:

 
CAMSYNC <SlaveAxis>, <TableNumber>, <SyncMode>

Example:

 
CAMSYNC Y, 0, 1

Table:     CAMSYNC Command Arguments

Argument

Type

Description

<SlaveAxis>

Axis

The axis that is set as the slave axis.

<TableNumber>

32-bit signed integer

The table number, from 0 to 99.

<SyncMode>

Other

The synchronization mode to use for camming.

Table:     Synchronization Modes

Mode

Description

0

Removes synchronization of a slave axis from a cam table. If the cam table is not currently synchronized to the slave, nothing is done.

1

Enables relative synchronization. If the slave axis is not at the position that corresponds to the current master axis position before synchronization occurs, the controller continues to operate as if the difference is an offset that applies to all points in the table.

For Example

The current master axis position is m1 and the cam table specifies that at m1 the slave axis position must be at s1. But the slave axis position is s2 before the controller executes the CAMSYNC command.

In this mode, no moves occur before synchronization. But the controller adds the value (s2-s1) to all of the slave axis positions in the cam table.

2

Enables absolute synchronization. If the slave axis is not at the position that corresponds to the current master axis position before synchronization occurs, the controller executes a RAPID adjustment move on the slave axis to move it to the correct absolute position.

For Example

The current master axis position is m1 and the cam table specifies that at m1 the slave axis position must be at s1. But the slave axis position is s2 before the controller executes the CAMSYNC command.

In this mode, the controller automatically does a RAPID move from s2 to s1 before synchronization occurs.

3

Enables velocity mode. In this mode, the controller reads the slave axis positions as velocities as an alternative to positions. The values that you specify for the master axis positions have an effect on how the cam table controls slave axis velocity. The synchronization occurs in relative mode.

For Example

The current master axis position is m1 and the cam table specifies that at m1 the slave axis velocity must be at the velocity v1. But the slave axis velocity is v2 before the controller executes the CAMSYNC command.

In this mode, no moves occur before synchronization. But the controller adds the value (v2-v1) to all of the slave axis velocities in the cam table. In the cam table, you must specify the slave axis velocities in units/millisecond for the Slave Units keyword at the start of the cam table. Refer to Cam Table Format.

Use the CAMSYNC command to synchronize a master axis to a slave axis for camming motion. Before you use this command, you must use the LOADCAMTABLE or LOADCAMVAR command to configure the cam table and master axis. Refer to Camming Motion for more information.

Warning: Before you use the CAMSYNC command to remove synchronization, use the ABORT Command on the master axis to make sure that the master axis is not moving. Then, make sure that the velocity is zero for the master axis and for the slave axis. If you remove synchronization from a slave axis that is moving, then the slave axis will suddenly stop.

WARNING: Acceleration and deceleration ramping does not apply to slave motion. When synchronization is enabled or removed, the slave axis immediately tries to follow the master axis. If the movement of the master axis implies that the slave axis should also be moving, the slave axis tries to immediately move at the specified speed and starts to jerk. When you issue the CAMSYNC command, make sure that the master axis is not moving or configure the cam table to prevent the slave axis from moving at the current master axis position.
