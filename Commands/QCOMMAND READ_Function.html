QCOMMAND READ Function

Reads the raw position and velocity command from the command queue of a task.

Syntax:

 
<return> = QCOMMAND READ <Task>, <Axis> [ <Axis> ... ], <PosArray>, <VelArray>, <Time>
 
<return> = QCOMMAND READ <Task>, <Axis> [ <Axis> ... ], <PosArray>, <VelArray>, <Time> TARGET <TargArray>
 
<return> = QCOMMAND READ <Task>, <Axis> [ <Axis> ... ], <PosArray>, <VelArray>, <Time> TARGET <TargArray> DATA <MiscArray>

Example:

 
$global[0] = QCOMMAND READ 1, X Y, $PosArray, $Velrray, 2

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Task>

The task on which to execute the command.

<PosArray>

String

The position data, in counts, of the motion profile of the specified axes.

<VelArray>

String

The velocity data, in counts per millisecond, of the motion profile of the specified axes.

<Time>

64-bit IEEE floating point

The time, in milliseconds.

<TargArray>

64-bit IEEE floating point

(Optional) The target position of the move, in primary units. This position is zero for points that are generated by PVT commands.

<MiscArray>

64-bit IEEE floating point

(Optional) To see the possible values, refer to Table: <MiscArray> Argument Values.

RETURNS

other

The variable that stores the return value. The value is either a string or a 64-bit IEEE floating point value.

Warning: If you use this command incorrectly, the axis can jerk and might cause damage to the equipment. Before you use this command on physical hardware, test it with simulated motion. Refer to Simulation of Motion.

Aerotech recommends that only advanced users use this command. Before you read this topic, read the following.

  • QCOMMAND Features and Limitations
  • Transformed Motion example program

Use the QCOMMAND READ function to get the raw position and velocity command from the command queue of a task. Use the QCOMMAND READ function with QCOMMAND WRITE to change or transform the motion profile in the command queue entry of a task.

Note: This function applies only to motions that use the queue of a task, which include RAPID, LINEAR, CW, CCW, and PVT motion.

The function returns a value that identifies a one entry that is a specific motion point in the queue. The value is returned in the variable that you specify in the <return> argument. Each task has a separate command queue. Thus, if axes X and Y are the input axes and are being commanded on task 1, the value of the <Task> argument must be set to 1.

The <Axis> arguments specify each axis that is used as the output of the transformation. If no valid queue entry is available for any of the specified axes, or if none of the specified axes are enabled, then the command returns a value of -1, the transforms do not occur and the <PosArray>, <VelArray>, <Time> variables are not assigned.

The following figure shows details of the positions and velocities that are returned. The red line is the profile path for a given axis. The return values are as follows, where x is the number for the specified axis.

<Time>   (T1-T0)

<PosArray>[x]  P1 (the area of the shaded region in the figure)

<VelArray>[x]  V1

The V0 is the <VelArray>[x] from the previous call to QCOMMAND READ.

The <Time> argument normally reflects the MotionUpdateRate parameter value that was active when the profile point was generated ( <Time> = 1.0 / MotionUpdateRate).

The following table shows the data in the <MiscArray> argument. Values [2] and above are zero during RAPID and PVT commands.

Note: The passed array must be at least 32 items long because the controller always writes 32 items into the <MiscArray> array. Make sure that you declare the array variables large enough to hold the returned data.

Table:     <MiscArray> Argument Values

Value

Description

0

User line (normally the line in the program file being executed is zero when executing immediate commands)

1

A mask of Motion flags, as follows.

0x01

First point in profile is executing.

0x02

RAPID motion is executing.

0x04

LINEAR motion is executing.

0x08

CW or CCW motion is executing.

0x10

PVT motion is executing.

2

0.0, except for CW and CCW motion when it is the start angle of the move in radians PI to -PI .

3

0.0, except for CW and CCW motion when it is the end angle of the move in radians PI to -PI.

4

Distance in this move so far (0 at beginning of move) is always in primary units.

5

Target velocity of this move (F/E command) times the MFO is always in primary units/sec.

6

Length of this entire move along path (0 at beginning of move) is always in primary units.

7

Percent complete of this move, based on path length 0 to 100 percent.

The data in the <PosArray>, <VelArray>, and <TargArray> array variables is not packed. For example, if the axis mask is X Z and you have not changed the default axis names, then <PosArray>[0] is the X position and <PosArray>[2] is the Z position.
