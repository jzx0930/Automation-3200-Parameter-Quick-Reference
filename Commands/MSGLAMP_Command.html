MSGLAMP Command

Shows text as illuminated text lamps in the Messages tab of the interface.

Syntax:

 
MSGLAMP <LampNumber>, <Color>, <ArgumentList>

Example:

 
MSGLAMP 1, RED, "Warning !"

Table:     MSGLAMP Command Arguments

Argument

Type

Description

<LampNumber>

64-bit IEEE floating point

The number of the lamp that you want to illuminate. The lamps are numbered 1 through 6.

<Color>

Other

The background color for the lamp. You can specify an RGB color value, or you can use one of the following predefined colors. GRAY, BLACK, RED, BLUE, MAGENTA, GREEN, YELLOW, CYAN, WHITE.

<ArgumentList>

Other

A mixture of comma-separated integer, double, and string arguments. The controller concatenates these arguments into one text string. The string shows in the text lamp. Thus, it is limited by the size of the display area in the lamp.

Use the MSGLAMP command to show warnings or other information as illuminated text lamps in the Messages tab of the operator interface. Use this command only for CNC Operator Interface, Motion Composer or Motion Simulator. If you use this command in Motion Composer or the Motion Simulator, you get a message that the command executed, but the command stays inactive. The illuminated text lamps do not show.

Figure: Lamps in the Operator Interface

Note: Motion Composer, CNC Operator Interface, Motion Simulator, or custom application must be open for this command to work. By default, Motion Composer, CNC Operator Interface, and Motion Simulator all try to register for these callbacks. The first application that registers will get the callbacks. The next application that tries to register will get an error. You can change the default behavior in Preferences of each application.
