WAIT MODE MOVEDONE Command

Sets the wait mode to move done.

Syntax:

 
WAIT MODE MOVEDONE

Example:

 
WAIT MODE MOVEDONE

Use the WAIT MODE MOVEDONE command to set the wait mode to move done. In this mode, the controller does not wait for in position before continuing to the next program block. However, the task will wait for the final velocity command sent to the drive to be zero before continuing to the next program block. The wait mode results in a one to three millisecond delay between consecutive moves.

When you issue this command, the controller sets the modal status of this command by setting the Wait For In Position bit of Task Mode to False and the Wait Auto bit of Task Mode to False.
