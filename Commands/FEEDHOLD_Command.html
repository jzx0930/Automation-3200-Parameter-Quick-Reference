FEEDHOLD Command

Activates and releases software feedhold control of a task.

Syntax:

 
FEEDHOLD ON
 
FEEDHOLD <Task> ON
 
FEEDHOLD OFF
 
FEEDHOLD <Task> OFF

Example:

 
FEEDHOLD ON
FEEDHOLD 2 ON

Table:     FEEDHOLD Command Arguments

Argument

Type

Description

<Task>

TaskID

(Optional) The task on which to activate or release feedhold. If you do not specify a task, the current task is used.

Use the FEEDHOLD command to supply software feedhold control of a task. A value of ON activates feedhold and a value of OFF releases feedhold.

This action is logically ORed with the state of the hardware feedhold inputs (as defined by the FeedholdInput and FeedholdSetup parameters). The feedhold state will not release until all feedhold conditions are false.

Feedhold does not have an effect on camming (CAMSYNC command) or HOME command motion.

Linear acceleration or deceleration is always used for MFO changes and feedhold state changes.
