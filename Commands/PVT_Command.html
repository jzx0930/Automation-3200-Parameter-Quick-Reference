PVT Command

Executes a vectorial time point move on the specified axes.

Syntax:

 
PVT <Axis> <Position>, <Velocity> [ <Axis> <Position>, <Velocity> ... ] TIME <Time>

Example:

 
PVT X 1.000, 0.000 Y 2.000, 0.000 TIME 1

Table:     PVT Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Position>

64-bit IEEE floating point

If the task is in ABSOLUTE mode, this argument specifies the absolute position to move to, in user units. If the task is in INCREMENTAL mode, this argument specifies the distance to move, in user units.

<Velocity>

64-bit IEEE floating point

The velocity, in user units/time base, at which to be moving at the end of the time interval. Use the SECONDS and MINUTES commands to specify which time base the controller uses.

<Time>

64-bit IEEE floating point

The total time of the move, in milliseconds.

Warning: To give more control to the programmer, this command removes some of the features that reduce jerk or unacceptable motion.

Use the PVT command to manually enter profile points from a program to make arbitrary velocity or acceleration profiles for a single or multiple axes. This command bypasses all automatic acceleration, and acceleration limiting used in RAPID, LINEAR, CW, CCW, BEZIER, MOVEABS, MOVEINC, and FREERUN command moves.

P is the position at the end of the time segment in user units. If the ABSOLUTE task mode is enabled, this value is entered as an absolute position. If the INCREMENTAL task mode is enabled, this value is entered as an incremental distance from the end of the last move.

V is the axis velocity at the end of the time segment and is in user units. The velocity must be analytic or the interpreted velocities are not continuous. If you do not have an analytic velocity, then use the PT Command.

T is the time interval in milliseconds. To specify a move duration of one millisecond or longer, enter any positive integer. To specify a move duration of less than a millisecond, enter 1/2, 1/4, 1/8, 1/16, 1/24, or 1/48. Do not enter any other fractional values for the time.

Note: If you specify a move duration of less than a millisecond for <Time>, all of the PVT moves in that same sequence must use the same <Time>.

If you specify a move duration for <Time> that is greater than 1.0 divided by the position update rate of an axis, the controller interpolates motion to the position update rate. To interpolate motion, the controller uses the cubic-spline polynomial algorithm that you specified for the MotionInterpolationMode Parameter. For more information, refer to the table that follows.

PVT commands do not behave like the other synchronous or asynchronous motion commands. After a PVT command is processed, the program immediately executes the next line in the program, much like an asynchronous motion command (MOVEINC, MOVEABS, FREERUN). However, unlike an asynchronous motion command, the PVT motion commands are buffered and executed one after another. If a stream of PVT commands is executed sequentially, it is possible to fill this buffer. If the buffer is full, subsequent PVT commands wait on the line until buffer space becomes available.

If you want your program to wait until PVT motion is complete before it continues, you must issue the WAIT MOVEDONE Command or the WAIT INPOS Command. Make sure that you do not use the WAIT MODE MOVEDONE command or the WAIT MODE INPOS command because PVT does not obey the setting of the mode commands. For more information, refer to Motion Done Overview.

PVT does not obey Acceleration and Deceleration Ramping. When you specify positions and velocities in a sequence of PVT commands, you must gradually change the axis positions and velocities from command to command to make smooth motion.

Warning: If the final PVT command in a sequence does not have a velocity of zero, the velocity instantaneously changes at the end of the command. This causes the axis to jerk.

When the controller combines PVT with LINEAR, CW, CCW, or BEZIER moves, the controller always decelerates to a stop on the LINEAR, CW, CCW, or BEZIER move that comes before the PVT move. Similarly, a PVT move that immediately comes before a LINEAR, CW, CCW, or BEZIER move must produce a zero velocity or the axis jerks.

Warning: To avoid velocity dropouts, enclose sequential PVT commands in a CRITICAL START and CRITICAL END pair.

PVT Limitations

  1. This command does not work with any transformation.
  2. The controller does not look ahead in the PVT mode and does not slow down for corners.
  3. MFO has no effect on PVT motion.
  4. MaxSpeedClamp parameter has no effect on PVT motion.
  5. Feedhold does not decelerate in PVT motion. It finishes the PVT command and waits until the queue is flushed. Then motion abruptly stops.

Example

'******************************************************
' Example PVT program.
'
' This program shows you how to use the PVT command on the A3200 
' controller. You can use the PVT command to generate arbitrary move 
' profiles. Each PVT command generates a move segment in a specified 
' time period of 1 ms or multiples of 1 ms. Move times greater than 
' 1 ms are curve fit with a 3rd order polynomial based on the 
' position and velocity specified in the PVT command. The ending 
' velocity of each segment is used as the starting velocity for 
' the next segment.
'
' You can directly calculate the velocity that you specify for the
' PVT command. But if you know only the positions, you can use an
' average velocity for this command. This program shows each of these
' methods for a sinusoidal move type. If you use an average velocity,
' a delay of one PVT segment time occurs.				
'*******************************************************
#define PI 3.141592654
' The position history values that you use to calculate the average velocity.
DVAR $xpos0
DVAR $xpos1
DVAR $xpos2
' Axis velocity in 1/s.
DVAR $xvel   
' PVT move segment time in ms.
DVAR $Ts
' Frequency of the sine wave in Hz.
DVAR $Freq
' Amplitude of the sine wave.
DVAR $Ampl
' Program internal variables.
' 2*PI*freq storage.				
DVAR $ang
' Time in seconds.
DVAR $time
' Loop counter.
DVAR $idx
' Enable average velocity (1) or analytic velocity (0) calculation mode.
DVAR $mode
' End index for loop. Average mode requires one more loop.        
DVAR $eidx  
              
' Make sure that the task executes a sufficient number of lines to calculate a new PVT target value.                        
ExecuteNumLines = 100    
                         
ENABLE X
' Set the units to primary units.
PRIMARY
' Set the time base to seconds.
SECONDS
' Enable absolute programming mode.
ABSOLUTE
' A sample time of the PVT command in ms ( >= 1 ).
$Ts = 3             

' Frequency of the sine wave in Hz.				                         
$Freq = 10              
' Specify the amplitude of the sine wave in units.
$Ampl = 100       
' Enable average velocity (1) or analytic velocity (0) calculation mode. 
$mode = 1                 

' Use Digital Scope to collect data.				                         
SCOPETRIG             

' Use the analytic velocity and average velocity calculation modes to generate a move.				
FOR $mode = 0 to 1  
                   
   ' Move the X axis to a location where PVT motion starts. 			  
   LINEAR X=$Ampl F10000    
   
   ' Initialize "last position" storage. This is required only for average mode. 		                         
   $xpos0 = $Ampl           
   $xpos1 = $Ampl         
   $xpos2 = $Ampl
   $ang = 2 * PI * $Freq
   ' Add one more loop for averaging mode.
   $eidx = 200 + $mode

   IF( $mode == 0 ) THEN 
      ' Analytic velocity is available. 
      ' Use the default interpolation mode.
      MotionInterpolationMode = 0
   ELSE
      ' Analytic velocity is not available.
      ' Use 3-position/1-velocity interpolation.
      MotionInterpolationMode = 1
   ENDIF

   ' Main loop, calculate a new PVT command each time.
   FOR $idx = 0 TO $eidx

      ' Calculate the current time in seconds.	
      $time = $idx * $Ts / 1000	
      IF( $mode == 1 ) THEN 
         ' ***** Use average velocity for the PVT command. *****

         ' Shift positions back one sample.
         $xpos0 = $xpos1                            
         ' Shift positions back one sample.
         $xpos1 = $xpos2      
         ' New commanded position.	     
         $xpos2 = $Ampl*cos( $ang * $time )    
         ' Average velocity near xpos1.		    
         $xvel = ($xpos2-$xpos0) * .5 * 1000/$Ts   
         
         ' Execute the move segment. 		                                           
         PVT X $xpos1, $xvel  TIME $Ts             

      ELSE	
         ' ****** Use calculated (analytic) velocity for the PVT command. *****
         ' New commanded position in units.
         $xpos0 = $Ampl*cos( $ang * $time )        
         
         ' Commanded velocity in units/s.		                                           
         $xvel = -$ang*$Ampl*sin( $ang * $time )  
         
         ' Execute the move segment.		                                        
         PVT X $xpos0, $xvel  TIME $Ts          
      ENDIF
   ' Move segment.
   NEXT $idx             
   DWELL .1
' $mode.
NEXT $mode           
END PROGRAM
