DATAACQ TRIGGER Command

Specifies which signal is monitored to collect data.

Syntax:

 
DATAACQ <Axis> 1 TRIGGER <TriggerSignal>

Example:

 
DATAACQ X 1 TRIGGER 2

Table:     DATAACQ TRIGGER Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<TriggerSignal>

32-bit signed integer

The signal to be triggered on.

Hardware: This command does not apply when used with the Ndrive FCL, FLS, QL, or QLe, the Nmark SSaM, or the Nstep.

Hardware: When you use a multi-axis drive such as the Npaq or Nservo, all axes in the drive share the same array space. You can specify any axis in the drive for the <Axis> argument.

Use the DATAACQ TRIGGER command to specify the signal that will cause a data acquisition sample to be captured. Options include dedicated high-speed opto-isolated inputs, PSO firing events, and the PSO output.

These DATAACQ commands must be issued in the following order.

  1. DATAACQ TRIGGER
  2. DATAACQ INPUT
  3. DATAACQ ARRAY
  4. DATAACQ OFF (optional)

When you use an opto-isolated input, you can capture samples on the rising or falling edges of the input. For an opto-isolated input, the rising edge is defined as the transition from the de-energized state to the energized state. The falling edge is defined as the transition from the energized state to the de-energized state.

There are two triggering options that use an output from the drive's PSO functionality: the PSO firing event and the PSO output. When you use the PSO Firing Event, samples are captured only once for each firing event. Firing events can include distance-based firing and entering or exiting a PSO window. (Refer to the PSO Tracking Diagram for the appropriate drive type, as shown on the PSO Block Diagrams, for more information about what can cause a PSO firing event.)

When you use the PSO Output as the trigger source, samples are captured each time the PSO output transitions from the inactive state to the active state. This means that you can capture multiple samples for a single PSO firing event depending on the number of output pulses generated for each event.
