RETURN Command

Returns to a program from an ONGOSUB routine, or exits from subroutines and programs.

Syntax:

 
RETURN 
 
RETURN <ReturnType>

Example:

 
RETURN

Table:     RETURN Command Arguments

Argument

Type

Description

<ReturnType>

String

(Optional) Specifies how the controller returns one or more axes after you issue an ONGOSUB command.

You can use the RETURN command to either return from an ONGOSUB command, or to exit subroutines and programs.

Returning from a Subroutine or Program

Use the RETURN command to exit subroutines or programs. In a subroutine, this command is optional because an ENDDFS command will automatically return. You can use the RETURN command to force an exit before the ENDDFS command.

After a RETURN command is executed, program execution continues on the line immediately after the CALL or FARCALL command that was executed. If you execute a RETURN in a program that was not called from another program, the program stops execution (as if an END PROGRAM were executed). If a task reaches the end of a program while called from another program, it will automatically execute a RETURN command.

Returning from an ONGOSUB Command

The <ReturnType> argument has an effect only when you return from an ONGOSUB command. If you do not specify a value for the <ReturnType> argument, the RETURN command uses RETURNTYPE_NULL.

Table:     <ReturnType> Argument Values

<ReturnType> How the Controller Returns the Axes Program Execution of the Interrupted Line

RETURNTYPE_NULL

The axes do not return to the absolute positions that they occupied when the interruption occurred.

If a RAPID, LINEAR, CW, CCW, or BEZIER move is interrupted, then the move does not complete on return from the interrupt and program execution continues on the line after the interrupted line.

For all other commands, program execution continues on the interrupted line.

RETURNTYPE_INTERRUPT

The axes return to the absolute positions that they occupied when the interruption occurred.

 If a RAPID, LINEAR, CW, CCW, or BEZIER move is interrupted, then the move completes on return from the interrupt.

For all other commands, program execution continues on the interrupted line.

RETURNTYPE_OFFSET

The axes do not return to the absolute positions that they occupied when the interruption occurred.

All of the moves that are made during the interrupt are treated as offsets. If a RAPID, LINEAR, CW, CCW, or BEZIER move is interrupted, then the move completes on return from the interrupt. For all other commands, program execution continues on the interrupted line.

This type also adds an offset to the subsequent moves that are specified in the ABSOLUTE programming mode. Refer to Positions Overview for more information about offsets.

For Example

If Axis Y moves +0.1 mm during the interrupt, then a subsequent move to the absolute position of zero, such as LINEAR Y(0) in ABSOLUTE mode, moves Axis Y to 0.1 mm. It does not move Axis Y to 0.0 mm.

The controller removes this offset when you home the axis or you issue a POSOFFSET CLEAR command.

RETURNTYPE_START

All of the axes that were part of an interrupted RAPID, LINEAR, CW, CCW, or BEZIER move return to the absolute positions that they occupied at the start of the move.

All other axes return to the absolute positions that they occupied when the interruption occurred.

If a RAPID, LINEAR, CW, CCW, or BEZIER move is interrupted, then the move executes again from the start of the line.

For all other commands, program execution continues on the interrupted line.

RETURNTYPE_END

All of the axes that were part of an interrupted RAPID, LINEAR, CW, CCW, or BEZIER move return to their absolute target positions, as if the line completed without being interrupted.

All other axes return to the absolute positions that they occupied when the interruption occurred.

If a RAPID, LINEAR, CW, CCW, or BEZIER move is interrupted, then the move continues on the line after the interrupted line.

For all other commands, program execution continues on the interrupted line.
