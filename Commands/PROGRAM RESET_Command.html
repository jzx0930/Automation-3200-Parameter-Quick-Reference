PROGRAM RESET Command

Resets the associated AeroBasic program to the first executable line.

Syntax:

 
PROGRAM RESET
 
PROGRAM <Task> RESET

Example:

 
PROGRAM RESET

Table:     PROGRAM RESET Command Arguments

Argument

Type

Description

<Task>

TaskID

(Optional) The task on which to reset the program. If you do not specify a task, this command resets the program that is running on the current task.

Use the PROGRAM RESET command to reset an AeroBasic program to the first executable line.

If you issue this command while the task that you specify for the <Task> argument is in the Program Paused or Program Complete Task State, this command puts the specified task into the Program Ready Task State. If you issue this command and the specified task is in the Error Task State, this command resets the AeroBasic program that is running on that task but it keeps the task in the Error Task State. You must clear all errors that occur. Refer to the Task Errors section of Fault and Error Overview for more information.

If you do not specify the <Task> argument, this command resets the AeroBasic program that is running on the task that executes this command. Then it puts that task into the Program Paused Task State.

If the PROGRAM RESET command changes the state of a task that is in a nested, called program, the controller starts to execute lines at the top of the first program and cancels all nested calls and loops. All modal task information, user-defined variables (DVAR), and the values of those variables stay unchanged.

To run the program, issue a Program Start.
