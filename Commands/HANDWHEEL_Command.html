HANDWHEEL Command

Moves an axis based on the position of an encoder input channel.

Syntax:

 
HANDWHEEL <Mode>, <Axis>, <Distance>

Example:

 
HANDWHEEL 1, X, .001

Table:     HANDWHEEL Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Mode>

64-bit IEEE floating point

Enables or disables the handwheel. To see the possible values, refer to Table: <Mode> Argument Values.

<Distance>

64-bit IEEE floating point

The distance to move the axis for each count of the handwheel channel. To reverse the direction of the axis, specify a negative value.

Use the HANDWHEEL command to move an axis based on the position of an encoder input channel. This is a form of gearing in which a slave axis follows a master encoder channel with a defined ratio. The ratio is set by the <Distance> argument which specifies the slave distance moved for every master encoder count.

Table:     <Mode> Argument Values

Value

Description

0

Disables the handwheel.

1

Enables the handwheel.

The GEAR MASTERCONFIG command must be executed prior to the HANDWHEEL command to define the master encoder channel. When using a single handwheel for multiple axes, use the GEAR MASTERCONFIG command initially to configure all desired axes and enable and disable each axis as required. To dynamically change the handwheel distance, always disable the handwheel and re-enable it with a new distance value.

The HANDWHEEL command automatically filters the master encoder channel. This is similar to the GEAR ON FILTERED command.

Warning: Before you use this command, use the ABORT Command on the master axis to make sure that the master axis is not moving. Then, make sure that the velocity is zero for the master axis and for the slave axis. If you disable the handwheel while the slave axis is moving, the slave axis will suddenly stop. If you enable the handwheel while the master axis is moving, the slave axis will suddenly follow the master axis.

Because this is an asynchronous command, you can sequentially execute multiple handwheel commands to permit simultaneous multi-axis positioning.

Example

' Configure the slave axis (X) to use the auxiliary encoder channel of the master axis (X).
GEAR X MASTERCONFIG X 2

' Configure the slave axis (Y) to use the auxiliary encoder channel of the master axis (X).
GEAR Y MASTERCONFIG X 2

' To put the slave axis (X) in the correct position, use the handwheel on the master axis (X).
HANDWHEEL 1 X .0001

' To put the slave axis (Y) in the correct position, use the handwheel on the master axis (X).
HANDWHEEL 1 Y .0001

MSGBOX DF_MSGBOX_OKONLY + DF_ICON_INFORMATION, "Press OK to change the handwheel speed."

' To put the slave axis (X) in the correct position, use the handwheel on the master axis (X).
HANDWHEEL 1 X .001

' To put the slave axis (Y) in the correct position, use the handwheel on the master axis (X).
HANDWHEEL 1 Y .001
MSGBOX DF_MSGBOX_OKONLY + DF_ICON_INFORMATION, "Press OK to change handwheels."

' Disable the X position.
HANDWHEEL 0 X .01

' Disable the Y position.
HANDWHEEL 0 Y 0.01

' Configure the slave axis (X) to use the auxiliary encoder channel of the master axis (Y).
GEAR X MASTERCONFIG Y 2

' To put the slave axis (X) in the correct position, use the handwheel on the master axis (Y).
HANDWHEEL 1 X .0001

MSGBOX DF_MSGBOX_OKONLY + DF_ICON_INFORMATION, "Press OK to change the handwheel speed."

' To put the slave axis (X) in the correct position, use the handwheel on the master axis (Y).
HANDWHEEL 1 X .001

MSGBOX DF_MSGBOX_OKONLY + DF_ICON_INFORMATION, "When the axes are in the correct positions, press OK."

' Disable the X position.
HANDWHEEL 0 X .01
