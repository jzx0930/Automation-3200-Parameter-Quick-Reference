ARRAY WRITE Command

Writes data into the PSO array.

Syntax:

 
ARRAY <Axis> WRITE <ControllerVariable> <StartIndex> <NumberOfElements>
 
ARRAY <Axis> WRITE <ControllerVariable> <StartIndex> <NumberOfElements> ABSOLUTE

Example:

 
ARRAY X WRITE $global[3] 10 100

Table:     ARRAY WRITE Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<ControllerVariable>

64-bit IEEE floating point variable

The name of the first controller variable to write to the drive array.

<StartIndex>

32-bit signed integer

The starting index of the drive array where controller variables are written.

<NumberOfElements>

32-bit signed integer

The number of elements to send.

Hardware: This command does not apply when used with the Nmark SSaM or the Nstep.

Hardware: When you use a multi-axis drive such as the Npaq or Nservo, all axes in the drive share the same array space. You can specify any axis in the drive for the <Axis> argument.

Use the ARRAY WRITE command to write sequential values from the controller variables to the drive array at a maximum of 10,000 elements per second. The drive array values are 32-bit signed integers. Because the controller array variables are 64-bit IEEE floating point variables, they will be rounded to 32-bit signed integers when they are written to the drive array. For more information about the drive array, refer to Drive Array Overview.

The ABSOLUTE keyword causes the values to be converted from absolute coordinates to incremental values. In the example, the drive array element 10 is always set to the value of $global[3]. If the ABSOLUTE keyword is specified, the second drive array element is set to the value of ($global[4] - $global[3]). This is useful when loading drive array elements for the PSODISTANCE command, which always works with incremental firing distances.

Special Considerations for Multi-Axis Drives

Because of the way the A3200 SMC communicates with multi-axis drives, it is possible to increase the maximum download rate by sending ARRAY WRITE commands to each axis of the drive. For example, on the Npaq, which uses six communications channels, you can increase the maximum download rate to approximately 30,000 elements per second. To increase this rate, use a Critical Section as shown in the example that follows.

Hardware: This example program applies only to the Npaq. It does not apply to the Npaq MR.

Example

' This #define specifies the number of global double variables that the ARRAY WRITE command
' writes to the drive array. For the example program to execute correctly, you must make
' this value a multiple of 60 because the controller downloads 60 values each time.
#define NUM_VALUES 3000

DVAR $Index
DVAR $EndTime

' Reset the clock value so you can monitor the time of the download process.
TIMER 0 CLEAR

' The FOR loop is within the CRITICAL START and CRITICAL END commands to
' make sure that the controller executes all contents in the loop once per millisecond.
' By executing the contents correctly, the controller increases throughput for the
' ARRAY WRITE commands to a maximum while decreasing the possible effects on real-time performance.
' The loop must contain a short DWELL command to prevent starving the controller. 
' The controller starves when one task does not release its time slice so that other tasks can run. 
				
' Start a critical section so that all the commands
' that follow execute within the same millisecond.
CRITICAL START

FOR $Index = 0 TO (NUM_VALUES - 1) STEP 60   
    ' Because you know that ARRAY WRITE can write only 10 values per
    ' millisecond, write 10 values to each of the six axes of the
    ' Npaq. This lets each ARRAY WRITE command complete within
    ' one millisecond, but lets all six commands execute at the same time.
    ARRAY X WRITE $global[$Index + 0], $Index + 0, 10
    ARRAY Y WRITE $global[$Index + 10], $Index + 10, 10
    ARRAY Z WRITE $global[$Index + 20], $Index + 20, 10
    ARRAY U WRITE $global[$Index + 30], $Index + 30, 10
    ARRAY A WRITE $global[$Index + 40], $Index + 40, 10
    ARRAY B WRITE $global[$Index + 50], $Index + 50, 10

    ' The program must use a DWELL here to make the task release its time slice and
    ' give it to other tasks. Without this DWELL command, the controller starves.
    DWELL 0.001

NEXT $Index

' Read the clock value. Because you reset the clock value at the start of this program, this value shows
' the quantity of time necessary for the controller to download all of the data.
$EndTime = TIMER(0)

' When the loop completes, end the critical section here.
CRITICAL END

' End of program.
END PROGRAM
