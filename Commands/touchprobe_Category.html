Touch Probe Command Category

You can monitor touch probe sensors on the A3200 through a virtual binary input or through a digital drive input such as a high-speed position latch. You can use the touch probe input to capture the position feedback of an axis when the controller triggers the probe input. If the controller detects the probe input, the controller generates a probe axis fault and stores the axis position.

You must set the Probe Input Fault bit in the FaultMask parameter so that the controller can detect the touch probe. You can also set the Probe Input Fault bits in the FaultMaskDecel, FaultMaskDisable, and FaultMaskOutput axis parameters to specify what the axis does if the controller generates a probe axis fault.

NOTE: If you do not set the Probe Input Fault bit in the FaultMaskDecel or FaultMaskDisable parameter, then the axis that you are using continues to move after the controller triggers a touch probe.

Table:     Touch Probe Commands

Command

Description

PROBE INPUT

Configures the drive and the input bit number where the touch probe is connected.

PROBE MODE

Specifies the type of scanning that you want to use.

PROBE ON

Turns on probe input monitoring.

PROBE OFF

Turns off probe input monitoring.

Example Programs

Example: Hardware Mode Scanning 
' The probe input changes to digital drive input 8 ($DI8.X).
PROBE INPUT PROBE_INPUT_DRIVE_DIGITAL X 8

' The scanning mode changes to hardware scanning mode.
' The controller writes the captured positions to $global[4].
PROBE MODE PROBE_HW_MODE 1 $global[4]
				
' Starts probe monitoring.
PROBE ON X
				
' Does motion on X.
G1 X100 F10
				
' Waits for the probe input to capture the axis positions.
WAIT ((TASKSTATUS(DATAITEM_TaskStatus0) & TASKSTATUS0_ProbeCycle) == 0) -1

' You can get the captured position in $global[4].
Example: Software Mode Scanning 
' The probe input changes to digital drive input 8 ($DI8.A).
PROBE INPUT PROBE_INPUT_DRIVE_DIGITAL A 8
				
' The scanning mode changes to software scanning mode.
' The controller writes the captured positions and
' starts at $global[0].
PROBE MODE PROBE_SW_MODE 1 $global[0]
				
' Starts probe monitoring. The probe input captures
' positions on axes X and Y.
PROBE ON X Y
				
' Starts Motion.
MOVEINC X 1000 10
MOVEINC Y 1000 10
				
' Waits for the probe input to capture the axis positions.
WAIT ((TASKSTATUS(DATAITEM_TaskStatus0) & TASKSTATUS0_ProbeCycle) == 0) -1
				
' $global[0] stores the captured X position.
' $global[1] stores the captured Y position.

  • PROBE MODE
  • PROBE ON/OFF
