PROBE INPUT Command

Configures the probe input.

Syntax:

 
PROBE INPUT <Type>, <Axis>, <InputNumber>

Example:

 
PROBE INPUT PROBE_INPUT_DRIVE_DIGITAL, X, 3

Table:     PROBE INPUT Command Arguments

Argument

Type

Description

<Type>

Other

The type of input that is scanned. To see the possible values, refer to Table: <Type> Argument Values.

<Axis>

Axis

The axis to which the touch probe connects.

<InputNumber>

64-bit IEEE floating point

The number of the input bit to which the touch probe connects.

Use the PROBE INPUT command to configure the touch probe input. The input can be a virtual binary input or a digital drive input. To get better performance when you use a digital drive input, specify a drive input that is also a high-speed position latch. Refer to your hardware manual for information about the locations of the digital drive inputs and the high-speed inputs.

If you specify the hardware scanning mode for the PROBE MODE command, you must also specify a digital drive input for the <Type> argument of the PROBE INPUT command. Make sure that you specify this input on the same axis on which the controller executes the PROBE ON command.

Table:     <Type> Argument Values

Value

Description

PROBE_INPUT_VIRTUAL_BINARY

Uses the virtual binary inputs. Refer to $BI. If you select this value, you must set the <Mode> argument of the PROBE MODE command to PROBE_SW_MODE.

PROBE_INPUT_DRIVE_DIGITAL

Uses the digital drive inputs. Refer to $DI.

To see example programs that use the touch probe commands, refer to the Examples section of Touch Probe Command Category.
