ON Command

Sets a variable, controller parameter, or I/O to a different specified value if a specified condition is true or if it is false.

Syntax:

 
ON <Condition> SET <Variant>
 
ON <Condition> SET <Variant>, <OnValue>, <OffValue>
 
ON <Condition> SET <Variant>, <OnValue>, <OffValue>, <Mode>
 
ON CLEAR 
 
ON CLEAR <Condition>

Example:

 
ON $global[0] > 5 SET $BO[2]
ON CLEAR $global[0] > 5

Table:     ON Command Arguments

Argument

Type

Description

<Condition>

Other

(Optional) The condition to examine. Refer to Conditional Expressions.

<Variant>

64-bit IEEE floating point

(Optional) The variable, controller parameter, or I/O to set. Refer to Variants.

<OnValue>

64-bit IEEE floating point

(Optional) The value to set the <Variant> argument to if the condition is true.

<OffValue>

64-bit IEEE floating point

(Optional) The value to set the <Variant> argument to if the condition is false.

<Mode>

Other

(Optional) Specifies when the <Variant> argument is set. To see the possible values, refer to Table: <Mode> Argument Values.

Use the ON command to specify a condition to examine and set the specified variable, controller parameter, or I/O to <OnValue> if the condition is true, or to <OffValue> if the condition is false.

You can use the <Mode> argument to specify when you want the <Variant> argument to be set. The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value

Description

ONSET_MODE_LEVEL

The <Variant> argument is continuously set to the <OnValue> or <OffValue> dependent on the result of the expression

ONSET_MODE_EDGE

The <Variant> argument is set to <OnValue> when the expression transitions from false to true and to the <OffValue> when the expression transitions from true to false

ONSET_MODE_LATCH

The <Variant> argument is set to <OnValue> when the expression transitions from false to true. This mode does not use the <OffValue> argument.

The monitors for the ON command are related to the task, not to the program. The monitors are active immediately when they are defined and until they are cleared. They are not automatically cleared when the program completes, or if no programs are running on the task. You must clear the monitor with the ON CLEAR command at the end of the program. If you specify a monitor in a program and you do not clear it, the monitor will be active in the next program that runs on that task.

Note: The limit of the number of established monitors is specified by the MonitorStatements parameter.

Example

' Set an ON monitor.
ON $global[0] > 5 SET $BO[2]
' Clear one ON monitor.
ON CLEAR $global[0] > 5
' Clear all of the ON monitors.
ON CLEAR
ON( $BO[0]==1 ) SET $global[0] 100 20 ONSET_MODE_LEVEL
ON( $BO[1]==1 ) SET $global[1] 100 20 ONSET_MODE_EDGE
ON( $BO[2]==1 ) SET $global[2] 100 20 ONSET_MODE_LATCH
ON( $global[0]==1 ) SET $DO[0].X 1 0
ON( $global[1]==1 ) SET $AO[0].X 5 -1
