PSOWINDOW CONTROL Command

Specifies that the two PSO window counters are either co-dependent or independent.

Syntax:

 
PSOWINDOW <Axis> CONTROL <Mode>

Example:

 
PSOWINDOW X CONTROL 1

Table:     PSOWINDOW CONTROL Command Arguments

Argument

Type

Description

<Axis>

Axis

The axis on which to execute the command.

<Mode>

Other

The window control mode to use. To see the possible values, refer to Table: <Mode> Argument Values.

Hardware: This command applies only when used with the Npaq or the Nservo. It does not apply to the Npaq MR.

Use the PSOWINDOW CONTROL command to specify if the two PSO window counters are co-dependent (used together for XY mode operation) or if the windows are independent.

When the windows work co-dependently, both of the axes that the windows are tracking must be inside the bounds that you specify. If either of the axes is not in its window, then PSO does not fire pulses.

When the windows work independently, the axes that the windows are tracking can go in and out of their windows and trigger the PSO pulse generator to generate pulses. For PSO to fire the pulses, it is not necessary for the two axes to be in their windows at the same time.

If you only need one window, then you do not need to use this command to change the default setting. If you need both windows for your program and you do not want interference between windows, then use the independent mode.

The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value

Description

0

Co-dependent Windows. When both windows are enabled, they work together.

1

Independent Windows. When both windows are enabled, they work separately.
