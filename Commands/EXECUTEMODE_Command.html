EXECUTEMODE Command

Changes the execution mode of a task.

Syntax:

 
EXECUTEMODE <Mode>
 
EXECUTEMODE <Task> <Mode>

Example:

 
EXECUTEMODE 1 TASKEXEC_AUTO_INTO

Table:     EXECUTEMODE Command Arguments

Argument

Type

Description

<Task>

TaskID

(Optional) The task on which to change the execution mode. If you do not specify a task, the command changes the execution mode of the current task.

<Mode>

Other

The execution mode to set on the specified task. To see the possible values, refer to Table: <Mode> Argument Values.

Use the EXECUTEMODE command to change the execution mode of the specified task. The table that follows shows the possible values for the <Mode> argument.

Table:     <Mode> Argument Values

Value

Description

TASKEXEC_AUTO_INTO

This is the default setting. When you issue a Program Start, the task consecutively executes program lines without pausing. The line number that is reported is the line number of the line that is currently executing.

TASKEXEC_AUTO_OVER

When you issue a Program Start, the task consecutively executes program lines without pausing. The line number that is reported is the line number of the line that is currently executing, except in the case of a subroutine call. In this case, the line number of the subroutine call is reported. If the program encounters a breakpoint when operating in this mode, the task execution mode changes to TASKEXEC_AUTO_INTO.

TASKEXEC_STEP_INTO

This mode causes a Program Start to behave the same as a Program StepInto.

TASKEXEC_STEP_OVER

This mode causes a Program Start to behave the same as a Program StepOver.
