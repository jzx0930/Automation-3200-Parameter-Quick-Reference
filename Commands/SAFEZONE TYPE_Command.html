SAFEZONE TYPE Command

Sets the safe zone type.

Syntax:

 
SAFEZONE <Zone> TYPE <Type>

Example:

 
SAFEZONE 0 TYPE SAFEZONETYPE_NoExit

Table:     SAFEZONE TYPE Command Arguments

Argument

Type

Description

<Zone>

32-bit signed integer

The zero-based index of the safe zone.

<Type>

Other

The safe zone type.

Use the SAFEZONE TYPE command to set the type of a safe zone. Safe zones are indexed from 0 to 31. Refer to SafeZoneType to see the types that are available.

A NoEnter zone is an n-cube that the axes cannot enter. NoEnter zones can be any size. A NoExit zone is an n-cube that the axes cannot exit. You can also combine NoEnter and NoExit safe zones.

The controller examines the safe zones at a rate that corresponds to the drive type of the specified axis. For more information, refer to the table that follows.

Table:     Safe Zone Examination Rates

Drive Category Safe Zone Examination Rate

GL4

48 kHz

Ndrive FCL and FLS

48 kHz

Ndrive QL and QLe

24 kHz

Nmark CLS, GCL, and SSaM

48 kHz

Servo Drives

8 kHz

XL4s

48 kHz

NOTE: If the motion occurs at a high rate of speed and the NoEnter zone is small, the motion might pass through the NoEnter zone.

The controller monitors every generated position to make sure that the position does not violate the safe zone. To generate a Safe Zone Fault after the motion stops, you must select the Safe Zone Fault bit in the FaultMask parameter and the SAFEZONETYPE_NoExitAxisFault or SAFEZONETYPE_NoEnterAxisFault. If you select SAFEZONETYPE_NoExitAxisFault or SAFEZONETYPE_NoEnterAxisFault type, and the axes in the safe zone are selected in the TaskTerminationAxes parameter, a safe zone violation causes a running task to stop. For multi-axis zones, each axis decelerates independently, so the axis faults might fire at different times. Refer to Safe Zone Fault Axis Fault for more information.
