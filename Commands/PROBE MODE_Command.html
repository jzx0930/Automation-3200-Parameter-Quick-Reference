PROBE MODE Command

Selects the scanning mode that you want to use to monitor the probe input and the active level of the probe input.

Syntax:

 
PROBE MODE <ScanningMode>, <ActiveLevel>, <Variable>

Example:

 
PROBE MODE PROBE_SW_MODE, 1, $Position[0]

Table:     PROBE MODE Command Arguments

Argument

Type

Description

<ScanningMode>

Other

The scanning mode to use to monitor the probe intput.

<ActiveLevel>

64-bit IEEE floating point

The active level of the touch probe drive.

<Variable>

Other

The variable that stores the captured position feedback. The position feedback is in the units mode that you specify for the task.

Table:     <Mode> Argument Values

Value

Description

PROBE_HW_MODE

Uses the hardware mode to scan for the probe input.

PROBE_SW_MODE

Uses the software mode to scan for the probe input.

When the probe input is at the <ActiveLevel>, the touch probe captures position feedback and stores it as a value in the specified variable. Before you can read this value from the variable, the controller must clear the Probe Cycle bit of the Task Status 0 status value. You can issue the PROBE OFF command to clear this bit. The controller also clears this bit when you stop a task or a task error occurs. To see example programs that use the touch probe commands, refer to the Examples section of Touch Probe Command Category.

If you issue the PROBE ON command on two or more axes, you must make the <Variable> into an array. For each axis that you specify in the PROBE ON command, the controller writes the position feedback from all of these axes into the array at the same time. Make sure that the size of the array is sufficient to contain all of the position feedback. In the array, the controller stores the position feedback in ascending order of communication channels. It does not store position feedback in the order in which you specify it for the PROBE ON command.

Scanning Modes

The scanning mode specifies how the controller scans for the probe input.

Hardware

NOTE: If you want to use this mode with two or more axes, you must make sure that each drive has access to the touch probe input.

Use the hardware scanning mode to monitor the touch probe input at the servo-loop update rate. In this mode, the drive scans for the probe input. When you use this mode, you must specify a digital drive input in the PROBE INPUT command.

Software

NOTE: Aerotech recommends that you use this mode if you want to capture position feedback on two or more axes and none of these axes have access to the same digital drive input.

Use the software scanning mode to monitor the touch probe input at a one millisecond rate. In this mode, the PC scans for the probe input. When you use this mode, you can specify a virtual I/O bit as the input in the PROBE INPUT command. This mode requires a larger quantity of time to complete than the hardware mode.
