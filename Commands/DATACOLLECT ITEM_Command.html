DATACOLLECT ITEM Command

Configures one data item to collect.

Syntax:

 
DATACOLLECT ITEM <ItemNumber>, <Index>, <DataItem>
 
DATACOLLECT ITEM <ItemNumber>, <Index>, <DataItem>, <AdditionalData>

Example:

 
DATACOLLECT ITEM 0, 0, DATAITEM_DataCollectionSampleTime
DATACOLLECT ITEM 2, Y, DATAITEM_VelocityCommand

Table:     DATACOLLECT ITEM Command Arguments

Argument

Type

Description

<ItemNumber>

32-bit signed integer

Specifies the item number that is being configured. The first DATACOLLECT ITEM command must specify item number zero. Increment the item number by one in each subsequent call to DATACOLLECT ITEM.

<Index>

Axis

Specifies the axis name, or task index, from which the data item is collected. Specify zero for system data items.

<DataItem>

Other

Specifies the name of the data item to be collected. All of the supported data items are shown in the following sections.

<AdditionalData>

Other

(Optional) Additional data for the specified data item. This argument is required by some data items.

Note: This command is executed as a callback and does not execute in real time. The time that it takes for this command to execute is different for different PC configurations. Refer to Callback Commands for more information.

Use the DATACOLLECT ITEM command to specify one data item that you want the DATACOLLECT START command to collect. To collect more than one data item, issue the DATACOLLECT ITEM command more than one time. The following table shows the items that you can collect.

For some <DataItem> values, you can supply an integer as an optional argument. If you can use an optional argument with the <DataItem> value, the Additional Data column gives information about the argument.
