Double Type Overview

Most commands and functions use the double precision, floating-point type as arguments.

Doubles and integers types can be used interchangeably. But some commands, operators, and functions require double inputs to represent exact integers (for example, the binary floating-point operations). In these cases, if a non-integer is used, the controller will truncate the value, round the value, issue a compile time error (for example, expected integer constant) or issue a runtime error (for example, value must be integer).

There is no limit to the number of elements allowed in an expression. But, the ON Command, ONGOSUB Command, and WAIT Command are limited to 20 elements within the conditional expression.

Examples

' Floating-point constant.
6.7

 

' Hexadecimal constant.
0Xabcdef

 

' Global variable.
$global[5]

 

' Parameter.
KPIGain.X

 

' Expression.
8 * ($global[7] + $global[9 + 2])
