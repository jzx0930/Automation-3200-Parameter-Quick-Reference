#define preprocessor directive

Syntax:

 
#define <targetWord> [<replacementText>]
#undef <targetWord>

The #define preprocessor directive lets you do simple text substitution in the source code before the source code is compiled. The #define statement defines a target word and its corresponding replacement text. On subsequent lines of the source file, all occurrences of the target word are substituted with the replacement text. This directive is only active for all lines that come after the #define statement. For all lines that come before the #undef#define statement, occurrences of the target word are not replaced. Refer to Recognition of the Target Word for more information.

White space can come before the # sign. The #define preprocessor directive must be separated from the target word by white space. The target word must be separated from the replacement string by white space.

You can override or deactivate a #define statement at any time. A #define statement can affect subsequent #define statements that contain the same target word. Refer to Replacement within Replacement Strings for more information.

The #undef statement deactivates all previous #define statements that have the same target word. On all lines that come after an statement, none of the occurrences of the target word are replaced. If you use the #undef statement with a target word that is not defined or if you use the #define statement with a target word that is already defined, then you get a warning from the compiler.

Examples

' After this statement, "LaserState" is replaced with "$BI[7]".
#define LaserState $BI[7]
' Define a constant.
#define PI 3.14159
' Clear a #define.
#undef PI

Replacement String Restrictions

The replacement string can contain white space and multiple lines, but cannot contain a back slash (\), comments, or line termination characters. You can add special characters by specifying them as hexadecimal codes in the format 0xnn. For example, you can specify the replacement string "A;Z" as "A\x3BZ". This is useful for adding spaces at the end of replacement strings, because trailing spaces are normally removed from replacement strings.

There is no limit to the length of a replacement string. Comments terminate the replacement string, which lets you comment on the #define statement lines.

For Example

#define counter $task[0] ; This comment is not in the replaced text.
$global[1] = counter ; This comment is retained.

becomes

$global[1] = $task[0] ; This comment is retained.

Target Word Restrictions

There is no limit to the length of a target word. The target word can contain each of the characters except for a back slash ("\") or white space. The target word is terminated by the first white-space character. Target words can be keywords such as FOR or LINEAR. But, if you redefine your keywords, the new keywords overwrite the default behavior of the controller for these lines.

Target words are case sensitive. Target words are not matched within comments or string literals.

Do not make the <targetWord> be the same as the name of a variable, which includes $global variables, $task variables, built-in variables, and program variables that you create. If the names of the <targetWord> and a variable are the same, a compile error occurs.

Order of Replacement

When the compiler processes a given line in a program, it evaluates all #define statements that come before that line, before any of the target words are replaced.

For Example

#define X Z
#define Z Y
ENABLE X

becomes

ENABLE Z

First, the compiler looks at ENABLE X to determine what to replace. The first #define statement (#define X Z) triggers because the target word (X) is present. The second #define statement (#define Z Y) does not trigger, because the target word (Z) is not present. Then the compiler replaces the first #define statement target word (X) with its replacement string (Z). The replacement from the first #define statement occurs only after the compiler determines that the second #define statement should not occur.
