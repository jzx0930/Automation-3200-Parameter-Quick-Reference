Program Automation

Use Program Automation to automatically run programs or include them within other programs. These programs are automatically run, included, or downloaded when you start or reset the A3200.

You can add programs to Program Automation through Configuration Manager or Motion Composer. In both applications all operations are the same. Configure Program Automation only from one of these two applications at one time to make sure the Program Automation settings are consistent.

To add a program, go to the Network Explorer. (By default, the Network Explorer is collapsed in Motion Composer.) Under Controller → A3200, right-click the Program Automation folder. Select Add... to open the Program Automation dialog.

Figure: Program Automation Dialog

  1. Click Browse... and select the program that you want to add.
  2. Select the Mode with which you want the program to be added.
  3. Select the Task if one is required for the selected Mode. If no task is required, then the task list is disabled.
  4. Click OK to add the program to Program Automation.

Files added are shown in either the Includes or the Programs folder under the Program Automation folder. If a program is added with the mode of Include, the program is shown in the Includes folder. All other programs are listed under the Programs folder.

After a program is added to Program Automation, you can change its settings by right-clicking the file under Includes or Programs and selecting Modify... In Configuration Manager, when you browse to a file in Program Automation you can view the Program Automation settings of the file in the viewer.

When you add a program in one application, the other application must be closed and reopened in order to see the new settings. To prevent this problem, only edit Program Automation settings from one application at one time.

In Program Automation, you can specify programs to be automatically included, downloaded, or executed on a single task or multiple tasks. You can specify multiple programs for the same task. In this case the programs are done in the order in which they appear under the Includes or Programs folders.

Because compiler error messages are more descriptive in Motion Composer, use Motion Composer to manually test and debug programs before you add them to program automation.

The following table describes the modes of Program Automation in detail.

Table:     Program Automation Modes

Execute Type

Description

Download

Loads a program (compiles and downloads) onto the controller to be called as a subroutine by another program. The program is not executed or associated to a particular task.

Load a program in this mode if it contains a standard error handler routine(s) used by several programs run on the machine. In this case, each program would use a FARCALL command or an OnGoSub command to call the routine on an error condition. This also allows one set of shared subroutines to be maintained.

Programs used by CNC Operator Interface for customizing button actions, running at startup, or running at shutdown need to be added to Program Automation with this Execute Type.

DownloadAndAssociate

 

Loads the specified program on the specified task, but does not start executing the program. The program is then ready to be executed from Motion Composer or CNC Operator Interface.

Include

Includes the contents of the program in all programs that are compiled. This is useful for including #define or #include preprocessor directives in all applications that you are running. There is no way to associate this program to a particular task.

Do not include other PGM programs that have executable lines of code. If your program contains executable lines such as subroutines, you should use the Download mode.

This program is also included on all immediate commands. If you change an Include file, you need to reset the A3200 for the immediate command line to see those changes.

Run

Program starts running on the task after you reset the A3200. When the program is complete, the task is left in the ProgramComplete state. This means that the program is still loaded in Motion Composer and CNC Operator Interface. If you do not want the program to be loaded, use the RunSilent mode.

When you have multiple programs in the Run Mode on the same task, each program must wait for the program before it to complete before executing.

The controller declares program automation complete when all programs have executed, not when all programs are complete. This means that you can open Motion Composer or CNC Operator Interface while a program from program automation is still running. If this occurs, the autorunning program is shown in the correct task in Motion Composer or CNC Operator Interface. You cannot load other programs onto the affected task until the autorunning program has completed or stopped.

RunSilent

Same as Run, except that if the program completes before Motion Composer or CNC Operator Interface are opened, the program is not shown. This option leaves the task in the ProgramIdle state, while the Run option leaves the program in the ProgramCompleted state.

When you have multiple programs in the Run Mode on the same task, each program must wait for the program before it to complete before executing.

The controller declares program automation complete when all programs have executed, not when all programs are complete. This means that you can open Motion Composer or CNC Operator Interface while a program from program automation is still running. If this occurs, the autorunning program will be shown in the appropriate task in Motion Composer or CNC Operator Interface. You cannot load other programs onto the affected task until the autorunning program has completed or stopped.
