Console Interface Overview

Console is a command line interface used for system hardware debugging. It reads commands from text files, similar to batch files, so that you can execute multiple Console commands as if they were one command. Specify a silent or block-terminal mode to deliver outputs to a file instead of the terminal.

Metric units (millimeters or millimeters/second) are the default unit for the controller's velocity and acceleration parameters. Commands and parameters in the Console utility are not case sensitive.

Note: You can concurrently run the Console utility with other applications. However, Console will allow you corrupt the motion being commanded by another application, so care must be exercised.
The Console Prompt

For the commands that apply only to an axis or a task (for example, PARMGET A displays all axis parameters), the command interface uses the default axis or task. The active task and axis is indicated in the prompt at the command line. Use the TK and AX commands to change the task or axis. View the following series of commands for examples.

Prompt and Command Defaults after Command Executed

<TK1\AX1>

Task 1, Axis 1

<TK1\AX1> TK 4

Select Task 4

<TK4\AX1> AX 16

Select Axis 16

<TK4\AX16> TK  

No task change

<TK4\AX16> AX

No axis change

<TK4\AX16>

Task 4, Axis 16
Determining Your Axis Configuration

Select the physical axis index with the AX command (AX <axis index>).

Use the INFO command to display the axis standard and auxiliary feedback type and channel assignment.

Entering Commands in Console

The command line is the line the cursor is on, or the line the user is currently entering text on. Alphabetic keystrokes (a to z, 0 to 9, "-", "+", ".") result in the corresponding character being echoed on the command line and the cursor moving one character to the right. Non-alphabetic keystrokes (i.e. "$","#", Esc) are considered special characters and may or may not be echoed. Also, the characters "_", "\", and ":" are considered alphabetic and may be used in file names. The case of alphabetic characters are ignored, lowercase characters are echoed back as uppercase. Parameter names may be entered in any case in Console. After the command is fully entered, press the enter (return) key to execute the command.

As the user types, the Status Screen will display all Console commands that match the current text on the command line. For example, if TSK has been typed on the command line, the status line will display: "TSKASSOC, TSKDEASSOC, TSKINFO..." which are all valid Console commands starting with TSK. If you press "?" it will scroll through the help information on these commands.

If the text on the line matches only one valid command, then a one-line description of the command appears to the right of the command. For example, if TSKA is on the command line, then "TSKASSOC - Associates program with current task..." appears on the task line. Pressing the "?" key at this point will display the full help text for the command in the help screen.

After the text on the command line matches one valid command, the user does not need to type the rest of the command. For example, "TSKA", "TSKASS", or "TSKASSOC" are equivalent to the full command name.

In most cases, if the user suppliess an invalid parameter and presses return, Console will reject the return (not execute anything) and display an error message on the status bar. For example, if the user types "AX 44" and hits return, Console will ignore the return key because 44 is not a valid axis.

Console Command Line Options

The following table lists the command line options for the Console utility where # is a number and filespec is a valid path and file name. When typing a number of options on the same command line, precede each by a space.

Table:     Console Command Line Options

Option Description

D###

UNIDEX Device Identification (where ### is 3200)

C#

Card # (where # is 1 Ц 4)

Ifilespec

Playback file at startup in continuous mode

Jfilespec

Playback file at startup in step mode

K 

Shut down the A3200

Ofilespec

Startup with OUTPUTON command active (echo mode)

Pfilespec

Startup with OUTPUTON command active (block terminal mode)

EXAMPLE

Console -JSetup.Ply; run PLAY file commands in Setup.Ply file in step mode
Special Keys in Console

The Console command line interface uses these keys as normal keyboard keys: alphabet, 0 to 9, +, -, _, :, \, and (.). Other keys are considered special keys, as listed in the following table.

All other keys are ignored and not recognized by the Console utility. For example, the } (curly bracket) key has no special meaning and is ignored. However, the " (double quotes) is used to ignore recognition of some special characters by the Console application when specifying string variable values or strings to be compiled.

Table:     Console Special Keys

Key Description

Enter

Execute the command line. After execution, move down a line and open new prompt line.

Esc

Discontinue the multiple screen output or discontinue monitoring (returns user to command prompt)

←

Move cursor one character to left in the command line

→

Move cursor one character to right in the command line

ннннн↑

Recall the previous command from the command buffer (erases active command line)

↓

Cycle thru the next command line in the command buffer (erases active command line)

Home

Move cursor to beginning of command line

End

Move cursor to end of command line

Insert

Toggles insert (overstrike) mode

Delete

Deletes character at cursor

Back Space

Deletes the character to the left of the cursor and moves the cursor back one space to the left

PageUp

Make the previous "PLAY" command the active command (erases active command line)

?

Retrieve help for the active command line

"

All ASCII characters between "" are NOT interpreted as special characters.

!

As first character in line, prefixes a direct memory access command. (Refer to Command Summary.)
