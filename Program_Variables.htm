Program Variables

Syntax:

 
$<letter><letter><alphanumeric>

Example:

 
DVAR $myVariable    ; define program variable
$myVariable = 9     ; set it
DWELL F$myVariable     ; use it

Program variables are user named variables defined at the beginning of a program. They are defined using the DVAR command and must be preceded by a $ symbol. Program variables are case sensitive and the first two characters must be letters. The remaining characters can be letters, numbers, or underscores. You cannot use axis names as program variable names.

Program variables are double precision floating point and may be defined as an array.

Program variables are accessible only from the declaring program and cannot be accessed from programs called by the declaring program. Program variables declared in a program running on two different tasks access different variables. If a program calls itself, the variables from the two instances of the program are independent and separate.

Program variables are initialized to zero when the program is downloaded, and they retain their values even if the program is exited and later reentered.

One or more variables can be defined on the same DVAR line. The maximum number of program variables or variable array elements that can be declared is limited only by the quantity of available memory on the controller.

Note: In a program, all of the DVAR commands must come before the first executable command. DVAR commands and preprocessor directives are non-executable commands.
