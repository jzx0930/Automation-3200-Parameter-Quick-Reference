MFO Overview

The MFO command specifies the manual feedrate override (MFO) percentage for the task and scales the programmed feedrate for all coordinated motion (LINEAR, CW, CCW) and RAPID motion. The MFO percentage multiplied by the programmed feedrate determines the true feedrate of the move. The MFO value can vary from the value set in the MinimumMFO parameter to the value set in the MaximumMFO parameter.

Note: The MFO value does not change if the MFOLock task mode is active. For information, refer to M48 Command.

The MFO that you set might be internally limited if satisfying that MFO setting will cause any of the following to occur:

  • Any individual axis speed will exceed the MaxSpeedClamp parameter setting.
  • Any individual axis speed in a CW or CCW move will exceed the CoordinatedCircularAccelLimit parameter setting.
  • A coordinated move duration is shorter than the CoordinatedMoveDurationMinimum parameter setting.
  • The MFO is set to greater than 100% and the controller is executing a RAPID move. In this case, the controller will clamp the MFO to 100% during the RAPID move.

If limiting occurs, the Limit MFO Active bit of Task Status 2 will be turned on for the duration of the move.

The MFO does not have an effect on coordinated moves if they are in the deceleration phase (end phase) of the move.

Normally, the operation of an MFO change during a coordinated move is delayed by a time interval in milliseconds, determined by the MotionUpdateRate parameter.

MFO does not have an effect on PVT, slave motion (handwheel axes, camming, or gearing slaves), or HOME motion. Refer to MSO Command to control spindle speed.

The OVERRIDEASYNC ON/OFF command determines whether MFO can affect asynchronous motion, such as jogging, MOVEINC, and MOVEABS.

If the AnalogMFOInput parameter is active, the controller overwrites the MFO value with the value obtained from the specified analog input.

Programmed acceleration rates (or times) are used during all MFO changes (acceleration or deceleration). MFO changes always result in linear ramping, even if the ramp mode is sinusoidal.

What occurs when MFO is not equal to 100 percent?

If the MFO is not set to one hundred percent when a movement starts, acceleration and deceleration ramping is altered.

If rate-based ramping is active, the A3200 will adjust the acceleration rate of a move to maintain the acceleration time that would have occurred if MFO had been equal to 100 percent.

If time-based ramping is active, the A3200 will determine the acceleration rate differently. (Refer to the following figure for more information.)

Figure MFO and Velocity-Time Motion Curves

In the figure the slope of the motion lines shows the acceleration. When MFO is equal to one hundred percent, the axis accelerates to the maximum velocity at a rate determined by acceleration ramping, which dictates the slope of the blue line. This acceleration occurs over a time (t), which is dependent on the feedrate and acceleration ramping.

When MFO is not equal to one hundred percent, the A3200 uses t (and the MFO value) to determine the acceleration. This results in an acceleration different that the one specified by acceleration and deceleration ramping (as seen in the yellow and green lines).

What occurs when MFO is negative?

Setting the MFO to a negative value activates Retrace mode. Refer to Retrace Overview for more information.
