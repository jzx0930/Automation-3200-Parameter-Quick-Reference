PC Modbus Overview

A dedicated thread in the real-time environment implements the Modbus/TCP protocol. To achieve deterministic I/O updates, a real-time TCP/IP stack implementation is used in place of the standard Windows stack. To use the real-time TCP/IP stack implementation, you must have the network interface card (NIC) driver running under real-time control.

There are six register sets per connection. These are InputWords, InputBits, OutputWords, OutputBits, OutputWordsStatus, and OutputBitsStatus. Word registers hold 16-bit values, and Bit registers hold 1-bit values. The values are indexed from 0 to 65,535. This is the maximum size defined in the protocol, but it is limited to a smaller range by the mappings defined in the parameter file.

There is a Modbus master and a Modbus slave implementation on the PC. The Professional Option supports up to 16 simultaneous master connections. Use Configuration Manager to map I/O on the Modbus device to AeroBasic variables on the A3200. When the mappings are defined, all I/O synchronization occurs automatically and you can access the data by using variables in AeroBasic. You can declare mappings as either simple variables or as arrays of contiguous values. Refer to Creating Fieldbus Mappings For PC Modbus Master or Creating Fieldbus Mappings for PC Modbus Slave for more information.

To Use PC Modbus Master

  1. Shut down the A3200. Click Start → All Programs → Aerotech → Automation 3200 → Shutdown A3200. A black Command Prompt window comes into view momentarily.
  2. Install one of the supported Ethernet network controller cards and add real-time support. Refer to PC Modbus Installation for more information.
  3. Use Configuration Manager to configure the real-time NIC driver. Refer to PC Modbus Configuration for more information.
  4. Configure the remote Modbus device. You must do this with a PC that has a Network Interface Card that is configured for Windows. Do not use a card that is configured for real-time control.
  5. The sections that follow give detailed procedures on how to configure the most commonly used Modbus slave devices (WAGO Fieldbus Controllers (FC) and Programmable Fieldbus Controllers (PFC)). Other Modbus compliant modules supply similar configuration tools that might be different from these procedures. For more information, refer to the documentation supplied by the manufacturer of the module you are using.
    1. Find a compatible IP address to assign to the WAGO FC/PFC.
    2. Assign an IP Address to the WAGO FC/PFC
      1. Power off the WAGO slave device.
      2. Install the WAGO BootP Server application at http://www.wagocatalog.com/okv3/Download_files/BootPSer.zip.
      3. Connect the WAGO slave device to the same network to which your PC is connected.
      4. Open the WAGO BootP Server application.
      5. Click the Edit Bootptab button. Bootptab.txt comes into view so you can edit this file.
      6. Edit the colon-separated sections of Bootptab.txt to define the operating parameters that you want to configure on the WAGO slave device. Refer to the example that follows.

      7. For Example

        To configure the IP address of the WAGO slave device as 10.101.0.24, the Gateway as 10.101.0.1, and the subnet mask as 255.255.0.0 to a fieldbus controller with a MAC address of 00:30:DE:00:11:03, edit the text as follows.

        WAGOFC1:ht=1:ha=0030DE001103:ip=10.101.0.24:sm=255.255.0.0:T3=0A.65.00.01:

        Where

        WAGOFC1 is an identifier. (You can change the name of WAGOFC1 to a name that you want to use.)

        and

        ht is the hardware type. (Set ht to 1.)

        and

        ha is the hardware address. (This is the MAC address.)

        and

        ip is the dot-separated IP address.

        and

        sm is the subnet mask.

        and

        T3 is the gateway. (You must enter the gateway in hexadecimal notation.)
      8. Make sure that you terminate the line with a colon (:).
      9. After you are done editing bootptab.txt, save and close the file.
      10. In the WAGO BootP Server dialog, click the Start button to scan for a bootp packet request.
      11. Power on the WAGO slave device. After a few seconds occur, the WAGO BootP Server dialog comes into view. If the "found" line shows, then the IP address is assigned the WAGO slave device.
        12. Figure: WAGO BootP Server

      12. To get the WAGO slave device to reply correctly, open a Windows command prompt and type the text that follows.

        ping <IP address>

      13. For Example

        C:\>ping 10.101.0.24
Reply from 10.101.0.24: bytes=32 time=10ms TTL=255
        Reply from 10.101.0.24: bytes=32 time<10ms TTL=255
        Reply from 10.101.0.24: bytes=32 time<10ms TTL=255
    3. Make sure the PC and this IP Address are on the same Subnet

      IF (REMOTE_IP XOR LOCAL_IP) AND NET MASK = 0, the two nodes are on the same subnet.

      For Example

      In this example, the two nodes are on different subnets and the packets are routed to a gateway from the side of the Host. When a network packet is routed to a gateway, delays in communication can be caused by overhead and packet collisions. Because of the possible delays, do not use an I/O module outside of its own subnet for time-critical applications.

  6. Use Configuration Manager to configure the slave device and to configure the correct I/O mappings for your device. Refer to the documentation of the remote device for correct configuration. Refer to Creating Fieldbus Mappings for PC Modbus Master for more information.
  7. After you configure the device, connect the device directly to the network card that is running under real-time control. Aerotech recommends that you connect directly by using a crossover cable or through a network switch.
