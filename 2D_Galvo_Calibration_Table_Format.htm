2D Galvo Calibration Table Format

A 2D axis calibration file can contain standard 2D calibration tables and calibration tables for an Aerotech galvo scanner that is connected to a GL4 or an Nmark CLS or GCL. The format of a 2D galvo calibration table is almost the same as the format of a standard 2D axis calibration table, with the differences and restrictions that follow.

  • A 2D galvo calibration table starts with a :GALVO2D token. A 2D axis calibration table starts with a :START2D token.
  • The ColumnAxis and OutputAxis1 arguments must be the first axis of the GL4 or the Nmark CLS or GCL.
  • The RowAxis and OutputAxis2 arguments must be the second axis of the GL4 or the Nmark CLS or GCL.
  • The SampDistRow and SampDistCol arguments must be positive.
  • You cannot use the following arguments.
    • OFFSETROW
    • OFFSETCOL
    • INTABLE
  • The table must have an odd number of rows and columns. The center of the table corresponds to the home position of the galvo scanner.
  • The table can have a maximum number of 138,000 points.

When you use a 2D galvo calibration table with the Nmark CLS, the restrictions that follow also apply.

  • The SampDistRow and SampDistCol arguments must be equal.
  • The table must have the same number of rows and columns.
  • The table can have a maximum number of 257 rows and 257 columns.

NOTE: Typically, axis indices are specified as zero-based integers. But calibration files specify axis indices as one-based integers.
