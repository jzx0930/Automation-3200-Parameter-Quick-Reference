I/O Overview

The following types of I/O are available.

The table that follows shows the number of analog and digital I/O points available on each drive type. For more information, refer to the hardware manuals for your drives. All I/O numbering is zero-based. For example, the first analog input is Analog Input 0.

Drive Modbus

The drive Modbus implementation supplies I/O over a TCP/IP connection using the local Ethernet port on the drive. This is supported on the Ndrive HPe and HLe, Npaq (not the Npaq MR), and Nservo drive types. Refer to Drive Modbus Overview for more information.

PC Modbus

The PC Modbus implementation supplies I/O over a TCP/IP connection using the Ethernet port on the PC. This supports one slave connection and one master connection. If you have the Professional Option, you can have a maximum of 16 master connections. Refer to PC Modbus Overview for more information.

Virtual I/O

Virtual I/O is not directly connected to physical I/O bits and is more like a register set that is used to communicate between tasks or other processes on the PC. For more information about binary and register connections, refer to the $BI and $BO functions and the $RI and $RO functions.
