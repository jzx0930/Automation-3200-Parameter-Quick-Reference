Imprecision of Generated Circular Motion

To avoid inaccuracies and to prevent the controller from causing a task error, Aerotech recommends that you supply six or more digits after the decimal point for all values that you specify to the LINEAR, CW, CCW, and BEZIER commands. When you specify numeric values in an AeroBasic motion program, those values are specified with a finite number of decimal digits. If you do not specify enough digits to your moves, the controller might not be able to generate the correct motion. Supplying more decimal digits does not have a negative effect on program execution and does not require more controller memory.

Circular Moves

If you do not specify a sufficient number of digits to the IJ method of the CW and CCW commands or the IJK method of the G2 and G3 commands, the controller might not generate a valid arc. The controller tries to correct inaccuracies by using the tolerance that you specify in the RadiusErrorThreshold task parameter. But errors that are higher than this tolerance cause a task error.

You can specify more decimal digits to your command moves to prevent most inaccuracies. But you can also use an alternative to the IJ method of the CW and CCW commands or the IJK method of the G2 and G3 commands to try to prevent inaccuracies when you specify an arc. Refer to the CW and CCW commands or the G2 and G3 commands for more information about the alternative methods.

For Example

The program that follows makes an arc. Only 2 decimal digits are supplied.

Example

ABSOLUTE
LINEAR X1 Y1
CW X3.67 Y1.00 I1.33 J-1.33

Assume that the arc is supposed to have a start coordinate of (1, 1), an end coordinate of (3.666666666, 1), and a center point of (2.333333333, - 0.3333333333). But, because the values that are specified to the command are not accurate, the CW move cannot be achieved because the distance between the center point and start point is not the same as the distance between the center point and end point.

Normalcy

When you use normalcy mode, slowdowns to zero velocity can occur if consecutive moves are not exactly tangent. Use the NORMALCY TOLERANCE command to avoid slowdowns between moves that are close to tangent, but not exactly tangent. Specify more decimal digits to your moves to prevent the controller from treating tangent moves as non-tangent moves. Also, you can use corner rounding so that the controller makes the motion tangent by inserting an arc between two consecutive LINEAR moves.

Cutter Radius Compensation

When you use cutter radius compensation with the CoordinatedAccelLimit parameter, small arcs are inserted if your moves are not exactly tangent. This can cause slowdowns in VELOCITY ON mode. Use the CutterTolerance parameter to avoid the insertion of small arcs when consecutive moves are close to tangent, but not exactly tangent. Specify more decimal digits to your moves to prevent the controller from treating tangent moves as non-tangent moves. Also, you can use corner rounding so that the controller makes the motion tangent by inserting an arc between two consecutive LINEAR moves.
