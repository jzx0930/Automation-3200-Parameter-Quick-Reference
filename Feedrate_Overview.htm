Feedrate Overview

You must specify a feedrate when you do coordinated motion. Dominant and dependent axes have different methods of specifying the feedrate. Feedrates are commonly specified in distance units per time (for example, millimeters/second and inches/minute). Feedrates are modal and remain in effect until a new feedrate is specified.

Coordinated types of motion such as LINEAR, CW, CCW, and BEZIER use one feedrate that specifies the combined vector speed, which is also called the coordinated speed. This feedrate is the speed of the path through space. It is not an individual axis speed.

Non-coordinated types of motion, such as RAPID, FREERUN, MOVEABS, and MOVEINC, use individual axis speeds.

Parameters can be used to set the default feedrates or read the current feedrate value. New feedrate values are set using the program syntax (E, F, XF, YF, and so on). Refer to the examples that follow.

Program Syntax Examples

' Specify the feedrate of the dominant axes.
F<dominant_axes_feedrate>
' Specify the feedrate of the dependent axes.
E<dependent_axes_feedrate>

Example Program

' Set the units to primary units.
PRIMARY
' Set the time base to seconds.
SECONDS
' Set the feedrate of the dominant axes to 100 mm/s.
F100
' Move the X axis at 100 mm/s.
LINEAR X24
' Move the X and Y axes at 100 mm/s combined vector speed and a speed of 70.7 mm/s for each axis.
LINEAR X24 Y24 
' Move the X axis at 200 mm/s.				            
LINEAR X24 F200
' This is a non-coordinated move. The X axis moves at 100 mm/s. The Y axis moves at 200 mm/s.
RAPID X24 Y10 XF100 YF200

The nominal feedrate (as programmed) can be read using the DefaultCoordinatedSpeed and DefaultDependentCoordinatedSpeed parameters. Actual feedrates may be different and are read using the TASKSTATUS Status Function.

Actual feedrate values can differ from the programmed values because:

  • Acceleration and deceleration ramping can cause the speed at the beginning and end of a move to be less than the programmed feedrate.
  • The Manual Feedrate Override (MFO) is not set to 100%. MFO can also be set through the MFO command (a value of 1.0 = 100%).
  • If the MaxSpeedClamp axis parameter is set to a nonzero value, that value is the maximum attainable feedrate.
  • If set, the TrajectoryIIRFilter axis parameter filters velocities and delays speed changes.
  • Acceleration limiting can force slower speeds (only applies to coordinated motion with VELOCITY ON commands). Refer to Acceleration and Deceleration Ramping Overview for more information.
Table 1: Coordinated Motion Feedrate Commands
Command Description

F

Specifies a feedrate value for a coordinated move on a dominant axis.

E

Specifies a feedrate value for a coordinated move on a dependent axis.
Table 2: Feedrate Task Parameters
Parameter Description

DefaultCoordinatedSpeed

Set default dominant feedrate.

DefaultDependentCoordinatedSpeed

Set default dependent feedrate.

Default feedrates used during independent (non-countoured) motion, such as FREERUN and homing, are specified in the axis parameters listed in Table 3. FREERUN move commands specify the axis feedrate directly.

Table 3: Independent Motion Feedrate Parameters
Parameter Description

MaxJogSpeed

Velocity of the axis on the jog screen

DefaultSpeed

The default feedrate for RAPID motion commands

HomeSpeed

Feedrate for HOME commands
