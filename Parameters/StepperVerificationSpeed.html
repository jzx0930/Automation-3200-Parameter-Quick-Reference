StepperVerificationSpeed Parameter

Default Value

0.0

Minimum Value

0.0

Maximum Value

N/A

Units

Primary Units /Second

Type

64-bit IEEE floating point

Note: When you reference this parameter in an AeroBasic program, the parameter value is in the units specified by the unit mode of the task. If the task is in primary units mode, this parameter is in primary units. If the task is in secondary units mode, this parameter is in secondary units.

hardware: This parameter applies only when used with stepper motors (the MotorType parameter is set to Stepper Motor) with a position feedback device. (The PositionFeedbackType parameter is set to something other than None.)

Use the StepperVerificationSpeed parameter to enable position verification for a stepper motor and to define the speed at which the axis does a required correction. The A3200 always commands a stepper motor in an open loop configuration. The A3200 can also do stepper position verification with a stepper motor that has a feedback device attached. When you use stepper verification, the feedback device verifies that the stepper motor has reached the commanded position and corrects the position if necessary. A correction move for stepper verification can only occur when the stepper axis is not being commanded to move. If the stepper motor fails to hold position during commanded motion, the correction can only occur when the move has completed.

On the A3200, there are two modes available for stepper verification that are controlled by the Stepper Verification setting of the ServoSetup parameter. In the ServoSetup parameter, the default value of the Stepper Verification setting is set to Triggered At Move End mode. When you use this mode, the position of the stepper motor is verified at the end of each commanded move. If the axis is not in-position, as defined by the InPositionDistance and InPositionTime parameters, a correction move is initiated. This correction move occurs at the speed that is defined by the StepperVerificationSpeed parameter and does not complete until the position error of the axis crosses zero.

In the ServoSetup parameter, the second value of the Stepper Verification setting is set to Triggered When Not In Position mode. When you use this mode, the controller monitors the in-position status of the stepper axis and initiates a correction move when it goes out of position. In this mode, if the axis stays in-position as defined by the InPositionDistance and InPositionTime parameters, stepper verification does not have an effect. If the axis is not in-position, the correction move occurs at the speed that is defined by the StepperVerificationSpeed parameter and does not complete until the position error of the axis crosses zero.

Selecting the Correct Value

To prevent overshooting the target position, you should set the StepperVerificationSpeed parameter to a value that is less than one count per millisecond. Because you set the value of this parameter in user units per second, you must convert the units to determine the correct value.
