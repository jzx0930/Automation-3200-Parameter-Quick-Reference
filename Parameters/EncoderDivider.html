EncoderDivider Parameter

Default Value

0

Minimum Value

0

Maximum Value

1024

Units

None

Type

32-bit signed integer

Hardware: This parameter applies only when used with the GL4, the Ndrive CP, FCL, HLe, HPe, ML, MP, QL, or QLe, the Nmark GCL, the XC2, the XC4, the XC4e, the XL4s, or the XR3.

Some supported drives are not valid with this parameter when you use them in specific drive type and feedback combinations because the echo encoder functionality is not available. Drive type and feedback combinations that the EncoderDivider parameter does not support include:

  • An Ndrive CP or MP when the axis is configured to use the optional primary encoder multiplier for feedback.
  • An XC2, XC4, or XR3 when the axis is configured to use an encoder multiplier with primary feedback from the -MX1 multiplier option.
  • An XC4e when the axis is configured to use an encoder multiplier with auxiliary feedback from the -MX3 multiplier option.

Use the EncoderDivider parameter to configure the bi-directional encoder interface on the auxiliary I/O connector. The default value of this parameter is 0, which configures the auxiliary encoder interface as an encoder input.

If you specify a nonzero value for the EncoderDivider parameter, the drive changes the auxiliary encoder interface to an output. It uses the specified value as a divisor for the encoder echo. The drive divides the encoder input stream by the specified value and echos it out of the auxiliary encoder interface. As a result, the output signals have a lower frequency.

NOTE: If your drive uses one of the primary feedback types that follow, you must set the EmulatedQuadratureDivider Parameter to a nonzero value:

  • Capacitance Sensor feedback for the Ndrive QL or QLe
  • -MXH feedback for other FireWire drives
  • Encoder Multiplier for the GL4 or the XL4s
  • Encoder Multiplier with the -MX2 or -MX3 multiplier option for the XC4e
  • Encoder Multiplier with the -MX2 or -MX4 multiplier option for the XR3

If you are using an XR3 that gets auxiliary feedback from an encoder multiplier with the -MX4 multiplier option, you must set the AuxiliaryEmulatedQuadratureDivider Parameter to a nonzero value.
