NOTE: The controller generates an axis fault only if you set the corresponding bit in the FaultMask parameter. When an axis fault occurs, you can configure what the axis does by changing the state of the corresponding bit in the FaultMaskDecel and FaultMaskDisable parameters.

Use the FaultMask parameter to define the axis fault conditions that are detected by the controller. This parameter is a bit mask where each bit corresponds to a specific axis fault.

For most faults, the controller only reports the fault condition if the axis is enabled. The Emergency Stop fault and the Communication Lost fault, however, can be reported when the axis is disabled. If you set the Feedback Fault Reporting setting of the FaultSetup parameter to Always, the Position Feedback and Velocity Feedback faults can also be reported when the axis is disabled.

You can independently configure what the axis does for each fault condition by using the settings in the following table.

Table:     Axis Fault Behaviors

If an axis is in a fault condition, the axis completes the action that you specify in the FaultMask, FaultMaskDecel, and FaultMaskDisable parameters. Before normal operations can continue, you must recover from the axis fault by doing the following.

  1. Correct the condition that caused the controller to generate an axis fault.
  2. Acknowledge the fault condition by doing one of the following operations.
    • Click the Acknowledge All button or issue an ACKNOWLEDGEALL command to clear all of the axis faults on the controller.
    • Issue the FAULTACK command to acknowledge the axis faults on the axis that you specify.

Examining Axis Faults

For more information about an axis fault that the controller generates, do one of the following options.

  • Look at the axis fault in CNC Operator Interface, Digital Scope, or Motion Composer. These applications show axis faults in red text.
  • In your AeroBasic program, use the DATAITEM_AxisFault item to see all of the axis faults that the controller generates. For more information about this data item, refer to the AXISSTATUS function.
  • For advanced diagnostics information about the axis fault, use the Fault tab in the Status Utility.

Note: For all axes that are used in a gantry configuration, you must set the FaultMask, FaultMaskDecel, FaultMaskDisable, and FaultMaskDisableDelay parameters to the same value.

For more information about axis faults, refer to Fault and Error Overview.

AXISFAULT_VelocityFbk

Velocity Feedback Fault

The drive detected a problem with the feedback device specified by the VelocityFeedbackType and VelocityFeedbackChannel parameters. Or, the drive detected an invalid velocity feedback configuration.

The drive detected an invalid state (all high or all low) for the Hall-effect sensor inputs on this axis.

Maximum Velocity Command Fault

The commanded velocity is more than the velocity command threshold. Before the axis is homed, this threshold is specified by the VelocityCommandThresholdBeforeHome parameter. After the axis is homed, this threshold is specified by the VelocityCommandThreshold parameter.

0x00000800

12

0x00001000

Velocity Error Fault

The absolute value of the difference between the velocity command and the velocity feedback exceeded the threshold specified by the VelocityErrorThreshold parameter.

14

Probe Input Fault

The user-defined touch probe input was triggered.

15

0x00008000

AXISFAULT_ExternalFault

External Fault

The external fault input, specified by the ExternalFaultAnalogInput, ExternalFaultDigitalInput, or SoftwareExternalFaultInput parameter, was triggered.

17

0x00020000

AXISFAULT_MotorTemp

Motor Temperature Fault

The motor thermistor input was triggered, which indicates that the motor exceeded its maximum recommended operating temperature.

18

0x00040000

AXISFAULT_AmplifierTemp

Amplifier Temperature Fault

The amplifier temperature is not within the operating range. For a FireWire drive, this fault occurs at temperatures greater than 75 C. For a HyperWire drive, this fault occurs at temperatures less than 0 C or greater than 75 C.

19

0x00080000

The encoder fault input on the motor feedback connector was triggered.

20

0x00100000

22

25

26

27

28

The power supply output has exceeded the allowable power or temperature threshold.

29
