Hardware: You cannot use Gantry mode with the GL4, the Ndrive FCL, FLS, QL, or QLe, the Nmark CLS, GCL, or SSaM, or the XL4s.

Note: For a complete list of the steps required to configure two (or more) axes in gantry mode, refer to the Configuring Parameters for Gantry Mode topic.

Use the GantrySetup parameter to enable gantry mode and select the gantry configuration. Set this parameter on only the slave axes of a gantry. Keep the master axis set to Gantry Mode Disabled, which is the default. You must set this parameter to the same value on all slaves in the gantry set or an error occurs. For more information, refer to GantryConfiguration in the Troubleshooting section of the Help file.

Gantry Mode Disabled

Gantry Mode Disabled is the default. All axes operate independently.

Position Gantry Mode

  • You cannot use Position Gantry Mode if you have more than one slave axis.
  • The master axis and slave axis each have a feedback device, Hall sensors, and limits.
  • The master axis and slave axis run servo loops and motor commutations.
  • The master axis and slave axis are sent the same motion commands, except in special situations such as homing.
  • The master axis and slave axis use the marker index pulse for alignment during the home cycle.
  • The master axis and slave axis use the HomeOffset parameter independently to adjust the final home location and orthogonality of the gantry.

Current Gantry Mode (Synchronized Commutation)

  • You can use Synchronized Commutation if you have one or more slave axes.
  • Only the master axis has a feedback device, Hall sensors, and limits.
  • Only the master axis runs the servo loop and motor commutations.
  • Each slave axis generates a current command that is the same as the current command of the master axis.

Current Gantry Mode (Offset Commutation)

  • You can use Offset Commutation if you have one or more slave axes.
  • Only the master axis has a feedback device and limits.
  • The master axis and slave axes have Hall sensors.
  • Only the master axis runs the servo loop.
  • Only the master axis runs motor commutation. But because each axis has Halls, you can offset the commutations between the master axis and slave axes.
  • Each slave axis generates a current command that is the same as the current command of the master axis.

Current Gantry Mode (Independent Commutation)

  • You can use Independent Commutation if you have one or more slave axes.
  • The master axis and slave axes each have a feedback device, a motor, and Hall sensors.
  • Only the master axis has limits.
  • Only the master axis runs the servo loop.
  • The master axis and slave axes run motor commutations.
  • Each slave axis generates a current command that is the same as the current command of the master axis.
