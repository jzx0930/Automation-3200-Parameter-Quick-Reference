Use the HomeSetup parameter to configure options related to the home cycle. The sections that follow give the possible options for this parameter.

Home Start Direction

Note: This setting does not apply when you set the HomeType parameter to either Home at Current Position or Home at Current Position Feedback because no motion occurs with these types.

This option specifies the direction of initial motion during the home cycle.

CCW/Negative Direction is the default. The initial movement during the home cycle is toward the CCW or negative encoder direction.

When you select CW/Positive Direction, the initial movement during the home cycle is toward the CW or positive encoder direction.

This option is based on the encoder direction instead of the programming direction. The ReverseMotionDirection parameter does not have an effect on the direction of the home cycle.

Limit Type

Note: This setting applies only when you set the HomeType parameter to Home Past Limit to Marker, Home to Limit and Reverse to Marker, or Home to Limit Only.

This option specifies the type of limit switch that is used for home types that require a limit.

Use End-of-Travel Limit Switch is the default. The home cycle uses the correct end-of-travel limit input based on the setting of the Home Start Direction.

When you select Use Home Limit Switch, the home cycle uses the home limit input.

Marker Input Source

NOTE: This setting applies only to axes configured in a dual-loop configuration.

NOTE: This setting applies only when you set the HomeType parameter to Home Past Limit to Marker, Home to Limit and Reverse to Marker, or Home to Marker Only.

This option specifies the marker input that is used for home types that require a marker.

Use Position Feedback is the default. The home cycle uses the marker input from the feedback device specified by the PositionFeedbackType and PositionFeedbackChannel parameters.

When you select Use Velocity Feedback, the home cycle uses the marker input from the feedback device specified by the VelocityFeedbackType and VelocityFeedbackChannel parameters.

Gantry Homing Limit Synchronization

Warning: Setting this parameter to Independent Abort can cause each axis of the gantry to have different motion and can cause damage to the hardware.

Note: This setting applies only when you set the HomeType parameter to Home Past Limit to Marker, Home to Limit and Reverse to Marker, or Home to Limit Only.

This option specifies if motion to a limit switch is synchronized during the home cycle when you configure the axis for gantry mode.

Synchronized Abort is the default. Motion to a limit switch during the home cycle is synchronized at all times. In this configuration, you must make sure that the limit-switch inputs will trigger at the same time on both the master and slave axes. The controller will synchronize the motion of the axes so that both axes abort at the same time.

When you select Independent Abort, motion to a limit switch can be aborted at different times on the master and slave axes. In this configuration, you can home axes whose limit switches are triggered at different times.
