EmulatedQuadratureChannel Parameter

Default Value

0

Minimum Value

0

Maximum Value

8

Units

None

Type

32-bit signed integer

Hardware: This parameter applies only when used with the Npaq with either the -MXR or the resolver options. It does not apply to the Npaq MR.

Use the EmulatedQuadratureChannel parameter to configure an MXR or resolver feedback channel to send real-time quadrature output to one of three available PSO channels. The optional MXR and resolver boards each contain up to six input channels, but only three of these channels can be configured to output real-time quadrature signals. This parameter is used to configure which channels generate quadrature, with the remaining channels being unable to generate quadrature outputs.

The default value of this parameter is None (a value of 0), which means that the MXR or resolver channel associated with this axis does not output quadrature. When you select To PSO Channel 6 (a value of 6), To PSO Channel 7 (a value of 7), or To PSO Channel 8 (a value of 8), the MXR or resolver channel that corresponds to this axis (that is, setting this on axis 0 affects input channel 0) outputs quadrature to the specified PSO channel. Values of 1 to 5 are invalid and result in no quadrature output. You can only configure one input channel to output quadrature to a given PSO channel, which means that two axes within a single Npaq cannot have this parameter set to the same (nonzero) value.

If you use an Npaq with MXR, Aerotech recommends that you do not change the value of the EmulatedQuadratureChannel parameter programmatically because this might require the Npaq to internally reprogram the MXR hardware. Unexpected quadrature outputs might also occur during the programming cycle. If you must change the value of this parameter in a program, change the value at a time when the quadrature outputs are not being used. You must also examine the state of the Programming MXH bit of the Drive Status to make sure that programming has completed before you continue.

Note: When you have a FireWire drive that uses -MXH or -MXR feedback, quadrature is generated only if the effective multiplication factor is greater than 4. The effective multiplication factor is equal to the value of the EncoderMultiplicationFactor parameter divided by the value of the EmulatedQuadratureDivider parameter.
