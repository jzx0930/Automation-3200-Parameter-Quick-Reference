Use the IOSetup parameter to configure options related to I/O. The sections that follow give the possible options for this parameter.

Software ESTOP Filtering

This option enables software-based filtering of the ESTOP input.

Disabled is the default.

When you select Enabled, the ESTOP input is filtered by a 10 millisecond software filter. This means that the ESTOP input must be active for a minimum of 10 milliseconds before the controller generates an ESTOP fault.

Software Opto Filtering

This option enables software-based filtering of the opto-isolated digital inputs.

Disabled is the default.

When you select Enabled, the opto-isolated inputs are filtered by a 2 millisecond software filter. This means that the input must be active for a minimum of 2 milliseconds before the controller recognizes the active state.

Opto Output Mode

Hardware: This setting does not apply when used with the Ndrive QL or QLe, the XC2, the XC4, the XC4e, or the XR3.

This option specifies the output mode for the opto-isolated digital outputs.

When you select Current Sourcing, the outputs are configured for current sourcing mode.

Current Sinking is the default. The outputs are configured for current sinking mode.

Opto Output Mode, Optional IO

Hardware: This setting applies only when used with the Ndrive CP, ML, or MP with the -IO option.

This option specifies the output mode for the opto-isolated digital outputs that are in the -IO option.

Same as Standard IO is the default. The controller configures the outputs in the -IO option for the same mode as the standard opto-isolated digital outputs. To change this mode, use the Opto Output Mode option of this parameter.

When you select Sourcing and Sinking, the controller configures the outputs to source and sink current. This configuration, which is sometimes referred to as a push-pull output, can improve the rise and fall times of the outputs when you connect them to a load that has a high resistance.

Analog Input 0 Range

Hardware: This setting applies only when used with the Npaq or the Nservo. It does not apply to the Npaq MR.

This option specifies the sensitivity of analog input 0.

When you select +/-1 Volt, the input is configured for a maximum range of +/-1 Volt with a resolution of 30.5 microvolts per ADC count.

When you select +/-2.5 Volts, the input is configured for a maximum range of +/-2.5 Volts with a resolution of 76.3 microvolts per ADC count.

When you select +/-5 Volts, the input is configured for a maximum range of +/-5 Volts with a resolution of 152.6 microvolts per ADC count.

+/-10 Volts is the default. The input is configured for a maximum range of +/-10 Volts with a resolution of 305 microvolts per ADC count.

Analog Input 1 Range

Hardware: This setting applies only when used with the Npaq or the Nservo. It does not apply to the Npaq MR.

This option specifies the sensitivity of analog input 1.

When you select +/-1 Volt, the input is configured for a maximum range of +/-1 Volt with a resolution of 30.5 microvolts per ADC count.

When you select +/-2.5 Volts, the input is configured for a maximum range of +/-2.5 Volts with a resolution of 76.3 microvolts per ADC count.

When you select +/-5 Volts, the input is configured for a maximum range of +/-5 Volts with a resolution of 152.6 microvolts per ADC count.

+/-10 Volts is the default. The input is configured for a maximum range of +/-10 Volts with a resolution of 305 microvolts per ADC count.

Npaq PSO Interlock

Hardware: This setting applies only when used with the Npaq. It does not apply to the Npaq MR.

Note: Because there is only one PSO output, this setting applies only when set on the first axis of the Npaq.

This option specifies the behavior of the PSO interlock input on the Npaq.

When you select Active, the PSO interlock input must be driven to allow firing of the PSO output.

Defeated is the default. PSO firing can occur regardless of the state of the PSO interlock input.

Npaq Internal Brake

Hardware: This setting applies only when used with the Npaq. It does not apply to the Npaq MR.

This option specifies the behavior of the brake signals on the Npaq.

Use BrakeOutput Parameter is the default. Use the BrakeOutput Parameter to specify the type of output that controls the brake.

When you select Use Sync Output (Active High), the controller uses the SYNCOUT signals to generate an active-high brake control signal. The axis index in the Npaq specifies the SYNCOUT output number and the controller ignores the value of the BrakeOutput parameter.

When you select Use Sync Output (Active Low), the controller uses the SYNCOUT signals to generate an active-low brake control signal. The axis index in the Npaq specifies the SYNCOUT output number and the controller ignores the value of the BrakeOutput parameter.

Fan Control

Hardware: This setting applies only when used with the GL4, the Ndrive FCL, FLS, or ML, the Nmark CLS or GCL, the Npaq, the XL4s, or the XR3. It does not apply to the Npaq MR.

This option specifies the behavior of the cooling fan of the drive.

Temperature Controlled is the default. The fan speed is based on the heat sink temperature of the drive.

When you select Always Full Speed, the fan always runs at full speed regardless of the heat sink temperature.
