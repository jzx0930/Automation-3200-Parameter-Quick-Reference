StepperResolution Parameter

Default Value

0

Minimum Value

0

Maximum Value

N/A

Units

Microsteps

Type

32-bit signed integer

Use the Stepper Resolution parameter to specify the microstepping resolution of a stepper motor. This parameter specifies the number of internal counts (microsteps) per revolution for a standard 50 pole pair stepper motor. Larger values of this parameter result in finer positioning resolution and smoother motor operation.

When you use a stepper motor that does not have a feedback device, the axis is commanded in terms of the microstepping resolution of the motor. In this configuration, you must set the CountsPerUnit parameter to the number of microsteps per unit.

Note: The Nstep uses clock and direction outputs to control an external stepper drive. The Nservo also uses clock and direction outputs to control an external stepper drive when the Nservo Stepping Mode setting of the ServoSetup parameter is set to Clock/Direction Output. When you do not use a feedback device, the StepperResolution parameter is ignored and the CountsPerUnit parameter is used to calculate the number of clock pulses that will be generated per unit.

When you use a stepper motor with a feedback device, the axis is commanded in terms of feedback resolution, and the CountsPerUnit parameter specifies the number of feedback counts per unit. The StepperResolution parameter specifies the microstepping resolution and is typically set equal to or greater than the effective feedback resolution. The drive uses the ratio of the StepperResolution and CountsPerUnit parameters to correctly step the motor.

If you are using a 50 pole pair (1.8°) stepper motor, the controller internally scales the StepperResolution parameter. If you are using a stepper motor with a different number of pole pairs, you must use the following equation to set the StepperResolution parameter.

For Example

To configure a 100 pole pair (0.9°) stepper motor for 100,000 microsteps per revolution, set StepperResolution parameter to 50000.
