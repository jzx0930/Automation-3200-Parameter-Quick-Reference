PiezoVoltsPerUnit Parameter

Default Value

1.0

Minimum Value

0.0

Maximum Value

N/A

Units

Volts / User Unit

Type

32-bit IEEE floating point

Warning: Setting this parameter to an incorrect value can cause the motor to fail and might cause damage to the motor.

Hardware: This parameter applies only when used with the Ndrive QL or QLe.

You can use this parameter only when the servo loop is turned off. Refer to the SERVO ON/OFF command.

Use the PiezoVoltsPerUnit parameter to specify the quantity of voltage that the controller must apply to a piezo actuator in order to generate one unit of motion. The hardware manual for the piezo stage shows the maximum voltage of the piezo actuator and the length of distance traveled of the actuator.

Programming in Volts

To command the piezo stage in volts, make sure that the PiezoVoltsPerUnit parameter is set to 1. A value of 1 is the default. When this parameter is set to 1, the units change to volts for all of the values that you specify in motion commands.

Programming in Position Units

If the data supplied by the manufacturer is not in the same units as your position units, you must convert the length of the distance traveled into the position units you are using. Set the value of this parameter to the maximum voltage divided by the length of distance traveled. Refer to the equation that follows.

The PiezoVoltsPerUnit parameter is almost the same as the GainPff parameter. The controller uses the PiezoVoltsPerUnit parameter only when the servo loop is off. The controller uses the GainPff parameter only when the servo loop is on. If you already configured the GainPff parameter, Aerotech recommends that you set the PiezoVoltsPerUnit parameter to the same value.
