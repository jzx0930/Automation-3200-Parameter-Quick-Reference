Use the ServoSetup parameter to configure options related to the servo loop. The sections that follow give the possible options for this parameter.

GainKpos Control

This option specifies the behavior of the GainKpos servo loop gain.

When you select Disabled During Motion, the effect of the GainKpos servo loop gain is disabled during commanded motion.

Always Enabled is the default. The GainKpos servo loop gain behaves normally.

Velocity Integrator Control

This option specifies the behavior of the integrator in the velocity part of the servo loop.

Always Enabled is the default. The integrator behaves normally.

When you select Disabled During Motion, the operation of the velocity integrator is disabled during commanded motion.

Feedforward Control

This option specifies the behavior of the feedforward terms (GainAff and GainVff) in the servo loop.

After Servo Filters is the default. The feedforward terms are added to the current command after the servo loop digital filters.

When you select Before Servo Filters, the feedforward terms are added to the current command before the servo loop digital filters.

Stepper Verification

Note: This setting applies only when you set the MotorType parameter to Stepper, the PositionFeedbackType parameter to a value other than None, and the StepperVerificationSpeed parameter to a nonzero value.

This option specifies the behavior of the stepper verification feature of the controller.

Triggered At Move End is the default. Stepper verification correction moves, if necessary, occur only one time at the end of each commanded move.

When you select Triggered When Not In Position, stepper verification moves will be triggered any time the axis is not in-position, as long as the axis is not being commanded to move.

Nservo Stepper Mode

Note: This setting applies only when used with the Nservo and you set the MotorType parameter to Stepper.

This option specifies the behavior of the Nservo when configured to control a stepper motor.

Clock/Direction Outputs is the default. The Nservo generates clock and direction signals to send to an external stepper motor amplifier.

When you select Current Command Outputs, the Nservo generates two phase current commands to control the stepper motor.

Motor Outputs When Disabled

Hardware: This setting applies only when used with the Ndrive MP, an Npaq MR with MP drives, or the XC2.

This option specifies the behavior of the motor outputs.

Open Circuit is the default. The motor outputs are not connected when the axis is disabled.

When you select Shorted to Negative Bus, all three motor outputs are shorted together and tied to the negative motor supply when the axis is disabled.

Nservo Current Command Mode

Hardware: This setting applies only when used with the Nservo.

This option specifies the behavior of the Nservo current command outputs.

Single-ended, +/-10 Volts is the default. The current command outputs will output a +/-10 Volt signal when connected in a single-ended configuration.

When you select Single-ended, 0-10 Volts, the current command outputs will output a 0-10 Volts signal when connected in a single-ended configuration.

When you select Differential, +/-10 Volts, the current command outputs will output a +/-10 Volt signal when connected in a differential configuration.

Tachometer Mode

Hardware: This setting applies only when used with the Npaq or the Nservo. It does not apply to the Npaq MR.

This option specifies the drive to operate using an external analog tachometer loop.

Disabled is the default. The servo loop operates normally.

When you select Enabled the velocity feedback is disconnected from the servo loop, causing it to operate using only the position loop. Then, the external tachometer controls the velocity loop.

Current Command Polarity

Hardware: This setting does not apply when used with the GL4, the Ndrive FLS, or the Nmark CLS or GCL.

This option specifies the polarity of the current command output from the servo loop.

Normal is the default. When the output of the servo loop is calculated, a positive position error causes a positive current command output.

When you select Inverted, when the output of the servo loop is calculated, a positive position error causes a negative current command output. To compensate for motor or feedback phasing errors without requiring changes to the wiring, use this setting with the Commutation Feedback Direction setting.

Commutation Feedback Direction

This option specifies how the feedback device is read by the A3200 for determining the motor commutation.

Normal is the default. The A3200 uses the velocity feedback signal to determine the motor commutation.

When you select Inverted, the velocity feedback signal is inverted before being used to determine the motor commutation. To compensate for motor or feedback phasing errors without requiring changes to the wiring, use this setting with the Current Command Polarity setting.

Hall Commutation Direction

This option specifies the sequence of Hall states during motor operation.

Normal is the default. During a clockwise rotation of the motor, Hall state sequences are the same as the sequences that Table: Hall States Expected at Different MSET Angles shows for the CommutationOffset parameter.

When you select Inverted, the drive commutates in the order of the reversed Hall state sequences. To compensate for Hall state wiring errors without requiring changes to the wiring, use this setting together with the CommutationOffset parameter to make sure the drive electrical commutation is correct.
