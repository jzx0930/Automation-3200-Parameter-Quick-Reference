Hardware: This parameter applies only when used with the Ndrive CP, HLe, or HPe, the XC2, the XC4, the XC4e, or the XR3.

Use the ResoluteEncoderSetup parameter to configure options related to the behavior of the drive in relation to the Resolute absolute encoder interface. The sections that follow give the possible options for this parameter.

Commutation Initialization

This option specifies if absolute position information from the encoder is used to initialize the motor commutation.

When you select Do Not Use Absolute Position, the motor commutation is initialized through the standard method, using either Hall-effect switches or Auto-MSET mode depending on the setting of the MotorType parameter.

Use Encoder Absolute Position is the default. The motor commutation is initialized using the absolute position information read from the Resolute encoder, and the commutation assumes that the zero position of the encoder corresponds to the motor zero. To compensate for misalignment between the encoder and the motor, use the CommutationOffset parameter.

Absolute Position Availability

This option specifies when the drive reads position information from an attached Resolute absolute encoder.

Only When Used For Feedback is the default. The drive reads positions from the encoder only when you set the PositionFeedbackType parameter to Resolute Absolute Encoder.

When you select Always Available, the drive always reads positions from the encoder. The setting that you specify for the PositionFeedbackType parameter does not have an effect. Use Always Available only for systems where you want to use the Resolute absolute encoder to give status from a device that you are not using as feedback for the servo loop.

Data Alignment

This option specifies the location of the encoder position data in the Resolute encoder serial packet when the drive reads position information from an attached Resolute absolute encoder.

Left Justified is the default. If the position data is before the status bits in the serial packet, select Left Justified.

If the position data is after the status bits in the serial packet, select Right Justified.

Initial Position

This option specifies how the drive initializes the position feedback of an axis when you use a Resolute absolute encoder for feedback.

Always Positive is the default. When the drive initializes the position of the axis, it interprets the value from the encoder as an unsigned number. Thus, the initial position is always positive. For example, if you set the ResoluteEncoderUserResolution Parameter to 16, the drive initializes the position feedback to a value between 0 and 65,535.

When you select Positive or Negative, the drive interprets the value from the encoder as a signed number when it initializes the position of the axis. Thus, the initial position can be positive or negative. For example, if you set the ResoluteEncoderUserResolution Parameter to 16, the drive initializes the position feedback to a value between -32,768 and 32,767.

CRC Error Suppression

This option controls if the drive ignores CRC errors that occur while it communicates with the encoder.

None is the default. The drive does not ignore CRC errors. If a CRC error occurs, the drive gives a Position Feedback fault.

When you select Single cycle events, the drive ignores a CRC error for only one servo cycle. If a CRC error occurs in two cycles, one after the other, the drive generates a Position Feedback fault. This setting is not recommended. This setting is only recommended for systems where electrical noise interrupts the encoder.
