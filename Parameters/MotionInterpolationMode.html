2-position / 2-velocity

3-position / 1-velocity

NOTE: Change the value of this parameter only if motion accuracy problems occur. As an alternative, you can make motion accuracy better by increasing the value of the MotionUpdateRate Parameter to prevent interpolation.

Use the MotionInterpolationMode parameter to change the cubic-spline polynomial algorithm that the controller uses to interpolate some motion commands to the position update rates of the axes. The controller interpolates the specified motion commands only for these conditions.

  • If the value of the MotionUpdateRate Parameter is less than the position update rate of an axis, the controller interpolates the BEZIER, CCW, CW, LINEAR, or RAPID motion command.
  • If the value of the <Time> argument that you specify to the PT or PVT motion command is greater than 1.0 divided by the position update rate of an axis, the controller interpolates the PT or PVT motion command.

Note: To use the PT command, you must set the MotionInterpolationMode parameter to 3-position / 1-velocity.

2-position / 2-velocity is the default setting of the MotionInterpolationMode parameter. The controller uses all available positions and velocities to generate a cubic spline interpolation. For this setting, the controller uses true velocities, which are the velocities that you specify. Aerotech recommends that you use this setting for BEZIER, CCW, CW, LINEAR, RAPID, and PVT motion.

When you set this parameter to 3-position / 1-velocity, the controller uses three positions and a previously estimated velocity to generate a cubic spline interpolation. For this setting, the controller uses an estimated velocity and ignores velocity that you specify. Aerotech recommends that you use this setting when true velocities are not available.
