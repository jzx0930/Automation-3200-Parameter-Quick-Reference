AuxiliaryEmulatedQuadratureDivider Parameter

Default Value

1.0

Minimum Value

0.0

Maximum Value

65536.0

Units

None

Type

32-bit IEEE floating point

Hardware: This parameter applies only when used with an XR3 that has an encoder multiplier with an -MX4 multiplier option.

Use the AuxiliaryEmulatedQuadratureDivider parameter to decrease the effective resolution of emulated quadrature signals. To generate emulated quadrature signals, your drive must have an optional auxiliary encoder multiplier module.

Specify a non-default value for this parameter when the system resolution and speed of motion cause the maximum encoder frequency of the axis to exceed the maximum quadrature output frequency of the drive. The non-default value does not change the resolution of the servo loop. It can be an integer or a non-integer value.

Specifying a Non-Integer Value for the AuxiliaryEmulatedQuadratureDivider Parameter

You can specify a non-integer value for the AuxiliaryEmulatedQuadratureDivider parameter when you use an optional auxiliary encoder multiplier module. Make sure the controller can use this value to generate an integer number of quadrature pulses per encoder cycle. This means that when you divide the value of the AuxiliaryEncoderMultiplicationFactor parameter by the value of the AuxiliaryEmulatedQuadratureDivider parameter, the result must be an integer.

If the result is a non-integer value, the controller rounds that value to the nearest integer and a small difference occurs between the expected quadrature resolution and the actual quadrature resolution.

To determine if you must specify a non-default value or the default value of 1 for this parameter, use the procedure that follows.

TO DETERMINE THE TYPE OF VALUE TO SPECIFY FOR THE AUXILIARYEMULATEDQUADRATUREDIVIDER PARAMETER

  1. Calculate the maximum encoder frequency of the axis. Use the equation that follows.

  2. Compare the value that you calculated for the maximum encoder frequency to the value of the maximum quadrature output frequency of the drive. Refer to the table that follows.
    3. Table: Maximum Quadrature Output Frequencies

    This table shows you the output frequencies at which the controller can send emulated quadrature signals out of the drives.

    Drive Type Multiplier Option Frequency (Counts per Second)

    XR3

    Using an Encoder Multiplier with -MX4

    30 MHz
  3. Based on the values that you compared, do one of the steps that follow:
    • If the maximum encoder frequency of the axis does not exceed the maximum quadrature output frequency of the drive, set the AuxiliaryEmulatedQuadratureDivider parameter to the default value of 1. It is not necessary for you to continue with this procedure.
    • If the maximum encoder frequency of the axis exceeds the maximum quadrature output frequency of the drive, continue to Step 4.
  4. Calculate the minimum frequency divider. This is the smallest possible value that you can set for the AuxiliaryEmulatedQuadratureDivider parameter. To make sure the value is compatible with this parameter, it must obey the conditions in Step 5.
    5. If You Are Not Using PSO

    Divide the maximum encoder frequency of the axis by the maximum quadrature output frequency of the drive. Refer to the equation that follows.

    If You Are Using the Emulated quadrature signals as a PSO Tracking Input
    1. In the Maximum PSO Encoder Input Data Rates section of PSO Specifications, find the encoder input data rate for the type of PSO tracking that you are using.
    2. Compare the encoder input data rate to the maximum quadrature output frequency of the drive.
    3. Use the smaller of the two values. Do one of the options that follow:
      • If the maximum quadrature output frequency is less than the encoder input data rate, use the equation that follows.

      • If the encoder input data rate is less than the maximum quadrature output frequency, use the equation that follows.

  5. Select a value for the AuxiliaryEmulatedQuadratureDivider parameter. Make sure it obeys the conditions that follow:
    • The value must result in an integer when you divide it into the AuxiliaryEncoderMultiplicationFactor parameter. An integer value makes sure that you get a whole number of counts per encoder cycle. Refer to the equation that follows.
    • The value must be greater than or equal to the minimum frequency divider that you calculated.
    • The value must be greater than 4 so the drive can generate emulated quadrature signals.

      • NOTE: If your HyperWire drive is configured to use an encoder multiplier with an -MX4 multiplier option, quadrature is generated only if the effective multiplication factor is greater than 4. The effective multiplication factor is equal to the value of the AuxiliaryEncoderMultiplicationFactor parameter divided by the value of the AuxiliaryEmulatedQuadratureDivider parameter.
  6. Set the AuxiliaryEmulatedQuadratureDivider parameter to the value that you selected. You can change the value of this parameter in Configuration Manager or from your AeroBasic program.
  7. The AuxiliaryEmulatedQuadratureDivider parameter decreases the effective resolution of the quadrature signals that are used for PSO tracking. If you set this parameter to a non-default value, divide your PSO firing distances by the value that you set for this parameter.

    8. For Example

    If you set the AuxiliaryEmulatedQuadratureDivider parameter to a value of 2, you must divide all distances specified in the PSODISTANCE command by a factor of 2.
