Use the FeedbackSetup parameter to configure options related to feedback. The sections that follow give the possible options for this parameter.

Position Feedback Direction

This option specifies how the drive reads the position feedback device.

Normal is the default. The drive reads the position feedback device normally. In this mode, the position feedback counts in a positive direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input.

When you select Inverted, the position feedback device is inverted. In this mode, the position feedback counts in a negative direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input. The function of this setting is to compensate for motor or feedback phasing errors without requiring changes to the wiring.

Velocity Feedback Direction

This option specifies how the drive reads the velocity feedback device.

Normal is the default. The drive reads the velocity feedback device normally. In this mode, the velocity feedback counts in a positive direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input.

When you select Inverted, the velocity feedback device is inverted. In this mode, the velocity feedback counts in a negative direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input. The function of this setting is to compensate for motor or feedback phasing errors without requiring changes to the wiring.

Limit Input Source

Hardware: This setting applies only when used with multi-axis drives. But it does not apply to the XR3.

This option specifies which feedback device is used as the source for the end-of-travel limit switches.

Velocity Feedback Channel is the default. The controller reads the limits from the feedback channel that you specify for the VelocityFeedbackChannel Parameter.

When you select Position Feedback Channel, the controller reads the limits from the feedback channel that you specify for the PositionFeedbackChannel Parameter.

Hall Input Source

Hardware: This setting applies only when used with multi-axis drives. But it does not apply to the XR3.

This option specifies which feedback device is used as the source for the Hall-effect switches.

Velocity Feedback Channel is the default. The controller reads the Hall-effect switches from the feedback channel that you specify for the VelocityFeedbackChannel Parameter.

When you select Position Feedback Channel, the controller reads the Hall-effect switches from the feedback channel that you specify for the PositionFeedbackChannel Parameter.

On the Nservo, the controller can also read the Hall-effect switches from the Auxiliary I/O connector, which you can configure by selecting Nservo Auxiliary I/O.

Gantry Slave Marker

This option specifies if the marker signal from a gantry slave axis is used during the home sequence.

Use this setting only on the gantry slave axis.

Used For Home Alignment is the default. The marker signals from both master and slave axes are used during the home sequence.

When you select Ignored, the marker signal from the slave axis is not used, thus bypassing the gantry de-skewing that normally occurs.

Nstep Encoder Inputs

Hardware: This setting applies only when used with the Nstep.

This option specifies the behavior of the encoder inputs on the Nstep.

Differential is the default. The encoder inputs are configured for RS-422 compatible differential signals.

When you select Single-Ended, the encoder inputs are configured for single-ended operation where negative input voltages indicate a logic low and positive input voltages indicate a logic high.

Feedback Averaging

This option enables the controller to average feedback signals from two or more encoder inputs based on the drive type. The controller uses the average of the signals as the position feedback for the axis.

To use the Feedback Averaging option, you must connect each encoder to the drive so that all the encoders count in the same direction. While Feedback Averaging is enabled, you cannot use the Position Feedback Direction to invert the direction of a single encoder input. If you try to use Position Feedback Direction during this time, the controller averages the encoder inputs and then inverts the position feedback value of the average.

Disabled is the default. The position feedback operates normally. The controller does not average the feedback signals.

When you select Enabled, the controller averages the feedback signals and uses the average as the position feedback for the axis. One of the operations that follow occurs:

For Single-Axis Drives

The controller averages the feedback signals from the primary and auxiliary encoder inputs. The controller averages these signals after you home the axis.

To configure this option, refer to Configuring Parameters for Position Feedback Averaging on Single-Axis Drives.

For Multi-Axis Drives

The controller averages the feedback signals from two or more encoder inputs that you specify for the PositionFeedbackChannel and PositionAveragingChannel parameters. The controller averages these signals after you home the axis.

To configure this option, refer to Configuring Parameters for Position Feedback Averaging on Multi-Axis Drives.

Auxiliary Feedback Direction

Hardware: This setting applies only when used with the GL4, the Ndrive FCL, FLS, QL, or QLe, the Nmark CLS, GCL, or SSaM, or the XL4s.

This option specifies how the drive reads the auxiliary feedback device.

Normal is the default. The drive reads the auxiliary feedback device normally. In this mode, the auxiliary feedback counts in a positive direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input.

When you select Inverted, the auxiliary feedback device is inverted. In this mode, the auxiliary feedback counts in a negative direction when the rising edge of the COS encoder input comes before the rising edge of the SIN encoder input. The function of this setting is to compensate for motor or feedback phasing without requiring changes to the wiring.

Auxiliary Marker Function

Hardware: This setting applies only when used with the XC4 or the XC4e.

This option enables a fail-safe differential PSO output mode on Auxiliary I/O Pins 19 and 20 of the Auxiliary I/O connector.

Auxiliary Marker Input is the default. The controller configures the pins as an RS-422 input and uses that input as the marker input of an auxiliary encoder.

When you select PSO Output, the controller configures the pins as an RS-422 output and puts that output into an inactive state.

IFOV Input

Hardware: This setting applies only when used with the GL4.

This option sets the input source for the encoder feedback to be used with IFOV.

Auxiliary Encoder is the default. The controller configures the auxiliary encoder connector as the encoder feedback input for IFOV.

When you select Sync Port, the controller configures the sync port connector as the encoder feedback input for IFOV. There are two Sync ports on the controller. If you select Sync Port on the first axis of the controller, then the SYNC A port is configured. If you select Sync Port on the second axis of the controller, then the SYNC B port is configured.

For more information on these connectors, refer to the GL4 Hardware Manual.
