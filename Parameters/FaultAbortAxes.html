FaultAbortAxes Parameter

Default Value

0x00000000

Minimum Value

N/A

Maximum Value

N/A

Units

None

Type

32-bit signed integer

Use the FaultAbortAxes parameter to specify the axes on which all motion aborts when a fault is generated on this axis. When a fault is generated, the axes that you specify will abort regardless of the custom fault handling that you defined (for example, using the ON or ONGOSUB commands). For more information about axis faults, refer to Fault and Error Overview.

If the axis that is being aborted is part of a coordinated move (LINEAR, CCW, or CW), then all the axes in the move will be aborted. In this case, the deceleration of the axes will take place in a coordinated fashion (along the coordinated path) and will use the currently active coordinated deceleration rate.

If the axis that is being aborted is not part of a coordinated move, its motion will be aborted using linear deceleration at the rate specified by the AbortDecelRate parameter.

If the axis that is being aborted is configured as a spindle (using the Spindle0Axis, Spindle1Axis, Spindle2Axis, or Spindle3Axis task parameters), the abort will occur regardless of the G100 or G101 setting for the task.

The FaultAbortAxes parameter ignores End of Travel limit faults encountered while homing and PROBE axis faults. The parameter will not abort any axes if these faults occur.
