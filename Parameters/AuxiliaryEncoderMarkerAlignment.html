AuxiliaryEncoderMarkerAlignment Parameter

Default Value

0

Minimum Value

0

Maximum Value

359

Units

Degrees

Type

32-bit signed integer

Hardware: This parameter applies only when used with an applicable drive type and multiplier option. To see the drive types and multiplier options that are available for this parameter, refer to the table that follows.

Table:     Applicable Drive Types and Multiplier Options

Drive Type Multiplier Option

XC4e

Using an Encoder Multiplier with -MX3

XR3

Using an Encoder Multiplier with -MX4

NOTE: This parameter applies only when you set one of these parameters to the specified value:

  • If you set the PositionFeedbackType Parameter to Encoder Multiplier.
  • If you set the VelocityFeedbackType Parameter to Encoder Multiplier.

Use the AuxiliaryEncoderMarkerAlignment parameter to change the behavior of the controller when you use an applicable multiplier option for your HyperWire drive. For information about the applicable multiplier options, refer to the hardware manual for your drive. To see a list of available hardware manuals, refer to Other Documentation. You can also refer to the Specifications tab for your drive at www.aerotech.com.

By default, the controller expects the marker input to align with the encoder sine and cosine inputs so that the center of the marker pulse occurs where the sine input is 0 and the cosine input is at the maximum value. This alignment represents an encoder angle of 0°, which corresponds to the encoder signals that show at the top center of the Lissajous pattern in the Feedback Tuning tab of Digital Scope.

If the marker signal for your encoder does not align correctly (refer to the above paragraph for a description of this alignment), the controller might not detect a repeatable home position. This typically occurs when the transitions of the marker pulse are very close to an encoder angle of 0°. If your system shows signs of this condition, you can use this parameter to change the alignment that the controller expects for the marker pulse. Aerotech recommends that you set the value of this parameter to the value, in degrees, of the encoder angle, where the center of the marker pulse occurs.
