Use the HomeType parameter to define the procedure that will be used to establish the home position of the axis. The following sections supply a detailed description of the procedure that will be used for each setting. The HomeSpeed parameter sets the speed of all homing related motion, and the HomeRampRate parameter sets the acceleration and deceleration rates.

Note: Virtual Axes will behave as if Home at Current Position is selected, regardless of the setting of the HomeType parameter.
Home Past Limit to Marker

Note: This mode should only be used when the Limit Type setting of the HomeSetup parameter is set to Use Home Limit Switch.

When you select this home type, the following procedure is used.

  1. If the limit input is currently active, the axis moves out of the limit condition by moving in the direction opposite of that specified by the Home Start Direction setting of the HomeSetup parameter. Once the limit condition is cleared, the axis decelerates to a stop. If the limit input is not active, no motion occurs during this step.
  2. The axis moves toward the limit by moving in the direction specified by the Home Start Direction setting of the HomeSetup parameter. This movement continues until the limit input goes active, at which point the axis decelerates to a stop.
  3. The axis then begins moving again, in the same direction as the previous move, and continues until the marker input goes active, at which point the position of the marker is latched in the hardware and the axis decelerates to a stop.
  4. The axis moves back to the position at which the marker was latched. If a nonzero value is set for the HomeOffset parameter, the axis moves the specified distance from where the marker was latched.
  5. The position command and the programmed position command are set to the value specified by the HomePositionSet parameter.

Figure: Home Past Limit to Marker Motion Diagram

Home to Limit and Reverse to Marker

Note: For home types that use limit switches to operate correctly, you must select the CW/Positive End-of-Travel Limit Fault and CCW/Negative End-of-Travel Limit Fault bits in the FaultMask parameter. If you do not select these bits, the limit switches will be ignored during the home cycle, preventing the home cycle from completing successfully.

When you select this home type, the following procedure is used.

  1. The axis moves toward the limit by moving in the direction specified by the Home Start Direction setting of the HomeSetup parameter. This movement continues until the appropriate limit input goes active, at which point the axis decelerates to a stop.
  2. The axis moves out of the limit by moving in the direction opposite that of the previous move. Once the limit condition is cleared, the axis decelerates to a stop.
  3. The axis then begins moving again, in the same direction as the previous move, and continues until the marker input goes active, at which point the position of the marker is latched in the hardware and the axis decelerates to a stop.
  4. The axis moves back to the position at which the marker was latched. If a nonzero value is set for the HomeOffset parameter, the axis moves the specified distance from where the marker was latched.
  5. The position command and the programmed position command are set to the value specified by the HomePositionSet parameter.

Figure: Home to Limit and Reverse to Marker Motion Diagram

Home to Marker Only

When you select this home type, the following procedure is used.

  1. The axis begins moving in the direction specified by the Home Start Direction setting of the HomeSetup parameter and continues until the marker input goes active, at which point the position of the marker is latched in the hardware and the axis decelerates to a stop.
  2. The axis moves back to the position at which the marker was latched. If a nonzero value is set for the HomeOffset parameter, the axis moves the specified distance from where the marker was latched.
  3. The position command and the programmed position command are set to the value specified by the HomePositionSet parameter.

Figure: Home to Marker Only Motion Diagram

Home to Limit Only

Note: This home type is not recommended for most systems as the repeatability of the home position is not as good as one of the home types that use the marker. In general, this type should only be used if no marker is present on the axis.

Note: For home types that use limit switches to operate correctly, you must select the CW/Positive End-of-Travel Limit Fault and CCW/Negative End-of-Travel Limit Fault bits in the FaultMask parameter. If you do not select these bits, the limit switches will be ignored during the home cycle, preventing the home cycle from completing successfully.

When you select this home type, the following procedure is used.

  1. The axis moves toward the limit by moving in the direction specified by the Home Start Direction setting of the HomeSetup parameter. This movement continues until the appropriate limit input goes active, at which point the axis decelerates to a stop.
  2. The axis moves out of the limit by moving the direction opposite that of the previous move. Once the limit condition is cleared, the axis decelerates to a stop.
  3. If a nonzero value is set for the HomeOffset parameter, the axis moves the specified distance from this location.
  4. The position command and the programmed position command are set to the value specified by the HomePositionSet parameter.

Figure: Home to Limit Only Motion Diagram

Home at Current Position

When you select this home type, the position command and the programmed position command are set to the value specified by the HomePositionSet parameter. Any position error present at the time of homing is maintained. No motion occurs using this type of home, including any offset move specified by the HomeOffset parameter.

Home at Current Position Feedback

When you select this home type, the position command, position feedback, and the programmed position command are set to the value specified by the HomePositionSet parameter. Any position error present at the time of homing will be cleared. No motion will occur using this type of home, including any offset move specified by the HomeOffset parameter.

Note: Due to the immediate clearing of the position error, this home type might cause unexpected motion if there is a large quantity of position error at the time of homing. This home type should only be executed when the position error is small.
Home at Current Position Absolute

When you select this home type, the position command and position feedback are not changed. The programmed position command is set to the absolute position of the axis. Any position error present at the time of homing is maintained. No motion occurs using this type of home, including any offset move specified by the HomeOffset parameter.

If an axis is configured to use this home type, the first time you enable the axis the controller automatically homes the axis and calibration becomes active on the axis.
