Use the EndOfTravelLimitSetup parameter to configure options related to the active state of the end-of-travel limits. The sections that follow give the possible options for this parameter.

Home Switch Polarity

This option specifies the active polarity of the Home input.

Active Low is the default. The software reports the limit input as active when the input level is low. When no external device is connected, the circuit for the limit inputs inside an Aerotech drive supplies a signal level of high to the internal logic. Thus if you use this setting when you do not have a limit switch connected, the controller does not report the home switch as active.

When you select Active High, the software reports the home input as active when the input level is high.

Aerotech recommends that you use a normally-closed limit switch and select Active High for this option. If you connect a normally-closed limit switch between the limit input and limit common and the switch is not activated, it stays closed and the limit inputs are pulled low by the limit switch. When the switch is activated, it opens and the limit input is pulled high by the circuit inside the drive.

Negative/CCW End-of-Travel Switch Polarity

This option specifies the active polarity of the negative or CCW end-of-travel limit input.

Active Low is the default. The software reports the limit input as active when the input level is low. When no external device is connected, the circuit for the limit inputs inside an Aerotech drive supplies a signal level of high to the internal logic. Thus, you can use this setting to prevent unnecessary limit faults when you do not have a limit switch connected. But this configuration is not fail-safe and is not recommended.

When you select Active High, the software reports the home input as active when the input level is high.

Aerotech recommends that you use a normally-closed limit switch and select Active High for this option. This configuration provides fail-safe limit behavior because the controller generates a fault if the external hardware is not connected or is not operating correctly. If you connect a normally-closed limit switch between the limit input and limit common and the switch is not activated, it stays closed and the limit inputs are pulled low by the limit switch. When the switch is activated, it opens and the limit input is pulled high by the circuit inside the drive.

LIF481 Mode

This option enables the integrated limit signals from a Heidenhain LIF481 encoder.

Disabled is the default. The limit inputs on the feedback connector are treated normally.

When you select Enabled, the limit inputs on the feedback connector correspond to the signals from the LIF481 encoder. The end-of-travel limit state is encoded into the CW and CCW hardware inputs. When the CCW input is high and the CW input is low, the software reports that the CW end-of-travel limit is active, and when both the CCW and CW inputs are high, the software reports that the CCW end-of-travel limit is active. When the CCW limit is low, the software reports that neither end-of-travel limit is active, regardless of the state of the CW input.

End-of-Travel Switch Swap

This option specifies a software reversal of the end-of-travel limit inputs.

Maintain CW and CCW Switches is the default. The limit inputs are configured normally.

When you select Swap CW and CCW Switches, the CW hardware input maps to the negative or CCW end-of-travel limit and the CCW hardware input maps to the positive or CW end-of-travel limit from a software perspective.

Opto-Isolated Limits

Hardware: This setting applies when used with the Ndrive CP, HLe, and HPe drives.

This option specifies the use of opto-isolated limit inputs.

Standard Limits is the default. The end-of-travel and home limits are read from the standard encoder input connector.

When you select Opto-Isolated Limits, the end-of-travel and home limits are read from the opto-isolated digital inputs. Input 8 is the negative or CCW end-of-travel limit input, input 9 is the positive or CW end-of-travel limit input, and input 10 is the home limit input.

Home Limit

NOTE: Home Limit applies only if you use the following settings for the HomeSetup and HomeType parameters in Configuration Manager.

  • For the HomeSetup parameter, set Limit Type to Use Home Limit Switch.
  • For the HomeType parameter, select a setting that uses the limit.

This option specifies which limit input is used as the home limit. Change this option from the default only if you are using a drive that does not supply a home limit input and you want to use one of the end-of-travel limit inputs as an alternative.

Use Home Input is the default. The home limit input is used as the home limit.

When you select Use CCW Input, the CCW end-of-travel limit input is used as the home limit. The axis cannot have a CCW end-of-travel limit and you must clear the CCW/Negative End-of-Travel Limit Fault bit of the FaultMask parameter. If you do not clear this bit in the FaultMask parameter, the controller generates a fault when it encounters the limit switch during the home cycle. As a result, the home does not complete.

When you select Use CW Input, the CW end-of-travel limit input is used as the home limit. The axis cannot have a CW end-of-travel limit and you must clear the CW/Positive End-of-Travel Limit Fault bit of the FaultMask parameter. If you do not clear this bit in the FaultMask parameter, the controller generates a fault when it encounters the limit switch during the home cycle. As a result, the home does not complete.

Current-Based Limits

Hardware: This setting applies only when used with the GL4, the Ndrive FCL or FLS, the Nmark CLS or GCL, or the XL4s.

This option specifies how the controller detects end-of-travel limits.

Use Limit Switches is the default. The controller uses the signals from the dedicated limit switch inputs to determine when an end-of-travel limit is reached.

When you select Use Current Thresholds, the controller compares the current command of the axis to the EndOfTravelCurrentThresholdHigh and EndOfTravelCurrentThresholdLow parameter values to determine when an end-of-travel limit is reached. The end-of-travel limits are used only until the axis has been homed, after which you must specify software limits in order to limit the travel of the axis.

Gantry End-of-travel Limits

Hardware: This setting applies only when used with multi-axis drives.

This option specifies if the end-of-travel limits from a gantry slave axis are used by the controller.

Use Master Limits is the default. The end-of-travel limit inputs from the master axis are used by both the master axis and the slave axis, and the limit inputs from the slave axis are ignored.

When you select Use Independent Limits, the master axis and the slave axis each use their own end-of-travel limit inputs.
