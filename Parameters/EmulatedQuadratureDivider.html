Use the EmulatedQuadratureDivider parameter to decrease the effective resolution of emulated quadrature signals. To generate emulated quadrature signals, your drive must have one of the primary feedback modules that follow:

  • Optional Encoder Multiplier Module
  • Capacitance Sensor Feedback

To determine if you must specify a non-default value or the default value of 1 for this parameter, refer to the instructions in the applicable section that follows.

Drives with an Optional Encoder Multiplier Module

Specify a non-default value for this parameter when the system resolution and speed of motion cause the maximum encoder frequency of the axis to exceed the maximum quadrature output frequency of the drive. The non-default value does not change the resolution of the servo loop. It can be an integer or a non-integer value.

Specifying a Non-Integer Value for the EmulatedQuadratureDivider Parameter

You can specify a non-integer value for the EmulatedQuadratureDivider parameter when you use an optional primary encoder multiplier module. Make sure the controller can use this value to generate an integer number of quadrature pulses per encoder cycle. This means that when you divide the value of the EncoderMultiplicationFactor parameter by the value of the EmulatedQuadratureDivider parameter, the result must be an integer.

If the result is a non-integer value, the controller rounds that value to the nearest integer and a small difference occurs between the expected quadrature resolution and the actual quadrature resolution.

To determine if you must specify a non-default value or the default value of 1 for this parameter, use the procedure that follows.

    • If the maximum encoder frequency of the axis does not exceed the maximum quadrature output frequency of the drive, set the EmulatedQuadratureDivider parameter to the default value of 1. It is not necessary for you to continue with this procedure.
    • If the maximum encoder frequency of the axis exceeds the maximum quadrature output frequency of the drive, continue to Step 4.
  4. Calculate the minimum frequency divider. This is the smallest possible value that you can set for the EmulatedQuadratureDivider parameter. To make sure the value is compatible with this parameter, it must obey the conditions in Step 5.
    5. If You Are Not Using PSO

    Divide the maximum encoder frequency of the axis by the maximum quadrature output frequency of the drive. Refer to the equation that follows.

    If You Are Using the Emulated quadrature signals as a PSO Tracking Input
    1. In the Maximum PSO Encoder Input Data Rates section of PSO Specifications, find the encoder input data rate for the type of PSO tracking that you are using.
    2. Compare the encoder input data rate to the maximum quadrature output frequency of the drive.
    3. Use the smaller of the two values. Do one of the options that follow:
      • If the maximum quadrature output frequency is less than the encoder input data rate, use the equation that follows.

      • If the encoder input data rate is less than the maximum quadrature output frequency, use the equation that follows.

  5. Select a value for the EmulatedQuadratureDivider parameter. Make sure it obeys the conditions that follow:
    • The value must result in an integer when you divide it into the EncoderMultiplicationFactor parameter. An integer value makes sure that you get a whole number of counts per encoder cycle. Refer to the equation that follows.
    • The value must be greater than or equal to the minimum frequency divider that you calculated.
    • The value must be greater than 4 so the drive can generate emulated quadrature signals.

      • Note: When you have a FireWire drive that uses -MXH or -MXR feedback or a HyperWire drive that uses an encoder multiplier with the -MX2, -MX3, or -MX4 multiplier option for primary feedback, quadrature is generated only if the effective multiplication factor is greater than 4. The effective multiplication factor is equal to the value of the EncoderMultiplicationFactor parameter divided by the value of the EmulatedQuadratureDivider parameter.
  6. Set the EmulatedQuadratureDivider parameter to the value that you selected. Do one of the options that follow based on the drive type that you are using:
    • For an Npaq with MXR feedback, Aerotech recommends that you change the value of this parameter in Configuration Manager only. If you must change the value programmatically, refer to Step 7.
    • For all other drives, you can change the value of this parameter in Configuration Manager or from your AeroBasic program.
  7. For an Npaq with MXR Only. To change the value of the EmulatedQuadratureDivider parameter from a program, do the steps that follow:

    8. NOTE: If you use an Npaq with MXR and change the value of this parameter from a program, it might be necessary for the Npaq to internally reprogram the MXR hardware. Unexpected quadrature outputs might also occur during the programming cycle.
    1. Change the value at a time when quadrature outputs are not being used.
    2. Examine the state of the Programming MXH bit of the Drive Status to make sure programming has completed.
  8. The EmulatedQuadratureDivider parameter decreases the effective resolution of the quadrature signals that are used for PSO tracking. If you set this parameter to a non-default value, divide your PSO firing distances by the value that you set for this parameter.

    9. For Example

    If you set the EmulatedQuadratureDivider parameter to a value of 2, you must divide all distances specified in the PSODISTANCE command by a factor of 2.

Piezo Drives Configured to Use Capacitance Sensor Feedback

Specify a non-default value for this parameter when the system resolution and speed of motion cause the maximum sensor frequency of the axis to exceed the maximum quadrature output frequency of the drive. The non-default value does not change the resolution of the servo loop. The capacitance sensor feedback module requires a non-default value to be an integer.

To determine if you must specify a non-default value or the default value of 1 for this parameter, use the procedure that follows.

  1. Calculate the maximum sensor frequency of the axis. Use the equation that follows.

  3. Based on the values that you compared, do one of the steps that follow:
    • If the maximum sensor frequency of the axis does not exceed the maximum quadrature output frequency of the drive, set the EmulatedQuadratureDivider parameter to the default value of 1. It is not necessary for you to continue with this procedure.
    • If the maximum sensor frequency of the axis exceeds the maximum quadrature output frequency of the drive, continue to Step 4.
  4. Calculate the value of the EmulatedQuadratureDivider parameter.
    5. If You Are Not Using PSO

    Divide the maximum sensor frequency of the axis by the maximum quadrature output frequency of the drive. Then round up the calculated value to the nearest integer. Refer to the equation that follows.

    If You Are Using the Emulated quadrature signals as a PSO Tracking Input
    1. In the Maximum PSO Encoder Input Data Rates section of PSO Specifications, find the encoder input data rate for the type of PSO tracking that you are using.
    2. Compare the encoder input data rate to the maximum quadrature output frequency of the drive.
    3. Use the smaller of the two values. Do one of the options that follow:
      • If the maximum quadrature output frequency is less than the encoder input data rate, use the equation that follows. Then round up the calculated value to the nearest integer.

      • If the encoder input data rate is less than the maximum quadrature output frequency, use the equation that follows. Then round up the calculated value to the nearest integer.

  5. Set the EmulatedQuadratureDivider parameter to the value that you calculated. You can change the value of this parameter in Configuration Manager or from your AeroBasic program.
  6. The EmulatedQuadratureDivider parameter decreases the effective resolution of the quadrature signals that are used for PSO tracking. If you set this parameter to a non-default value, divide your PSO firing distances by the value that you set for this parameter.

    7. For Example

    If you set the EmulatedQuadratureDivider parameter to a value of 2, you must divide all distances specified in the PSODISTANCE command by a factor of 2.
