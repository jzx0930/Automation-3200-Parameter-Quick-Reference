Analyzing A3200 Performance

To get real-time controller performance, the Automation 3200 runs with Windows in the TenAsys INtime real-time operating system. The A3200 runs on a PC. Thus controller performance, like general Windows performance, depends on the PC hardware configuration and the software configuration. Both the wide variety of available PCs and the different system configurations of those PCs make it important for you to analyze your system before you use the A3200. You can also analyze the A3200 controller software.

Analyzing PC Performance

Use the Real-Time Performance Analyzer application to detect if your PC is or is not optimized for real-time use.

  1. Close all A3200 applications.
  2. Shut down the A3200. Click Start → All Programs → Aerotech → Automation 3200 → Shutdown A3200. A black Command Prompt window comes into view momentarily.
  3. Open Real-Time Performance Analyzer. Click Start → All Programs → Aerotech → Automation 3200 → Real-Time Performance Analyzer.
  4. Let the application run for 30 minutes or more.
  5. Compare your results with the results that are shown in the following figures. If your PC is optimized for real-time use, then the Real-Time Performance Analyzer dialog should look almost the same as Figure: Good Real-Time Performance.

Analyzing Controller Performance

Use the Status Utility to detect if the controller software is or is not optimized for real-time use.

  1. Close all A3200 applications.
  2. Open Configuration Manager. Click Start → All Programs → Aerotech → Automation 3200 → Configuration Manager.
  3. If Configuration Manager is not already connected to the A3200, select Controller → Connect to the A3200.
  4. Open the Status Utility. Click Start → All Programs → Aerotech → Automation 3200 → Status Utility.
  5. In the Status Utility, select the Controller tab.
  6. Compare your results with the results that are in the figure that follows. If the A3200 controller is optimized for real-time use, then the Controller tab in the Status Utility will look almost the same as Figure: Good Status Utility.
    7. Figure: Good Status Utility

    Table:     Good Status Utility

    Figure Label

    Name

    Description

    Communication Polling Timers

    Communication Polling Min (usec) and Communication Polling Max (usec) are near 100 for FireWire and near 1000 for HyperWire.

    Communication Activities Maximum

    Communication Activities Max (usec) is less than 35.

    Processor Usage Average

    While motion does not occur, the Processor Usage Avg (%) of the A3200 is under 15%.

    Internal Counters

    All Internal Counters are 0.

  7. If your controller software is not optimized for real-time use, refer to Optimizing A3200 Performance for information about how to optimize the controller.
