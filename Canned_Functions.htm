Canned Function Overview

Canned functions supply flexibility on how and when to call a subroutine. They have the following purposes.

  • Allow one task to execute a subroutine on a different task
  • Allow a subroutine to be executed synchronously with respect to coordinated (LINEAR, CW, CCW, BEZIER) and RAPID motion
  • Implement canned cycles or other actions that must occur at the end of moves

Canned functions are defined like regular subroutines using the DFS Command.

The ways that you can use canned functions are as follows.

  • Use the SYNCCALL Command to explicitly execute a canned function without having to configure the canned function first.
  • Use the SETCANNEDFUNCTION Command to configure a canned function. Use the EXECCANNEDFUNCTION Command to execute a configured canned function.

Because every canned function must be present on the controller, you can download all canned functions to the controller by creating one or more programs that contain the canned functions, and then use the "Download" Execute Type of Program Automation to automatically download the program(s).

Canned functions do not execute if a task error is present. Use the ONGOSUB Command to customize actions during fault and error conditions.

A canned function cannot be interrupted by another canned function. Use the ONGOSUB command when nested interrupting is required.

The state of canned functions on a task is indicated by the Canned Function Executing bit of Task Status 0.

Canned functions are executed synchronously with respect to coordinated motion and RAPID motion, which means that the controller does not execute the canned function until the previous motion command has completed.

SYNCCALL Command

Use the SYNCCALL command to explicitly execute a canned function.

SETCANNEDFUNCTION And EXECCANNEDFUNCTION Commands

Use the SETCANNEDFUCTION command to configure a canned function. Use the CannedFunctions Parameter to define the maximum number of configurable canned functions. You can disable a configurable canned function by specifying "" for the <ProgramName> argument and "" for the <Label> argument.

For example

SETCANNEDFUNCTION 1, "", ""

After you use the SETCANNEDFUNCTION command to configure a canned function, use the EXECCANNEDFUNCTION command to explicitly execute the canned function.

Canned functions can also be configured to automatically execute at the end of every LINEAR command move. These types of canned functions are generally referred to as canned cycles. To configure this mode, set the <ID> argument to CANNEDFUNCTION_ID_ENDOFBLOCK in the SETCANNEDFUNCTION command. In this mode, the controller decelerates all motion to a stop before executing the canned function. You cannot pass parameters to the canned function with this calling method. Refer to Implementing Canned Cycles.
