Non-Windows Operating System Communication

Viewing the A3200 Help File

You can use any of the following to view the main help file under GNU/Linux:

  • Xchm ( http://xchm.sourceforge.net/ )
  • GnoCHM ( http://gnochm.sourceforge.net/ )
  • KchmViewer ( http://www.kchmviewer.net/ )

Limitations of a Non-Windows Operating System

You cannot run any Aerotech A3200 applications or libraries

Because the libraries and applications shipped with the A3200 must run under Microsoft Windows, you must complete the system setup — including programming, tuning, and parameter setting — by using Microsoft Windows. However, after you complete the setup, you can use other production environment interfaces from other operating systems.

Non-Windows Options for Communicating with the A3200

ASCII Interpreter (recommended)

The A3200 is shipped with an ASCII Interpreter that interprets AeroBasic commands received from any of the following available media types.

The advantage to the ASCII protocol is that the A3200-side of it is done and communicating with it is easy (refer to ASCII Command Interface).

Available Media

Ethernet TCP/IP

Example Setup

This example illustrates the minimal setup of the ASCII Interface over TCP/IP.

Note: Because much of the A3200-side work is done by the ASCII interpreter task, the ASCII Interface is the recommended means of communication from non-Windows operating systems. The other options require a custom application for the A3200-side.

In this example, a computer running either Linux (or Unix) or a Java-capable operating system communicates with an A3200 over TCP/IP and interacts with the ASCII interpreter shipped with the A3200.

Required Windows Setup

The minimal setup required from a Windows machine is to setup the ASCII interpreter to use an Ethernet socket. Then set the socket to use TCP/IP and listen on a specific port.

Code

Any socket-capable language or framework can be used to open a socket to the A3200, send commands in ASCII, and retrieve status information. The following two examples are supplied with the installation.

C++ in Linux

This example was written specifically for Linux, but should be able to compile and run under any Unix system. The example is in the A3200 installation in samples\cpp\ASCII\OperatorInterface\Lin. (Refer to the example for more specific documentation.)

Java

This example can be run under any operating system that supports Java. The example is in the A3200 installation in java\ASCII\Toggle. Refer to the example for more specific documentation.

Limitations

Some of the setup must be completed under Windows.

Modbus

Modbus is a protocol for transferring information between two devices. There are libraries that implement this protocol in many different languages. The Modbus protocol is very generic and using it requires that you design an additional protocol. For example, you can transfer two numbers but because the interpretation of these numbers is unknown, you must also write an AeroBasic program running on the A3200 to interpret the numbers.

Available Media

Ethernet TCP/IP by configuration. Refer to Modbus Protocol.

Example Setup

This example illustrates the minimal setup of Modbus/TCP.

Note: If you use the Modbus protocol, the A3200-side must be a custom application.

In this example, a computer sends data to the A3200 over Modbus/TCP. A program on the A3200 reads the data and executes the appropriate commands.

The Modbus protocol requires the computer program to adhere to the Modbus/TCP standard. Libraries that supply an abstraction of the protocol are available for several different operating systems. An advantage to using this protocol is that because Modbus is supported by many PLCs, you can connect to an A3200 from computers, as well as PLCs, I/O devices, and so on.

Required Windows Setup

Modbus requires more Windows Setup than the ASCII Interpreter protocol. You must configure the Modbus/TCP in the Configuration Manager and write a user program to process the data sent over the protocol.

Code

Several examples are shipped with the A3200 installation and are in the samples/Modbus folder. Although all of the examples are written in .NET, the Modbus/TCP protocol is not specific to .NET and there are Modbus/TCP libraries for in C++, Java, .NET, and COM.

Limitations

  • Much of the configuration, including a custom program, must be done under Windows.
  • The commands are limited by AeroBasic.
Custom

A custom protocol can be created by opening the sockets directly. Although using this method supplies the most flexibility, it typically requires more time and labor than the other protocols.

Available Media

Ethernet TCP/IP

Example Setup

This example shows the minimal setup.

Note: If you use the Custom protocol, the A3200-side must be a custom application.

In a custom protocol, you must write a custom program that will open a socket and will execute its own communication protocol.

Windows Setup

Setup communication for the socket and create the program to support the protocol.

Code

No custom code examples are currently supplied.

Limitations

  • Much of the configuration, including a custom program, must be done under Windows.
  • The commands are limited by AeroBasic.
Web Interface

You can connect to the A3200 controller from a device that has a recent Web browser. Smartphones and tablets are supported.

Available Media

Ethernet TCP/IP

Example Setup

The Web Interface requires minimal setup. An example is shipped with the A3200 installation and is in the samples/HTTP/REST folder.
