Arithmetic and Comparison Operators

Syntax:

 
<Operand> <Operator> <Operand>

Example:

 
$VA = $VB + $VC      ' Addition.
 
$VA = $VB ** 2       ' Raise to power of 2.
 
$global[0]=(3 EQ 9)  ' Assigns zero to global variable 0.

Comparison operations always evaluate to 0.0 or 1.0.

For both arithmetic and comparison operators, the operator expression is evaluated, or converted, into a double precision, floating-point value when the controller executes the line that contains that operator expression. Operations that have violations of the input domain, such as division by zero, cause a compiler error or a runtime error.

Double Comparison Precision

Because of the internal representation of double-precision floating point numbers, some decimal numbers, such as 0.2, cannot be exactly represented. Mathematical operations with such values can result in very small differences between the actual values and the expected values. This small difference can cause unexpected results for mathematical comparisons. Thus, do not do "equal" (==, EQ) or "not equal" (!=, NE) comparisons that use two floating point numbers. If you must do this type of comparison, compare the difference between the two values to an acceptable threshold, as shown in this example.

Example

#define COMPARE_THRESHOLD 0.0001
DVAR $Value1, $Value2

' See if two doubles are equal in the specified threshold.
IF (ABS($Value1 - $Value2) < COMPARE_THRESHOLD) THEN

' They are equal.

ELSE

' They are not equal.

END IF
