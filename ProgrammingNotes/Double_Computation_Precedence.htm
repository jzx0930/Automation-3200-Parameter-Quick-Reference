Double Computation Precedence

The AeroBasic language uses this precedence order of conventions during evaluation of double precision, floating-point expressions.

  • An expression in parentheses is evaluated until everything in those parentheses can be shown as one constant.
  • All functions have precedence over operators.
  • When an operand is compressed between two binary operators, the operator with the higher precedence is computed first. However, if the operators are of the same precedence, then the left operator is computed first. For example, 1 + 2 − 3 is evaluated as (1 + 2) − 3.
