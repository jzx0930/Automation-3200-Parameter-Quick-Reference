Motion Overview

The A3200 SMC controller supplies a maximum of 32 axes of motion that you can use on five tasks. You can use these axes on a maximum of 32 tasks if you have the Professional Option installed. The types of motion that you can use are synchronous and asynchronous.

Synchronous Motion

Most move types are synchronous which means that the controller does not execute the next program line until the current line has completed. Synchronous motion can be coordinated or non-coordinated. Coordinated motion refers to moves that follow a well-defined path in space. These commands use the F or E commands and start and stop axes at the same time. They can execute in velocity profiling mode (VELOCITY ON). Examples of this type of motion are LINEAR, CW, CCW, and BEZIER.

Non-coordinated motion is another form of synchronous motion in which axes start at the same time but do not necessarily end at the same time. Each axis moves at its own velocity specified in the command or by axis parameters. Program execution does not continue to the next line until all axes in the move command have completed motion. Examples of this type of motion are RAPID, ABORT, and HOME.

Synchronous motions can also be configured for various "in position" modes. These modes specify the criteria for move completion and are described in Motion Done Overview.

Asynchronous Motion

Asynchronous motion commands begin when executed in a program. Program flow then continues to the next line without waiting for the motion to complete. Asynchronous motion offers versatility that lets you do other tasks during a time-consuming move. These moves are potentially more complicated because you must coordinate their completion. Refer to the Asynchronous Motion Overview topic for more information.

Examples of this type of motion command are FREERUN, MOVEABS, MOVEINC, HOMEASYNC, and OSCILLATE.
