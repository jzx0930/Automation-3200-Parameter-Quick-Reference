Assigning Drive Communication Channels

NOTE: The Drive Configuration dialog applies to HyperWire drives only. If you are using FireWire drives, refer to the FireWire information in the Getting Started section of the Help file.

If you purchased an integrated system, the communication channel numbers are typically configured at the factory and do not require changes. But if you purchased an independent system, all of the communication channel numbers have a value of 0. Thus you must assign a unique communication channel number to each HyperWire drive that is connected to the HyperWire card installed in your PC. Some drives, such as the GL4 or XR3, must have a unique range of communication channel numbers.

If you are troubleshooting the communication channel numbers, refer to the Troubleshooting Communication Channel Numbers section of this topic. Then refer to the applicable procedure in this topic to resolve the error.

To assign communication channel numbers to the HyperWire drives that are connected to your PC, refer to the applicable procedure that follows.

TO AUTOMATICALLY ASSIGN COMMUNICATION CHANNEL NUMBERS
  1. Open Configuration Manager.
  2. On the Controller menu, select Drive Configuration. The Drive Configuration dialog connects to the HyperWire network and identifies all of the connected drives.

    		3.

    WARNING: When you open the Drive Configuration dialog, Configuration Manager disables all of the enabled axes and stops all of the AeroBasic programs that are running. It also stops programs that monitor safety-type conditions.
  3. In the Drive Configuration dialog, select the Communication Channels tab.
  4. Click the Auto-Assign Channels button to automatically assign communication channel numbers to all of the connected drives.
  5. Optional. Review the changes that were made to the communication channel numbers assigned to the drives.
  6. In the Message Log, a message shows the number of drives for which the assigned communication channel numbers changed. For communication channel numbers that have changed but must be committed to the memory of the controller, the Communication Channels Editor shows each of these numbers with an asterisk.
  7. To commit the communication channel numbers to the HyperWire drives, do one of the steps that follow. You can refer to the Message Log to see the operations that occur. The Message Log shows a message after Configuration Manager successfully commits the communication channel numbers.
    • If a dialog comes into view and tells you to commit the communication channel numbers, click Yes.
    • Click Commit.

  8. Now the communication channel numbers are applied and they are saved if you reset the drives.

TO MANUALLY ASSIGN COMMUNICATION CHANNEL NUMBERS
  1. Open Configuration Manager.
  2. On the Controller menu, select Drive Configuration. The Drive Configuration dialog connects to the HyperWire network and identifies all of the connected drives.

    		3.

    WARNING: When you open the Drive Configuration dialog, Configuration Manager disables all of the enabled axes and stops all of the AeroBasic programs that are running. It also stops programs that monitor safety-type conditions.
  3. In the Drive Configuration dialog, select the Communication Channels tab.
  4. In the Communication Channels Editor, select a HyperWire drive for which you want to assign a communication channel number.
  5. Double-click the Channel box.
  6. Do one of the steps that follow.
    • If the drive requires only one communication channel number, type a new number in the Channel box.
    • If the drive requires a range of communication channel numbers that occur in a sequence, type the first number of the range that you want to assign in the Channels box.
  7. Press the Enter key. For communication channel numbers that have changed but must be committed to the memory of the controller, the Communication Channels Editor shows each of these numbers with an asterisk.
  8. Repeat Steps 4 through 7 for the other HyperWire drives.
  9. To commit the communication channel numbers to the HyperWire drives, click Commit. You can refer to the Message Log to see the operations that occur. The Message Log shows a message after Configuration Manager successfully commits the communication channel numbers. Now the communication channel numbers are applied and they are saved if you reset the drives.
Troubleshooting Communication Channel Numbers

If HyperWire drives have the same communication channel number or if a multi-axis HyperWire drive has a range of communication channel numbers that do not occur in a sequence, Configuration Manager identifies these numbers by putting a red border around the Channel or Channels boxes that contain them. You must resolve these errors before you can commit the communication channel numbers to the drives.

Table:     Communication Channel Numbers Problems and Solutions

Problems Solutions

Two or more HyperWire drives have the same communication channel number.

Make sure that all of the HyperWire drives have unique communication channel numbers. Then commit the communication channel numbers to the drives. Refer to the To Automatically Assign Communication Channel Numbers or the To Manually Assign Communication Channel Numbers procedure in this topic.

One or more multi-axis HyperWire drives have a range of communication channel numbers that do not occur in a sequence. As a result, the Message Log of the Drive Configuration dialog shows a message that has a yellow warning icon.

Make sure that each multi-axis HyperWire drive has a range of communication channel numbers that occur in a sequence. Then commit the communication channel numbers to the drives. Refer to the To Automatically Assign Communication Channel Numbers or the To Manually Assign Communication Channel Numbers procedure in this topic.
