#include preprocessor directive

Syntax:

 
#include ["]<filename>["]

Example:

 
#include "c:\Program Files\Aerotech\a3200\file1.pgm"
  
#include file1.pgm
  
#include "..\file1.pgm"

The #include preprocessor directive inserts the entire text of the specified file into the program. The inserted text receives the same treatment as the actual text in the program. This means that all preprocessor directives that are found in the included text are followed. For example, you can include files within included files and there is no limit to the nesting level of #include preprocessor directives.

If there is no directory specification in the file name, the included file is assumed to be in the same folder as the current program. This also applies if a ".." is used in the file name, which specifies to move up one directory level from the same folder in the current program. (The current program is the program that contains the #include statement.)

You can also surround the file name with double quotes, which have no effect on the function of the #include statement.

If you have a program that only contains #define statements, you can use program automation to automatically include that file instead of using the #include statement.

Note: The following conditions apply to all of the lines that are included with this statement.

  1. A program pause does not pause the program until all of the lines that are included are done executing.
  2. A program stepinto or program stepover does not pause the program until all of the lines that are included are done executing.
