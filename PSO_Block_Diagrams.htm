PSO Block Diagrams

Hardware: PSO does not apply when used with the Nmark SSaM or the Nstep.

The block diagram that follows shows the basic design of the PSO Hardware. Links to detailed diagrams for distance tracking hardware, window hardware, and output hardware are listed by drive type following Table: Related PSO Commands.

Figure: PSO Block Diagram and Related Commands

Table:     Related PSO Commands

Command Name

Description

PSOCONTROL

Control the overall state of the PSO hardware including resetting the state, forcing a firing event, or arming the distance-based tracking feature.

PSODISTANCE

Configure the PSO for distance-based firing and specify the distance between firing events.

PSOOUTPUT

Specify the internal PSO signal that will be used as the PSO output.

PSOPULSE

Specify the parameters (on time, total time, number of cycles) for the PSO pulse generator.

PSOTRACK

Specify the input signals that will be used to generate distance-based PSO firing events.

PSOWINDOW

Configure the optional PSO window hardware by specifying the input signal and the range for each of the PSO windows.

Ndrive FCL, FLS, HLe, HPe, ML, and QLe, Nmark CLS and GCL, Npaq, and Nservo
