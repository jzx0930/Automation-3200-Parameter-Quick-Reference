Data Collection Overview

The A3200 can collect real-time information about the performance of the motion system, including position errors during motion, the currents necessary to do motion, analog and digital input status, and controller status. The A3200 has the following types of data collection.

  • Use Digital Scope to configure, collect, and view the data. You can save the collected data to a text file and then you can open the file in Digital Scope. You can also configure the rate at which Digital Scope collects data. In a program, use the SCOPETRIG Command to start and stop data collection in Digital Scope. For more information, refer to Digital Scope Application Overview.
  • Use the DATACOLLECT commands to collect and write sampled data from a program without using Digital Scope. The file format is compatible with Digital Scope. You can collect many types of data and specify the sample time at which the DATACOLLECT commands collect data. For more information, refer to Data Collection Command Category.
  • Use the DATAACQ commands to collect information at the hardware level with low latency. Data acquisition is triggered by an external signal or an internally generated PSO signal. The controller stores the samples in a drive array. You can retrieve these samples into system variables in your program. For more information, refer to DataAcquisition Command Category.
  • Use Signal Log to continuously collect a predefined set of diagnostic data. You can use this data to find axis faults and other error conditions. Signal Log is enabled by default and runs as a background process. For more information, refer to Signal Log Overview.
  • Write an AeroBasic program that collects information about the controller and then save this information to a file. Use the AXISSTATUS Function, TASKSTATUS Function, and SYSTEMSTATUS Function to collect controller information and then save this data to program variables. Use the FILEWRITE Command to write the variables to a file. You can create a deterministic sample interval of 1 millisecond or multiples of 1 millisecond by issuing the CRITICAL START and CRITICAL END commands in a critical section of your program.

If the controller finds an error during data collection, some data signals might report NaN (not a number). Digital Scope shows a warning if a signal reports NaN.

To prevent invalid signals during data collection

  1. If you are collecting data on Virtual Axes, make sure that the data collection rate is 1 kHz or slower. Virtual axes do not support collecting some signals at rates greater than 1 kHz.
  2. Make sure that real-time errors do not occur on the controller during data collection. Real-time errors can cause signals to report NaN. You can review real-time errors in the Diagnostics Tab of the Status Utility.
