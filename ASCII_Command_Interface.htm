ASCII Command Interface

Use the ASCII command interface to send ASCII text strings to the controller and to do actions that include commanding motion and retrieving diagnostic information. The ASCII command interface receives ASCII text strings over a TCP/IP connection. The following sections show you how to configure the ASCII command interface.

After you configure the interface to receive ASCII text strings, you must make or download a program (for example, PuTTY) to send ASCII text strings to the ASCII command interface through a TCP/IP connection.

NOTE: The ASCII command interface supports all of the AeroBasic commands.
To Enable the ASCII Command Interface

The ASCII command interface is not enabled by default. To enable the interface, use the following procedure.

  1. In Configuration Manager, go to the CommandSetup Parameter.
  2. In the ASCII Command Interface Options section of the Editor pane, select Enabled.
  3. Configure the CommandPort Parameter.

NOTE: You must reset the controller for these changes to have an effect.

For more information about how to configure the CommandSetup and CommandPort parameters, refer to the Parameters and Interfaces section of this topic.

Parameters and Interfaces

Use the CommandSetup parameter to enable the ASCII command interface. You can also configure other ASCII communication settings. Refer to CommandSetup Parameter. The following table shows the values of this parameter.

Table:     CommandSetup Parameter Values

Parameter Value Description

0x00000004

Enabled When you set the parameter to this value, the ASCII command interface listens for client connections on the CommandPort.

0x00010000

Always Send EOS When you set the parameter to this value, the ASCII command interface attaches the End of String (EOS) character to the end of each response that it sends from the server to the client.

The CommandPort parameter specifies the TCP/IP port that the ASCII command interface uses to listen for incoming client connections. The ASCII command interface accepts client connections only when the A3200 SMC is running.

The ASCII Command Character parameters control which characters the ASCII command interface uses as outputs to send information back to the client about the result of some request sent by the client.

Table:     ASCII Command Character Parameters

Parameter Name Description

CommandTerminatingCharacter

Specifies the ASCII character that terminates all command strings that the client sends and all return string data that the ASCII command interface returns to the client. This is the "End of String" (EOS) character.

CommandSuccessCharacter

Specifies the ASCII character that the ASCII command interface returns when a command executes successfully.

CommandInvalidCharacter

Specifies the ASCII character that the ASCII command interface returns when a command is not syntactically correct.

CommandFaultCharacter

Specifies the ASCII character that the ASCII command interface returns when a command is formatted correctly and identified by the controller, but cannot execute successfully.

When a command executes successfully, the ASCII command interface returns the CommandSuccessCharacter. If the ASCII command interface also returns data for that command, such as axis status or position feedback, the interface returns the information in the following order.

  1. CommandSuccessCharacter
  2. Response Data
  3. CommandTerminatingCharacter (EOS)
Command Sequence

The ASCII command interface supports the following command modes.

  • Server commands
  • Controller commands

All server and controller commands must be terminated by the CommandTerminatingCharacter. This is the "End of String" (EOS) character.

The <Response Character> argument is the single-character response code that tells you about the status of a command that executes.

  • If a command executes successfully, the ASCII command interface returns the CommandSuccessCharacter.
  • If a command does not execute successfully or is not valid, the ASCII command interface returns the CommandInvalidCharacter.
  • If a command does not execute successfully, the ASCII command interface returns the CommandFaultCharacter.
Server Commands

The ASCII command interface executes server commands as special server commands that change the behavior of the interface or retrieve information. On a controller task, the ASCII command interface does not execute these commands as AeroBasic.

By default, the ASCII command interface uses the Library task to execute commands. To change the task, use the ~TASK server command.

Each server command has the prefix of ~ (tilde character).

Table:     Server Commands

Server Command Description

~GETVARIABLE <variable name> [<TaskID>]

Use the ~GETVARIABLE command to retrieve the value of a program, task, or global variable. The <TaskID> argument is optional. If you specify the <TaskID> argument, the ~GETVARIABLE command retrieves the variable on the specified task. If you do not specify the <TaskID> argument, this command retrieves the variable on the current task of the client. You do not have to include the $ in the name of the variable that you retrieve.

Different from the ~STATUS command, you can specify a <TaskID> argument when you use the ~GETVARIABLE command to retrieve global variables. Using this command is the same as if you get a variable from an AeroBasic program or use Variable Watch.

NOTE: If you use this command to retrieve strings that contain <CommandTerminationCharacter> characters, Aerotech recommends that you change the <CommandTerminationCharacter> to a non-printable character.

Example

' The ASCII command interface ignores the <TaskID> argument and 
' does not cause an error.
~GETVARIABLE $global[5] 2
%3.14

~INITQUEUE [<TaskID>]

Use the ~INITQUEUE server command to initialize queue mode on a task. The <TaskID> argument is optional. If you specify the <TaskID> argument, the ~INITQUEUE command initializes queue mode for the specified task. If you do not specify the <TaskID> argument, this command initializes queue mode on the current task of the client. Refer to Queue Mode Overview.

NOTE: Do not issue the ~INITQUEUE command on Task 0. Task 0 is the Library task.

Example

' Initializes queue mode on Task 3.
~INITQUEUE 3
%

~LASTERROR

Use the ~LASTERROR command to retrieve a string representation of the last error that occurred when the client executed a server command or a controller command. When you issue this command, the ASCII command interface only returns errors that occur when the current client executes commands. The interface does not make a report about errors that occur when other clients execute commands. At this time during the client connection, if no error has occurred the ASCII command interface returns "No Error".

~RESETCONTROLLER

Use the ~RESETCONTROLLER command to reset the controller. This command has the same functionality as when you click the Reset Controller button in Motion Composer. The ASCII command interface does not respond to this command until the reset completes.

NOTE: Do not issue commands from other clients while the controller is resetting because the behavior of commands is not defined during this period.

~STATUS

Use the ~STATUS command to retrieve more than one status item at a time. This command collects information about the syntax of Axis, System, Task, and Variable categories.

The ~STATUS server command is different from the DATACOLLECT ITEM command because it can retrieve more than one item at a time and also because it can retrieve a Variable status item.

The required arguments are different for each of the categories. But, you can call the status items in any order. Do not include the <AdditionalData> argument unless the <DataItem> that you call specifies this argument. For more information about lists for possible data items, refer to DATACOLLECT ITEM Command.

Results are separated by spaces and returned in the same order in which you call them. For more information about syntax and examples, refer to Table: ~STATUS Server Command Syntax Examples.

NOTE: In the DATACOLLECT ITEM command, all data items start with DATAITEM_, which is removed in the ~STATUS server command.

Example

' Retrieves more than one status item at a time. 
~STATUS (X, PositionFeedback) (1, QueueStatus, 1)
(Timer) (T01, MFO) ($global[0])
%19.671 0 50495 100 50

~STOPTASK [<TaskID>]

Use the ~STOPTASK command to stop a task and reset all of the states for that task. This command has the same functionality as when you click the Stop button in Motion Composer. The <TaskID> argument is optional. If you specify the <TaskID> argument, the ~STOPTASK command stops the specified task. If you do not specify the <TaskID> argument, this command stops the current task of the client.

~TASK <TaskID>

Use the ~TASK command to change the task on which controller commands execute. By default, controller commands execute on the Library task. If you change the executing task, the executing task changes only for that specific client. The executing tasks of other clients do not change.

If the value that you specify for the <TaskID> argument is negative, or if the value is greater than the number of possible tasks, the ASCII command interface returns the CommandInvalidCharacter. If you specify a valid value for the <TaskID> argument and the specified task cannot execute commands, for example the task is in the PLCReserved state, the interface returns the CommandFaultCharacter.

Example

' Sets the executing task to Task 3.
~TASK 3
%

~VERSION

Use the ~VERSION command to retrieve the version of the A3200 software that is running on the server. The ASCII command interface returns version information in the format of MAJOR.MINOR.REVISION.BUILD.

Controller Commands

The syntax for controller commands is the same as the syntax for AeroBasic code that you can write in a PGM program or execute in the Execute Command box of Motion Composer. Controller commands execute on the Library task by default. You can use the ~TASK command to change the task on which these commands execute. Controller commands do not have character prefixes.

The blocking behavior of controller commands is the same as the blocking behavior of the AeroBasic command that the controller is executing. If the controller executes the AeroBasic command through the ASCII command interface, the blocking behavior of the AeroBasic command is the same as if the controller executes the AeroBasic command in a PGM program. After the controller executes the AeroBasic command through the ASCII command interface, the interface returns the response character and can then accept more commands from the client.

For Example

The AeroBasic HOME command blocks until you home the axis. If you issue HOME X through the ASCII command interface, the interface does not return the response character to the client until the home sequence completes.

To constantly retrieve status information while doing blocking moves, connect one more client to the ASCII command interface and use that client to continuously request status information.

Controller Command Examples

Enables Axis X.

Example

ENABLE X
%

When a controller command executes successfully, the ASCII command interface returns the CommandSuccessCharacter. If the ASCII command interface also returns data for that command, such as axis status or position feedback, the interface returns the information in the following order.

  1. CommandSuccessCharacter
  2. Response Data
  3. CommandTerminatingCharacter (EOS)

Example

$glob0 = 75
%
$glob0
%75
