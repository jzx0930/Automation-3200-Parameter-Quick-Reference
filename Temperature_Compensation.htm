Temperature Compensation

Axis calibration can also be used to correct for thermal expansion. Using inputs from the calibration file and the AXCALPARM function, the SMC can calculate a temperature correction value. The SMC adds this temperature correction value to the correction value applied to the commanded axis position by axis calibration.

The SMC uses four inputs to determine the temperature correction value. The following table shows these inputs.

Table:     Temperature Compensation Inputs

Input

Origin

Equivalent in Axis Calibration File

AXISCAL_EXPAND_COEFF

Axis Calibration File

EXPANDCOEFF

AXISCAL_CURRENT_EXPAND_COEFF

AXCALPARM Command

Not applicable

AXISCAL_REFERENCE_TEMP

Axis Calibration File

MATERIALTEMP

AXISCAL_CURRENT_MATERIAL_TEMP

AXCALPARM Command

Not applicable

When the axis calibration file is downloaded, the value in the ":START" line of the axis calibration file for EXPANDCOEFF is copied to AXISCAL_EXPAND_COEFF and the value in the ":START" line for MATERIALTEMP is copied to AXISCAL_REFERENCE_TEMP. The AXISCAL_CURRENT_EXPAND_COEFF and AXISCAL_CURRENT_MATERIAL_TEMP values are set to AXISCAL_INVALID_EXPAND_COEFF and AXISCAL_INVALID_MATERIAL_TEMP, respectively. This disables axis calibration temperature correction.

For default Aerotech calibration files:

  • The AXISCAL_EXPAND_COEFF and EXPANDCOEFF are set to the expansion coefficient of the feedback device.
  • The AXISCAL_REFERENCE_TEMP and MATERIALTEMP are set to 20 °C.

NOTE: Typically, axis indices are specified as zero-based integers. But calibration files specify axis indices as one-based integers.

Note: To see the calibration file data, use the DUMPAXISCAL Console command in Console.

Using Temperature Compensation

To enable temperature compensation, AXISCAL_CURRENT_MATERIAL_TEMP must be set to something other than AXISCAL_INVALID_MATERIAL_TEMP. If the axis is homed and axis calibration is active, the axis will be corrected for temperature when AXISCAL_CURRENT_MATERIAL_TEMP does not equal AXISCAL_INVALID_MATERIAL_TEMP.

Use the AXCALPARM command to set AXIS_CURRENT_MATERIAL_TEMP to the current temperature in °C. Typically, you will have a task that monitors an input from some temperature sensor and uses the AXCALPARM command to continually update the value of AXIS_CURRENT_MATERIAL_TEMP based on that input.

The AXISCAL_CURRENT_EXPAND_COEFF parameter is used to change whether temperature correction is done relative to the feedback device or relative to the expansion coefficient of some other material.

Temperature Correction Value Calculation

The SMC calculates temperature correction as follows:

DeltaTemp = AXISCAL_CURRENT_MATERIAL_TEMP - AXISCAL_REFERENCE_TEMP;
If(AXISCAL_CURRENT_EXPAND_COEFF==AXISCAL_INVALID_EXPAND_COEFF) DeltaExpandCoeff = -1 * AXISCAL_EXPAND_COEFF;
Else
    DeltaExpandCoeff=AXISCAL_CURRENT_EXPAND_COEFF - AXISCAL_EXPAND_COEFF;
EndIf
TempCorrection = DeltaExpandCoeff * DeltaTemp * (CurrentMasterPosition - TableStartPosition);

TableStartPosition is the position of the first point in the calibration table after all offsets are applied. This position cannot be zero if an offset value is specified in the calibration file or if you have changed the HomeOffset parameter relative to the home offset specified in the calibration file.
