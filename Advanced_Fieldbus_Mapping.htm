Advanced Fieldbus Mapping

The easiest way to map your Fieldbus I/O to AeroBasic is to map a simple array of words or bits. If you understand the Fieldbus protocol that you are using and the addresses that are used by the remote device, you can write AeroBasic programs by using those mappings.

Fieldbus Mapping Dialog in Configuration Manager

If your system is more advanced, use the Fieldbus Mapping Dialog to map any type of data to any valid address on the Fieldbus and to give it a unique name so that it can be accessed in AeroBasic programs. It is easy to work with data structures and aliases because the mappings can overlap. There is a mapping dialog for each Fieldbus that is supported by A3200.

For Example

You have eight lamps connected to the output bits of a Modbus slave device. By using the mapping dialog, you can give each lamp a unique name so that you can use it in your AeroBasic program. You can also test or set all of the lamps at one time.

Figure: PC Modbus Master Configuration for Lamps

In the code, you can see that the mapping for lamps overlaps the mappings for the individual lamps.

Example

' Clear all lamps.
$lamps = 0

' Specify the lamp that you want and configure it.
$faultLamp = AXISSTATUS(X, DATAITEM_AxisFault)

' Test for a lamp.
if ($lamps != 0)
MSGDISPLAY 0, "One or more lamps are on."
endif

Non-Contiguous Data

The mapped Fieldbus do not have to be contiguous. A3200 automatically synchronizes only the data sections that are mapped. Gaps in the mapped address space are not read from or written to the Fieldbus device. Data is grouped into the largest possible contiguous sections. Each section is synchronized individually using round-robin scheduling. If possible, individual mappings are synchronized atomically, which means that all of the data in the mapping is updated in the same millisecond and all values are fully written. The maximum array size that A3200 can atomically synchronize is different for each Fieldbus protocol. Scalar data values are always synchronized atomically.
