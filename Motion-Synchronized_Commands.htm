Motion-Synchronized Commands

When lookahead is not active, all commands are synchronized to motion. Some AeroBasic commands are synchronized to motion when lookahead is active and when it is not active. During the lookahead pre-execution of the AeroBasic program, the program executes through the synchronized command, but motion-synchronized commands are not executed until the motion before it has completed. For more information about lookahead, refer to Program Block Lookahead Overview.

These are the motion-synchronized commands:

  • Axis speed commands (XF,YF,…)
  • DWELL
  • EXECCANNEDFUNCTION
  • GALVO LASEROVERRIDE AUTO
  • GALVO LASEROVERRIDE OFF
  • GALVO LASEROVERRIDE ON
  • I/O sets ($AO, $BI, $BO, $DO, $RI, $RO, $WO, $PORT)
  • MOVEDELAY
  • PSOANALOG ARRAY
  • PSOANALOG OFF
  • PSOANALOG ON
  • PSOANALOG OUTPUT
  • PSOANALOG TRIGGER
  • PSOCONTROL ARM
  • PSOCONTROL ARM GALVO
  • PSOCONTROL ARM PWM
  • PSOCONTROL FIRE
  • PSOCONTROL FIRE CONTINUOUS
  • PSOCONTROL OFF
  • PSOCONTROL ON
  • PSOCONTROL PULSE DISABLE
  • PSOCONTROL RESET
  • PSODISTANCE
  • PSOOUTPUT
  • PSOPULSE
  • PSOPULSE2
  • PSOTRACK
  • PSOWINDOW CONTROL
  • PSOWINDOW INPUT
  • PSOWINDOW LOAD
  • PSOWINDOW OFF
  • PSOWINDOW ON
  • PSOWINDOW RANGE
  • PSOWINDOW RANGE ARRAY
  • PSOWINDOW RESET
  • RAMP MODE
  • RAMP RATE
  • RAMP TIME
  • SCOPETRIG
  • SCURVE
  • SYNCCALL
  • TRACE END

Note: I/O reads are not motion-synchronized. For information about how this changes the behavior of the command during lookahead, refer to Program Block Lookahead Overview.

To synchronize the execution of other AeroBasic commands with motion, use the SYNCCALL Command.
