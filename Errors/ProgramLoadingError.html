An error occurred when you tried to download, associate, or run a program file. This error can also occur when you use immediate commands.

The sections that follow tell you what might cause the error and give you solutions.

The program cannot be associated because a program is already associated with the specified task.

Cause

You tried to associate a program on a task that already had a program associated.

Solution

Issue a Program Stop to disassociate the existing program from the task before you associate a new program on the task.

A program is not associated with the specified task.

Cause

You tried to start a program on a task when there was no program associated with the task.

Solution

You must associate a program with a task before you can issue a Program Start to run a program on that task. If you add a program to a task by issuing an Add Program, before you associate the program with a task, issue a Program Associate to associate the program with the task. You can also issue a Program Load to add and associate the program or you can issue a Program Run to add, associate, and start the program.

A program is not active on the specified task.

Cause

An error occurred executing a program in queue mode. Refer to Queue Mode Overview.

Solution

Restart the program and try again. If the problem continues to occur, contact Aerotech Global Technical Support.

The program cannot be removed because it is associated with a task.

Cause

You tried to remove a program by issuing a Remove Program and the program was associated with a task.

Solution

You must disassociate a program from a task by issuing a Program Stop before you can remove the program. 

An immediate command cannot be executed while another immediate command is executing.

Cause

You tried to execute an immediate command while another immediate command was executing.

Solution

You can only execute one immediate command on a task at one time. Wait until the command completes executing before you try to execute another immediate command. Refer to Immediate Commands for more information.

An invalid task execution mode was specified.

Cause

You specified an invalid execution mode argument to the EXECUTEMODE command.

Solution

Change your program to use a valid execution mode. Refer to EXECUTEMODE command for a list of valid execution modes.

A program with the specified name already exists on the controller.

Cause

You tried to add a program to the controller, but a program with the same name already exists.

Solution

Use a different name for the program or issue a Remove Program to remove the existing program before you add the new program.

You tried to load more than 100 programs onto the controller.

Cause

The maximum number of programs that can be loaded onto the controller is 100. You tried to load more than 100 programs.

Solution

Issue a Remove Program to remove programs from the controller before you add a new program.

The specified program was not allocated.

Cause

The controller tried to execute a program and memory for the program was not allocated.

Solution

Restart the program and try again. If the problem continues to occur, contact Aerotech Global Technical Support.

The program load is not complete.

Cause

The controller tried to load a program line but the program line was already loaded.

Solution

Restart the program and try again. If the problem continues to occur, contact Aerotech Global Technical Support.

The program code line is full.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

Insufficient memory is available to load the specified program line.

Cause

The controller tried to load a program line but sufficient memory was not available.

Solution

Restart the controller and try again. If the problem continues to occur, contact Aerotech Global Technical Support.

The program label list is full.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

A program queue is already associated with this task.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

A jump outside of the program queue range was detected.

Cause

You executed a program in queue mode, and the program tried to jump to a line that has not been downloaded to the controller or to a line that has been executed and discarded by the controller. Refer to Queue Mode Overview.

Solution

You must use sequential, single line commands when you run a program in queue mode. Refer to PROGRAM BUFFEREDRUN command for more information.

The last program line has been executed.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

This operation cannot be completed in the current task state.

Cause

You tried to do a program operation that is not valid in the current task state.

Solution

Some program operations are prohibited in certain task states. Refer to Program Control Commands for more information.

The specified immediate command cannot be executed while a program is running.

Cause

You issued an immediate command while an AeroBasic program was running on the same task.

Solution

Not all immediate commands can be executed while an AeroBasic program is running. Do one of the solutions that follow.

  • Wait until the AeroBasic program completes before you issue an immediate command.
  • Issue the immediate command on a different task.

Refer to Immediate Commands for more information.

The immediate command cannot be executed while a task is in queue mode.

Cause

You tried to issue an immediate command while a task was running in queue mode. Refer to Queue Mode Overview.

Solution

If a program is running in queue mode on a task, do one of the following.

  • On that same task, wait until the program completes before you issue an immediate command.
  • On that same task, stop the program that is running in queue mode before you issue an immediate command.
  • On a different task, issue an immediate command.

For more information about immediate commands, refer to Immediate Commands.

The specified program operation cannot be performed while an immediate command is executing.

Cause

You issued a program control command on a task while an immediate command was in progress. Refer to Program Control Command Category for a list of program control commands.

Solution

Do one of the solutions that follow.

  • Wait until the immediate command completes before you issue a program control command.
  • Issue the program control command on a different task.
  • Stop the immediate command with PROGRAM STOP. Then issue the program control command.
The immediate command operation failed to execute.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

The controller failed to create the large program execution thread.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

A buffered program is already running on this task.

Cause

You tried to run a program in queue mode while a buffered program was in progress on the same task.

Solution

Do one of the solutions that follow.

  • Run the program on another task.
  • Wait for the current program to complete before you run a new program.

Refer to Queue Mode Overview for more information about queue mode.

The immediate command contains too many lines to be executed.

Cause

You tried to execute too many lines as a single immediate command.

Solution

Break the command into smaller sections and execute them separately.

The program cannot be loaded because the task is stopping.

Causes

  • You tried to load a program to a task, but that task is in the process of stopping an existing program.
  • You tried to load a program to a task while one or more of the axes that you selected in the TaskStopAbortAxes parameter were moving.

Solutions

  • Execute the program on another task or wait for the current program to stop before you load a new program.
  • Issue an ABORT on all of the axes that are selected in the TaskStopAbortAxes parameter. Wait for the abort to complete and then load a new program. If you do not want a task stop to cause the controller to abort those axes, then clear the selections for those axes in the TaskStopAbortAxes parameter.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
