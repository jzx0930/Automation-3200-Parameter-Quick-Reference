The sections that follow tell you what might cause the error and give you solutions.

An error occurred creating the Modbus register mutex.

Cause

An unexpected error occurred during PC Modbus initialization.

Solution

Restart the PC and continue normal operation of the controller. 

The Modbus register size is invalid.

Cause

An invalid size was specified for a Modbus register.

Solution

Examine the Modbus settings and verify that the register sizes do not exceed 65,536 words or bits.

An error occurred while allocating memory for the Modbus registers.

Cause

An error occurred while trying to allocate memory for the PC Modbus registers.

Solution

Examine the sizes for all PC Modbus registers. Do not set the size of a register to a larger value than you need. If you are not using a specific set of registers, set the size to zero.

No Modbus registers are declared.

Cause

You have enabled the PC Modbus Slave, but all Modbus registers sizes are set to zero.

Solution

When you configure the connection details for the PC Modbus Slave, you must declare a minimum of one of the register types to a non-zero size.

A timeout occurred while waiting to acquire a mutex.

Cause

An unexpected error occurred during PC Modbus initialization.

Solution

Restart the PC and continue normal operation of the controller. 

A socket error occurred while sending data.

Cause

An error occurred while trying to send data to a Modbus device.

Solution

Confirm the Ethernet connection between the PC and Modbus device. Verify that the Modbus device is powered on.

An exception occurred while receiving data.

Cause

An error occurred while trying to receive data from a Modbus device.

Solution

Confirm the Ethernet connection between the PC and Modbus device. Verify that the Modbus device is powered on.

The Modbus register address is out of bounds.

Cause

You tried to access a Modbus register that has not been allocated.

Solution

Examine the configured Modbus register sizes. For a PC Modbus Slave, the mapped addresses must not be outside the bounds of the configured register sizes. Make sure to reset the controller after all changes that you make to the Modbus configuration.

The specified Modbus connection number is invalid.

Cause

The modbus connection number that was specified is invalid.

Solution

If you make changes to the Modbus configuration, you must recompile your AeroBasic program. If you are using obsolete MODBUS commands, you must manually specify the correct connection number. When you connect as a Modbus Master, you must specify a number from 0 - 15 as the connection number. (Connection numbers 1 - 15 are only available with the Professional Option.) When you connect as a Modbus Slave, you must specify 0 for the connection number.

The specified Modbus register is invalid for the given command.

Cause

The set of Modbus registers that you are trying to access is not valid for the given command. This error occurs if you do one of the following.

  • You try to write to InputWords or InputBits.
  • You try to use a MODBUSREG command to access InputBits or OutputBits.
  • You try to use a MODBUSBIT command to access InputWords or OutputWords.
  • You try to access SlaveInputWords, SlaveOutputWords, SlaveInputBits or SlaveOutputBits when using Drive Modbus.

Solution

Verify that you are accessing the correct register sets when you use the obsolete MODBUS commands.

The specified Modbus register is read-only.

Cause

You tried to write to a mapped Modbus variable in a read-only register space.

Solution

You cannot write to master input and status registers because they are read-only. Make sure that your I/O is mapped to the correct register space and make all necessary changes to your AeroBasic program.

The maximum number of configured Modbus slave connections has been exceeded.

Cause

You configured too many Modbus slave connections.

Solution

You can configure a maximum of one A3200 Modbus slave at one time. Remove one or more of the configurations and reset the controller.

The maximum number of configured Modbus master connections has been exceeded.

Cause

You configured too many Modbus master connections.

Solution

If you have the standard license options, you can only configure one Modbus Master connection at one time. If you have the Professional Option, you can configure up to 16 Modbus Master connections. Remove one or more of the configurations and reset the controller.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
