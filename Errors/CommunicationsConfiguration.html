Possible causes include:

  • The FireWire or HyperWire card is malfunctioning.
  • The FireWire or HyperWire cable is malfunctioning.
  • The PCI slot is malfunctioning.
  • The FireWire or HyperWire network is configured incorrectly.
  • The installed FireWire or HyperWire card is not supported.

Until you find a solution, you can run virtually without a FireWire or HyperWire card.

To configure the system to run without a FireWire or HyperWire card (virtual)

  1. In the Network Explorer of Configuration Manager, right-click Controller and then click Connection Settings.
  2. The Connection Settings dialog comes into view. Select the Drive Communication tab.
  3. Select Operate without a card.
  4. Click OK and then close Configuration Manager. Close all other A3200 applications.
  5. Shut down the A3200. Click Start → All Programs → Aerotech → Automation 3200 → Shutdown A3200. A black Command Prompt window comes into view momentarily.
  6. Open Configuration Manager or any other A3200 application. All axes then operate as virtual.

The sections that follow tell you what might cause the error and give you solutions.

A FireWire or HyperWire card was not found on the PCI bus.

Cause

No FireWire or HyperWire card is installed in the PC or the A3200 does not recognize the card.

Solution

If you have a FireWire or HyperWire card installed but it is not recognized, then you must configure the card using the Connection Settings dialog in Configuration Manager. Refer to Configuring and Verifying the FireWire or HyperWire™ Card for more information.

If you do not have a FireWire or HyperWire card installed, then you must install a card or you must configure the system to run without a FireWire or HyperWire card as described previously.

An invalid PCI slot number was specified.

Cause

The PCI slot number specified in the FireWire or HyperWire configuration is outside the range of valid slot numbers.

Solution

  • Verify that you have the correct FireWire or HyperWire card selected in the Drive Communication Interface settings in the Connection Settings dialog of Configuration Manager.
  • Do the procedure that follows.
    1. Move the FireWire or HyperWire card to a different PCI slot. Refer to Installing the FireWire or HyperWire™ Card .
    2. Reconfigure the FireWire or HyperWire card. Refer to Configuring and Verifying the FireWire or HyperWire™ Card.
    3. Open Configuration Manager and try to connect to the controller. FireWire or HyperWire related error messages should not come into view.
An invalid communications speed was requested.

Cause

The FireWire card does not support the requested communications speed.

Solution

  • If you are using the -S SMC argument, change the specified speed or remove the -S argument. Refer to SMC Arguments for more information.
  • Replace your FireWire card with one that supports speeds of 400 MHz.
The controller was unable to verify the BusID in the OHCI register space.

Cause

Your FireWire card is not OHCI compliant, which is a requirement of the A3200.

Solution

Replace your FireWire card with a card that is 1394 OHCI compliant.

A timeout occurred while waiting for the FireWire soft reset to complete.

Cause

A timeout occurred waiting for the FireWire card to reset.

Solution

Increase the soft reset timeout value by using the -Tf SMC argument. Refer to SMC Arguments for more information.

An error occurred while configuring the PC as a valid FireWire node.

Cause

This error is usually caused by the FireWire cables forming an infinite loop (there are two paths in the FireWire cabling to the drives). This error can also by caused by a defective drive, defective FireWire card, or defective FireWire port.

Solution

  • Look at the FireWire cabling and make sure there is only one forward path in the cabling between the PC and the drives.
  • If your FireWire card is a PC bus card, verify that the card is correctly seated in the bus.
  • Disconnect all drives from the FireWire bus and reset the A3200. If this succeeds, then the problem is with the drive or FireWire cable. If it fails the problem is with the PC or FireWire card.

This error can also be caused by a PC power supply instability that occurs during PC startup. Aerotech recommends a specially modified FireWire interface card (Aerotech part number NFIRE-PCI/ES15588) in this case.

An error occurred while establishing the PC as the master FireWire node.

Cause

This error usually shows that either the FireWire card or the FireWire cable is defective.

Solution

Shut down and restart the PC. If the problem continues, then replace the FireWire cable and try to connect again. If the problem still occurs, then replace the FireWirecard.

A duplicate communications channel was detected.

Cause

Two or more drives are configured with the same communications channel.

Solution

The communication channels are set using small mechanical switches on the drive. Verify that all drives have unique communication channels. For more information, refer to the hardware manuals for your drives.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
