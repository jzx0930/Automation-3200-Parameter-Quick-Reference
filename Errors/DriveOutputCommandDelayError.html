DriveOutputCommandDelayError

A DriveOutputCommandDelay error has occurred: <Location>

Table:     Location Codes

Location Code Interpretation

1

This action cannot be done while an output command is being delayed.

2

The number of available delay buffers has been exceeded.

3

This action cannot be done while DriveOutputCommandDelay is active.

The sections that follow tell you what might cause the error and give you solutions.

This action cannot be done while an output command is being delayed.

Cause

You tried to change the DriveOutputCommandDelay parameter on an axis that is currently delaying one or more of the $AO, $DO, PSO, or ARRAY WRITE commands.

Solution

Before you change the DriveOutputCommandDelay parameter, wait for all of the $AO, $DO, PSO, or ARRAY WRITE commands to be delayed. You can do this by issuing a DWELL command before you change the DriveOutputCommandDelay parameter.

The number of available delay buffers has been exceeded.

Cause

You tried to enable the DriveOutputCommandDelay parameter on more than 4 axes. You can only configure the DriveOutputCommandDelay parameter on 4 axes at one time.

Solution

Set the DriveOutputCommandDelay parameter to 0 on axes that do not need to delay the $AO, $DO, PSO, or ARRAY WRITE commands.

This action cannot be done while DriveOutputCommandDelay is active.

Cause

The IFOV ON command was issued and the DriveOutputCommandDelay parameter was set on one or more axes that were configured for Infinite Field of View (IFOV).

Solution

You cannot use IFOV and the DriveOutputCommandDelay parameter at the same time because IFOV already synchronizes drive outputs. Set the DriveOutputCommandDelay parameter to 0 on all axes that are in your IFOV configuration.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
