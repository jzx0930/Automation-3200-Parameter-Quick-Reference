CamTableError

An error occurred with a cam table command: <Location>

Table:     Location Codes

Location Code Interpretation

1

The cam table number is not valid.

2

The cam synchronization mode is not valid.

3

The tracking mode is not valid.

4

The controller did not load cam points for this cam table.

5

The controller is using the cam table.

6

The interpolation type is not valid.

7

The specified slave axis is not configured to a master axis.

8

The slave axis is currently synchronized to a table.

The sections that follow tell you what might cause the error and give you solutions.

The cam table number is not valid.

Cause

The cam table number that you specified to the <TableNumber> argument is not valid for a CAMSYNC, FREECAMTABLE, LOADCAMTABLE, or LOADCAMVAR command.

Solution

Change your AeroBasic program so that the CAMSYNC, FREECAMTABLE, LOADCAMTABLE, or LOADCAMVAR command specifies a <TableNumber> argument that is greater than or equal to 0 and less than or equal to 99.

The cam synchronization mode is not valid.

Cause

The cam synchronization mode that you specified to the <SyncMode> argument is not valid for the CAMSYNC command.

Solution

Change your AeroBasic program so that the CAMSYNC command specifies a valid <SyncMode> argument. For more information about the synchronization modes, refer to Table: Synchronization Modes in this command.

The tracking mode is not valid.

Cause

The mode that you specified to the <TrackingMode> argument is not valid for a LOADCAMTABLE or LOADCAMVAR command.

Solution

Change your AeroBasic program so that the LOADCAMTABLE or LOADCAMVAR command specifies a <TrackingMode> argument that has a value of 0, 1, or 2.

The controller did not load cam points for this cam table.

Causes

  • You did not issue the LOADCAMTABLE or LOADCAMVAR command.
  • The value of the <TableNumber> argument that you specified to the LOADCAMTABLE or LOADCAMVAR command is not the same as the value of the <TableNumber> argument that you specified to the CAMSYNC command.

Solutions

  • Issue the LOADCAMTABLE or LOADCAMVAR command to load a cam table.
  • Issue the CAMSYNC command with a <TableNumber> argument that has the same value as the <TableNumber> argument that you specified to the LOADCAMTABLE or LOADCAMVAR command.
The controller is using the cam table.

Cause

You tried to issue the FREECAMTABLE command on a cam table that is synchronized to a master axis.

Solution

Issue the CAMSYNC command with a <SyncMode> argument that has a value of 0 and a <TableNumber> argument that has the number of the cam table that you want to release. Then issue the FREECAMTABLE command.

The interpolation type is not valid.

Causes

  • The mode that you specified to the <InterpolationType> argument is not valid for a LOADCAMTABLE or LOADCAMVAR command.
  • The interpolation type that you specified in the cam table is not valid.

Solutions

  • Change your AeroBasic program so that the LOADCAMTABLE or LOADCAMVAR command specifies an <InterpolationType> argument that has a value of 0 or 1.
  • Change your cam file so that all data lines have valid interpolation type values. Refer to Cam Table Format for more information.
The specified slave axis is not configured to a master axis.

Cause

You issued the CAMSYNC command with a cam table that the controller either loaded incorrectly or did not load.

Solution

Issue a LOADCAMTABLE or LOADCAMVAR command to load the cam table that you want to use. Then issue the CAMSYNC command with a <TableNumber> argument that has the cam table that you want to use.

The slave axis is currently synchronized to a table.

Cause

You issued the CAMSYNC command with a slave axis that is currently synchronized to a cam table.

Solution

Issue the CAMSYNC command with a <SlaveAxis> argument that has the synchronized slave axis and a <SyncMode> argument that has a value of 0. Then issue the CAMSYNC command with a <TableNumber> argument that has the cam table that you want to use.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
