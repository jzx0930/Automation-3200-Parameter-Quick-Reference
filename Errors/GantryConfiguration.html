The sections that follow tell you what might cause the error and give you solutions.

The GantryMasterAxis and/or GantrySetup parameters are invalid.

Causes

  • On a specified axis, the GantrySetup parameter is set to a nonzero value, but the value specified for the GantryMasterAxis parameter is invalid.
  • On a specified axis, the GantryMasterAxis parameter is set to a nonzero value, but the value specified for the GantrySetup parameter is invalid.
  • On a specified axis, the GantryMasterAxis parameter is set to an invalid value.

Solution

Review the settings for the GantrySetup and GantryMasterAxis parameters and change them to specify a valid gantry configuration. Refer to Configuring Parameters for Gantry Mode for more information on gantry parameter configuration.

The gantry master must have a lower communication channel than the gantry slave.

Cause

The gantry master communication channel is set to a higher communication channel number than the gantry slave(s).

Solution

Each drive is set to a unique communications channel, which corresponds to the axis parameters defined in the Configuration Manager. Check the communication channel settings on the gantry master and gantry slave drives and make sure the gantry master has a lower channel number than the slave(s). The communication channels are set using small mechanical switches on the drive. For more information, refer to the hardware manuals for your drives.

The gantry master and gantry slave must be the same type drive.

Cause

The gantry parameters are configured such that the gantry master and gantry slave(s) are using incompatible drive types.

Solution

Change the gantry configuration to use axes with compatible drive types.

Note: Mixed drive types will not trigger a fault or error, but Aerotech recommends that you always use the same drive type.

Multi-Axis Drives
  • Two axes of an Npaq
  • Two axes of an Nservo
  • Two axes of an XR3
Single-Axis Drives
  • Two Ndrive CPs
  • Two Ndrive HLes
  • Two Ndrive HPes
  • Two Ndrive MLs
  • Two Ndrive MPs
  • Any combination of two of these Ndrive types (not recommended)
  • Two XC2s
  • Two XC4s
  • Two XC4es
  • Any combination of two of these HyperWire drive types (not recommended)
The GantryMasterAxis and/or GantrySetup parameters are invalid.

Cause

An internal error has occurred in the SMC configuring the gantry parameters.

Solution

Review the settings for the GantrySetup and GantryMasterAxis parameters. If you continue to encounter this error, contact Aerotech Global Technical Support.

The GantrySetup parameter setting is invalid.

Cause

On a specified axis, the GantrySetup parameter is set to an invalid value.

Solution

For the specified axis, change the GantrySetup parameter to a valid setting.

A gantry cannot span across two multi-axis drives.

A gantry cannot be configured to use axes on two different multi-axis drives. For example, if a gantry master axis is set on one Npaq, and a gantry slave axis is set on a different Npaq, unexpected behavior will occur.

Cause

The Gantry parameters are configured to create a gantry using axes on two different multi-axis units.

Solution

Change the gantry configuration to use axes that are in the same multi-axis unit and update the GantryMasterAxis and/or GantrySetup parameters as necessary.

One or more drives do not support gantry mode.

Cause

The gantry parameters are configured to use one or more of the following drives. These drives do not support a gantry configuration.

  • GL4
  • Ndrive FCL
  • Ndrive FLS
  • Ndrive QL
  • Ndrive QLe
  • Nmark CLS
  • Nmark GCL
  • Nmark SSaM
  • XL4s

Solution

Change the gantry parameters to exclude drives that do not support a gantry configuration.

A position gantry must have exactly two axes.

Cause

More than two axes are configured in a position gantry chain. Position gantry mode supports only two axes per gantry chain.

Solutions

Use one of the following solutions.

  • Change the GantrySetup and GantryMasterAxis parameters to configure a position gantry with exactly two axes.
  • Change the GantrySetup parameter on the gantry axes to use current gantry mode instead of position gantry mode. Current gantry mode supports more than two axes per gantry chain.
A gantry cannot contain multiple GantrySetup types.

Cause

The GantrySetup parameter is not set to the same value on every slave axis in a gantry chain.

Solution

Change the GantrySetup parameter to be the same on every slave axis in the gantry chain.

The gantry master and gantry slave must have the same FaultMask configuration.

Cause

A gantry slave axis has one or more of the following parameters set differently than the gantry master axis.

  • FaultMask
  • FaultMaskDecel
  • FaultMaskDisable
  • FaultMaskDisableDelay

Solution

In Configuration Manager, make the values of the following parameters the same on the gantry master axis and the gantry slave axis.

  • FaultMask
  • FaultMaskDecel
  • FaultMaskDisable
  • FaultMaskDisableDelay

If you are not sure about how to configure the fault behavior of the controller, contact Aerotech Global Technical Support.

A gantry has an invalid MotorType parameter.

Cause

The MotorType parameter specifies a motor type that is not compatible with gantry mode. Gantry mode does not support the motor types that follow.

  • Stepper Motor
  • Piezo Actuator

Solution

Use one of the solutions that follow.

  • Disable gantry mode with the GantrySetup and GantryMasterAxis parameters.
  • Use a motor type that gantry mode supports.
A gantry cannot span across two different HyperWire chains

Cause

A gantry slave axis is on a different HyperWire chain than its designated master axis.

Solution

Connect HyperWire cables so that gantry slave axes are on the same HyperWire chain as their designated master axes.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
