CamFileError

An error occurred while processing a cam file: <Location>

Table:     Location Codes

Location Code Interpretation

1

Master position units were not specified.

2

Slave position units were not specified.

3

A number of points that is not valid was specified.

4

A point number that is not valid was specified.

5

A master position value that is not valid was specified.

6

The controller detected a master position value that is out-of-sequence.

7

A slave position value that is not valid was specified.

8

A number of points that is not sufficient was specified.

9

A number of points that is too large was specified.

The sections that follow tell you what might cause the error and give you solutions.

Master position units were not specified.

Cause

The Master Units header line in the cam file is either missing or not valid.

Solution

Specify a valid Master Units header line in the cam file. Refer to Cam Table Format for more information.

Slave position units were not specified.

Cause

The Slave Units header line in the cam file is either missing or not valid.

Solution

Specify a valid Slave Units header line in the cam file. Refer to Cam Table Format for more information.

A number of points that is not valid was specified.

Causes

  • The Number of Points header line in the cam file is either missing or not valid.
  • You specified a value to the <VariableNumber> argument that is not valid for the LOADCAMVAR command.

Solutions

  • Specify a valid Number of Points header line in the cam file. Refer to Cam Table Format for more information.
  • Issue the LOADCAMVAR command with a <VariableNumber> argument that has a valid value. Refer to LOADCAMVAR command for more information.
A point number that is not valid was specified.

Cause

You specified a point number that is not valid on a data line in the cam file.

Solution

Specify point numbers as monotonically ascending or monotonically descending positive integers in the cam file. Refer to Cam Table Format for more information.

A master position value that is not valid was specified.

Cause

The master position value is either missing or not valid on a data line in the cam file.

Solution

Make sure that each data line in the cam file has a valid master position value. Refer to Cam Table Format for more information.

The controller detected a master position value that is out-of-sequence.

Cause

A master position value that you specified in the cam file causes the master position values to not be strictly ascending or strictly descending.

Solution

Specify master position values in the cam file so that they are all strictly ascending or strictly descending. Refer to Cam Table Format for more information.

A slave position value that is not valid was specified.

Cause

The slave position value is either missing or not valid on a data line in the cam file.

Solution

Make sure that each data line in the cam file has a valid slave position value. Refer to Cam Table Format for more information.

A number of points that is not sufficient was specified.

Cause

The cam file has less data lines than the value that you specified for the Number of Points header line in the cam file.

Solution

Decrease the value of the Number of Points header line to be the same as the number of data lines in the cam file. Refer to Cam Table Format for more information.

A number of points that is too large was specified.

Cause

The cam file has more data lines than the value that you specified for the Number of Points header line in the cam file.

Solution

Increase the value of the Number of Points header line to be the same as the number of data lines in the cam file. Refer to Cam Table Format for more information.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
