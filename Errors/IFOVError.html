The sections that follow tell you what might cause the error and give you solutions.

An internal configuration error occurred.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

A valid galvo axis must be specified.

Cause

The FireWire cable, HyperWire cable, or one or more drive cables are not connected correctly.

Solution

Make sure that the FireWire or HyperWire cable is connected and that all drives are connected and powered on.

 

Cause

A galvo axis that is not valid was specified for the IFOV AXISPAIR Command.

Solution

Make sure that you specify the galvo axis of a GL4 or an Nmark CLS, GCL, or SSaM as part of the AeroBasic command.

A valid servo axis must be specified.

Cause

A servo axis that is not valid was specified for the IFOV AXISPAIR command.

Solution

Make sure that you specify the servo axis of a drive that is not a GL4 or an Nmark CLS, GCL, or SSaM.

The IFOV configuration must have two galvo axes and two servo axes.

Cause

  • Zero or one IFOV axis pairs were configured instead of two axis pairs.
  • The IFOV axis pairs do not specify different axes.

Solution

Issue the IFOV AXISPAIR command two times to configure the two axis pairs. All four axes that you specify must be different.

A maximum of six axes can be commanded with IFOV.

Cause

More than two axes were specified for the IFOV SYNCAXES command.

Solution

The controller lets you configure a total of six axes with IFOV. This includes the four axes in the IFOV axis pairs and a maximum of two more axes for the IFOV SYNCAXES command. Do one of these solutions.

  • If you want to synchronize only the four axes in the two IFOV axis pairs, specify NONE for the IFOV SYNCAXES command.
  • If you want to synchronize more than four axes, specify 1 or 2 for the IFOV SYNCAXES command.
The IFOV size must be greater than 0.

Cause

  • The IFOV SIZE command was not issued.
  • A value of 0 was specified for the IFOV SIZE command.

Solution

Use the IFOV SIZE command to specify the field of view size, in user units, of the galvo head. This value must be greater than 0.

The search time is not valid.

Cause

  • The IFOV TIME command was not issued.
  • The value specified for the IFOV TIME command is not divisible by 5 milliseconds.
  • The value specified for the IFOV TIME command exceeds the value of the IFOVMaximumTime parameter.

Solution

Use the IFOV TIME command to specify a search time that is divisible by 5 milliseconds.

The operation cannot be done because IFOV is active.

Cause

IFOV is enabled and a command was issued that cannot be used during IFOV.

Solution

  • Change your AeroBasic program to issue the command before you issue IFOV ON.
  • Change your AeroBasic program to issue the command after you issue IFOV OFF.
  • Use the IFOV OFF command to disable IFOV. Then try to issue the command again.
The operation cannot be done because no IFOV configurations are available.

Cause

  • IFOV ON was issued while the IFOVConfigurations parameter was set to 0.
  • IFOV ON was issued while the controller was using other IFOV configurations.

Solution

  • To specify the number of configurations that are available, change the IFOVConfigurations parameter to 1 or 2 in Configuration Manager. Then reset the controller.
  • Do not use more than two IFOV configurations at the same time. On each task, you can specify a different IFOV configuration. But you can enable IFOV only on two tasks at the same time.
The tracking speed must be greater than 0.

Cause

  • The IFOV TRACKINGSPEED command was not issued.
  • A value of 0 was specified for the IFOV TRACKINGSPEED command.

Solution

Use the IFOV TRACKINGSPEED command to specify the maximum speed of the servo axes while in IFOV mode. You must specify a speed that is greater than 0.

The tracking acceleration must be greater than 0.

Cause

  • The IFOV TRACKINGACCEL command was not issued.
  • A negative value was specified for the IFOV TRACKINGACCEL command.

Solution

Use the IFOV TRACKINGACCEL command to specify the acceleration and deceleration of the servo axes while in IFOV mode. You must specify a positive value for the acceleration.

The IFOV axis pair number must be 0 or 1.

Cause

The IFOV AXISPAIR command specified an axis pair number that was not 0 or 1.

Solution

Use the IFOV AXISPAIR command to specify the horizontal and vertical axis pairs. To specify the horizontal axis pair, use an axis pair number of 0. To specify the vertical axis pair, use an axis pair number of 1.

The commanded motion exceeds the IFOV tracking speed.

Cause

The motion commanded in the AeroBasic program is moving too fast for IFOV.

Solution

Make these changes to your AeroBasic program:

  1. Use the F command to set the coordinated speed to a speed that is less than the maximum permitted IFOV speed.
  2. Use the <AxisName>F<Speed> syntax to set the axis-based speed command, for the two galvo axes, to a speed that is less than the maximum permitted IFOV speed.

For information about the maximum permitted speed in IFOV, refer to IFOV: Calculating the Maximum Permitted Speed.

The IFOV encoder scaling is not valid.

Cause

The encoder scaling that is specified for the IFOV AXISPAIR command is not valid.

Solution

Change your AeroBasic program so that the IFOV AXISPAIR command specifies an <EncoderScaleFactor> argument that is greater than -32,768 and less than 32,787.

One or more of the axes in the move cannot be commanded because IFOV is enabled.

Cause

IFOV is enabled on this task and a move was commanded that contained axes that were not configured for IFOV.

Solution

  • If you do not want to continue using IFOV, issue the IFOV OFF command to disable IFOV.
  • Change your AeroBasic program so that you command only axes that are configured for IFOV.
  • Use the IFOV SYNCAXES command to synchronize more axes that you want to command while in IFOV mode.
  • To command motion independently from IFOV, issue the motion commands on a different task.
The granularity time is not valid.

Cause

An argument that is not valid was passed to an internal IFOV command.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
