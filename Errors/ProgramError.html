An error occurred with a program command.

The sections that follow tell you what might cause the error and give you solutions.

An invalid opcode was encountered.

Cause

An unsupported command was executed.

Solution

Please contact Aerotech Global Technical Support and supply a description of how the problem occurred.

An invalid axis mask was specified.

Cause

An invalid axis mask was detected while executing a command.

Solution

Please contact Aerotech Global Technical Support and supply a description of how the problem occurred.

Cannot execute ARRAY READ FAST during data collection.

Cause

You tried to use the ARRAY READ FAST command while data collection was in progress.

Solution

Wait until data collection is complete before executing the ARRAY READ FAST or use ARRAY READ without the FAST argument.

Cannot perform QCOMMAND operations at trajectory rates greater than 1 kHz.

Cause

You tried to use QCOMMAND READ or QCOMMAND WRITE function and the trajectory rate is greater than 1 kHz.

Solution

Change the MotionUpdateRate parameter to 1 kHz or less.

Cannot perform FIBER operations at trajectory rates greater than 1 kHz.

Cause

You tried to use a FIBER command while the trajectory rate is greater than 1 kHz.

Solution

Change the MotionUpdateRate parameter to 1 kHz or less.

Cannot execute multiple ARRAY READ FAST operations on the same axis.

Cause

The ARRAY READ FAST command was issued on a drive that is already doing the ARRAY READ FAST command from another task.

Solution

Wait for the previous ARRAY READ FAST command to complete before you issue another ARRAY READ FAST command.

FIBER commands require a wait mode of WAIT MODE INPOS or WAIT MODE MOVEDONE.

Cause

A FIBER command was issued on a task and the task is configured for the WAIT MODE AUTO wait mode.

Solution

Change the wait mode of the task by issuing the WAIT MODE MOVEDONE command or WAIT MODE INPOS command. Then issue the FIBER command.

FIBER commands cannot be executed while Cutter Compensation or Corner Rounding is active.

Cause

A FIBER command was issued while cutter compensation or corner rounding was on.

Solution

Turn off cutter compensation and corner rounding before issuing a FIBER command. See the Fiber Optic Overview topic for information on how to align fiber optic devices using FIBER commands.

FIBER commands cannot be executed while Normalcy is active.

Cause

A FIBER command was issued while Normalcy was active.

Solution

Turn off Normalcy before issuing a FIBER command on the same axis. See the Fiber Optic Overview topic for more information on how to align fiber optic devices using FIBER commands.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
