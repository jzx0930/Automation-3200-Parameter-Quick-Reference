FileError

An error occurred while executing a FILE command: <Location>

Table:     Location Codes

Location Code Interpretation

1

A file name that is not valid was specified for the file command.

2

An argument that is not valid was specified for the file command.

3

A file handle that is not valid was specified for the file command.

4

The file is too large.

The sections that follow tell you what might cause the error and give you solutions.

A file name that is not valid was specified for the file command.

Cause

A file name or a file path that is not valid was specified.

Solution

If you do not specify an absolute file path in the file name, the file path is relative to the program that is currently executing. Make sure you can find the specified directory. In this directory, make sure you can find the file that you specified.

An argument that is not valid was specified for the file command.

Cause

  • If the argument was specified for the FILEOPEN function, the value of the <Mode> argument is incorrect.
  • If the argument was specified for the FILEREAD function, the value of the <Mode> argument is incorrect.
  • If the argument was specified for the FILEREADARRAYBINARY function, the value of one or more of these arguments is incorrect.
    • <DataType>
    • <ControllerVariable>
    • <NumVariables>
  • If the argument was specified for the FILEWRITEARRAY command, the value of the <Mode> argument is incorrect.

Solution

  • For the FILEOPEN function, change the value of the <Mode> argument to 0, 1, or 2. To open a serial port, change the value of the <Mode> argument to 2.
  • For the FILEREAD function, change the value of the <Mode> argument to 0, 1, 2, 3, or 4.
  • For the FILEREADARRAYBINARY function, make one or more of these changes.
    • Change the value of the <DataType> argument to a valid Data Type Keyword.
    • Change the value of the <ControllerVariable> argument to specify a valid program, task, or global variable.
    • Change the value of the <NumVariables> argument to a positive integer.
  • For the FILEWRITEARRAY command, change the value of the <Mode> argument to 0 for global doubles or 1 for global string variables.
A file handle that is not valid was specified for the file command.

Cause

A file handle that is not valid was specified.

Solution

Issue the FILEOPEN function to get a valid file handle.

The file is too large.

Cause

The file is too large to do the required operation.

Solution

You must select a smaller file to open.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
