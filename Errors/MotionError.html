MotionError

An error occurred with a motion command: <Location>

Table:     Location Codes

Location Code Interpretation

1

Circular offsets are not allowed when using G0 or RAPID commands.

2

Circular offsets are not allowed when using G1 or LINEAR commands.

3

Circular offsets were not specified to a G2 or G3 command.

4

The Bezier start position is different than the controller position.

5

The BEZIER command requires the ABSOLUTE programming mode.

The sections that follow tell you what might cause the error and give you solutions.

Circular offsets are not allowed when using G0 or RAPID commands.

Cause

You tried to execute RAPID motion but specified circular offsets using the R or IJ keywords as part of the RAPID command.

Solution

Change your program to specify the correct arguments to the command.

Circular offsets are not allowed when using G1 or LINEAR commands.

Cause

You tried to execute LINEAR motion but specified circular offsets using the R or IJ keywords as part of the LINEAR command.

Solution

Change your program to specify the correct arguments to the command.

Circular offsets were not specified to a G2 or G3 command.

Cause

You did not specify circular offsets as arguments to a G2 or a G3 command.

Solution

You must specify the appropriate arguments when commanding a circle. (Refer to the G2 or G3 commands for more information about required arguments.)

The Bezier start position is different than the controller position.

Cause

You issued a BEZIER command and Control Point P0 (the first control point) is not that same as the programmed position of the controller.

Solution

Change your program using one of the following procedures.

  • Use the POSOFFSET command before the BEZIER command to make the programmed position of the controller the same as <P0_H> and <P0_V> values.
  • Use the LINEAR or the RAPID command before the BEZIER command to move the horizontal and vertical axes to the correct <P0 H> and <P0_V> value.
  • Change the control points of the BEZIER command so that the starting position is the same as the programmed position of the controller.
The BEZIER command requires the ABSOLUTE programming mode.

Cause

You issued a BEZIER command and the task is in the INCREMENTAL programming mode. The task must be in the ABSOLUTE programming mode.

Solution

The BEZIER command only supports absolute coordinates. Change the programming mode of the task to absolute by issuing the ABSOLUTE command in your AeroBasic program.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
