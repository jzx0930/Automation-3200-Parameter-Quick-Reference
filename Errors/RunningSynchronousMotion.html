RunningSynchronousMotion

The synchronous motion command was aborted.

Cause

The controller was executing a synchronous motion command, such as a LINEAR, CW, CCW, or BEZIER command, and one or more axes in the command were aborted. During synchronous motion, all axes in the motion command must complete the move for the motion command to complete. When this error occurs, the motion command does not complete and the task stops.

For more information about synchronous motion commands, refer to Motion Overview.

Solutions

  • Examine your program to make sure that a different task did not execute the ABORT command on an axis in the synchronous move.
  • If you stopped a task, a task error occurred, or the controller generated an axis fault, you might have configured specific task and fault parameters so that the controller aborted the axes. Refer to the following procedure.
    • To Reconfigure the Task and Fault Parameters
    1. Examine the configurations for the TaskStopAbortAxes, TaskErrorAbortAxes, and FaultAbortAxes parameters.
    2. Make all necessary changes to prevent the configurations of these parameters from causing the controller to abort the axes.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
