The sections that follow tell you what might cause the error and give you solutions.

An invalid task number was specified.

Cause

You specified an invalid task number for the argument of a command.

Solution

Valid task numbers are from 0 to 31. Change the command argument to a number that is in the permitted range.

An invalid pointer type was specified.

Cause

An invalid pointer type was used with a library function.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

A variable array index is out of range.

Cause

You specified an index value that is not in the permitted limits for an array variable.

Solution

Specify an index value that is in the permitted limits of the array variable.

An invalid pointer offset was specified.

Cause

An invalid pointer offset was specified when getting or setting a variable.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid pointer type for extra data was specified.

Cause

An error occurred when the system tried to read a program variable.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid parameter was specified.

Cause

The controller cannot identify a specified parameter.

Solution

Make sure that the specified parameter is in Configuration Manager.

An invalid program queue mode command was specified.

Cause

You specified a RETURN or CALL command in a program that is in queue mode.

Solution

Remove RETURN and CALL commands from the program that is in queue mode.

The specified program was not found on the SMC.

Cause

The specified program is not on the controller.

Solution

Add the specified program to the controller. Refer to Program Automation for information about how to add programs to the controller.

An invalid number of program lines was specified.

Cause

The number of program lines in a program cannot be counted.

Solution

Compile and run the program again.

An invalid program queue size was specified.

Cause

An invalid value was specified for the QueueBufferSize Parameter.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid program line number was specified.

Cause

The specified program line number is not valid.

Solution

Compile and run the program again.

An invalid program label was specified.

Cause

You used an incorrect program label.

Solution

  1. Make sure that you use correct syntax for the program label. Refer to Labels for more information about label syntax.

  2. Compile and run the program again.

An invalid callstack parameter was specified.

Cause

You specified an invalid callstack parameter.

Solution

  1. Refer to ParameterNotPassed for more information about callstack argument errors.

  2. Compile and run the program again.

An invalid axis was specified.

Cause

You specified an axis name that is not in the active parameter file.

Solution

To make sure that you use a correct axis name, do the following:

  • Use only axis names that are in the active parameter file.
  • Make sure that you typed the axis name correctly.
An invalid spindle number was specified.

Cause

You specified a spindle number that is not a valid spindle.

Solution

Make sure that the spindle number is 0, 1, 2, or 3. Refer to Spindle Overview for more information about spindles.

An invalid spindle operation was detected.

Cause

The controller tried to execute an invalid spindle command.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid set of rotation axes was specified.

Cause

In a G84 command, you used the same rotation axes more than one time.

Solution

Make sure that all of the axes in the G84 command are different.

 

Cause

In the G84 command, you specified an axis incorrectly.

Solution

Make sure that you specify the axis correctly in the G84 command.

An invalid set of RTheta axes was specified.

Cause

You specified one or more incorrect axes for a G46 or G47 command.

Solution

Make sure that you specify the correct axes for the G46 or G47 command.

An invalid axis cutter was specified.

Cause

You specified more than one of the same axes in the cutter compensation or corner rounding commands.

Solution

Make sure that all of the axes in the cutter compensation or corner rounding commands are different. For more information about how to use these features, refer to the Corner Rounding Overview, Cutter Offset Compensation Overview, or Cutter Radius Compensation Overview.

The offsets (IJK) in the circular move are not in the correct coordinate plane.

Cause

You specified an invalid argument for the G17, G18, or G19 command.

Solution

Make sure that you use the correct coordinate plane (G17, G18, and G19) for the circular move.

The axes in the circular move are not in the correct coordinate plane.

Cause

The axis in a G2 or G3 command does not use the same coordinate plane as the move.

Solution

Make sure that you use the correct coordinate plane (G17, G18, and G19) for the circular move.

An invalid normalcy axis was specified.

Cause

You specified an invalid axis in a normalcy command.

Solution

Make sure that you specify the correct normalcy axes in the NORMALCY AXES Command.

An invalid handwheel mode was specified.

Cause

You specified an invalid argument value to the HANDWHEEL command.

Solution

Specify 0 to disable the handwheel or 1 to enable the handwheel. Refer to HANDWHEEL Command for more information.

An invalid return type was specified.

Cause

You specified an incorrect <ReturnType> argument for a RETURN command.

Solution

Use a valid <ReturnType> argument with the RETURN command. To see the valid <ReturnType> arguments, refer to the Returning from an ONGOSUB Command section of RETURN Command.

An invalid priority level was specified.

Cause

You specified an invalid value for an ONGOSUB priority argument.

Solution

Use one of the priority argument values (ONGOSUB Command) that follow:

  • PRIORITY_MEDIUM
  • PRIORITY_FAULT
  • PRIORITY_HIGHEST
An invalid corner rounding argument was specified.

Cause

You specified an invalid value for the ROUNDING TOLERANCE command.

Solution

Specify a valid value for the ROUNDING TOLERANCE Command.

An invalid axis group was specified.

Cause

An error occurred because of one or all of the causes that follow:

  • You specified an axis group value that is not in the range of 0 to 7.
  • You did not increase the axis group value by one for more than one axis group.

Solution

Use one or all of the solutions that follow:

  • Specify a value from 0 to 7 for the axis group number.
  • If you are using more than one axis group, start with a value of 0 for the first axis group. Then increase the axis group number by one for each axis group that follows. Refer to JOYSTICK Command for more information.
The value that was specified is out of range.

Cause

You specified a value that is too small or too large for a command.

Solution

Make sure that the value of the command is in the permitted limits for that command. Refer to the help topic for the command to see the permitted range of values for arguments.

The specified value must be an integer.

Cause

You specified a non-integer value where an integer value is necessary.

Solution

Change non-integer values to integer values where integer values are necessary.

A non-integer array index was specified.

Cause

You specified a non-integer value for an array index.

Solution

Change non-integer values to integer values for each array index.

An invalid internal move type was specified.

Cause

The controller incorrectly identified a move type.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An AeroBasic command argument was out of range.

Cause

You specified a value that is too small or too large for an AeroBasic command.

Solution

Make sure that the value of the AeroBasic command is in the permitted limits for that command. Refer to the help topic for the command to see the permitted range of values for arguments.

The variable index is out of range.

Cause

You specified an index value that is not in the permitted limits for a global variable.

Solution

Specify an index value that is in the permitted limits of the global variable.

The specified value must be positive.

Cause

You specified a negative number for a speed, time, or rate value.

Solution

Change negative numbers to positive numbers for speed, time, and rate values.

A required argument was not passed to an AeroBasic command.

Cause

You used an AeroBasic command that did not include a necessary argument.

Solution

Refer to the help topic for the command to see the arguments that the command must have.

An invalid value was specified for a PSO command.

Cause

You specified an invalid argument for a PSO command.

Solution

Refer to the help topic for the PSO command that you want to use. Then make sure that you use permitted arguments for that command. To see a list of available PSO commands, refer to PSO Command Category.

An invalid FireWire packet index was specified.

Cause

A communication packet has an invalid index.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid data item was passed to the AXISSTATUS, TASKSTATUS, or SYSTEMSTATUS function.

Cause

You used an incorrect data item with the AXISSTATUS, TASKSTATUS, or SYSTEMSTATUS function.

Solution

Refer to AXISSTATUS Function, TASKSTATUS Function, and SYSTEMSTATUS Function to see the data items that you can use with these functions.

An invalid timer index was specified.

Cause

You specified a timer value that is not in the range of 0 to 31.

Solution

Change the timer value to a number from 0 to 31.

An invalid CountsPerUnit or SecondaryUnitsScaleFactor was specified.

Cause

You specified a value of 0 for a CountsPerUnit or SecondaryUnitsScaleFactor parameter.

Solution

Change the value of the CountsPerUnit Parameter or SecondaryUnitsScaleFactor Parameter to a number that is greater than 0.

The specified range is invalid. Maximum must be greater than minimum.

Cause

The <HighValue> argument that you specified for the PSOWINDOW RANGE command is not greater than the <LowValue> argument.

Solution

For the PSOWINDOW RANGE Command, change the <HighValue> argument to a number that is greater than the <LowValue> argument.

An invalid operation was tried.

Cause

You used an invalid type of drive motion.

Solution

Make sure that you complete motion with valid AeroBasic syntax.

An invalid type cast was specified.

Cause

You specified an invalid type cast when getting or setting a Fieldbus I/O variable.

Solution

Refer to Using Fieldbus I/O in AeroBasic for information about Fieldbus I/O.

The operation is not supported on the specified axis.

Cause

You tried to use the PSOOUTPUT PULSE LASER MASK command on a drive that is not a GL4 or an Nmark CLS, GCL, or SSaM.

Solution

Do laser masking on a GL4 or an Nmark CLS, GCL, or SSaM. Refer to PSOOUTPUT PULSE LASER MASK Command for more information.

 

Cause

You tried to use the PSOOUTPUT PULSE LASER MASK command on an axis that is not a virtual axis.

Solution

Do laser masking on a virtual axis. Refer to PSOOUTPUT PULSE LASER MASK Command for more information.

The specified index must be an integer that ranges from 0 to 31.

Cause

You used an index value that is not in the range of 0 to 31.

Solution

Valid index values are from 0 to 31. Change the invalid index values to numbers that are in the permitted range.

The size of the specified mask must be 1.

Cause

An axis mask that was configured for a single axis is being used with multiple axes.

Solution

Reset the controller. To do this, open Configuration Manager. On the Controller menu, select Reset Controller. Then run the program again.

An invalid data item was passed to the AXISSTATUSFAST or TASKSTATUSFAST command.

Cause

You specified an incorrect data item for the AXISSTATUS or TASKSTATUS function.

Solution

Refer to AXISSTATUS Function and TASKSTATUS Function to see the data items that you can use with these functions.

An axis was specified more than one time.

Cause

You included the same axis more than one time in an argument to a command.

Solution

Include an axis only one time in a list of axes that are part of an argument to a command. Remove an axis if it is included more than one time.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
