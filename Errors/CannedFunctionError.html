CannedFunctionError

An error occurred with a canned function: <Location>

Table:     Location Codes

Location Code Interpretation

1

The canned function ID is invalid.

2

The canned function table is full.

3

A canned function is already active.

4

A canned function is already executing.

6

The specified command cannot be executed in a canned function.

The sections that follow tell you what might cause the error and give you solutions.

The canned function ID is invalid.

Cause

You tried to execute a canned function that does not exist or has not been set.

Solution

Use the SETCANNEDFUNCTION command to set the canned function before trying to execute the canned function.

The canned function table is full.

Cause

You tried to set more canned functions than are currently available.

Solution

Increase the value of the CannedFunctions parameter to the number of canned functions that you require.

A canned function is already active.

Cause

You tried to execute a canned function on a task that is already executing a canned function.

Solution

Only one canned function can execute on a task at a time. Change your program to wait for the canned function to complete before executing the next canned function, or execute the next canned function on another task.

A canned function is already executing.

Cause

You tried to execute a canned function on a task that is already executing a canned function.

Solution

Only one canned function can execute on a task at a time. Change your program to wait for the canned function to complete before executing the next canned function, or execute the next canned function on another task.

A canned function is already set.

Cause

You tried to execute a canned function on a task that is already executing a canned function.

Solution

Only one canned function can execute on a task at a time. Change your program to wait for the canned function to complete before executing the next canned function, or execute the next canned function on another task.

The specified command cannot be executed in a canned function.

Cause

You issued a command in a canned function subroutine, but that command cannot be executed in a canned function.

Solution

Do one of these:

  • Issue the command outside of the canned function subroutine.
  • Issue the command on a different task.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
