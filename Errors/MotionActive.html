The sections that follow tell you what might cause the error and give you solutions.

This action cannot be performed while the axis is profiling.

Cause

You tried to execute motion on an axis that is either currently executing a coordinated move or is in Infinite Field of View (IFOV).

Solution

  • If you are not using IFOV, change your programs so that they command axes that are being moved by different tasks.
  • If you are using IFOV, issue the IFOV OFF command to disable IFOV.
This action cannot be performed while the axis is homing.

Cause

You tried to execute motion on an axis that is currently executing a home.

Solution

Wait for the home to complete before executing a motion command. The HOME command will not return until the home cycle is complete; however, you could get this error if you try to command the axis from another task while the axis is executing a home. The HOMEASYNC command returns immediately. If you are using this command, then you can either change your program to use the HOME command or wait for the home to complete by using the WAIT MODE MOVEDONE  command.

This action cannot be performed while the axis is executing drive motion.

Cause

You tried to execute motion on an axis that is currently executing drive-based motion. Drive-based motion occurs when the drive is controlling the motion of the axis instead of the SMC. This occurs in the following cases.

  • An axis is enabling
  • An axis is aborting due to a fault or the ABORT command
  • An axis is moving out of a limit
  • Autofocus is active on an axis
  • Stepper verification is occurring on an axis
  • An axis is executing an OSCILLATE command
  • An axis is executing an MSET command

Solution

Avoid issuing a motion command on an axis while it is executing drive-based motion.

This action cannot be performed while the axis is executing asynchronous motion.

Cause

You tried to execute motion on an axis that is currently executing an asynchronous move such as a MOVEINC, MOVEABS, or FREERUN.

Solution

Wait for the asynchronous move to complete before commanding a different type of motion on that axis. You can use the WAIT MODE MOVEDONE or WAIT MODE INPOS command to wait for the move to complete.

This action cannot be performed while the axis is moving.

Cause

You tried to execute motion on an axis that is either currently executing a motion command or is in Infinite Field of View (IFOV).

Solution

  • If you are not using IFOV, wait for the motion to complete before executing another motion command. You can use the WAIT MODE MOVEDONE or WAIT MODE INPOS command to wait for the motion to complete.
  • If you are using IFOV, issue the IFOV OFF command to disable IFOV. Then try to issue the motion command again.
This action cannot be performed while the axis is synced to another axis.

Cause

You tried to do motion on an axis that is a slave to another axis.

Solution

Only do motion on the master axis when using gearing, camming, or a gantry.

This action cannot be performed while the spindle is active.

Cause

You tried to change the axis of a spindle while the spindle was active.

Solution

Each task has up to four spindles available. You can either use a different spindle number for each axis, or you must make sure the spindle is not active (in motion) when you change the spindle axis.

This action cannot be performed while parts rotation is active.

Cause

You tried to change the axis of a rotation while the rotation was active.

Solution

Disable the rotation before changing any axes that are part of the rotation.

This action cannot be performed while a polar or cylindrical transformation is active.

Cause

You tried to change the axis or radius of a polar or cylindrical transformation while the transformation was active. 

Solution

Disable the transformation before changing the settings of the transformation.

This action cannot be performed while cutter compensation is active.

Cause

You tried to change the axes or settings of cutter compensation while cutter compensation was active.

Solution

Disable cutter compensation before changing the settings of cutter compensation.

This action cannot be performed while normalcy is active.

Cause

You tried to change the axes or settings of normalcy while normalcy was active.

Solution

Disable normalcy before changing the settings of normalcy.

The same axis was specified in multiple motion commands.

Cause

You tried to command the same axis using multiple motion commands. This can occur if you are commanding a helix or dual circular motion, and you specify the same axis in more than one of the motion commands.

Solution

Change your program so that the same axis is not specified in more than one motion command at a time.

This action cannot be performed while corner rounding is active.

Cause

You tried to change the corner rounding or cutter compensation axes with the G44 command while corner rounding was active.

Solution

Change your program so that it does not issue the G44 command while corner rounding is active.

This action cannot be performed while the axis is feedheld.

Cause

You tried to execute motion on an axis that is currently being feedheld by a task.

Solution

Change the feedhold state of the task by releasing feedhold with the FEEDHOLD OFF command.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
