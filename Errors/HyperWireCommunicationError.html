HyperWireCommunicationError

An error occurred during HyperWire communication: <Location>

Table:     Location Codes

Location Code Interpretation

1

The controller tried to send an asynchronous command to an invalid node.

2

The controller tried to send an asynchronous command during an invalid communication state.

3

The drive cannot identify an asynchronous command.

4

The communication channel of the drive is out of bounds.

5

You cannot make an isochronous connection while you configure drive communication channels.

6

The controller tried to send an asynchronous communication packet that is too large.

7

The controller did not receive an asynchronous response packet.

8

The controller received an asynchronous response packet that is too large.

9

10

11

12

The sections that follow tell you what might cause the error and give you solutions.

The controller tried to send an asynchronous command to an invalid node.

Cause

The controller tried to send a blinking or channel-set command to an invalid node number.

Solution

Make sure that all of the HyperWire network cables are connected correctly and all of the drives are powered on. Reset the controller. Then try to do the task again.

The controller tried to send an asynchronous command during an invalid communication state.

Cause

The controller tried to send an asynchronous command while the controller is in isochronous mode.

Solution

Make sure that all of the HyperWire network cables are connected correctly and all of the drives are powered on. Reset the controller. Then try to do the task again.

The drive cannot identify an asynchronous command.

Cause

The controller sent an asynchronous command that the drive cannot identify.

Solution

Run Firmware Loader on all of the drives on the HyperWire network. Then make sure that all of the drives have the same firmware version and that version matches the software version of the controller.

The communication channel of the drive is out of bounds.

Cause

You tried to assign a drive communication channel that is out of bounds.

Solution

After you assign a communication channel to a multi-axis drive, make sure that a sufficient number of communication channels are available to keep all other axes of the drive in bounds. The applicable communication channels are 0 - 31. Refer to Assigning Drive Communication Channels for more information.

You cannot make an isochronous connection while you configure drive communication channels.

Cause

You tried to connect to the drive network through a different application while the Drive Configuration dialog is open.

Solution

When you assign drive communication channels, close all other programs and applications to prevent them from trying to connect to the drive network.

The controller tried to send an asynchronous communication packet that is too large.

Cause

The controller made an asynchronous communication packet that is too large to send.

Solution

Make sure that all of the HyperWire network cables are connected correctly and all of the drives are powered on. Reset the controller. Then try to do the task again.

The controller did not receive an asynchronous response packet.

Cause

An asynchronous bus-cycle retry occurred on the HyperWire network.

Solution

Make sure that all of the HyperWire network cables are connected correctly and all of the drives are powered on. Reset the controller. Then try to do the task again.

The controller received an asynchronous response packet that is too large.

Cause

A drive sent an asynchronous response packet to the controller, but the packet is too large to process.

Solution

Run Firmware Loader on all of the drives on the HyperWire network. Then make sure that all of the drives have the same firmware version and that version matches the software version of the controller.

A timeout occurred while waiting for the network reset to occur.

Cause

The HyperWire network reset did not complete in a sufficient quantity of time.

Solution

Make sure that all of the HyperWire network cables are connected correctly. Power cycle all of the drives on the HyperWire network. Then reset the controller.

The HyperWire card detected that it is not the network master.

Cause

The HyperWire card incorrectly identified itself during bus initialization.

Solution

Make sure that all of the HyperWire network cables are connected correctly. Power cycle all of the drives on the HyperWire network. Then reset the controller.

The controller received an asynchronous response packet that is empty.

Cause

A drive sent an asynchronous response packet that is empty.

Solution

Make sure that all of the HyperWire network cables are connected correctly. Power cycle all of the drives on the HyperWire network. Then reset the controller.

The controller did not receive an enumeration response packet.

Cause

The controller could not detect the full number of drives on the HyperWire network.

Solution

Make sure that all of the HyperWire network cables are connected correctly. Power cycle all of the drives on the HyperWire network. Then reset the controller. If this error continues to occur, contact Aerotech Global Technical Support.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
