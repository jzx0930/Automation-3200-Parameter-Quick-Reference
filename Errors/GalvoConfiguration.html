Table:     Location Codes

Location Code Interpretation

4

The sections that follow tell you what might cause the error and give you solutions.

A valid galvo axis must be specified.

Cause

The FireWire cable, HyperWire cable, or one or more drive cables are not connected correctly.

Solution

Make sure that the FireWire or HyperWire cable is connected and that all drives are connected and powered on.

 

Cause

You issued a command that requires a GL4 or an Nmark axis, but the axis that you specified is not a galvo axis.

Solution

Make sure that you specify the galvo axis of a GL4 or an Nmark CLS, GCL, or SSaM as part of the AeroBasic command.

 

Cause

In Configuration Manager, you defined a 2D galvo calibration file on an axis that is not a galvo axis.

Solution

Make sure that you define your 2D galvo calibration file on a galvo axis.

The specified GALVO command is not supported.

Cause

An internal error occurred.

Solution

Record the text of the error message and contact Aerotech Global Technical Support.

A negative value for GALVO LASERONDELAY can be used only at trajectory rates of 1 kHz or greater.

Cause

The value of the MotionUpdateRate parameter is less than 1 kHz and you specified a negative value for the GALVO LASERONDELAY command.

Solution

Increase the value of the MotionUpdateRate parameter to 1 kHz or greater.

This action cannot be done while the controller uses marking on the fly functionality.

Cause

"Marking on the fly" functionality is enabled and you issued a HOME, SETPOSCMD, SETPOSFBK, or IFOV ON command.

Solutions

  • Deactivate the "marking on the fly" functionality by issuing the GALVO ENCODERSCALEFACTOR command with a value of 0. Then issue a HOME, SETPOSCMD, SETPOSFBK, or IFOV ON command.
  • Before you issue the GALVO ENCODERSCALEFACTOR command, issue a HOME, SETPOSCMD, SETPOSFBK, or IFOV ON command.
You must issue a GALVO WOBBLE CONFIG before you issue a GALVO WOBBLE ON.

Cause

You issued the GALVO WOBBLE ON command but did not configure the galvo wobble feature by using the GALVO WOBBLE CONFIG command.

Solution

Use the GALVO WOBBLE CONFIG command to configure the galvo wobble feature. Then issue the GALVO WOBBLE ON command.

The specified value for GALVO LASERONDELAY can be used only at trajectory rates of 2 kHz or greater.

Cause

You issued a LINEAR, CW, CCW, or BEZIER move at a trajectory rate of 1 kHz in the automatic laser mode, but the value that you specified for the GALVO LASERONDELAY command is less than -975 microseconds.

Solution

When the laser is automatically controlled at 1 kHz, the value that you specify for the GALVO LASERONDELAY command must be -975 microseconds or greater. To use the larger range of delay, increase the value of the MotionUpdateRate parameter to 2 kHz or greater. For more information about the automatic laser mode, refer to the GALVO LASEROVERRIDE AUTO command.

The specified value for GALVO LASERONDELAY cannot be used when IFOV is enabled.

Cause

You issued a LINEAR, CW, CCW, or BEZIER move in the automatic laser mode while IFOV was enabled, but the value that you specified for the GALVO LASERONDELAY command is less than -975 microseconds.

Solution

If you are using IFOV, increase the value that you specify for the GALVO LASERONDELAY command to -975 microseconds or greater.

The specified value for GALVO LASERONDELAY requires you to use the automatic laser mode.

Cause

The value that you specified for the GALVO LASERONDELAY command is less than -975 microseconds and you are controlling the laser in the manual laser mode.

Solutions

  • To specify values that are less than -975 microseconds to the GALVO LASERONDELAY command, change the laser mode to the automatic laser mode by issuing the GALVO LASEROVERRIDE AUTO command.
  • To continue manually controlling the laser, increase the value that you specify for the GALVO LASERONDELAY command to -975 microseconds or greater. For more information about the manual laser mode, refer to the GALVO LASERONDELAY and the GALVO LASEROFFDELAY commands.
This feature is available only if you are using a GL4 or an Nmark GCL.

Cause

The FireWire cable, HyperWire cable, or one or more drive cables are not connected correctly.

Solution

Make sure that the FireWire or HyperWire cable is connected and that all drives are connected and powered on.

 

Cause

You issued a command that requires a GL4 or an Nmark GCL, but the axis that you specified is not a GL4 or Nmark GCL axis.

Solution

Make sure that you specify a GL4 axis or an Nmark GCL axis as part of the AeroBasic command.

If you cannot find the cause of this error, or if you tried the recommended solution and receive the error again, record the text of the error message and contact Aerotech Global Technical Support.
