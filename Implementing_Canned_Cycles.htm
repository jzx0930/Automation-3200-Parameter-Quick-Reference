Implementing Canned Cycles

Canned Cycles are a special type of canned function, which can be used to do operations such as drilling and boring. Unlike other canned functions, canned cycles are not explicitly called with the EXECCANNEDFUNCTION command, but rather they are automatically executed at the end of every LINEAR move. There can only be one canned cycle active at any time.

Use the Program Automation folder in Configuration Manager or Motion Composer to set up the program containing the subroutine. Set up the program so that it is downloaded to the SMC upon reset. Then you can use the SETCANNEDFUNCTION command to register the subroutine as a canned cycle. Refer to the example that follows for a more detailed explanation of this process.

Example: Implementing Canned Cycles

Copy this code into a program titled Drill.pgm.

' This line prevents a compiler warning.
END PROGRAM
				
DFS DRILL
    ' Activate output to lower the drill.
    $DO[0].0
ENDDFS

Verify the program for correct operation. Set up Drill.pgm as Download Only in the Program Automation folder in Configuration Manager or Motion Composer.

Execute the following command to register the canned cycle by putting the command line in a program and adding it to the Program Automation Page:

SETCANNEDFUNCTION CANNEDFUNCTION_ID_ENDOFBLOCK, "DRILL.PGM", "DRILL"

The DRILL subroutine will automatically be called at the end of every LINEAR motion command until the canned function is disabled.
