Optimizing Performance in VELOCITY ON Mode

In VELOCITY ON mode, cycle time is minimized because the controller does not decelerate to a full stop between consecutive moves. But, VELOCITY ON mode can cause high axis accelerations that can negatively affect servo loop performance and overall part quality. VELOCITY ON mode can also increase the workload of the controller. If the controller becomes overloaded, part quality might suffer. To create a balance between part quality, cycle time, and the workload of the controller, you must consider the following.

Improve Overall A3200 Performance

To prevent overloading the controller, refer to Optimizing A3200 Performance.

Improve Mechanical Structure

Large acceleration rates can excite structural resonances and degrade part quality. Make sure that your mechanical structure is rigid. If your mechanical structure is not rigid, improve the structure until it becomes rigid.

Tune the Servo Loop

Tune the servo loop gains of the axes for optimal servo loop tracking. Refer to Loop Transmission.

To automatically tune and optimize an axis in a new or existing system, you can use the EasyTune feature in Digital Scope. When EasyTune is finished, the optimized servo gains and servo filters are committed to the controller. For other tuning options, refer to Autotuning an Axis and Using Calculators to Set Parameters.

Select an Optimal Coordinated Feedrate and Ramp Rate

When you increase the coordinated feedrate and coordinated ramp rate, cycle time improves, but causes higher axis accelerations that can lead to defective part quality. Also, if you set the coordinated feedrate and coordinated ramp rate too high, and there are too many moves to process, you can overload the controller.

Use Automatic Feedrate Reduction Features

The controller supplies features that automatically decrease the feedrate when the controller finds situations that cause high axis accelerations. The controller lets you set a high coordinated feedrate, and prevents high accelerations by automatically decreasing the feedrate when necessary.

Use the CoordinatedAccelLimit parameter to prevent high axis accelerations caused by changes in direction between consecutive moves.

Use the CoordinatedCircularAccelLimit parameter to prevent high axis accelerations around circular arcs.

Use the DependentCoordinatedAccelLimit parameter to prevent high axis accelerations on dependent axes.

Use Corner Rounding (available only with CNC Option)

The CoordinatedAccelLimit parameter prevents high axis accelerations by decelerating between consecutive, non-tangent moves. But, decelerating between each move can increase cycle time. To prevent an increase in cycle time, you can remove non-tangencies between consecutive moves by using corner rounding. Corner rounding causes the controller to automatically insert a circular arc between consecutive, non-tangent moves.

Select an Optimal Motion Update Rate

To improve the quality of your part without decreasing the feedrate, increase the value of the MotionUpdateRate parameter. For systems that can achieve high acceleration rates, the MotionUpdateRate parameter lets the controller more accurately follow the coordinated path. But, if you make the value of the MotionUpdateRate parameter too large, the larger value might cause the workload of the controller to increase.

Prevent Short Duration Moves

Moves that have a duration less than the reciprocal of the MotionUpdateRate parameter are not strictly followed by the controller (refer to the following figure) and can increase the workload of the controller. Use the CoordinatedMoveDurationMinimum parameter to automatically decrease the feedrate for these moves so that the duration is longer. The CoordinatedMoveDurationMinimum parameter increases cycle time.
