Resolving FlashConfig Mismatches

When you change which FlashConfig drives and stages are connected to your controller, you must change the FlashConfig settings in Configuration Manager. Each time the controller is powered on or is reset, it senses the following conditions for each axis.

  • A FlashConfig stage was added.
  • A FlashConfig stage was connected to a different drive.
  • A FlashConfig stage is no longer connected.

These conditions cause a mismatch between the serial number of the stage and the value of the RequiredStageSerialNumber Parameter. You cannot enable the axis until you open the FlashConfig dialog box in Configuration Manager and resolve the mismatch condition. Until you resolve the mismatch condition, when you try to enable the axis you get a RequiredStageSerialMismatch error.

To open the FlashConfig dialog box

  1. Open Configuration Manager and connect to a controller.
  2. In the Network Explorer, select the controller.
  3. On the toolbar, click the FlashConfig button .

The dialog box that shows is different if you connected new FlashConfig stages or if you changed or disconnected FlashConfig stages.

New FlashConfig Stages Detected

Figure: New FlashConfig Stages Detected

When you connect a new FlashConfig stage to your controller and click the FlashConfig button in Configuration Manager, you see the New FlashConfig Stages Detected dialog box. The axes with new stages are identified with a warning icon in the grid at the top of the dialog box. You have the following options.

  • Transfer the configuration data from the new stages to the controller. For each axis, the configuration data on the controller is overwritten by the FlashConfig data from the stage.
    • To transfer FlashConfig data
    1. If FlashConfig contains valid calibration data, review the changes that you want to make to Existing Calibration.
    2. In the New FlashConfig Stages Detected dialog box, click Transfer >.
      3. Figure: New FlashConfig Stages — Transfer

    3. Review the types of data that are kept in FlashConfig for each new stage.
      • Valid When you click Transfer, all of the data that is identified as Valid transfers from the stage and overwrites the data on the controller.
      • No data The stage does not contain that type of data. When you click Transfer, the data on the controller stays the same for that data type on that axis.
    4. Click Transfer. The data transfers from the stage to the controller.
    5. If you are prompted to commit parameters or reset the controller, click Yes. After the controller resets, the transfer is complete. You can enable and use the axis.

    You can change the parameters that were transferred from the stage to the controller by editing the parameters in Configuration Manager or by tuning the axis in Digital Scope.

  • Ignore the configuration data from the new stages. For each axis, the configuration data on the controller stays the same and no data is transferred from the stages. You can always restore the controller with the data used at the factory to initially configure the axis. Refer to Restoring FlashConfig Data
    • To Ignore FlashConfig data
    1. In the New FlashConfig Stages Detected dialog box, click Ignore.
    2. If you are prompted to commit parameters or reset the controller, click Yes. No data is transferred. After the controller resets, you can enable and use the axis.

Changed or Disconnected FlashConfig Stages

Figure: Stage Serial Number Mismatch

If you connect a different stage to a drive or if you disconnect a stage, when you click the FlashConfig button in Configuration Manager you see the Stage Serial Number Mismatch dialog box. The axes that are not connected as expected and axes that are missing are identified with a warning icon in the grid at the top of the dialog box. Use the information supplied in the grid to correct the connection of these FlashConfig stages to your controller.

Warning: If you purchased integration for your system, always use the information in the interconnect drawing supplied by Aerotech to connect the stages to the drives.

If you replaced a FlashConfig stage with an identical stage, or if you want to stop using a FlashConfig stage, you might want to use the stages as they are connected. Before you can enable these axes, you must resolve the mismatch condition. You have the following options.

  • Transfer the configuration data from the stage to the controller for the stages that are connected to different drives. For each axis, the configuration data that is on the controller is overwritten by the FlashConfig data from the stage.
    • To transfer FlashConfig data
    1. In the Stage Serial Number Mismatch dialog box, click More Options.
    2. If FlashConfig contains valid calibration data, review the changes that you want to make to Existing Calibration.
    3. Click Transfer >.
    4. Figure: Unexpected Stages — Transfer

    5. Review the types of data that are kept in FlashConfig for each mismatched axis.
      • Valid When you click Transfer, all of the data that is identified as Valid transfers from the stage and overwrites the data on the controller.
      • No data The stage does not contain that type of data. When you click Transfer, the data on the controller stays the same for that data type on that axis.
    6. Click Transfer. The data transfers from the stage to the controller.
    7. If you are prompted to commit parameters or reset the controller, click Yes. After the controller resets, the transfer is complete. You can enable and use the axis.

    You can change the parameters that were transferred from the stage to the controller by editing the parameters in Configuration Manager or by tuning the axis in Digital Scope.

    If you disconnect a FlashConfig stage from the controller and do not connect another stage for that axis, when you select the transfer option, the data types for that axis are identified as No data. The Transfer option has the same result for that axis as if you selected the Ignore option.

  • Ignore the configuration data from the stages. For each axis, the configuration data that is on the controller stays the same and no data is transferred from the stages. You can always restore the controller with the data used at the factory to initially configure the axis. Refer to Restoring FlashConfig Data.
    • To Ignore FlashConfig data
    1. In the Stage Serial Number Mismatch dialog box, click More Options.
    2. Click Ignore.
    3. If you are prompted to commit parameters or reset the controller, click Yes. No data is transferred. After the controller resets, you can enable and use the axis.
