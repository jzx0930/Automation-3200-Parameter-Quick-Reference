Argument Lists and Format Specifiers

Syntax:

 
[<FormatSpecifierString>,] <ArgumentList>

Example:

 
' Shows the time and Global Variable 0.
MSGBOX 1, "{#F3}", "Time is #TM. Global Variable 0 is", $global[0]

The MSGDISPLAY, MSGMENU, FILEWRITE, and MAKESTRING commands accept a string as an argument. This string can have one or more string constants that are contained in double quotes (for example, "text") and one or more variables. Before this string, you can put an optional format specifier that is contained in double quotes and braces (for example, "{format specifier}"). The format specifier controls how the controller converts variables and expressions into text in the string. If you do not use a format specifier, the controller shows numeric data with six decimal places by default.

The Format Character and Multi-Character Specifiers table lists valid format character and multi-character specifiers that you can use in a format specifier. A format character or multi-character specifier corresponds to one variable or expression. If the number of variables or expressions is more than the number of format character or multi-character specifiers, then the format specifier repeats from the start until it formats all of the numeric arguments in the <ArgumentList> argument. If you specify too many arguments in the <ArgumentList> argument, an error occurs in the AeroBasic compiler that tells you to decrease the number of arguments in the list.

The following table lists format character and multi-character specifiers that you can include in a format specifier. These characters control how the controller shows variables and expressions in the string.

Table:     Format Character and Multi-Character Specifiers

String Sequence

Description

#C

ASCII character.

#D

Signed 32-bit integer, -231 ≤ X < 231.

#Fn

Floating-point. (Optional. You can use this character to specify the number of decimal places. For example, the default value for n is 6 and the maximum value for n is 15.)

#H

Hexadecimal number.

#U

Unsigned 32-bit integer, 0 ≤ X < 232.

%

Indicates that an ANSI C printf code follows.

Do not use % in a format specifier list that also includes the # specifiers.

#SA

Converts axis faults into ASCII string code.

#ST

Converts an error code into an ASCII string code.

The Special Text table lists special text sequences that you can use in a string constant in the <ArgumentList>. When you put special text sequences into a string constant argument to the MSGDISPLAY, MSGMENU, FILEWRITE, or MAKESTRING command, the controller replaces the special text sequence with the special text. Do not put special text sequences into the format specifier string.

Table:     Special Text

String Sequence

Description

#DT

Current date. (You can configure the format of the date string in Control Panel → Region and Language.)

#PGn

Program (no path) running on task n ("" if no program running; n is 1-based).

#PMn

Program (with path) running on task n ("" if no program running; n is 1-based).

#PNn

Program line number running on task n (0 if no program running; n is 1-based).

#TM

Time without seconds. (You can configure the format of the time string in Control Panel → Region and Language.)

#TS

Time with seconds. (You can configure the format of the time string in Control Panel → Region and Language.)

For examples of format specifiers and special text, refer to MSGDISPLAY Command.
