Importing Custom Items into the Catalog Manager

You can import custom items from parameter files or catalog files.

Parameter Files If a parameter file contains custom stages or motors that you want to make changes to or use again, then you must import the custom items from the parameter file.

Catalog Files If you want to use one or more of the custom stages or motors from an exported catalog, then you must import the custom items from the catalog file.

To Import custom items into the Catalog Manager

  1. On the File menu of the Catalog Manager dialog, select Import. Or click the Import From File button on the toolbar. A menu that shows the types of files from which you can import items comes into view.
  2. On this menu, select Import Parameter File or Import Catalog File so that you can import items from the correct file type.
  3. In the Open dialog, find and select the file from which you want to import items. Then click Open. The Importing Items dialog that shows all of the custom items that you can import comes into view.

  4. NOTE: If you import items from a parameter file, the Catalog Manager ignores standard Aerotech motors and stages.
    5. Figure: Importing Custom Items

  5. Optional. In the Review valid items section of the Importing Items dialog, clear the box that is adjacent to each item that you do not want to import.
  6. Click OK to add the selected items to your catalog. If a conflict occurs while you try to import custom items, refer to the following section for more information.

Conflicts That Can Occur When You Import Items

Different types of conflicts, such as name or version conflicts, can occur when you try to import custom items from parameter files or catalog files. If a conflict occurs, you must do one of the following.

  • Clear the box that is adjacent to the item causing a conflict. Catalog Manager does not import this item which stops the conflict from occurring.
  • Correct the conflict before you try to import the item. Refer to the sections that follow for information about the types of conflicts that can occur.
Figure: Importing Items That Have Conflicts

Resolve version conflicts

Cause

Your catalog currently contains the items in this section. But the properties of the items in this section are different from the properties of the corresponding items in your catalog.

Solutions

  • Update the properties of each item in your catalog so they are the same as the items that you want to import from the parameter file or catalog file.
  • Save a copy of each item from the parameter file or catalog file. If you do this, you must give each new item a unique name.
Rename items to avoid name conflicts

Cause

The items in this section have the same names as the items in your catalog.

Solution

You must give each item a unique local name.

The Catalog Manager uses local names for display purposes to prevent two items that have the same name from causing a conflict. If you add an item to a parameter file and that item has a local name, the Catalog Manager shows the local name adjacent to the original item name. But it does not save the local name to the parameter file.

If you save changes to an item that has a local name, the Catalog Manager saves the local name as the new item name.

Figure: Item That Has a Local Name

Provide names for new items

Cause

In a previous version of software, the items in this section were configured as Other Stage or Other Motor and do not have specified names.

Solution

You must give each item a unique name.
