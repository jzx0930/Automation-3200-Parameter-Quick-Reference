Asynchronous Motion Overview

Asynchronous motion commands cause program execution to continue on to the next program block immediately after the move starts. The controller does not wait for the move to end before continuing to the next command; therefore, the task can execute other commands while the motion is proceeding. A significant exception is the ABORT command which waits until the decelerations have completed before continuing onto the next program block.

Table:     Asynchronous Motion Commands

Command

Action

HOMEASYNC

Home an axis without waiting for completion.

FREERUN

Free run an axis at the specified velocity.

MC

Use only with brushless motors for debugging purposes to output a fixed current to the specified axis.

MOVEABS

Move to an absolute position.

MOVEINC

Move incrementally.

MOVEOUTLIM

Move an axis out of a limit condition.

MOVETOLIM

Move an axis into a limit condition.

MSET

Use with brushless motors to output the specified electrical vector to the specified axis, typically for resolver or Hall-effect device alignment or debugging purposes.

OSCILLATE

Oscillate (cycle at the specified distance and velocity).

Table:     Related Commands

Command

Action

ABORT

Decelerate axis motion to a stop.

WAIT INPOS

Waits for the commanded velocity to get to zero and for the position error to get to a specified value.

WAIT MOVEDONE

Waits for the commanded velocity to get to zero.

Use asynchronous motion to complete other actions while doing a time consuming move. However, you must make sure that the asynchronous motion has actually been completed before a subsequent command is issued if it depends on the completion of this motion. Use the WAIT MOVEDONE or WAIT INPOS command to wait for asynchronous motion to complete.

If you issue a FREERUN, MOVEABS, or MOVEINC command on an axis while the controller is executing a FREERUN, MOVEABS, or MOVEINC command that you previously issued on that axis, the controller immediately stops executing the previously issued command and starts to execute the new command that you issued. The axis accelerates or decelerates smoothly to the new velocity and moves to the new target position.

Although they do not wait for the end of the move, asynchronous motion commands wait until the following conditions are satisfied before continuing to the next command.

  1. FEEDHOLD is not active.
  2. The Wait Done bit of the Axis Status status item is off for each axis in the move.

Asynchronous motion never includes the following transformations.

  • Cutter compensation
  • Corner rounding
  • Normalcy

If you purchased the CNC Option, refer to Transformation Overview for more information.

Asynchronous Motion Status

You can wait for motion to complete by issuing the WAIT MOVEDONE or WAIT INPOS commands. To find out if a move is done without waiting, look at the Move Done bit of the Axis Status status item. To find out if a move is done and in-position, look at the Move Done bit of the Axis Status status item and the In Position bit of the Drive Status status item.
