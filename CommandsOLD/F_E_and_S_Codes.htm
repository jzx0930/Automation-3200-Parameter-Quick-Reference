F and E Commands

Syntax:

 
F<CoordinatedSpeed>
E<CoordinatedSpeed>

Example:

 
F ($global[0]+1) ; set dominant axes feedrate
E100             ; set dependent axes feedrate

Using the F and E Commands

F and E commands specify feedrates for dominant and dependent axes, respectively. The feedrate specifies the speed for the line that it is on and remains in effect until a new feedrate command is given. The task will interpret the feedrate in the user units of the task when executing coordinated motion. Refer to Feedrate Overview.

The F and E commands apply only to coordinated motion (LINEAR, CW, CCW, and BEZIER) and specify the desired feedrate along the path of that coordinated motion. F and E commands can be specified on their own line, or with other commands, or with coordinated motion. If they are specified on the same line with coordinated motion, then the current feedrate is set to the desired value before the motion begins.

The F command specifies the feedrate for dominant axes. The E command specifies the feedrate for dependent axes. For coordinated moves that combine dominant and dependent axis motion, the controller uses only one of the two current feedrates and bases the selection on the feedrate dominance mode. Refer to Axis Type Overview. When you select Ignore E Word in the SystemCompatibility parameter, the F command is always used for coordinated move speeds and the E command is ignored.

The rate at which the motion changes to the new feedrate is controlled by the Automatic Acceleration Ramping. Refer to Acceleration and Deceleration Ramping Overview. With that ramping, the move might not have enough distance to achieve the new feedrate. In VELOCITY OFF mode, the motion will decelerate to zero speed before it reaches the new feedrate. However, in VELOCITY ON mode, the specified feedrate will not be achieved in that move, but the motion will continue to ramp toward the new feedrate in subsequent moves unless another feedrate change is made on a subsequent line.

The MaxSpeedClamp parameter can prevent the controller from achieving the feedrates that are specified by the F and E commands. In this case, the controller lowers the coordinated speed of the F and E commands so that every axis involved in the coordinated motion moves at or below the speed that is specified by MaxSpeedClamp.
