FlashConfig Data

The Restore FlashConfig dialog box lists the status of all data that is available from the stage. If you purchased integration from Aerotech, you can restore axis parameters, calculator information, and calibration data (if applicable). If you did not purchase integration, you can restore calculator information. For more information, refer to Restoring FlashConfig Data.

The data stored in FlashConfig is also used to sense if you connect a new stage, connect a stage to a different drive, or disconnect a stage. If the controller senses one of these conditions, you must resolve the mismatch condition before you can restore individual axes with the data that is stored in FlashConfig on the stage. Refer to Resolving FlashConfig Mismatches.

Types of FlashConfig Data

The table that follows lists the types of data that are kept in a FlashConfig stage and describes how the data is restored to the controller.

Table:     Types of FlashConfig Data

Type of Data Data Status Description Operation

Parameters

No data

The stage does not contain parameters.

The axis parameters on the controller stay the same.

Invalid

The stage contains parameters that are for a different motion controller and cannot be restored.

The axis parameters on the controller stay the same.

Valid

The stage contains parameters.

The axis parameters on the controller are overwritten with the parameters from the stage.

Calculator

Valid

The stage contains calculator information.

If the active parameter file on the controller contains calculator information, it is overwritten by the data from the stage.

1D Calibration

No data

The stage does not contain 1D calibration tables.

In most conditions, the 1D calibration tables on the controller stay the same.

Invalid

The stage contains orthogonality or straightness correction tables.

The tables cannot be restored because the tables for the master axis or corrected axis will not be restored.

 The 1D calibration tables on the controller are removed.

Partial

In most conditions, the stage contains accuracy correction tables and orthogonality or straightness correction tables.

A minimum of one calibration table will be restored. Some of the tables cannot be restored because the tables for the master axis or corrected axis will not be restored.

The 1D calibration tables on the controller are overwritten with the 1D tables from the stage.

Valid

The stage contains accuracy correction tables, orthogonality correction tables, or straightness correction tables.

All 1D calibration tables can be restored.

 The 1D calibration tables on the controller are overwritten with the 1D tables from the stage.

2D Calibration

No data

The stage does not contain 2D calibration tables.

In most conditions, the 2D calibration tables on the controller stay the same.
Invalid

The stage contains 2D calibration tables.

The tables cannot be restored because the tables for one or more of the master axes or the correction axes will not be restored.

The 2D calibration tables on the controller are removed.

Partial

The stage contains 2D calibration tables.

A minimum of one of the calibration tables will be restored. Some of the tables cannot be restored because the tables for one or more of the master axes or the correction axes will not be restored.

The 2D calibration tables on the controller are overwritten with the 2D tables from the stage.

Valid

The stage contains 2D calibration tables.

All 2D calibration tables can be restored.

The 2D calibration tables on the controller are overwritten with the 2D tables from the stage.

Note: You might not be able to restore 2D calibration from a stage that supports FlashConfig if the number of corrections used to initially calibrate the stage is greater than the maximum calibration table size that can be stored in FlashConfig.

Removal of Existing Calibration from Other Axes

When you restore data from FlashConfig, calibration might be removed from axes on the controller that you do not select in the Restore FlashConfig dialog box. Before you restore data, you can review the list of axes in the Existing Calibration section of the dialog. Refer to the examples that follow.

Example 1

The controller has 1D calibration for Axes X and Y. Each axis has an accuracy correction table. There is also an orthogonality correction table so that Axis X corrects Axis Y.

Both axes are connected to stages that support FlashConfig. The same types of calibration tables (accuracy correction and orthogonality correction) are kept in FlashConfig on the stages.

In the Restore FlashConfig dialog box, you restore configuration data from the stage connected to Axis X. You do not restore data from Axis Y.

The orthogonality correction table for Axis X and Axis Y on the controller is removed. The accuracy correction table for Axis X is transferred from FlashConfig to the controller. The accuracy correction table for Axis Y stays the same on the controller.

Example 2

The controller has 2D calibration for Axes X, Y, and Z. One table contains corrections for Axis X and Axis Y. A different table contains corrections for Axis Y and Axis Z.

You connect stages that support FlashConfig to Axis X and Axis Z. The stages contain a 2D table that corrects Axis X and Axis Z.

In the Restore FlashConfig dialog box, you restore configuration data from the stages connected to Axis X and Axis Z. Axis Y does not show and cannot be selected because a FlashConfig stage is not connected to it.

Both tables on the controller are removed, which has an effect on the calibration of Axis Y. The table that corrects Axis X and Axis Y must be removed because data is being restored for Axis X. The table that corrects Axis Y and Axis Z must be removed because data is being restored for Axis Z.

The calibration data that corrects Axis X and Axis Z is transferred from FlashConfig to the controller.
