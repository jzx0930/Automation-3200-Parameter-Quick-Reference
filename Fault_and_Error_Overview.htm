Fault and Error Overview

The controller continuously monitors for problems and generates task errors and axis faults. Examples of these problems include division by zero from an AeroBasic program and an axis fault that the controller generates while you issue a motion command.

Axis Faults

Warning: To prevent damage to the hardware, configure the FaultMask parameter before you enable or move the axes.

An axis fault shows that the drive detected a fault condition for which you must do the necessary steps to correct. Examples of axis faults include a Position Feedback Fault and a CW End Of Travel Limit Fault.

An axis can be in one or more fault conditions at the same time. If you want the controller to abort one or more axes when the controller generates an axis fault, specify those axes in the FaultAbortAxes parameter.

Axis faults can have an effect on program execution in the following ways.

  • If the controller generates an axis fault while the task is executing a synchronous motion command on that axis, the controller stops executing your AeroBasic program and puts the task into the Error Task State.
  • If you specify a mask of axes in the TaskTerminationAxes parameter and the controller generates an axis fault on one or more of those axes, the controller stops executing your AeroBasic program and puts the task into the Error Task State.

You can customize axis fault handling by using the ONGOSUB command in an AeroBasic program.

You can independently configure what the axis does for each fault condition by using the settings in the following table.

If an axis is in a fault condition, the axis completes the action that you specify in the FaultMask, FaultMaskDecel, and FaultMaskDisable parameters. Before normal operations can continue, you must recover from the axis fault by doing the following.

  1. Correct the condition that caused the controller to generate an axis fault.
  2. Acknowledge the fault condition by doing one of the following operations.
    • Click the Acknowledge All button or issue an ACKNOWLEDGEALL command to clear all of the axis faults on the controller.
    • Issue the FAULTACK command to acknowledge the axis faults on the axis that you specify.

Examining Axis Faults

For more information about an axis fault that the controller generates, do one of the following options.

  • Look at the axis fault in CNC Operator Interface, Digital Scope, or Motion Composer. These applications show axis faults in red text.
  • In your AeroBasic program, use the DATAITEM_AxisFault item to see all of the axis faults that the controller generates. For more information about this data item, refer to the AXISSTATUS function.
  • For advanced diagnostics information about the axis fault, use the Fault tab in the Status Utility.
Task Errors

A task error stops a program when an error condition occurs in that program. For example, a task error occurs if the controller tries to generate motion while the axis is disabled. When a task error occurs, the controller stops executing your AeroBasic program and puts the task into the Error Task State.

If you want the controller to abort one or more axes when a task error occurs on a task, specify those axes in the TaskErrorAbortAxes parameter.

You can customize task error handling by using the ONGOSUB command in an AeroBasic program.

Examining Task Errors

For more information about a task error that occurs, do one of the following options.

  • Look at the error message in CNC Operator Interface or Motion Composer.
  • To find the error message in your AeroBasic program, use the ERRORDECODE function. If you issue this function, you must use the DATAITEM_TaskErrorCode item to get the error code and the DATAITEM_TaskErrorLocation item to get the location code of the task error. For more information about these data items, refer to the TASKSTATUS function.
  • For advanced diagnostics information about the task error, use the Tasks tab in the Status Utility.

Clearing Task Errors

To clear task errors, do one of the following operations.

  • Click the Acknowledge All button or issue an ACKNOWLEDGEALL command to clear task errors on all tasks.
  • Click the Stop button or issue a PROGRAM STOP command to stop the current task and clear a task error on the current task.
  • Click the Abort button to abort all motion, stop the current task, and clear a task error on the current task.
  • Issue the TASKERROR command to clear a task error on a specific task.
