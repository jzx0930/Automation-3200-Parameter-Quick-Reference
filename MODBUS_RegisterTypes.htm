Modbus Register Types

NOTE: The information in this topic is for use with the MODBUSREGXXX and MODBUSBIT commands, which are marked as obsolete and will be removed in a future version. Refer to the Drive Modbus Overview or the PC Modbus Overview.

Register Names

When you use the MODBUSREGxxx, or MODBUSBIT commands, you must specify the register set on which the command should operate.

Each of the following register sets is used for a different purpose.

MasterInputBits

You can divide each register name into three parts to understand exactly what each name is used for.

  1. Master or Slave
    • Indicates the type of modbus interface you are commanding.
    • The A3200 PC Modbus supports both Master and Slave configurations.
      • There is only one Slave connection. You must make the <ConnectionNumber> argument 0 or omit it.
      • There are 16 configurable Master connections. The <ConnectionNumber> argument specifies which connection is being commanded. If you omit the <ConnectionNumber> argument, the default connection is 0.
    • The A3200 Drive Modbus supports only one Master configuration per drive.
  2. Input or Output
    • Specifies which set of the Modbus function code is used to transmit the data.
    • Indicates the direction of the data transfer from the perspective of the master device in the system (Inputs from the Slave to the Master, Outputs from the Master to the Slave). However, some devices do not follow this convention and use Outputs for everything.
    • The Outputs support both read and write functions (from the Master).
    • The Inputs can only be read by the Master (written to by the Slave).
  3. Words or Bits
    • Indicates the type of data being transmitted, and how the data is indexed.
    • Words are 16-bit chunks of data. Each 16 bits of data has an index number.
      • Commonly used to store analog data.
      • Data can be interpreted as each of the data types with any data packing.
        • Your ability to Interpret data depends on the implementation and interpretation of the slave device.
        • You can store larger data types by combining multiple consecutive words and interpreting the data as that type.
    • Bits are 1-bit sized values, where each bit has an index number.
      • Used to store binary data.
        • Lights
        • Switches
        • Buttons
      • Can only store a value of true or false.

When you use the Modbus commands, you must specify the correct registers. The Modbus commands do not let you specify incorrect registers. The registers that you use also depend on the other Modbus device that the A3200 communicates with. Many devices do not implement the entire Modbus specification and use a subset from the registers that are available. To determine the correct registers to use for a specific purpose, refer to the documentation for the Modbus device you are using.
