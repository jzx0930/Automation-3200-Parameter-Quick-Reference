Configuring the Compiler Output Directory

In Configuration Manager, you can set the output directory that the AeroBasic compiler uses when it compiles an AeroBasic program. By default, the AeroBasic compiler generates all compiled output files, which are .ogm files, to the same directory as the program that you compile. If you do not have write permission to compile to the same directory as the program that you compile, you can set the Compiler Output Directory to a directory that has write permission.

To Configure the Compiler Output Directory

  1. Open Configuration Manager.
  2. In the Network Explorer, select the A3200 node.
  3. In the Viewer pane, select Compiler Output Directory.
  4. Select or clear the directory. Do one of these options.
    • To select the directory. In the Editor pane, click Browse to specify a directory that the AeroBasic compiler uses to generate all compiled output files.
      • Figure: Compiler Output Directory — Selected

    • To clear the directory. Clear the directory from the Compiler Output Directory box. When this box is clear, the AeroBasic compiler generates all compiled output files to the same directory as the program that you compile.
      • Figure: Compiler Output Directory — Cleared

  5. Click Set to save the changes.
