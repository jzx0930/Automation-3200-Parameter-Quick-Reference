Limits Overview

There are two types of limits: hardware end of travel limits and software limits. Software limit and hardware limit response is configured using the FaultMask parameters.

Hardware end of travel limits

Hardware end of travel limits are connected to an electrical switch and are used to prevent motion past a certain point. They can also be used to find reference positions during homing. Refer to Homing Overview. The active level of the end of travel limits is configured using the EndOfTravelLimitSetup.

Virtual Axes do not generate hardware limit faults.

Software limits

NOTE: Software Limits are not active until you home an axis.

The default parameter settings decelerate the axis to a stop and keep the axis enabled when an end of travel hardware limit is reached. Software limits are disabled by default and must be configured by the user, if desired. Use the SoftwareLimitSetup Parameter, the SoftwareLimitHigh Parameter, and the SoftwareLimitLow Parameter to configure the behavior of the software limits.

There are four modes of Software Limit operation, described in the following sections.

Drive Detection

To enable Drive Detection mode set the Target Scanning and Clamping settings of SoftwareLimitSetup to Disabled. Check the CW Software Limit and the CCW Software Limit bits in FaultMask, and check the CW Software Limit and the CCW Software Limit bits in FaultMaskDecel or FaultMaskDisable.

In this mode the axis will start to decelerate after the limit threshold is crossed and it will go past the software limit threshold. The axis will decelerate at the rate specified by the AbortDecelRate Parameter, but the LimitDecelDistance Parameter can limit the deceleration distance. If you use LimitDecelDistance, the deceleration rate can be higher than the rate specified by AbortDecelRate. The deceleration uses a linear ramp type.

The controller generates a software limit fault the same millisecond that the threshold is crossed. Depending on the setting of the TaskTerminationAxes Parameter, the program will terminate with an error.

Virtual axes do not generate software limit faults in Drive Detection mode.

Clamping with No Fault

To enable Clamping with No Fault mode set the Target Scanning setting of SoftwareLimitSetup to Disabled and set the Clamping setting of SoftwareLimitSetup to Enabled. Clear the CW Software Limit and the CCW Software Limit bits in FaultMask.

In this mode the axis will stop one count short of the software limit threshold. The axis will decelerate at the rate specified by the AbortDecelRate Parameter, but the LimitDecelDistance Parameter can limit the deceleration distance. If you use LimitDecelDistance, the deceleration rate can be higher than the rate specified by AbortDecelRate. The deceleration uses a linear ramp type.

If the axis being clamped is part of a multi-axis asynchronous move, then the other axes are unaffected. If the axis being clamped is part of a multi-axis synchronous move (RAPID, LINEAR, CW, CCW or BEZIER), then the other axes in the move are aborted at the rate specified by the AbortDecelRate parameter. If the axis being clamped is a slave to another axis (for example, handwheel, gearing, camming), then the master axis is aborted at the rate specified by the AbortDecelRate parameter and the master-slave relationship is removed.

The controller does not generate an axis fault, and programs are unaffected, although the controller will not do a move that would go past the software limit threshold. The Clamped bit of Axis Status will indicate that an axis is in the clamped state (stopped one count from the limit).

Clamping with Fault Afterwards

To enable Clamping with Fault Afterwards mode set the Target Scanning setting of SoftwareLimitSetup to Disabled and set the Clamping setting of SoftwareLimitSetup to Enabled. Check the CW Software Limit and the CCW Software Limit bits in FaultMask, and check the CW Software Limit and the CCW Software Limit bits in FaultMaskDecel or FaultMaskDisable.

In this mode the axis will stop one count short of the software limit threshold. The axis will decelerate at the rate specified by the AbortDecelRate Parameter, but the LimitDecelDistance Parameter can limit the deceleration distance. If you use LimitDecelDistance, the deceleration rate can be higher than the rate specified by AbortDecelRate. The deceleration uses a linear ramp type.

If the axis being clamped is part of a multi-axis asynchronous move, then the other axes are unaffected. If the axis being clamped is part of a multi-axis synchronous move (RAPID, LINEAR, CW, CCW or BEZIER), then the other axes in the move are aborted at the rate specified by the AbortDecelRate parameter. If the axis being clamped is a slave to another axis (for example, handwheel, gearing, camming), then the master axis is aborted at the rate specified by the AbortDecelRate parameter and the master-slave relationship is removed.

The controller generates a software limit fault one millisecond after the clamp deceleration completes. Depending on the setting of the TaskTerminationAxes Parameter, the program will terminate with an error. The Clamped bit of Axis Status will indicate that an axis is in the clamped state (stopped one count from the limit).

Target Scanning

To enable Target Scanning mode set the Target Scanning setting of SoftwareLimitSetup to Enabled. Check the CW Software Limit and the CCW Software Limit bits in FaultMask.

In this mode the controller will not do a move that would go past the software limit threshold. Instead of performing the move, the controller generates a software limit fault. Depending on the setting of the TaskTerminationAxes Parameter, the program will terminate with an error.

Note: This mode has no effect on handwheel, gearing, camming, or freerun type motions because these motions have no target position.

Recovery from a Limit Fault

After a limit fault occurs, follow these steps:

  1. In any application, click the Acknowledge All button. If the FaultAckMoveOutOfLimit Parameter is set to Yes, the axis will move out of the limit at the HomeSpeed speed until the limit becomes inactive. Regardless of the FautlAckMoveOutOfLimit setting, the fault status is automatically cleared.
  2. If the axis was set to disable on the limit fault, re-enable the axis and click the Acknowledge All button in any application.
  3. You can also jog out of a limit without acknowledging the fault if you jog away from the limit. You cannot use a RAPID or LINEAR command to move out of a limit fault.
