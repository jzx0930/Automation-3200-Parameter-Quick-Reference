Kinematic Transformation Overview

Kinematic transformation occurs when a dedicated task, hereafter called the "kinematic transform task," runs a special program that changes or overrides motion generated by other programs, hereafter called "program tasks." The kinematic transform task can change motion on any axes, remove motion on any axes, or add motion on new axes. Kinematic transformation is extremely powerful, but should only be used by advanced users. Mistakes in the transformation program can cause jerks in motion, and they can cause your system to stop responding.

For best motion results, Aerotech strongly recommends that you set the MotionUpdateRate Parameter to 0.5.

The kinematic transform task can transform motion generated from any program task as long as the kinematic transform task number is higher than the program task number.

Kinematic transformation programs use these commands to accomplish the transformation. For more information, refer to Transformed Motion Example
