PC Modbus Communication with a WAGO I/O Module

Use the following tutorial to configure PC Modbus to communicate with a WAGO slave device. The information for this tutorial is specifically written for the WAGO 750-842 slave device, configured with four WAGO 750-504 modules and four WAGO 750-402 modules. But, you can use much of the information in this tutorial with other WAGO slave devices that are almost the same as these modules.

Configure the IP Address of the WAGO Slave Device

NOTE: In this section, use a standard network card under Windows control to configure the WAGO slave device. Connect the WAGO slave device through a network switch with a standard Ethernet cable, or directly connect the WAGO slave device to your PC with a crossover cable.

For specific instructions on how to configure the IP address and subnet mask of the Modbus device, refer to the documentation from the manufacturer. You can find the documentation for the WAGO 750-842 slave device on the WAGO website.

Use the following procedure to configure PC Modbus to communicate with a WAGO slave device.

  1. Install the WAGO BootP Server application at http://www.wagocatalog.com/okv3/Download_files/BootPSer.zip.
  2. Connect the WAGO slave device to the same network to which your PC is connected.
  3. Open the WAGO BootP Server application.
  4. Click the Edit Bootptab button. Bootptab.txt comes into view so you can edit this file.
    1. Edit the colon-separated sections of bootptab.txt to define the operating parameters that you want to configure on the WAGO slave device. Refer to the following example.

    2. For Example

      To configure the IP address of a WAGO slave device as 10.101.37.8, the Gateway as 10.101.0.1, and the subnet mask as 255.255.0.0 to a WAGO slave device with a MAC address of 00:30:DE:00:11:03, edit the text as follows.

      WAGOFC1:ht=1:ha=0030DE001103:ip=10.101.37.8:sm=255.255.0.0:T3=0A.65.00.01:

      Where

      WAGOFC1 is an identifier. (You can change the name of WAGOFC1 to a name that you want to use.)

      and

      ht is the hardware type. (Set ht to 1.)

      and

      ha is the hardware address. (This is the MAC address.)

      and

      ip is the dot-separated IP address.

      and

      sm is the subnet mask.

      and

      T3 is the gateway. (You must enter the gateway in hexadecimal notation.)
    3. Find the MAC address of the WAGO slave device. You should see that the MAC address is printed on this device.
    4. Select the correct IP address and subnet mask for the WAGO slave device. You must configure the PC Modbus card and WAGO slave device on the same subnet mask so the PC Modbus card and WAGO slave device can communicate correctly.

    5. NOTE: The PC Modbus card and WAGO slave device can communicate over a variety of network structures. For performance reasons, Aerotech recommends that you connect the PC Modbus card and WAGO slave device directly with a crossover cable and assign static IP addresses for both the PC Modbus card and WAGO slave device. Refer to PC Modbus Overview for more information.
    6. Make sure that you terminate the line with a colon (:).
    7. After you are done editing bootptab.txt, save and close the file.
  5. Make sure that the WAGO slave device is connected to the network card of the PC and that the WAGO slave device is powered off.
  6. In the WAGO BootP Server dialog, click the Start button to scan for a bootp packet request.
  7. Power on the WAGO slave device. After a few seconds occur, the WAGO BootP Server dialog comes into view. If the "found" line is shown, then you have successfully assigned an IP address to the WAGO slave device.
  8. Figure: WAGO BootP Server

  9. On the WAGO BootP Server dialog, click the Stop button.
  10. Close the WAGO BootP Server application.

Configure the A3200 PC Modbus Network Card

In the following procedure, you assign a network card for the A3200 that you can use under INtime control.

  1. Configure a network card that is compatible with INtime and put it under INtime control.
  2. For more information on how to configure a network card, refer to the procedure for Installing the Ethernet Card in PC Modbus Installation.
  3. Assign a valid IP address to the network card.
    1. Make sure that the IP address that you assign to the network card is compatible with the IP address that you assigned to the WAGO slave device in the Configure the IP Address of the WAGO Slave Device section of this tutorial.
    2. Refer to the procedure To Configure the Ethernet Stack in PC Modbus Configuration to configure the INtime TCP/IP stack. The following figure shows an example configuration.
    3. Figure: Connection Settings Dialog

  4. Connect the WAGO slave device to this network card directly with a crossover cable or through a network switch with a standard Ethernet cable.

Configure the A3200 PC Modbus Master Connection

Use the following procedure to configure a PC Modbus Master to communicate with the WAGO slave device.

  1. Open Configuration Manager.
  2. Create or open the active parameter file.
  3. In the active parameter file, expand Fieldbus and click PC Modbus Master.
  4. Click the Edit PC Modbus Master button.
  5. Click the Add Connection button to add a new connection.
  6. In the PC Modbus Master Configuration dialog, enter the connection specifications for the WAGO slave device.
  7. Figure: PC Modbus Master Configuration - WAGO Slave Device Connection

    1. Look at the Name section to see the name of the WAGO slave device you are using. In this section, you can change the name WAGO to a name that you want to use.
    2. The Slave ID is not used by the WAGO slave device. Set the value of the Slave ID to 0 or 1.
    3. The Slave IP Address should be the same as the IP address that you configured for the WAGO slave device using the WAGO BootP Server application. Refer back to the Configure the IP Address of the WAGO Slave Device section of this tutorial for more information.
    4. Make sure that the Slave Port is set to 502. This is the default Modbus port.
  8. Add the correct variable mappings to correspond to the I/O of the WAGO slave device. For specific instructions on how to add variable mappings, refer to Creating Fieldbus Mappings for PC Modbus Master.
    1. This WAGO Slave device has 16 digital outputs: 4 digital outputs on each of its four 750-504 modules.
      1. Write to the outputs of the WAGO slave device through the Output Bits section, starting at address 0.
      2. Add the following variable mapping.

  9. NOTE: The mappings that you configured in step 7 of this section create the most simple and generic interface in AeroBasic. A simple array variable is generated where each array index accesses a single bit. You can create mappings that are more complex and specific to your project settings.

  10. Click OK to apply the configuration.
  11. Save the parameter file.
  12. Reset the controller for the changes to have an effect.

Examples of Using Mappings in an AeroBasic Program

The AeroBasic example programs that follow show the various ways in which you can use Fieldbus mappings. Each mapping that you define generates a corresponding AeroBasic variable (that starts with a '$') that you can use in a program in the same way that you use other variables in a program. The I/O is automatically synchronized in the background by the controller as quickly as possible. Inputs are automatically requested by the controller once per millisecond. Outputs are written to by the controller only when the outputs have been changed.

Example

' Use a FOR loop to change each of the output bits.
DVAR $iii
for $iii = 0 to 15
    $OutBits[$iii] = !$OutBits[$iii]
next $iii

Example

' Use an input bit as the condition to an action.
if ($InBits[4] == true)
    MSGDISPLAY 0, "Bit on!"
endif
