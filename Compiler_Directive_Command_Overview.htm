Preprocessor Directive Overview

Preprocessor directives are instructions that tell the compiler how to interpret program text.

All preprocessor statements begin with #. The syntax and semantics of A3200 preprocessor directives are almost the same as the equivalent ANSI C language standard for preprocessor directives. The following are differences between the A3200 and ANSI C Standard.

  • In the A3200, you cannot use # or ## operators in replacement strings.
  • In the A3200, you do not need to use double quotes around file names.
  • Multiline substitution syntax rules have differences.
  • Substitution in preprocessor directive text rules has differences.

The preprocessor (the component of the compiler that executes directives) executes all of the preprocessor directives in the program text before giving the program to the compiler.
