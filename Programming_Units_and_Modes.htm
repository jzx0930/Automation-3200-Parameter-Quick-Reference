Programming Units and Modes

Distance Units

The distance modes available on an axis depend on its type. The AxisType parameter determines if an axis is dominant or dependent. Most applications use the dominant type. The following sections describe the distance modes for each type of axis. Refer to Axis Type Overview for more information on axis type.

Dominant Type Axes

Dominant type axes work in either the primary or the secondary distance modes. Use the PRIMARY and SECONDARY commands to specify which distance mode the controller will use.

The primary distance mode uses the CountsPerUnit parameter to define the number of encoder counts per distance unit. In this mode the distance in units is equal to the distance in counts divided by the value of the CountsPerUnit parameter.

The secondary distance mode uses the CountsPerUnit parameter, divided by the SecondaryUnitsScaleFactor parameter, to define the number of encoder counts per distance unit. In this mode the distance in secondary units is equal to the distance in primary units divided by the value of the SecondaryUnitsScaleFactor parameter. Typically the CountsPerUnit parameter is configured for millimeters and the SecondaryUnitsScaleFactor for inches, but you can configure these parameters for any unit.

Dependent Type Axes

Dependent type axes work in the primary distance mode. The CountsPerUnit parameter defines the number of encoder counts per distance unit, and the distance in units is equal to the distance in counts divided by the value of the CountsPerUnit parameter. Typically with dependent axes, the CountsPerUnit parameter is configured for degrees, but you can configure it for any unit. The PRIMARY and SECONDARY commands do not affect dependent type axes, which will always use the primary distance mode.

Note: To change the program direction of an axis, use the ReverseMotionDirection parameter.

Velocity Units

The number of encoder counts per distance unit is also used to define velocities. In addition, velocities have time units, which are either seconds or minutes. Use the SECONDS and MINUTES commands to specify which time unit mode the controller uses.

Because dependent type axes also contain the DependentSpeedScaleFactor parameter (a feedrate scaling parameter), the dependent feedrate that is specified by the E command and the resulting commanded velocity can have different units than the dependent distance. Having different units for feedrate or velocity and distance is useful for rotational axes, when axis velocities must be programmed in revolutions per minute (rpm), but units are in degrees. Refer to Configuring a Rotational Axis for more information.

The AxisType parameter determines if an axis is dominant or dependent.

User Units

User units refer to the active units of a task and are always either primary units or secondary units. User units are indicated by Bit 0 of the Task Mode status item.

Configuring Units

If Aerotech preconfigured your system, the necessary information for converting encoder counts is supplied in the parameter file. If the system is not preconfigured, use the following procedure.

  1. Select the CountsPerUnit parameter in Configuration Manager.
  2. Click Calculator to open the Axis Parameter Calculator. Enter your information in the appropriate fields. Refer to Using Calculators to Set Parameters for more information.
  3. Set the AxisType parameter in Configuration Manager.
  4. Set the SecondaryUnitsScaleFactor and SecondaryUnitsName parameters in Configuration Manager.
  5. If you are using dependent axes, set the DependentSpeedScaleFactor parameter in Configuration Manager.
