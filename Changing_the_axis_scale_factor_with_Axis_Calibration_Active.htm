Changing the Axis Scale Factor with Axis Calibration Active

You must update the calibration table for a particular axis if the machine resolution of the axis changes (if the encoder hardware or multiplication factor changes). For example, if the multiplication factor for an axis is doubled, the CountsPerUnit parameter will be doubled to reflect this change, which might require you to manually update the calibration table.

If the axis calibration table is in primary units (the POSUNIT=PRIMARY argument is present in the :START line), you will not need to change the axis calibration table values after the multiplication factor increase. The values in the axis calibration table will still be correct because they are specified in primary units and the CountsPerUnit parameter has already been updated to account for the multiplication factor increase.

However, if the axis calibration table is in counts (the POSUNIT argument in not present in the :START line), you must update the axis calibration table after the multiplication factor increase. The position values in the axis calibration table (specified in encoder counts) are no longer correct because the machine resolution of the axis has changed. The correction values, sample distance, and OFFSET entry must be multiplied by a scale factor.

The scale factor is equal to the new value of the CountsPerUnit parameter divided by the old value of the CountsPerUnit parameter. So, if the new value divided by the old value is equal to two, the values in the axis calibration table must be multiplied by two.

You can change the CountsPerUnit value for an axis while its machine resolution stays the same. For example, to change the unit from millimeters to micrometers, the CountsPerUnit parameter can be divided by 1000. If the position and correction values in the calibration table are specified in primary units, you must use the optional divisor argument to POSUNIT and CORUNIT so that the position and correction values in the axis calibration table do not need to be changed.
