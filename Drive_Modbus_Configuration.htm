Drive Modbus Configuration

You can use the Ndrive, Npaq, and Nservo drives as a Modbus master. Before you connect the master drive to a Modbus slave device, you must configure the axis parameters of the drive. You must also configure the Modbus slave device so that it has an IP address on the same subnet as the A3200 drive. Refer to the instructions for the slave device for information on configuring the parameters and the slave device.

Hardware: On Npaq and Nservo drives, configure the first axis of the drive as the Modbus master.

The procedure that follows tells you how to configure the most used Modbus slave devices (WAGO Fieldbus Controllers (FC) and Programmable Fieldbus Controllers (PFC)). Other Modbus-compliant modules supply equivalent configuration tools that might be different from this procedure. For more information, refer to the documentation supplied by the manufacturer of the module you are using.

To configure the drive Modbus communication

  1. Configure the Modbus slave device with a compatible IP address for your subnet.
    2. Assign an IP Address to the WAGO Slave Device
    1. Power off the WAGO slave device.
    2. Install the WAGO BootP Server application at http://www.wagocatalog.com/okv3/Download_files/BootPSer.zip.
    3. Connect the WAGO slave device to the same network to which your PC is connected.
    4. Open the WAGO BootP Server application.
    5. Click the Edit Bootptab button. Bootptab.txt comes into view so you can edit this file.
    6. Edit the colon-separated sections of Bootptab.txt to define the operating parameters that you want to configure on the WAGO slave device. Refer to the example that follows.

    7. For Example

      To configure the IP address of the WAGO slave device as 10.101.0.24, the Gateway as 10.101.0.1, and the subnet mask as 255.255.0.0 to a fieldbus controller with a MAC address of 00:30:DE:00:11:03, edit the text as follows.

      WAGOFC1:ht=1:ha=0030DE001103:ip=10.101.0.24:sm=255.255.0.0:T3=0A.65.00.01:

      Where

      WAGOFC1 is an identifier. (You can change the name of WAGOFC1 to a name that you want to use.)

      and

      ht is the hardware type. (Set ht to 1.)

      and

      ha is the hardware address. (This is the MAC address.)

      and

      ip is the dot-separated IP address.

      and

      sm is the subnet mask.

      and

      T3 is the gateway. (You must enter the gateway in hexadecimal notation.)
    8. Make sure that you terminate the line with a colon (:).
    9. After you are done editing bootptab.txt, save and close the file.
    10. In the WAGO BootP Server dialog, click the Start button to scan for a bootp packet request.
    11. Power on the WAGO slave device. After a few seconds occur, the WAGO BootP Server dialog comes into view. If the "found" line shows, then the IP address is assigned the WAGO slave device.
      12. Figure: WAGO BootP Server

    12. To get the WAGO slave device to reply correctly, open a Windows command prompt and type the text that follows.

      ping <IP address>

    13. For Example

      C:\>ping 10.101.0.24
Reply from 10.101.0.24: bytes=32 time=10ms TTL=255
      Reply from 10.101.0.24: bytes=32 time<10ms TTL=255
      Reply from 10.101.0.24: bytes=32 time<10ms TTL=255
    Make sure the A3200 drive and this IP address are on the same subnet

    IF (REMOTE_IP XOR LOCAL_IP) AND NET MASK = 0, the two nodes are on the same subnet.

    For Example

    In this example, the two nodes are on different subnets and the packets are routed to a gateway from the side of the Host. When a network packet is routed to a gateway, delays in communication can be caused by overhead and packet collisions. Because of the possible delays, do not use an I/O module outside of its own subnet for time-critical applications.

  2. Assign an IP address to the drive that is different but compatible with the IP address of the Modbus slave device. The subnet mask should be the same as the subnet mask on the slave. Refer to Step 1. The default gateway should normally be the address of your router, but might not be required. Configure the axis parameters that follow. You can find these parameters in Configuration Manager under Axis → Communication.

    3. DriveIPAddress: TCP/IP address of the drive.

    DriveSubnetMask: TCP/IP subnet mask of the drive.

    DriveDefaultGateway: TCP/IP gateway for the drive.

  3. Add the connection configuration to the active parameter file. In Configuration Manager under Fieldbus → Drive Modbus Master, click the Edit Drive Modbus Master button. Click the Add Connection button to add a new connection, and then supply the correct Axis, Slave ID, Slave IP Address, and Slave Port. The Slave ID is typically not used. You can usually set the Slave ID to 0 or 1.
  4. Add the correct I/O mappings for this Modbus device. Refer to the documentation supplied by the manufacturer of the module for the correct register types and addresses to use. You can only configure the drive to read and write the range of I/O that is mapped. The mapped names are automatically generated into AeroBasic variables that you can access directly in your programs. Refer to Creating Fieldbus Mappings for Drive Modbus for more information.
  5. Connect the Modbus slave device to the drive Ethernet port with an Ethernet crossover cable (Aerotech P/N NCONNECT-IO-XXX). If you are making a direct connection, do not use a standard, non-crossover cable. As an alternative, you can use a standard Ethernet cable through a network switch, but this might add some latency.
