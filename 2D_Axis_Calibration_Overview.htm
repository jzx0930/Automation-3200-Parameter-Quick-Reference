2D Axis Calibration Overview

Use 2D axis calibration to correct two or three axes based on the position of any two axes. The A3200 SMC supports a single 2D calibration file (extension .cal) that contains up to ten 2D calibration tables. Each calibration table is delimited by :START2D and :END tokens. The table size is limited only by available memory. Each calibration point uses 24 bytes of controller memory.

The first point in the calibration table corresponds to the home position (0,0) of the row and column input axes. You can shift the table using the OFFSETROW and OFFSETCOL keywords. You can shift the table start position using the negative of the specified offset value, which is viewed through the Status Utility.

Note: The values in the calibration file, including sample distances, offsets, and corrections, follow the polarity of the encoder and are not affected by the ReverseMotionDirection parameter.

Note: To view the nth calibration table (starting with 0) loaded into memory, use the DUMP2DAXISCAL n Console command in Console.

2D axis calibration is active when:

  • A 2D axis calibration table has been downloaded to the A3200 SMC.
  • The 2D Calibration File value has been set in Configuration Manager .
  • The controller has been reset.
  • Both the row and column input axes have been homed.
