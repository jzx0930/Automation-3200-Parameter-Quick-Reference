Selecting a Parameter File

A parameter file contains information that configures the software to work with the drive, motor, feedback device and stage that are connected to the system. One parameter file must be selected as the active file.

When you open Configuration Manager for the first time, you are prompted to specify the active parameter file.

Figure: Specify Active Parameter File

Integrated Systems

If you purchased integration from Aerotech, the parameter file is installed with your software.

To select the custom controller configuration file that was SUPPLIED by Aerotech

  1. In the Specify Active Parameter File dialog, select Select an existing parameter file and click OK. The Select Active Parameter File dialog comes into view.
  2. Select the file with the file extension .prma and click Open. The file shows in Configuration Manager in the Network Explorer under Open Files. The file is in bold text with <Active> adjacent to the file name.
  3. Figure: Configuration Manager — Active Parameter File

More information about your custom parameter file and your Aerotech system

If you purchased a complete and integrated system that includes the controller, drives, motors, stages, wiring, and so on, it was fully tested and configured at the factory. When you install the software, a custom controller configuration (.prma) file is installed in the Shared Documents\Aerotech\A3200\User Files folder on you machine. This file configures your controller. If necessary, you can reinstall the file by copying it from the \A3200\Ini folder on the software disc to the Shared Documents\Aerotech\A3200\User Files folder on your machine. The parameter file follows this naming convention: 132123-A-1-1.prma, where "132123" is the Aerotech customer order number. Make sure that you select this file as the active parameter (.prma) file.

If you are sure that you purchased integration and you do not have a custom parameter file, contact Aerotech Global Technical Support.

Note: If the system was tested without loading, you must do some servo loop tuning to adjust for load and to meet accuracy requirements.

The system specifications are on your software disc and are specified as ES, which means Engineering Specification. These system specifications give the system resolution and maximum feedrate. System settings can have an effect on the values of your parameters.

For more information about how to configure parameters, refer to Configuration Manager Overview and Working with Parameters.

For information about calculators that help you calculate the correct parameter values, refer to Using Calculators to Set Parameters.

Note: If you are not sure if you purchased integration, look for the custom parameter file. If you do not have a customer parameter file, refer to the information for independent systems.

Independent Systems

If you did not purchase integration, or if you are using third-party drives, motors, or stages, you must create and configure the parameter file for your controller.

To create a new parameter file

  1. In the Specify Active Parameter File dialog, select Create a new parameter file and click OK. The Select a Destination Path dialog comes into view.
  2. Specify the filename and the path where you want the file to be stored. The default name is Untitled.prma. The default path is C:\Users\Public\Documents\Aerotech\A3200\User Files\ .
  3. Click Save. The file shows in Configuration Manager in the Network Explorer under Open Files. The file is in bold text with <Active> adjacent to the file name.
  4. Figure: Configuration Manager — Active Parameter File

More information about configuring your independent system

These resources can help you start to configure your system.

  • The standard documentation in the Product Manuals, Standard Cable Drawings, and Motor Drawings sections on your installation device
  • Controller Configuration
  • Using Calculators to Set Parameters

The table that follows lists the categories of parameters that you must configure for minimum operation. Refer to the parameter categories and the topics on each parameter for more information. You might also need to configure parameters that are not in this table.

Table:     Minimum Parameter Configuration

Parameter Category Description

Current Loop

Use the Current Loop parameters to set the digital current loop gains. You must know the motor resistance and inductance. For more information, refer to Configuring the Current Loop and the Servo Loop.

Faults

Use fault masks to configure how the controller responds to fault conditions. To prevent damage to your system, make sure that you set the PositionErrorThreshold parameter and the AverageCurrentThreshold parameter.

Feedback

The Feedback parameters configure the feedback device. If you need to invert the polarity, use the FeedbackSetup parameter.

Other Information

To see the feedback counters, use Status Utility.

To plot the feedback signals, use Digital Scope.

To adjust the signal amplitude for analog encoders, use Feedback Tuning in Digital Scope.

Limits

Use the Limit parameters set the active polarity of the end of travel limits and to configure the software limits.

Other Information

To see the limit signals, use Status Utility.

Motion

Use the Motion parameters to configure the default ramp rates and the home cycle operation.

Motor

Use the Motor parameters to select the type of motor that you are using, such as AC, DC, or Stepper. You can also configure commutation parameters for three-phase motors and configure parameters for stepper motors.

While you are configuring your system for the first time, make sure that the value of the MaxCurrentClamp parameter is low.

Other Information

To see the hall-effect signals, use Status Utility.

To test the hall-effect signals, use the MotorVerification.pgm example program in the [Install directory]\Programs folder. The default install directory is C:\Program Files\Aerotech\A3200.

Servo Loop

Use the Gains parameters to configure the gains for servo loop. You can also use Autotune in Digital Scope.

Digital Scope Application Overview

Loop Transmission

Units

Use the Units parameters to specify the number of feedback counts per programming unit.
