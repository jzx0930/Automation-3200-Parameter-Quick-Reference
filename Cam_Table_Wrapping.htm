Cam Table Wrapping

If cam table wrapping is invoked, and if the master axis travels outside of the range of the master positions in the cam table, it is important to construct a cam table that does not instantaneously jerk the slave axis at the rollover point within the cam table. Use the following example to understand this problem.

To simplify the following example, it is assumed that the SLAVEADVANCE and SLAVEOFFSET commands set a value of 0, and that camming was synchronized at the point where m=m0, s=s0. It is also assumed that the points between {m0,s0} and {m9,s9} in the table are linear in slave and master values, and the following are true:

 

m

=

master position
 

s

=

slave position
 

sv

=

slave velocity
 

s'

=

slave commanded position value, derived from the table and m
 

 

 

 
 

mi

=

the i master position value in the table (i=0 is first point)
 

si

=

the i slave position value in the table (i=0 is first point)
 

mL

=

last master position value in the table
 

sL

=

last slave position value in the table
  1. The master axis always rolls over and starts indexing into the beginning of the table again. The slave axis does not roll over unless you are in CAMSYNC mode 3 (in CAMSYNC mode 3, s’ is a velocity). For example if {m0,s0}={0,0}, {m1,s1}={1,2} and {mL,sL}={100,200} and m=101, then if in CAMSYNC mode 3, s’=2, otherwise s’=202. This behavior always occurs, regardless of whether the master or slave axis has a rollover defined or not.
  2. If the master axis has a rollover (refer to RolloverCounts) defined, you must set the span of the master positions to the rollover. That is, (mL-m0) must be an even multiple of the RolloverDistance or the slave axis moves incorrectly.
  3. You must set sL=s0 or the slave axis is abruptly jerked when crossing either boundary of the table.
