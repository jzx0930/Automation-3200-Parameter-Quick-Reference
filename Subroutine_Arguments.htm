Subroutine Arguments

Syntax:

 
CALL Subname [<ArgumentIdentifier><Value> ...]

Example:

 
CALL MYSUB X1 Y($global[2])
;...
END PROGRAM
 
DFS MYSUB
   $global[0]=$X
   $global[1]=$Y
ENDDFS

Subroutine arguments are passed by using an argument identifier followed by the desired value. You can only use argument identifiers from the following lists.

  • Any of the 32 axis names defined in the active parameter file. (Refer to Axis Naming Conventions for more information on axis names.)

    Note: You can only use axis names if they are currently defined in the active parameter file. If, for example, you rename the X axis to MyAxis in the active parameter file, you can no longer use X as an argument identifier.
  • Any of the following case insensitive reserved letters: O, P, Q, R, L, I, J, K, F, E, S, and T.

Values are accessed in the subroutine body by using a dollar sign ($) followed by the same argument identifier. Up to 32 arguments can be passed to a subroutine.

The details concerning argument passing apply equally to the FARCALL command and the CALL command. However, you cannot pass arguments to an ONGOSUB command.

Arguments are passed by value and the value is not passed back to the calling program.

A subroutine can be called with a variable number of arguments. If an argument is used in a subroutine or program to which it has not been passed, a callstack parameter error occurs. Test each argument to see if a value was passed into the subroutine, using the DEFINED keyword. For more information, refer to Subroutine Argument Existence Testing.
