PC Modbus Troubleshooting

Note: If one or both of your network cards (Windows and real-time) are not functioning, uninstall the Windows card in Device Manager. Select Scan for Hardware Changes. Windows reinstalls the network card.

If the TXD/RXD LED on the Wago module is not continuously blinking after an SMC reset, communication with the module is not established. Use the following procedure to establish communication.

  1. Open the Status Utility and click the Ethernet tab to view the status of PC Modbus connections.
  2. The Status Utility shows all connections that are currently active under the Ethernet tab. The Master Connections Mask shows the connected or disconnected state of all 16 possible PC Modbus Master connections, starting with connection 0 on the right. A value of 1 shows that the connection is actively communicating. A value of 0 shows that the connection is either not configured or is not able to communicate. You can find more information from the error code of the PC Modbus Master connection that you are using. The Status Utility shows the connection status, errors that occur for the first two master connections, and the slave connection. Refer to PCModbusError for more information about all errors that are shown.
  3. If communication was not established correctly, verify that the IP address of the PC and the IP address of the WAGO module are configured correctly to operate on the same subnet. Refer to PC Modbus Overview for more information about selecting the correct subnet mask.
  4. You can verify the IP address of the module by issuing a Ping command and using a regular PC with a network card controlled by Windows. Assign the network card of the PC to an IP address that is on the same subnet as the WAGO module. Connect the network card and the WAGO module through the same network switch or use a crossover cable to make a direct connection. From a Windows command prompt, issue the Ping command with the IP address of the WAGO module. If you receive a response, then you have the correct IP address and subnet mask for that module.
  5. Open Configuration Manager and make sure that the mappings are correctly configured in the Fieldbus section of the parameter file. Some I/O modules, such as the WAGO module, do not communicate successfully if I/O is mapped and that I/O does not physically exist on the device. Verify that you have the correct I/O addresses for your Modbus module.
  6. If the module is a WAGO PFC/FC, and the I/O LED flashes red, the I/O LED is showing an error. Count the blink sequences to find and decode the blink error code. For more information, refer to the WAGO PFC/FC manual.
