PSO Overview

Hardware: PSO does not apply when used with the Nmark SSaM or the Nstep.

The A3200 controller contains encoder tracking and synchronized pulse generation features referred to as the PSO (Position Synchronized Output). The PSO option can be used to track the position of one to three encoder channels and is typically used for laser power modulation or triggering of external hardware devices.

If encoder feedback is nonlinear or is not available, refer to Part-Speed PSO.

The PSO consists of two main functional blocks, the distance counter(s), and the pulse generator. The distance counters generate a "firing event" after a specified distance is traveled. The firing event triggers the pulse generator, which in turn generates a pulse sequence. The distances can be either fixed or contained in a drive array. This array is tightly coupled to the PSO hardware for high frequency updates of the tracking distance.

The PSO hardware also contains up to two window functions that can independently track encoder channels and further qualify the main tracking counters. The window signal can also be used as the main firing output.

For more PSO connection information, refer to the drive hardware manual on your software installation device. To see a list of available hardware manuals, refer to Other Documentation.

NOTE: If the axes use the -MXU option or the -MX1 option for feedback, you cannot use the PSO feature. But you can use Part-Speed PSO.
