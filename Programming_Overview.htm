Programming Overview

The Automation 3200 programming language has many AeroBasic commands. Refer to Command List for a list of the available AeroBasic commands.

Some AeroBasic commands do actions that were carried out with G-Code or M-Code commands in previous versions of the A3200 . Refer to G-Code and M-Code Equivalents for an overview of which G-Codes and M-Codes correspond to AeroBasic commands.)

Sample programs are in the \A3200\Samples\AeroBasic folder installed with the A3200 software.

NOTE: For .NET (C# and Visual Basic), C, LabVIEW, or MATLAB library Help, refer to the Programming Help file (Start → Programs → Aerotech → Automation 3200 → A3200 Programming Help).

The AeroBasic programming section is separated into function-based subsections.

Acceleration and Deceleration Ramping

Linear, sinusoidal, rate-based, or time-based methods of ramping. Refer to Acceleration and Deceleration Ramping Overview.

Feedrate (Controlling Axis Speeds)

Controlling axis speeds. Refer to Feedrate Overview.

Programming Units and Modes

Controls distance and time programming units as well as absolute or incremental (relative) modes. Refer to Programming Units and Modes.

Motion

All motion commands. Refer to Motion Overview.

I/O

All input and output commands. Refer to I/O Overview.

Functions and Operations

Operators and math functions.

Program Flow Control

Conditional commands, loops, and subroutines. Refer to Program Flow Command Category.

Variables

Variable types and scope. Refer to Variables..

Status

Check axis and task information from a user program (that is, read axis command and feedback positions): AXISSTATUS, TASKSTATUS, SYSTEMSTATUS Functions

PSO (Laser Firing)

Position synchronized output programming. Refer to PSO Command Overview.

Advanced Commands

List of advanced program control and communication features.

Advanced Types of Motion Asynchronous, PVT, Master/Slave motion, QCommand (kinematic transforms), Slice

Transformations of Motion Backlash, Rotation, Mirroring, Polar, Cylindrical, Normalcy, POSOFFSET
Note: A program line must be less than 1024 characters. Do not use the line continuation character.
Note: Always end your programs with an END PROGRAM  command.

If you are upgrading from a 2.xx version of A3200 to a 3.0x or higher version, refer to Upgrading from a 2.xx Version for a list of significant changes.
