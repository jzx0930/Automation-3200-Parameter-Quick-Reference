Cam Table Format

The cam table file contains data lines and comment lines. Only the quantity of memory that is available on the controller can limit the size of this file.

A line is a string of characters. You can use a number of consecutive line terminator characters, such as a carriage return or line feed, to terminate a line.

A data line is a string of characters that has a numeric character as the first nonwhite space character. All other lines are comment lines.

White space characters include the tab, form feed, comma, and space characters.

Numeric characters include the digit, decimal point, minus sign, and plus sign characters.

The following sections give the correct format of a cam table file. All keywords and phrases in these sections are case sensitive.

Comment Lines

Your cam table file requires three special comment lines that include the following information. You must put these lines before the first data line in the file. You can put these lines in the sequence that you want to use.

  • The number of points in the file.
  • The units of the master position values.
  • The units of the slave position values.

Comment line Keywords

You must put a keyword on the same line with each of the special comments in the file. Make sure that you put the keyword before the special comment in the line.

  • Put the Number of Points keyword before the number of target points. The first numeric character that follows this keyword starts the number of cam points. This number must be a positive integer that is greater than one. Make sure that the number of data lines in the cam table file is equal to the number of points that you specify for this line.
  • Put the Master Units keyword before the units of the master position values. The first nonwhite space character that is not a ( starts the description of the master position units.
  • Put the Slave Units keyword before the units of the slave position values. The first nonwhite space character that is not a ( starts the description of the slave position units.

Units

You can specify unit descriptions of PRIMARY, SECONDARY, DEGREE, or COUNTS for the Master Units keyword and the Slave Units keyword.

You can also specify a multiplier that the controller applies to the master or slave position units by including a separator / that follows the description of those units.

For Example

If the PRIMARY units are millimeters and the slave positions in the cam table file are microns, then your multiplier value is 0.001. Specify this in the cam table file by entering SLAVE UNITS (PRIMARY/0.001).

Data Lines

Each data line must contain three or four numbers. You must separate these numbers by one or more white space characters. The controller ignores all text that follows these numbers.

  • The first number is the point number and must be a positive integer. Specify point numbers in a monotonically ascending or monotonically descending sequence.
  • The second number is the master position value. The controller reads this value in the units that you specify for the Master Units keyword. Specify master position values in a strictly monotonically increasing or a strictly monotonically decreasing sequence. It is not necessary to make these values evenly spaced.
  • The third number is the slave position value that corresponds to the master position value. The controller reads this value in the units that you specify for the Slave Units keyword.
  • The fourth number is optional and is the interpolation type for the point. This number lets the controller apply a different interpolation type for different points in the cam table. If you include this number, the controller uses this number to override the interpolation mode that you specify for the <InterpolationType> argument in the LOADCAMTABLE or LOADCAMVAR command.

  • NOTE: Aerotech recommends that you do not apply different interpolations for different points because this can cause discontinuities in the motion.
