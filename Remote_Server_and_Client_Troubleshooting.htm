Remote Server and Client Troubleshooting

If you encounter problems while you configure the server-to-client connection, refer to the following troubleshooting information.

Server-to-Client Connection

  • Make sure that the connection between the server PC and the client PC is a dedicated Ethernet connection. For this type of connection, Aerotech supports only a configuration that uses an Ethernet crossover cable or a dedicated high-speed switch. Aerotech does not support server-to-client connections that use hubs.
  • If you cannot connect to the server PC from the client PC, use the Ping command in the Windows command prompt to make sure that the connection is valid. Do the following for each PC.
    To Test the connection for the server PC and for the client PC in windows
    1. On the Start menu, type Run. Then press Enter.
    2. In the Run dialog, type cmd. Then press Enter. The Command Prompt window comes into view.
    3. In the Command Prompt window, do one of the following.
      • If you are testing the connection on the server PC, type ping. Then type the IP address of the client PC.
      • If you are testing the connection on the client PC, type ping. Then type the IP address of the server PC.
    4. Press Enter. The Command Prompt window does one of the following.
      Figure: Ping Command — Valid Connection

      • If the connection is valid, the Command Prompt window shows a response from the Ping command.
      • If the connection is not valid, the Command Prompt window shows a communication timeout.
  • If a communication error occurs after you configure the server-to-client connection, restart the server PC and the client PC to prevent communication errors.
  • If the client PC cannot connect to the server PC after an error condition occurs, such as a watchdog timeout or a communication error, close all A3200 applications on the client PC. Then open one of the A3200 applications to automatically connect to the server PC again.

Connection Settings in Configuration Manager

  • If you make changes in the Connection Settings dialog of Configuration Manager for the server PC and for the client PC but do not restart the two PCs, you must restart the server PC and the client PC for the changes to have an effect.
  • If a server PC and a client PC were previously configured for a server-to-client connection and you want to disable that connection, do the following for each PC.
    To Disable the server-to-client connection for the server pc and for the client pc
    1. Open Configuration Manager.
    2. In the Network Explorer, right-click Controller and select Connection Settings.
    3. In the Connection Settings dialog, select the Connection Mode tab.
    4. In the Connection Mode section of this tab, change the mode to Local.
      Figure: Connection Mode Tab — Local Connection

    5. Click OK.

TCP Ports

  • If you have firewall software enabled or you are using Windows Firewall, either disable the firewall software or configure it to allow inbound and outbound communications through the TCP ports that you want to use. Refer to Step 6 in the Configuration procedure of Remote Server and Client Installation and Configuration for more information.
  • Make sure that the communication ports are configured correctly on the server PC and the client PC.
    • On the server PC, open the Connection Settings dialog and make sure that the number for the Communication Port is the same as the number for the Server Communication Port that is configured on the client PC.
    • On the client PC, open the Connection Settings dialog and make sure that the number for the Server Communication Port is the same as the number for the Communication Port that is configured on the server PC.

Running A3200 Applications on the Server PC

You can run A3200 applications directly on the server PC only if the following conditions are true.

  • There are no A3200 applications running on the client PC.
  • None of the AeroBasic commands that you issue are callback commands. Refer to Callback Commands for more information.
