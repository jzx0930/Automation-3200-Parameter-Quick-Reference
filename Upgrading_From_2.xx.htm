Upgrading From a 2.xx Version of A3200

Aerotech implemented significant changes in the 3.xx release of the A3200 software that you need to be aware of. If you are upgrading from a version of A3200 software that was made before the 3.xx release, some of your existing program files, parameter files, and other source code might not be compatible. The major changes that were made in the 3.xx release include changes to the Hardware, User Interface, Configuration, and Programming.

Hardware

  1. There are problems with upgrading a Revision A Npaq.
  2. If you have a Revision A Npaq that was manufactured before May 2007, you might encounter FireWire errors when you upgrade to 3.0x. Npaqs manufactured in May 2007 have a slower DSP and are not capable of running all 3.0x features. If you must upgrade to 3.0x, send the Npaq back to Aerotech so we can apply an ECN to correct the problem. You can contact Aerotech Global Technical Support for more information.
  3. Some drive types are obsolete.
  4. The following drive types are obsolete.
    • Ndrive HP (previously known as Ndrive), all revisions
    • Ndrive HL (previously known as NLdrive), all revisions
    • Ndrive CL, all revisions
    • Ndrive CP (previously known as NCdrive), revision dash
    • Npaq, revision dash

    If you have one of these obsolete drives, use one of the following options.

    1. Continue to use a 2.XX version of the A3200 software.
    2. Contact your Aerotech sales representative to purchase a more recent drive.

    Aerotech discontinued software support for these obsolete drives in order to make maintenance of the A3200 easier and to add more features more quickly.

User Interface

  1. Most of the functionality of NView has been replaced with CNC Operator Interface. Program editing was replaced with Motion Composer. System setup, such as configuring the path to the active parameter file, was replaced with Configuration Manager.
  2. NParam was replaced with Configuration Manager.
  3. NScope was replaced with Digital Scope.
  4. NLoad was replaced with Firmware Loader.
  5. NEnetCfg and NReg were replaced with the Connection Settings dialog in Configuration Manager.
  6. NStat was renamed Status Utility.
  7. NDebug was renamed Console.

Configuration and Programming

  1. The names of most of the parameters have changed. The names are now more descriptive and more intuitive to the programmer.
  2. Many AeroBasic commands were changed and reorganized. They are more descriptive and easier to find.
    • Some commands changed in naming or syntax only.
    • Some commands replaced entire features. (For example, the RI/RO Action was replaced with PROGRAM commands.)
  3. The Operating System support changed. Refer to the Specifications tab for A3200 at www.aerotech.com.
  4. A3200 PC Modbus and Drive Modbus implementation changed.
    • Modbus I/O points are mapped with a unique name in Configuration Manager and can be accessed directly as variables in AeroBasic.
    • Many PC Modbus and Drive Modbus parameters were removed. Modbus connections and mappings are now configured using the Modbus Configuration tool in Configuration Manager.
    • There is no graphical interface that you can use to set or get Modbus data.
  5. Nmark programming changed.
  6. ActiveX controls were removed in order to serve the motion control aspect of the libraries.
  7. No new graphical controls are exposed in 3.0x.
  8. The interfaces for .NET, C, C++, LabVIEW, and Visual Basic 6 were replaced with new .NET, native C, and LabVIEW libraries. These changes increase the usability of our programming interfaces and decrease our time to make the A3200 software updates available.
  9. Data Collection is more flexible and straightforward.

Aerotech wants to supply the best possible products and user experience for our customers. If you have any questions regarding conversion, contact Aerotech Global Technical Support.
