PC Modbus Communication with an Automation Direct I/O Module

Use the following tutorial to configure PC Modbus to communicate with an Automation Direct slave device. The information for this tutorial is specifically written for the Automation Direct Terminator I/O T1H-EBC100 device, connected with T1K-16TD1 and T1K-16ND3 modules. But, you can use much of the information in this tutorial with other Automation Direct slave devices that are almost the same as these modules.

Configure the IP Address of the Automation Direct Slave Device

NOTE: In this section, use a standard network card under Windows control to configure the Automation Direct slave device. Connect the Automation Direct slave device through a network switch with a standard Ethernet cable, or directly connect the Automation Direct slave device to your PC with a crossover cable.

For specific instructions on how to configure the IP address and subnet mask of the Modbus device, refer to the documentation from the manufacturer. You can find the documentation for the Automation Direct Terminator I/O T1H-EBC100 slave device on the AutomationDirect website.

Use the following procedure to configure PC Modbus to communicate with an Automation Direct slave device.

  1. Install the NetEdit 3 application at http://www.hosteng.com/SW-Products/NetEdit3.htm.
  2. Connect the Automation Direct slave device to the same network to which your PC is connected.
  3. Open NetEdit 3.
  4. In the NetEdit 3 dialog, click the Scan Network button. The Automation Direct slave device that you are using comes into view.
  5. Figure: NetEdit 3 Dialog - Automation Direct Slave Device

  6. NOTE: In the NetEdit 3 dialog, if your Automation Direct slave device is not found, you might need to manually select the network adapter of your PC by going to Network → Adapter and selecting the correct device.
  7. In the NetEdit 3 dialog, double-click the Automation Direct slave device. The General Settings dialog comes into view.
  8. Figure: General Settings Dialog - Automation Direct Slave Device

    1. In the General Settings dialog, configure the correct IP Address and Subnet mask for the Automation Direct slave device. You must configure the PC Modbus card and Automation Direct slave device on the same subnet mask so the PC Modbus card and Automation Direct slave device can communicate correctly.

    2. NOTE: The PC Modbus card and Automation Direct slave device can communicate over a variety of network structures. For performance reasons, Aerotech recommends that you connect the PC Modbus card and Automation Direct slave device directly with a crossover cable and assign static IP addresses for both the PC Modbus card and Automation Direct slave device. Refer to PC Modbus Overview for more information.
    3. In the General Settings dialog, click OK to apply these settings.
  9. After you correctly configure the Automation Direct slave device, close the NetEdit 3 application.

Configure the A3200 PC Modbus Network Card

In the following procedure, you assign a network card for the A3200 that you can use under INtime control.

  1. Configure a network card that is compatible with INtime and put it under INtime control.
  2. For more information on how to configure a network card, refer to the procedure for Installing the Ethernet Card in PC Modbus Installation.
  3. Assign a valid IP address to the network card.
    1. Make sure that the IP address that you assign to the network card is compatible with the IP address that you assigned to the Automation Direct slave device in the Configure the IP Address of the Automation Direct Slave Device section of this tutorial.
    2. Refer to the procedure To Configure the Ethernet Stack in PC Modbus Configuration to configure the INtime TCP/IP stack. The following figure shows an example configuration.
    3. Figure: Connection Settings Dialog

  4. Connect the Automation Direct slave device to this network card directly with a crossover cable or through a network switch with a standard Ethernet cable.

Configure the A3200 PC Modbus Master Connection

Use the following procedure to configure a PC Modbus Master to communicate with the Automation Direct slave device.

  1. Open Configuration Manager.
  2. Create or open the active parameter file.
  3. In the active parameter file, expand Fieldbus and click PC Modbus Master.
  4. Click the Edit PC Modbus Master button.
  5. Click the Add Connection button to add a new connection.
  6. In the PC Modbus Master Configuration dialog, enter the connection specifications for the Automation Direct slave device.
  7. Figure: PC Modbus Master Configuration - Automation Direct Slave Device Connection

    1. Look at the Name section to see the name of the Automation Direct slave device you are using. In this section, you can change the name Automation Direct to a name that you want to use.
    2. You should see that the Slave ID for your Automation Direct slave device is the same as the Module ID for this device that is located in the General Settings dialog of the NetEdit 3 application.
    3. The Slave IP Address should be the same as the IP address that you configured for the Automation Direct slave device using the NetEdit 3 application. Refer back to the Configure the IP Address of the Automation Direct Slave Device section of this tutorial for more information.
    4. Make sure that the Slave Port is set to 502. This is the default Modbus port.
  8. Add the correct variable mappings to correspond to the I/O of the Automation Direct slave device. For specific instructions on how to add variable mappings, refer to Creating Fieldbus Mappings for PC Modbus Master.
    1. This Automation Direct slave device has 16 digital outputs on the T1K-16TD1 module.

  10. Click OK to apply the configuration.
  11. Save the parameter file.
  12. Reset the controller for the changes to have an effect.

Examples of Using Mappings in an AeroBasic Program

The AeroBasic example programs that follow show the various ways in which you can use Fieldbus mappings. Each mapping that you define generates a corresponding AeroBasic variable (that starts with a '$') that you can use in a program in the same way that you use other variables in a program. The I/O is automatically synchronized in the background by the controller as quickly as possible. Inputs are automatically requested by the controller once per millisecond. Outputs are written to by the controller only when the outputs have been changed.

Example

' Use a FOR loop to change each of the output bits.
DVAR $iii
for $iii = 0 to 15
    $OutBits[$iii] = !$OutBits[$iii]
next $iii

Example

' Use an input bit as the condition to an action.
if ($InBits[4] == true)
    MSGDISPLAY 0, "Bit on!"
endif
