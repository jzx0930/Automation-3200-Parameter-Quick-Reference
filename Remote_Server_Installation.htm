Remote Server and Client Installation and Configuration

To control the A3200 SMC remotely through an Ethernet interface, install the software in server mode on one PC and in client mode on a different PC. The PCs must be connected through Ethernet. In this configuration, Motion Composer and other applications on the client PC transparently communicate with the A3200 SMC that is running on the server PC, which is the PC that controls the motion. Because the real-time operating system is isolated from the client PC, more resources are available for your Windows applications.

NOTE: For the server-to-client connection, you must use Ethernet cards that are under standard Windows Ethernet control. Do not use Ethernet cards that are under A3200 real-time control.

Licensing

To use a Remote Server configuration, you must purchase a license that includes the Remote option. To discuss your licensing needs, speak with your Aerotech sales representative or contact Aerotech Global Technical Support.

For more information about licensing, refer to Licensing Options.

Installation

  1. On the server PC, install the A3200 software by using the applicable license key. When you are prompted to select the Setup Type, select Complete. Refer to Step 1: Installing the Software for installation instructions.
  2. On the client PC, install the A3200 software by using the applicable license key. When you are prompted to select the Setup Type, select Client Only.

Activation

After you complete the installation procedure, you must activate the software on both the server PC and the client PC. Refer to Activating Your Software.

Configuration

Aerotech strongly recommends and supports only a configuration where the A3200 remote server PC to client PC connection is isolated from the Internet and all intranet networks.

  1. In Windows, configure the network card for the server PC and the network card for the client PC to use static IP addresses. The IP address on the server PC must be different from the IP address on the client PC. Aerotech recommends that you use the same subnet mask on the two PCs.
    To Configure Network Cards for the server PC and for the client PC

    NOTE: For the server-to-client connection, you must use Ethernet cards that are under standard Windows Ethernet control. Do not use Ethernet cards that are under A3200 real-time control.

    1. On the Start menu, select Control Panel.
    2. In Control Panel, select Network and Internet → Network and Sharing Center.
    3. In the list on the left, select Change adapter settings.
    4. Right-click Local Area Connection and select Properties.
    5. In the Local Area Connection Properties dialog, double-click Internet Protocol Version 4 (TCP/IPv4).
      Figure: Local Area Connection Properties Dialog

    6. In the Internet Protocol Version 4 (TCP/IPv4) Properties dialog, select Use the following IP address.
      Figure: Internet Protocol Version 4 (TCP/IPv4) Properties Dialog

    7. Type the IP address, Subnet mask, and Default gateway that you want to use.
    8. Click OK.
  2. To connect the server PC to the client PC, do one of the following.
    • Connecting through a Dedicated High-Speed Switch Connect one end of an Ethernet cable to an Ethernet port on the server PC and connect the other end of the cable to an Ethernet port on a dedicated high-speed switch. Then connect one end of an Ethernet cable to an Ethernet port on the client PC and connect the other end of the cable to an Ethernet port on the same high-speed switch.
    • Connecting Directly Connect one end of an Ethernet crossover cable to an Ethernet port on the server PC and connect the other end of the crossover cable to an Ethernet port on the client PC.
  3. Configure the server PC and the client PC.
    To Configure the Server PC

    The server PC has the Drive Communication interface to the drives.

    1. Open Configuration Manager.
    2. In Network Explorer, right-click the Controller node and select Connection Settings. For more information about the Connection Settings dialog, refer to Connection Settings.
    3. In the Connection Mode tab, select Server.
      Figure: Connection Mode Tab — Server PC

    4. Communication Port Type the Communication Port number. The Communication Port number on the server PC must be the same as the Server Communication Port number on the client PC. Almost all systems can use the default value of 10024.
    5. Click OK to save the server PC configurations. Then close Configuration Manager and all other A3200 applications.
    To Configure the Client PC

    The client PC has the Ethernet connection to the server.

    1. Open Configuration Manager.
    2. In Network Explorer, right-click the Controller node and select Connection Settings. For more information about the Connection Settings dialog, refer to .
    3. In the Connection Mode tab, select Client.
      4. Figure: Connection Mode Tab — Client PC

    4. Client Callback Port Type the Client Callback Port number. Almost all systems can use the default value of 10025.
    5. Server IP Address Type the server IP address. If you do not know the IP address of the server PC, run IPConfig from a command prompt on the server PC.
    6. Server Communication Port Type the Server Communication Port number. The Server Communication Port number on the client PC must be the same as the Communication Port number on the server PC. Almost all systems can use the default value of 10024.
    7. Click OK to save the client PC configurations. Then close Configuration Manager and all other A3200 applications.
  4. Restart the server PC and the client PC.
  5. Turn off Windows Automatic Updates on the server PC and the client PC.
    To Turn Off Windows Automatic Updates
    1. On the Start menu, select Control Panel.
    2. In Control Panel, select System and Security.
    3. In the Windows Update section, select Turn automatic updating on or off.
    4. Click the Important updates arrow and select Never check for updates (not recommended).
      Figure: Turn Off Windows Automatic Updates

    5. In the Recommended Updates section, make sure that the Give me recommended updates the same way I receive important updates box is selected.
    6. Click OK.
  6. If you have firewall software enabled or you are using Windows Firewall, do one of the following for the server PC and the client PC. If you are not using a firewall, continue to the next step.
    • Disable the firewall software.
      To Turn Off Windows Firewall
      1. On the Start menu, select Control Panel.
      2. In Control Panel, select System and Security.
      3. In the Windows Firewall section, select Check firewall status.
      4. In the list on the left, select Turn Windows Firewall on or off.
      5. Select Turn off Windows Firewall (not recommended) for all applicable networks.
        Figure: Turn Off Windows Firewall

      6. Click OK.
    • Configure the firewall software to allow inbound and outbound communications through TCP ports that you want to use for server PC and client PC communications.
      To Allow Inbound Communications Through TCP Ports in Windows Firewall
      1. On the Start menu, select Control Panel.
      2. In Control Panel, select System and Security → Windows Firewall.
      3. In the list on the left, select Advanced Settings.
      4. On the left pane of the Windows Firewall with Advanced Security dialog, select Inbound Rules.
        Figure: Inbound Rules in Windows Firewall

      5. Do one of the following for each of the TCP ports that you want to use.
        • If the TCP port is in the list of Inbound Rules, select that port. Then in the Actions pane, select Enable Rule.
        • If the TCP port is not in the list of Inbound Rules, go to the Actions pane and select New Rule. In the New Inbound Rule Wizard dialog, add and configure each TCP port that you want to use.
      To Allow outbound communications through TCP Ports in Windows Firewall
      1. On the Start menu, select Control Panel.
      2. In Control Panel, select System and Security → Windows Firewall.
      3. In the list on the left, select Advanced Settings.
      4. On the left pane of the Windows Firewall with Advanced Security dialog, select Outbound Rules.
        Figure: Outbound Rules in Windows Firewall

      5. Do one of the following for each of the TCP ports that you want to use.
        • If the TCP port is in the list of Outbound Rules, select that port. Then in the Actions pane, select Enable Rule.
        • If the TCP port is not in the list of Outbound Rules, go to the Actions pane and select New Rule. In the New Outbound Rule Wizard dialog, add and configure the TCP port that you want to use.
  7. On the client PC, run Motion Composer or one of the other A3200 applications such as Console, Digital Scope, or the Status Utility.
  8. If the application is configured correctly, it starts and communicates with the A3200 SMC as if the SMC is running on the client PC although it is actually running on the server PC.
  9. To test the callback events, issue an AeroBasic command such as MSGDISPLAY to cause the server PC to generate the callback events.
  10. Callback events are sent from the server PC and processed by the client PC. The server PC sends the callback events to the client PC through the Client Callback Port, which is a dedicated Ethernet port.

After you configure the server PC and the client PC, you can view the status of the server-to-client connection in the Ethernet Tab of the Status Utility.

Figure: Server-to-Client Connection in the Status Utility

  • Local Client Outgoing quantity on the client PC shows the number of outgoing connections to the server PC.
  • Local Server Incoming quantity on the server PC shows the number of incoming connections from the client PC.

NOTE: If the client PC cannot connect to the server PC after an error condition occurs, refer to the Server-to-Client Connection section of Remote Server and Client Troubleshooting for information about how to correct this connection.

Customizing Safety Operations

You can customize safety operations that the server software uses to respond to sudden communication losses and timeouts by writing these operations in an AeroBasic program. For example, you can write this watchdog program to execute the ABORT command on all of the axes that you want to abort before the A3200 SMC stops.

To enable the watchdog program

  1. Write an AeroBasic program that includes the safety operations that you want to use.
  2. Save this program to [Install directory]\Programs\EnetWDogTimeout.pgm on the server PC. The default installation directory is C:\Program Files\Aerotech\A3200.

When the server software detects communication losses and timeouts, the A3200 SMC executes the EnetWDogTimeout.pgm watchdog program. After this watchdog program completes, the A3200 SMC stops for safety.
